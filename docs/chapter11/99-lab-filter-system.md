# 💻 Lab: ระบบคัดกรองข้อมูลอัจฉริยะ (Filter System)

> 💡 **เป้าหมายของ Lab นี้:** ยินดีต้อนรับสู่บททดสอบสำคัญก่อนเข้าสู่มหาเวทย์ LINQ! Lab นี้จะทดสอบความสามารถในการผสมผสาน `Func/Predicate` เข้ากับ `Lambda Expressions` เพื่อสร้าง "ระบบแกนกลาง" ที่สามารถกรอง ค้นหา หรือจัดการข้อมูลได้อย่างอิสระ โดยไม่ต้องพึ่งพา `if/else` นรกอีกต่อไป!

## 📖 ภาพรวมของโปรแกรม (Program Overview)

บริษัท **MegaCorp** มีพนักงาน 500 คน แผนก HR ปวดหัวมากเพราะต้องทำฟังก์ชันแยกกันยุ่บยั่บ เช่น `GetItEmployees()`, `GetHighSalary()`, `GetYoungStaff()` ฯลฯ ทำให้ไฟล์โค้ดยาวเหยียดและแก้ไขยาก

หน้าที่ของคุณในฐานะสถาปนิกคือ การสร้าง **"ท่อร่อนอัจฉริยะ (The Universal Filter)"** เพียงแค่รูเดียว! แล้วปล่อยให้ HR (ใน Main) เป็นคน "สอดไส้ตรรกะ Lambda" เข้ามาเองว่าวันนี้อยากร่อนเอาข้อมูลอะไร!

### ตัวอย่างหน้าจอเมื่อรันโปรแกรมสำเร็จ

```text
================================================
         [ MEGACORP HR FILTER SYSTEM ]
================================================
กำลังโหลดข้อมูลพนักงาน...

>> 1. กรองเอาเฉพาะ 'พนักงานฝ่าย IT':
- รหัส: E01 | อลิซ | แผนก: IT | เงินเดือน: 55,000 บาท
- รหัส: E04 | เดวิด | แผนก: IT | เงินเดือน: 32,000 บาท

>> 2. กรองเอาเฉพาะ 'เงินเดือนเกิน 50,000':
- รหัส: E01 | อลิซ | แผนก: IT | เงินเดือน: 55,000 บาท
- รหัส: E03 | ชาลี | แผนก: HR | เงินเดือน: 60,000 บาท

>> 3. กรองเอาเฉพาะ 'อายุระหว่าง 25 ถึง 30':
- รหัส: E01 | อลิซ | อายุ: 28 
- รหัส: E02 | บ๊อบ | อายุ: 25

================================================
✨ โบนัส: ระบบอัปเดตเงินเดือน (Action Injection)
>> อัปเดตเงินเดือนพนักงาน IT เพิ่ม 10% เสร็จสิ้น!
================================================
```

## 📋 กฎกติกาและเกณฑ์การประเมิน (Business Requirements)

คุณต้องวางโครงสร้างสถาปัตยกรรมให้เป๊ะตามนี้:

1. **คลาสฐานข้อมูล (Model): `Employee`**
   - มี Properties: `Id` (string), `Name` (string), `Age` (int), `Department` (string), `Salary` (decimal)
   - สร้าง Constructor รับค่าทั้ง 5 ตัว
   - สร้างเมธอด `public void PrintInfo()` ให้ปริ้นรูปแบบสวยงามตามตัวอย่าง

2. **คลาสเครื่องจักรคัดกรอง (The Engine): `EmployeeFilter` (ให้ทำเป็น `static class`)**
   - 🌟 สร้างเมธอด: `public static List<Employee> Filter(List<Employee> list, Predicate<Employee> condition)`
   - ลอจิกข้างใน: วนลูปเช็คสภาพพนักงานทีละคน โยนเข้าเครื่องพิพากษา `condition` ถ้าได้จริง (true) ให้เก็บเข้า List ใหม่ แล้วส่งกลับ

3. **การสั่งการใน Main (The UI Level):**
   - สร้าง `List<Employee> staff = new List<Employee>();` และยัดข้อมูลจำลองลงไปสัก 4-5 คน (ผสมแผนก อายุ และเงินเดือนให้หลากหลาย)
   - 🌟 เรียกใช้ `EmployeeFilter.Filter` 3 ครั้งติด โดย **"ห้ามไปสร้างฟังก์ชันเช็คเงื่อนไขไว้ข้างนอกเด็ดขาด"** ให้ใช้ชนักติดหลัง `Lambda =>` โยนเข้าไปในพารามิเตอร์ดื้อๆ เลย!
     - ภารกิจ 1: หาคนแผนก "IT"
     - ภารกิจ 2: หาคนเงินเดือน > 50000
     - ภารกิจ 3: หาคนอายุ >= 25 **และ (&&)** <= 30
   - นำผลลัพธ์ที่ได้แต่ละรอบ มาวนลูป `foreach` แล้วสั่ง `.PrintInfo()` โชว์หน้าจอ

## 📝 ขั้นตอนการพัฒนา (Step-by-Step Implementation)

### ขั้นตอนที่ 1: เตรียมโครงสร้าง Model 
- [ ] สร้างคลาส `Employee` มี Property ครบ 5 ตัวและ Constructor
- [ ] เขียน `PrintInfo()` ใช้ String Interpolation จัดรูปแบบให้สวย

### ขั้นตอนที่ 2: สร้างระบบตะแกรงร่อน (The Universal Filter)
- [ ] สร้างคลาส `EmployeeFilter`
- [ ] เขียนฟังก์ชัน `Filter` ที่รับลิสต์พนักงาน + รับ `Predicate<Employee>`
- [ ] ไส้ใน วนลูป `foreach` เช็คเงื่อนไขและสะสมคนรอดชีวิตคืนค่า

### ขั้นตอนที่ 3: บททดสอบ Lambda หน้างาน
- [ ] ใน `Main` เสกพนักงาน 4-5 คน ยัดลง List ต้นฉบับ
- [ ] ลองเรียก `EmployeeFilter.Filter(staff, emp => ...);` เอามาเก็บใส่ลิสต์ใหม่ทีละอัน
- [ ] วนปริ้นผลลัพธ์เช็คดูว่ามันกรองมาแม่นไหม

## 🔥 Challenge (ภารกิจท้าทายขีดจำกัด)

สำหรับผู้ที่กระหายพลังแห่ง Functional Programming!

**ภารกิจระบบควบคุมแบบเบ็ดเสร็จ (The Polymorphic Processor):**
ถ้าเราทำได้แค่ "กรอง (Read/Find)" มันก็ดูธรรมดาไป... เรามาเพิ่มความสามารถให้ "ลงมือทำ (Execute/Update)" ด้วยเถอะ!

1. ในคลาส `EmployeeFilter` ให้เพิ่มเมธอดใหม่ชื่อ `ProcessEmployees`
2. **สเปคพารามิเตอร์:** รับ `List<Employee> list` และรับมือสังหาร **`Action<Employee> action`**
3. **ลอจิกข้างใน:** ไม่ต้องเช็คอะไรทั้งนั้น แค่วนลูป `foreach` ดึงพนักงานมาทีละคน แล้วโยนไปให้มือสังหารจัดการ! `action(emp);` 
4. **ใน Main (การเรียกใช้งาน):**
   - โจทย์: "จงเพิ่มเงินเดือนให้พนักงานแผนก IT ทุกคน คนละ 10%"
   - วิธีทำ: คุณเอารายชื่อพนักงานฝ่าย IT ที่กรองได้จากข้อ 1 (ได้เป็น List) มาโยนใส่ท่อ `ProcessEmployees`
   - พารามิเตอร์ตัวที่ 2 (Action Lambda): ให้เขียนท่า `emp => emp.Salary = emp.Salary * 1.10m;`
   - ลองปริ้นรายชื่อพนักงาน IT อีกรอบดูสิ... คุณจะเห็นว่าเงินเดือนพวกเขาถูกอัปเดตเรียบร้อยแล้ว โดยใช้โค้ดแค่บรรทัดเดียว!

นี่คือสถาปัตยกรรมระดับเทพ ที่เปลี่ยนแนวทางการเขียนลูปนรกแบบเดิมๆ ให้กลายเป็น "Data Pipeline" ที่ไหลลื่นและสะอาดที่สุดครับ! (และนี่คือเบื้องหลังการทำงานของ LINQ `.ForEach()` ที่คุณจะได้เจอในบทหน้าครับ!)
