# Lambda Expressions (ลูกศรแห่งความตาย)

> 💡 **เป้าหมาย:** สัมผัสฟีเจอร์ที่ถูกใช้งานมากที่สุดใน C# ยุคใหม่! ถึงเวลาเปลี่ยนลุคโปรแกรมเมอร์ให้กลายเป็นสถาปนิกจอมเวท เรียนรู้วิธีย่อฟังก์ชันขนาดมหึมา ให้เหลือเพียงสัญลักษณ์ลูกศร `=>` (Fat Arrow) เพียงบรรทัดเดียว! โค้ดจะสั้นลงกว่า 80% ทันทีที่อ่านบทนี้จบ!

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. วิวัฒนาการของการย่อฟังก์ชัน

การที่เราจะส่งฟังก์ชันไปให้ `Action` หรือ `Predicate` ในสมัยก่อน เราต้อง "แยกตัว" ไปเขียนเป็น `static void ...` อยู่ข้างนอกให้รกไฟล์ (เหมือนในหัวข้อที่แล้ว) 
ต่อมาไมโครซอฟต์เห็นว่า "ถ้าฟังก์ชันมันสั้นแค่นี้ ทำไมไม่เขียนมัน 'แทรก' ลงไปตรงบรรทัดนั้นเลยล่ะ!?"

**ยุคที่ 1: Named Method (ฟังก์ชันปกติ)**
ต้องตั้งชื่อ ต้องประกาศชนิดข้อมูลวุ่นวาย
```csharp
static bool IsEven(int x) { return x % 2 == 0; }
Predicate<int> p = IsEven;
```

**ยุคที่ 2: Anonymous Method (ไร้ชื่อ - C# 2.0)**
ไม่ต้องตั้งชื่อฟังก์ชันแล้ว พิมพ์ดื้อๆ เลย
```csharp
Predicate<int> p = delegate(int x) { return x % 2 == 0; };
```

**ยุคที่ 3: Lambda Expression (ปัจจุบัน - C# 3.0+)**
ตัดคำว่า `delegate` ทิ้ง! ตัดชนิดข้อมูลทิ้ง (เพราะมันรู้จาก Predicate อยู่แล้ว)! เปลี่ยนปีกกาเป็นลูกศร **`=>`** !
```csharp
Predicate<int> p = (x) => x % 2 == 0; 
// ถ้ามีพารามิเตอร์ตัวเดียว ถอดวงเล็บออกได้อีก!
Predicate<int> p = x => x % 2 == 0; 
```
*แม่เจ้า! จาก 2 บรรทัดเหลือแค่ 12 ตัวอักษร!*

### 2. ไวยากรณ์แห่งลูกศร (The Fat Arrow Syntax)

โครงสร้างของ Lambda มีแค่ฝั่งซ้ายและฝั่งขวา:
**`(พารามิเตอร์) => { ไส้ในฟังก์ชัน }`**
(อ่านว่า "Goes to" หรือ "ชี้ไปยัง")

**รูปแบบต่างๆ ที่พบบ่อย:**
1. **แบบไม่มีพารามิเตอร์ (โยนวงเล็บเปล่า):**
   ```csharp
   Action sayHi = () => Console.WriteLine("Hi");
   ```
2. **แบบมี 1 พารามิเตอร์ (ไม่ต้องใส่วงเล็บ):**
   ```csharp
   Action<string> print = msg => Console.WriteLine(msg);
   ```
3. **แบบมีหลายพารามิเตอร์ (ต้องมีวงเล็บ):**
   ```csharp
   Func<int, int, int> add = (a, b) => a + b;
   ```
4. **แบบหลายบรรทัด (Statement Lambda):** 
   ถ้าไส้ในคุณยาว ต้องกางปีกกา `{ }` และ **ต้องใส่คำว่า `return` เองนะ!**
   ```csharp
   Func<int, string> getGrade = score => {
       if (score > 80) return "A";
       else return "F";
   };
   ```

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่าง: สถาปัตยกรรมกรองข้อมูล (Re-written with Lambda)

เราจะเอาโค้ดตะแกรงร่อนตัวเลขในบทที่แล้ว (ที่ต้องประกาศ `IsEven`, `IsGreaterThan50` ให้ยุ่งยาก) มาเขียนใหม่ด้วย Lambda! คุณจะเห็นเลยว่าทำไมโปรแกรมเมอร์ทั่วโลกถึงคลั่งไคล้มัน!

::: code-group
```csharp [Program.cs]
using System;
using System.Collections.Generic;

namespace LambdaConcept
{
    class NumberManager
    {
        // เมธอดแกนกลางยังเหมือนเดิม (รับ Predicate)
        public List<int> Filter(List<int> numbers, Predicate<int> condition)
        {
            List<int> passed = new List<int>();
            foreach (int num in numbers)
            {
                if (condition(num)) passed.Add(num);
            }
            return passed;
        }
    }

    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบตะแกรงร่อนตัวเลข (พลังแห่ง Lambda) ===\n");

            List<int> rawData = new List<int> { 12, 45, 68, 91, 24, 55, 8 };
            NumberManager manager = new NumberManager();

            // 🌟 เวทมนตร์เริ่มขึ้นที่นี่! (Inline Injection)
            // เราไม่ต้องไปสร้างฟังก์ชัน IsEven ข้างนอกอีกแล้ว... 
            // เราเขียนหน้างาน "อัดฉีด (Inject)" สดๆ ลงไปในพารามิเตอร์เลย!
            
            // 1. กรองเอาเฉพาะ "เลขคู่" 
            // อ่านว่า: สำหรับ x แต่ละตัว ให้พิจารณาว่า x หาร 2 ลงตัวหรือไม่?
            List<int> evens = manager.Filter(rawData, x => x % 2 == 0);
            Console.WriteLine("เลขคู่: " + string.Join(", ", evens));

            // 2. กรองเอาเฉพาะ "เลขที่มากกว่า 50"
            // อ่านว่า: สำหรับ n แต่ละตัว ให้พิจารณาว่า n > 50 หรือไม่?
            List<int> bigNums = manager.Filter(rawData, n => n > 50);
            Console.WriteLine("มากกว่า 50: " + string.Join(", ", bigNums));

            // 🌟 พลังความยืดหยุ่นหน้างาน (Ad-hoc Logic)
            // 3. กรองสเปคประหลาดๆ: "เลขคี่ ที่น้อยกว่า 30"
            // (ถ้าเป็นสมัยก่อน ต้องไปเปิดบรรทัดสร้างฟังก์ชันใหม่ให้เมื่อยตุ้ม)
            List<int> weirdNums = manager.Filter(rawData, num => (num % 2 != 0) && (num < 30));
            Console.WriteLine("เลขคี่ที่น้อยกว่า 30: " + string.Join(", ", weirdNums));
        }
    }
}
```
:::

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

### **โจทย์ที่ 1: การลดรูปขั้นสุด (The Minifier)**
มีโค้ดเก่าแก่สมัยโบราณเขียนไว้ดังนี้:
```csharp
Func<int, int, bool> checkLogic = delegate(int a, int b) {
    return a >= b;
};
```
**คำถาม:** จงแปลงร่างโค้ดด้านบนให้กลายเป็น Lambda Expression ที่สั้นที่สุดเท่าที่จะเป็นไปได้! (บอกเลยว่าเหลือความยาวไม่ถึง 1 ใน 4)

### **โจทย์ที่ 2: ความผิดพลาดของปีกกา (The Braces Trap)**
มีโปรแกรมเมอร์ฝึกหัด พยายามเขียน Lambda แบบนี้:
```csharp
Func<int, int> multiplyByTwo = x => { x * 2; };
```
**คำถาม:** โค้ดนี้ Compile ไม่ผ่านและแดงเถือก! เขาทำผิดกฎไวยากรณ์ข้อไหนของการใช้ปีกกา `{ }` ใน Lambda? และมีวิธีแก้อย่างไรบ้าง (ตอบมา 2 วิธี: วิธีถอดปีกกา กับ วิธีใส่ให้ถูก)?

::: details 💡 คำใบ้และแนวทาง (Hints)
- **สำหรับโจทย์ 1:** 
  คำตอบคือ: **`Func<int, int, bool> checkLogic = (a, b) => a >= b;`**
  (ตัด delegate ทิ้ง ตัดชนิดข้อมูล int ทิ้งเพราะมันรู้จาก Func อยู่แล้ว ตัดปีกกากับคำว่า return ทิ้งเพราะมีบรรทัดเดียว โคตรเท่!)
- **สำหรับโจทย์ 2:** **แดงเถือกเพราะลืมคำว่า `return` ครับ!**
  กฎเหล็ก: "เมื่อไหร่ที่คุณกางปีกกา `{ }` ครอบไส้ใน Lambda... คอมไพเลอร์จะถือว่าคุณเข้าสู่โหมดจริงจัง (Statement Mode) มันจะยกเลิกการคืนค่าอัตโนมัติ คุณต้องบังคับพิมพ์คำว่า `return` ด้วยตัวเองเท่านั้น!"
  - **วิธีแก้ที่ 1 (สายขี้เกียจ - Expression):** ถอดปีกกาทิ้งให้หมด `x => x * 2;` (มันจะแอบใส่ return ให้เอง)
  - **วิธีแก้ที่ 2 (สายจริงจัง - Statement):** ใส่ให้ครบ `x => { return x * 2; };`
:::

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: ศูนย์บัญชาการปุ่มกด (The Button Command Center)**

**ภารกิจของคุณ:** สร้างระบบปุ่ม (Button) ในเกม ที่สามารถบรรจุ "ท่าโจมตี" อะไรลงไปก็ได้ผ่าน Lambda!

1. สร้างคลาส `Button`
   - มี Action 1 ตัว: `public Action OnClick;`
   - มี Method: `public void Press()` ซึ่งถ้าโดนเรียก จะปริ้นว่า `[ดังจึ๊ก!]` แล้วเช็คว่า `OnClick` ว่างไหม ถ้าไม่ว่าง ให้รัน `OnClick` 
2. ใน `Main` เสก `Button btnA = new Button();` และ `Button btnB = new Button();`
3. 🌟 **ความท้าทาย:** ให้ใช้การกำหนดค่าด้วย Lambda (ห้ามสร้าง static method แยกเด็ดขาด)
   - เอาท่า **"กระโดด (Jump!)"** ยัดใส่ `btnA.OnClick`
   - เอาท่า **"ร่ายเวท (Casting Spell...)" และหน่วงเวลา (แอบปริ้น 1, 2, 3) รวม 4 บรรทัด** ยัดใส่ `btnB.OnClick` (ใบ้: ต้องกางปีกกา!)
4. สั่ง `btnA.Press();` และ `btnB.Press();` ทดสอบดูว่าปุ่มทำงานตามที่ตั้งไว้เป๊ะๆ ไหม!

ถ้าคุณทำได้... นี่แหละครับคือพื้นฐานของระบบ **UI Event Handling** ที่คุณจะต้องใช้ใน Unity 3D, Windows Forms, และ React/Blazor อย่างแน่นอน!

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** ในบรรทัด `x => x > 50` คอมพิวเตอร์มันตรัสรู้ได้อย่างไรครับว่าไอ้ตัวแปร `x` มันเป็นชนิด `int`? ในเมื่อเราไม่เคยพิมพ์คำว่า int นำหน้ามันเลย? (ผมกลัวมันจะนึกว่าเป็น string จัง)
**แนวคำตอบ:** **มันใช้ "วิชาอ่านใจ (Type Inference)" ครับ!**
ความดีความชอบต้องยกให้ C# Compiler ครับ มันจะหันกลับไปดู "ตัวรับ" ฝั่งซ้าย (เช่น เราบอกว่ามันกำลังจะถูกโยนเข้าพารามิเตอร์ `Predicate<int>`) พอมันเห็น `<int>` ปุ๊บ... มันจะย้อนกลับมาเสกให้ตัว `x` ของคุณมีคุณสมบัติเป็น `int` แบบลับๆ ทันทีครับ! (นี่คือเหตุผลว่าทำไมคุณพิมพ์ `x.` แล้วมันมีฟังก์ชันของตัวเลขเด้งขึ้นมาให้ใช้เป๊ะๆ)

**คำถาม 2:** ลูกศร `=>` อันนี้ มันทำงานเหมือนกับลูกศร `->` ในภาษา C++ (ที่เอาไว้ชี้ Pointer) ไหมครับ?
**แนวคำตอบ:** **คนละเรื่องเดียวกันเลยครับ! ห้ามจำสับสนเด็ดขาด!**
- `->` ใน C/C++ คือ Operator สำหรับ "ชี้ลงไปในหน่วยความจำ (Memory Dereference)"
- `=>` ใน C# คือ "เครื่องหมายแยกส่วน (Lambda Declaration)" ซ้ายคือพารามิเตอร์ ขวาคือไส้ใน 
(อย่างไรก็ตาม ปัจจุบันใน C# รุ่นใหม่ๆ ตัว `=>` ถูกเอาไปใช้เขียน Property แบบย่อด้วยนะครับ เรียกว่า Expression-bodied members เช่น `public string Name => "Arthur";` ซึ่งลึกๆ แล้วมันก็คือ Lambda ตัวนึงนั่นแหละครับ!)
:::
