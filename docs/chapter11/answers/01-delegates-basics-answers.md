### **โจทย์ที่ 1: ตรวจจับสเปคฟังก์ชัน (Signature Matching)**
**เป้าหมาย:** ทำความเข้าใจความเข้มงวดของ Type-Safety ใน C# Delegates

**เฉลยคำตอบระดับสถาปนิก:**
มีเพียง **ข้อ 1. `RepeatText`** ข้อเดียวเท่านั้นครับที่รอด!
นี่คือเหตุผลที่ข้ออื่นโดนเตะก้านคอ:
- **ข้อ 2. `CountChars` ผิด!** แม้จะมีพารามิเตอร์ 2 ตัวเหมือนกัน แต่ "ลำดับของชนิดข้อมูล" ไม่ตรงกันครับ (สัญญาเอา string ขึ้นก่อน แต่เจ้านี่เอา int ขึ้นก่อน คอมไพเลอร์ไม่ยอมเสี่ยงเด็ดขาด!)
- **ข้อ 3. `GetText` ผิด!** เพราะมันทะลึ่ง `return string` ออกมา ทั้งๆ ที่ใบสัญญาบังคับว่าต้องเป็น `void` (ไม่มีการคืนค่าใดๆ)
- **ข้อ 4. `Print` ผิด!** เพราะรับพารามิเตอร์ไม่ครบตามจำนวนที่สัญญาเรียกร้อง (ขาดตัว int)

### **โจทย์ที่ 2: ปัญหาผลลัพธ์ของ Multicast (The Return Value Trap)**
**เป้าหมาย:** ระวังภัยแฝงของการเอาฟังก์ชันมาต่อคิวในขณะที่มีการ Return ค่า

**เฉลย:**
ค่าสุดท้ายที่เก็บในตัวแปร `answer` คือ **เลข 2** ครับ!
**เหตุผล:** 
เมื่อคุณเอาฟังก์ชันหลายตัวมาต่อคิวกัน (Multicast) แล้วสั่งรัน... คอมพิวเตอร์จะไล่รันตั้งแต่ตัวที่ 1 ไปจนถึงตัวสุดท้าย
ถ้าฟังก์ชันพวกนี้ดันมีการ "ส่งค่ากลับ (Return)" ออกมาด้วย... คอมพิวเตอร์จะนำค่าใหม่ไป "ทับ (Overwrite)" ค่าเก่าเสมอ!
- คิว 1 รันเสร็จ โยนเลข 1 มาให้
- คิว 2 รันเสร็จ โยนเลข 2 มาทับเลข 1 ของเดิม
- ผลลัพธ์สุดท้ายที่เด้งออกมาเข้าตัวแปร จึงเป็นของคนสุดท้ายเสมอครับ! 
*(นี่คือข้อควรระวัง สถาปนิกเลยมักจะใช้ Multicast เฉพาะกับ `void` เท่านั้น เพื่อจะได้ไม่ต้องมานั่งปวดหัวเรื่องแย่งกัน Return ครับ)*

## 🔥 เฉลย Challenge (ระบบเซนเซอร์แจ้งเตือนภัย The Alarm System)

**เป้าหมาย:** สัมผัสความทรงพลังของ Event-Driven Programming! คุณไม่ต้องเขียน `if/else` เช็คสถานะไฟไหม้อีกต่อไป คุณแค่กระจายข่าวสาร แล้วให้หน่วยงานที่เกี่ยวข้อง (ฟังก์ชันที่เอามาต่อคิวไว้) เด้งขึ้นมารับผิดชอบหน้าที่ของตัวเองโดยอัตโนมัติ!

**กระบวนการคิดระดับสถาปนิก (Publish-Subscribe Pattern):**
คลาส `Sensor` มีหน้าที่แค่ "ประกาศเตือน (Publish)" โดยมันไม่จำเป็นต้องรู้เลยว่าโลกภายนอกมีรถดับเพลิงกี่คัน หรือมีพนักงานกี่คนที่ต้องอพยพ มันรู้แค่ว่า "ฉันมีรายชื่อคิวอยู่ในกล่อง ถ้าเกิดไฟไหม้ ฉันจะกดปุ่มเรียกคิวทั้งหมดทีเดียวจบ!"

```csharp
using System;

namespace EventDrivenAlarm
{
    // =======================================================
    // 1. ประกาศสัญญา (ใบรับรองมาตรฐานการรับมือเหตุฉุกเฉิน)
    // =======================================================
    public delegate void AlarmHandler(string dangerLevel);

    // =======================================================
    // 2. ศูนย์กระจายข่าวสาร (The Publisher)
    // =======================================================
    class Sensor
    {
        // 🌟 กล่องเก็บคิวฟังก์ชัน (จุดรับลงทะเบียน)
        public AlarmHandler OnFireAlert;

        public void TriggerFire()
        {
            Console.WriteLine("🚨 [Sensor]: ตรวจพบกลุ่มควันและความร้อนผิดปกติ!!!");
            
            // กฎเหล็ก: ก่อนจะกดปุ่มเรียกคิว ต้องเช็คก่อนว่ามีคนมาลงทะเบียนไว้ไหม (ป้องกัน NullReferenceException)
            if (OnFireAlert != null)
            {
                // กดปุ่มเรียกคิวทั้งหมดพร้อมกัน! และส่งระดับความรุนแรงไปให้ทุกคิวด้วย!
                OnFireAlert("ระดับ 5 อพยพด่วน!");
            }
            else
            {
                Console.WriteLine("  -> ไม่มีหน่วยงานไหนรับแจ้งเหตุเลย... ตึกไหม้เกรียม!");
            }
        }
    }

    // =======================================================
    // 3. หน่วยงานรับมือเหตุ (The Subscribers)
    // =======================================================
    class Program
    {
        static void CallFireStation(string lvl)
        {
            Console.ForegroundColor = ConsoleColor.Red;
            Console.WriteLine($"🚒 โทรเรียกหน่วยดับเพลิง! แจ้งภัย: {lvl}");
            Console.ResetColor();
        }

        static void TurnOnSprinklers(string lvl)
        {
            Console.ForegroundColor = ConsoleColor.Cyan;
            Console.WriteLine($"💦 เปิดระบบสปริงเกอร์ฉีดน้ำฉุกเฉินทั่วอาคาร! (ภัย {lvl})");
            Console.ResetColor();
        }

        static void EvacuateBuilding(string lvl)
        {
            Console.ForegroundColor = ConsoleColor.Yellow;
            Console.WriteLine($"🏃‍♂️ เสียงไซเรนดังหวอออ! ประกาศอพยพพนักงานทันที! ({lvl})");
            Console.ResetColor();
        }

        // =======================================================
        // EXECUTION (จุดลงทะเบียน)
        // =======================================================
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบป้องกันอัคคีภัยอัจฉริยะ (Event Broadcasting) ===\n");

            Sensor smokeDetector = new Sensor();

            // 🌟 เวทมนตร์ Multicast: นำฟังก์ชันจากหน่วยงานต่างๆ มาต่อคิวลงทะเบียน!
            smokeDetector.OnFireAlert += CallFireStation;
            smokeDetector.OnFireAlert += TurnOnSprinklers;
            smokeDetector.OnFireAlert += EvacuateBuilding;

            Console.WriteLine(">> ระบบพร้อมทำงาน... (สมมติเวลาผ่านไป 2 ชั่วโมง)\n");

            // เมื่อเซนเซอร์ถูกกระตุ้น มันจะรัน 3 ฟังก์ชันด้านบนให้เองโดยอัตโนมัติ!
            smokeDetector.TriggerFire();
            
            /* 
               ผลลัพธ์ที่จะออกมาติดกันรัวๆ:
               🚨 [Sensor]: ตรวจพบกลุ่มควันและความร้อนผิดปกติ!!!
               🚒 โทรเรียกหน่วยดับเพลิง! แจ้งภัย: ระดับ 5 อพยพด่วน!
               💦 เปิดระบบสปริงเกอร์ฉีดน้ำฉุกเฉินทั่วอาคาร! (ภัย ระดับ 5 อพยพด่วน!)
               🏃‍♂️ เสียงไซเรนดังหวอออ! ประกาศอพยพพนักงานทันที! (ระดับ 5 อพยพด่วน!)
            */
        }
    }
}
```

**สิ่งที่ได้เรียนรู้ (Architecture Lesson):**
ถ้าวันพรุ่งนี้ บริษัทต้องการ "ส่งอีเมลหา CEO" เวลาไฟไหม้ด้วย... คุณก็แค่สร้างฟังก์ชัน `SendEmailToCEO` แล้วเอาไป `+=` เข้ากล่อง `OnFireAlert` เท่านั้นเอง! 
ไม่ต้องไปยุ่ง ไม่ต้องไปแก้โค้ดในคลาส `Sensor` เลยแม้แต่บรรทัดเดียว! โคตรหล่อ โคตรงาม และโคตรยืดหยุ่น! นี่แหละความอลังการของ OOP ที่แท้ทรู!
