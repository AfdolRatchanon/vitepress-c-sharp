# Action, Func & Predicate (สามทหารเสือสำเร็จรูป)

> 💡 **เป้าหมาย:** ทิ้งความน่าเบื่อของยุคเก่า! เรียนรู้วิธีข้ามขั้นตอนการประกาศ `delegate` แบบแมนนวล ด้วยการใช้ 3 ทหารเสือครอบจักรวาล (`Action`, `Func`, `Predicate`) ที่ไมโครซอฟต์เตรียมไว้ให้ ซึ่งจะทำให้โค้ดของคุณสั้นลง อ่านง่ายขึ้น และพร้อมรับมือกับโลกของ LINQ!

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. วิกฤติการประกาศ Delegate ซ้ำซาก (The Boilerplate Problem)

จากหัวข้อที่แล้ว ถ้าคุณอยากได้ตัวแปรเก็บฟังก์ชันบวกเลข คุณต้องพิมพ์:
`public delegate int MathOp(int a, int b);`
ถ้าอยากได้ฟังก์ชันโชว์ข้อความ คุณต้องพิมพ์:
`public delegate void PrintMsg(string s);`

จินตนาการดูสิครับว่าโปรเจกต์ขนาดใหญ่ที่มีฟังก์ชัน 1,000 รูปแบบ คุณต้องมานั่งตั้งชื่อประกาศ `delegate` เป็นพันๆ บรรทัด! (รกหูรกตาที่สุด) ไมโครซอฟต์จึงออกพระราชกฤษฎีกาใหม่ว่า: *"ต่อไปนี้ เลิกตั้งชื่อ Delegate กันเองได้แล้ว! จงใช้พิมพ์เขียว 3 ตัวนี้แทน!"*

---

### 2. ⚔️ ทหารเสือคนที่ 1: `Action` (หน่วยปฏิบัติการไร้ร่องรอย)

**`Action`** เกิดมาเพื่อรับจบฟังก์ชันที่ **"ไม่มีการคืนค่า (Return `void`)"**
คุณสามารถใช้เครื่องหมายวงเล็บเหลี่ยม `< >` (Generics) เพื่อกำหนดว่าฟังก์ชันนี้ต้องรับพารามิเตอร์อะไรบ้าง (ใส่ได้สูงสุด 16 ตัว!)

- ถ้าเป็นฟังก์ชัน `void พูด()` -> ใช้ **`Action`** (ไม่มีวงเล็บเหลี่ยม)
- ถ้าเป็นฟังก์ชัน `void ปริ้น(string ข้อความ)` -> ใช้ **`Action<string>`**
- ถ้าเป็นฟังก์ชัน `void เซฟไฟล์(string ชื่อ, int ขนาด)` -> ใช้ **`Action<string, int>`**

```csharp
// แบบเก่า: delegate void MyLogger(string msg);
// แบบใหม่:
Action<string> logger = Console.WriteLine;
logger("สั้นลงเยอะเลย!");
```

---

### 3. 🛡️ ทหารเสือคนที่ 2: `Func` (นักปราชญ์ผู้คายคำตอบ)

**`Func` (ย่อมาจาก Function)** เกิดมาเพื่อรับจบฟังก์ชันที่ **"ต้องมีการคืนค่า (Return ของกลับมาเสมอ)"**
เคล็ดลับการจำ: **ชนิดข้อมูลตัวสุดท้ายใน `< >` คือ "ชนิดของสิ่งที่จะ Return" เสมอ!**

- ถ้าเป็นฟังก์ชัน `int สุ่มเลข()` -> ใช้ **`Func<int>`** (รับ 0 ตัว, ทิ้ง int 1 ก้อน)
- ถ้าเป็นฟังก์ชัน `int บวกเลข(int ก, int ข)` -> ใช้ **`Func<int, int, int>`** (รับ int 2 ตัว, ทิ้ง int 1 ก้อน)
- ถ้าเป็นฟังก์ชัน `string ประกอบคำ(string ก, int ข)` -> ใช้ **`Func<string, int, string>`**

```csharp
// แบบเก่า: delegate int MathOp(int a, int b);
// แบบใหม่:
Func<int, int, int> addFormula = Add;
int ans = addFormula(5, 10);
```

---

### 4. 🏹 ทหารเสือคนที่ 3: `Predicate` (ผู้พิพากษาแห่งความจริง)

**`Predicate`** เป็นร่างอวตารพิเศษของ `Func` มันเกิดมาเพื่อรับจบฟังก์ชันจำพวก **"ตรรกะเช็คเงื่อนไข (รับของ 1 ชิ้น แล้วตอบว่า `true` หรือ `false` เท่านั้น)"**

- ถ้าฟังก์ชันคือ `bool ตรวจสอบว่าสอบผ่านไหม(int คะแนน)` -> แทนที่จะเขียน `Func<int, bool>` เราสามารถใช้ชื่อสั้นๆ ว่า **`Predicate<int>`** แทนได้เลย! (ให้ความหมายเชิงสถาปัตยกรรมชัดเจนกว่ามาก)

```csharp
// แบบเก่า: Func<int, bool> isPass = CheckScore;
// แบบใหม่:
Predicate<int> isPass = CheckScore;
bool result = isPass(80); // ตอบ true หรือ false
```

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่าง: สถาปัตยกรรมกรองข้อมูลอัจฉริยะ (The Smart Data Filter)

นี่คือโครงสร้างที่ทรงพลังที่สุดของการใช้ Delegate สำเร็จรูป! เราจะสร้างเมธอดที่คอยวนลูปตรวจตัวเลข แต่เราจะไม่เขียนเงื่อนไข `if` ตายตัวไว้ข้างใน... เราจะ "รับเงื่อนไข (Predicate)" เข้ามาเป็นพารามิเตอร์แทน!

::: code-group
```csharp [Program.cs]
using System;
using System.Collections.Generic;

namespace BuiltInDelegates
{
    class NumberManager
    {
        // 🌟 ไฮไลต์: เมธอดนี้รับชุดตัวเลข (List) และรับ "ผู้พิพากษา (Predicate)" เข้ามา
        // ใครก็ตามที่อยากจะกรองเลข ต้องโยนฟังก์ชันที่รับ int 1 ตัว แล้วคาย bool มาให้เรา!
        public List<int> FilterNumbers(List<int> numbers, Predicate<int> condition)
        {
            List<int> passedNumbers = new List<int>();

            foreach (int num in numbers)
            {
                // โยนตัวเลขให้ผู้พิพากษาตัดสิน! ถ้าตอบ true แปลว่าผ่านเงื่อนไข
                if (condition(num) == true)
                {
                    passedNumbers.Add(num); // เก็บลงกล่อง
                }
            }

            return passedNumbers; // คืนค่าตัวเลขที่รอดชีวิตทั้งหมด
        }
    }

    class Program
    {
        // =======================================================
        // ฟังก์ชันเงื่อนไขต่างๆ (กฏหมาย 3 ฉบับ)
        // =======================================================
        static bool IsEven(int x) { return x % 2 == 0; } // เลขคู่?
        static bool IsGreaterThan50(int x) { return x > 50; } // มากกว่า 50?
        
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบตะแกรงร่อนตัวเลขอัจฉริยะ ===\n");

            List<int> rawData = new List<int> { 12, 45, 68, 91, 24, 55, 8 };
            NumberManager manager = new NumberManager();

            // 1. กรองเอาเฉพาะ "เลขคู่" (โยนกฎ IsEven เข้าไป)
            Console.WriteLine(">> ตะแกรงร่อนเลขคู่:");
            List<int> evens = manager.FilterNumbers(rawData, IsEven);
            Console.WriteLine(string.Join(", ", evens)); // ผล: 12, 68, 24, 8

            // 2. กรองเอาเฉพาะ "เลขที่มากกว่า 50" (โยนกฎ IsGreaterThan50 เข้าไป)
            Console.WriteLine("\n>> ตะแกรงร่อนเลขเกิน 50:");
            List<int> bigNums = manager.FilterNumbers(rawData, IsGreaterThan50);
            Console.WriteLine(string.Join(", ", bigNums)); // ผล: 68, 91, 55

            // เห็นไหมครับ!? โค้ดลอจิกวนลูปใน FilterNumbers ไม่ต้องถูกแก้เลย!
            // เราแค่สลับเปลี่ยน "ไส้ในของเงื่อนไข" หน้างานได้อย่างอิสระ! (Open-Closed Principle)
        }
    }
}
```
:::

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

### **โจทย์ที่ 1: วิเคราะห์สเปคทหารเสือ (Signature Matching)**
จงจับคู่ชนิดข้อมูล Built-in Delegate ต่อไปนี้ ให้เข้ากับฟังก์ชันที่กำหนดให้ (ห้ามมั่ว!):
1. `void Greet(string name) { ... }`
2. `string GetGreeting(string name) { return "Hello " + name; }`
3. `bool IsValidEmail(string email) { ... }`
4. `int GenerateRandomCode() { return 1234; }`

**ตัวเลือกที่มีให้:** 
`Action<string>`, `Func<int>`, `Predicate<string>`, `Func<string, string>`

### **โจทย์ที่ 2: จุดจบของ Func (The Last Parameter)**
มีคนประกาศตัวแปรดังนี้: `Func<int, double, string, bool> myFunc;`
**คำถาม:** ฟังก์ชันที่จะยัดลงกล่องนี้ได้ จะต้องรับพารามิเตอร์กี่ตัว ชนิดอะไรบ้างตามลำดับ? และที่สำคัญ ฟังก์ชันนั้นต้องมี `return` เป็นชนิดข้อมูลอะไร?

::: details 💡 คำใบ้และแนวทาง (Hints)
- **สำหรับโจทย์ 1:** 
  1. คู่กับ `Action<string>` (เพราะเป็น void)
  2. คู่กับ `Func<string, string>` (รับ string 1 ตัว, และคาย string ออกมา)
  3. คู่กับ `Predicate<string>` (รับ string 1 ตัว, คาย bool) *จริงๆ ใช้ Func<string, bool> ก็ได้ แต่ Predicate ดูโปรฝุดๆ*
  4. คู่กับ `Func<int>` (ไม่รับอะไรเลย, แต่คาย int 1 ก้อน)
- **สำหรับโจทย์ 2:**
  **รับพารามิเตอร์ 3 ตัวครับ!** คือ `(int, double, string)` ตามลำดับ
  และ **ต้อง Return เป็น `bool` ครับ!** (จำกฎเหล็ก: ชิ้นสุดท้ายในวงเล็บเหลี่ยมของ Func คือของที่จะถูกคายทิ้ง (Return) เสมอ! ที่เหลือด้านหน้าคือของที่กินเข้าไปทั้งหมด)
:::

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: โรงงานแปรรูปข้อความ (The String Transformer)**

**ภารกิจของคุณ:** สร้างระบบแปรรูปข้อความที่รับการเปลี่ยนแปลงกี่ตลบก็ได้!

1. ใน `Main` ประกาศตัวแปรรับค่า `string text = "  hello WORLD  ";`
2. สร้างตัวแปร Delegate จำนวน 3 ตัว (ห้ามประกาศคลาสใหม่ ให้ประกาศใน Main เลย):
   - ตัวที่ 1: ฟังก์ชันตัดช่องว่างหัวท้าย (`Trim`) และทำเป็นตัวพิมพ์เล็กทั้งหมด -> ต้องใช้ `Func` อะไร?
   - ตัวที่ 2: ฟังก์ชันนับตัวอักษร -> ต้องใช้ `Func` อะไร?
   - ตัวที่ 3: ฟังก์ชันปริ้นข้อความพร้อมขีดเส้นใต้ (`====\nข้อความ\n====`) -> ต้องใช้ `Action` อะไร?
3. เขียนฟังก์ชัน 3 ตัว (static method) ให้สอดคล้องกับสเปคด้านบน
4. นำ Delegate มาเชื่อมการทำงาน (ไม่ต้องใช้ `+=` นะครับ เอาผลลัพธ์จากตัว 1 ส่งเข้าตัว 2 แบบเป็นทอดๆ)
   - ใช้ตัวที่ 1 คลีนข้อความ (จะได้ "hello world")
   - เอาผลลัพธ์ไปปริ้นด้วยตัวที่ 3
   - เอาผลลัพธ์ไปโยนให้ตัวที่ 2 นับว่ามีกี่ตัวอักษร (ควรได้ 11) แล้วปริ้นความยาวออกมา

**เป้าหมายแฝง:** ให้คุณคุ้นชินกับการใช้สัญลักษณ์วงเล็บเหลี่ยม `< >` ในการตีกรอบ Type-Safety ก่อนที่คุณจะก้าวเข้าสู่โลกของ Generics ของจริงครับ!

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** ในโลกของมืออาชีพ เค้ายังมีการประกาศ `public delegate ...` ด้วยตัวเองอยู่ไหมครับ หรือใช้แต่ Action/Func กันหมดแล้ว?
**แนวคำตอบ:** **ใช้ Action/Func เป็นหลัก 99% เลยครับ!**
ยกเว้นกรณีเดียวจริงๆ ที่สถาปนิกจะยอมประกาศ Delegate ชื่อแปลกๆ ขึ้นมาเอง คือตอนที่มีพารามิเตอร์เยอะมากๆ (เช่น 5 ตัว)
ถ้าคุณใช้ `Func<string, int, bool, double, object, string>`... คนในทีมมาอ่านโค้ด จะด่าคุณเละเทะเลยครับ เพราะเขาจำไม่ได้ว่า bool คือตัวแทนของอะไร? int ตัวแทนของอะไร? (มันมีแต่ชนิดข้อมูล ไม่มีชื่อกำกับ)
ในกรณีแบบนี้ เขาจะยอมเสียเวลาประกาศ `public delegate string DataProcessor(string id, int age, bool isActive...);` แทนครับ เพื่อให้มี "ชื่อพารามิเตอร์" กำกับให้เพื่อนร่วมทีมอ่านเข้าใจง่ายขึ้นครับ! (แต่ถ้าแค่ 1-2 ตัว ใช้ Action/Func เถอะครับ สบายกว่าเยอะ!)

**คำถาม 2:** โค้ดด้านล่างนี้ เกิดข้อผิดพลาดอะไร ทำไมมันถึงไม่ยอมให้ใช้ Predicate?
```csharp
Predicate<int> checkScore = delegate(int score) { return score > 50; };
Func<int, bool> checkScore2 = checkScore; // 💥 พังตรงนี้! 
```
**แนวคำตอบ:** **อคิลลิสฮีล (จุดอ่อน) ของ C# ครับ! (Delegate Variance Problem)**
ถึงแม้ว่า `Predicate<int>` กับ `Func<int, bool>` จะมีรูปร่างหน้าตาสเปค "เหมือนกันเป๊ะ 100% (รับ int คาย bool)"... แต่ในสายตาของคอมไพเลอร์ C# **มันถือว่าเป็นกล่องคนละยี่ห้อกันครับ!** (เหมือนอะแดปเตอร์ไอโฟนกับซัมซุง) คุณจะเอาตัวแปรมันมายัดข้ามกันดื้อๆ ไม่ได้เด็ดขาด!
ถ้าอยากแปลงร่างจริงๆ คุณต้องใช้ความรุนแรง (Cast) หรือไม่ก็เขียนฟังก์ชันครอบมันอีกทีครับ... แต่ทางที่ดีที่สุดคือ: เลือกใช้ยี่ห้อใดยี่ห้อหนึ่งให้เป็นมาตรฐานในโปรเจกต์เดียวกันไปเลยครับ!
:::
