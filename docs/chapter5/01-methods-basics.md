# Methods Basics (พื้นฐานเมธอดและกฎ DRY)

> 💡 **เป้าหมาย:** ทำความเข้าใจปรัชญา "ห้ามทำซ้ำ (DRY)" เรียนรู้ไวยากรณ์ในการสร้างและเรียกใช้ Method พื้นฐานแบบที่ไม่มีการรับส่งข้อมูล (`void`) รวมถึงกฎการตั้งชื่อมาตรฐานสากล

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. ปรัชญา DRY (Don't Repeat Yourself)

ในวงการ Software Engineering มีหลักการสูงสุดข้อหนึ่งที่โปรแกรมเมอร์ทุกคนต้องท่องให้ขึ้นใจ นั่นคือ **DRY (Don't Repeat Yourself)** ซึ่งมีความหมายตรงข้ามกับ **WET (We Enjoy Typing - หรือ เขียนโค้ดซ้ำซาก)**

เมื่อไหร่ก็ตามที่คุณลากเมาส์คลุมโค้ด กด `Ctrl+C` แล้วไปกด `Ctrl+V` ที่อื่น นั่นคือสัญญาณอันตราย (Code Smell) ที่บอกว่าคุณกำลังแหกกฎ DRY
ผลเสียของการทำแบบนี้คือ เมื่อมีข้อผิดพลาด (Bug) คุณต้องตามไปแก้ทุกจุดที่คุณก๊อปปี้ไปวาง ซึ่งมักจะก่อให้เกิดบั๊กใหม่ตามมาเสมอ

วิธีแก้ปัญหาตามหลักวิศวกรรมคือ การหยิบโค้ดก้อนนั้นมา **"ตั้งชื่อให้มัน"** แล้วเก็บใส่แคปซูล (Encapsulation) แคปซูลนั้นเรียกว่า **Method (เมธอด)** หรือ **Function (ฟังก์ชัน)**

*(หมายเหตุ: ในภาษา C# เรามักเรียกฟังก์ชันว่า "Method" เพราะมันถูกสร้างและอาศัยอยู่ภายใน "Class" เสมอ ตามหลัก OOP)*

### 2. สถาปัตยกรรมของ Method (The Anatomy of a Method)

เพื่อให้เข้าใจได้ง่ายที่สุด เมธอดก็เหมือน **"ตู้กดน้ำอัตโนมัติ"** 
ตู้กดน้ำที่ดีต้องมีป้ายชื่อบอกชัดเจน, มีช่องหยอดเหรียญ (Input), มีกลไกภายในที่คนนอกมองไม่เห็น, และมีช่องรับกระป๋องน้ำ (Output)

มาดูไวยากรณ์การสร้าง Method แบบละเอียด (Method Signature):

```csharp
[Access Modifier] [static] [Return Type] [MethodName]([Parameters])
{
    // Body (กลไกการทำงานภายใน)
}
```

สำหรับหัวข้อพื้นฐานนี้ เราจะยังไม่ใส่เหรียญ (ไม่มี Parameter) และไม่เอากระป๋องน้ำออกมา (ไม่มี Return) เราจะโฟกัสแค่การให้มัน **"ทำงานเปล่าๆ"** โครงสร้างที่ง่ายที่สุดจึงหน้าตาแบบนี้:

```csharp
static void SayHello()
{
    Console.WriteLine("สวัสดีครับบบบ!");
}
```

มาผ่าตัดดูทีละชิ้นส่วน:
1. **`static`**: เป็นคีย์เวิร์ดบังคับ (ในเบื้องต้น) เพราะเนื่องจาก `Main()` ของเราเป็นแบบ `static` (สถิต/ไม่มีวิญญาณ) มันจึงเรียกใช้ได้เฉพาะ Method ที่เป็น `static` เหมือนกันเท่านั้น (คุณจะได้เรียนเจาะลึกเรื่องนี้ในบท OOP)
2. **`void`**: แปลว่า "ว่างเปล่า / สุญญากาศ" เป็นการบอก C# ว่า เมธอดนี้ทำงานเสร็จแล้ว **จบเลย ไม่ต้องส่งข้อมูลอะไรกลับมาให้คนเรียก** (คล้ายกับเครื่องฉายโปรเจคเตอร์ ที่ฉายภาพเฉยๆ ไม่ได้คืนเงินให้เรา)
3. **`SayHello`**: ชื่อเมธอด (Naming Convention)
4. **`()`**: วงเล็บว่างเปล่า บ่งบอกว่าไม่ต้องโยนข้อมูลอะไรเข้าไปให้มันประมวลผลทั้งสิ้น

---

### 3. Naming Convention (มารยาทการตั้งชื่อเมธอด)

ต่างจากตัวแปร (Variables) ที่เราใช้ `camelCase` (ตัวพิมพ์เล็กนำหน้า) 
ในภาษา C# ไมโครซอฟต์กำหนดมาตรฐานอุตสาหกรรมไว้ว่า **"ชื่อเมธอดต้องเป็น PascalCase เสมอ" (ตัวแรกของทุกคำต้องเป็นพิมพ์ใหญ่)**

นอกจากนี้ กฎเหล็กของการตั้งชื่อที่ดีคือ: **"ชื่อเมธอดต้องขึ้นต้นด้วย กริยา (Verb) เสมอ"** เพราะเมธอดคือการ "กระทำ"
- ✅ **ถูกต้อง:** `CalculateTax()`, `PrintReport()`, `ShowMenu()`, `GetData()`
- ❌ **ผิดมาตรฐาน:** `taxRate()` (ไม่ใช่กริยา และไม่พิมพ์ใหญ่), `Show_Menu()` (ห้ามใช้ขีดล่างในสไตล์ C#)

---

### 4. การอัญเชิญเมธอด (Method Invocation / Calling)

การสร้างเมธอดไว้เฉยๆ มันจะไม่ยอมทำงานเลยแม่แต่บรรทัดเดียว (เหมือนคุณสร้างตู้กดน้ำไว้ตั้งโชว์) 
มันจะทำงานก็ต่อเมื่อคุณ **"เรียกใช้ (Call / Invoke)"** มันเท่านั้น!

วิธีการเรียกใช้ก็แสนง่าย เพียงแค่พิมพ์ชื่อเมธอด และที่ขาดไม่ได้คือ **วงเล็บ `()` และเซมิโคลอน `;`**
```csharp
// การเรียกใช้
SayHello(); // คอมพิวเตอร์จะกระโดดไปทำโค้ดในปีกกาของ SayHello ทันที
```

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่างที่ 1: การกำจัดความซ้ำซ้อนด้วย `void` Method

โค้ดนี้เปรียบเทียบให้เห็นชัดเจนว่า การเขียนโค้ดแบบสปาเก็ตตี้ (บนลงล่าง) กับการแยกส่วน (Modular) แตกต่างกันอย่างไร

::: code-group
```csharp [Program.cs]
using System;

namespace MethodBasics
{
    class Program
    {
        // ==========================================
        // พื้นที่ส่วนการนิยาม Method (ต้องอยู่นอก Main แต่ยังอยู่ใน Class)
        // ==========================================

        // สร้าง Method สำหรับวาดหัวกระดาษ (Header)
        static void PrintCompanyHeader()
        {
            // ทุกครั้งที่ถูกเรียกใช้ มันจะพิมพ์ 3 บรรทัดนี้
            Console.WriteLine("========================================");
            Console.WriteLine("    🏢 C# ENTERPRISE SYSTEMS CO.,LTD    ");
            Console.WriteLine("========================================");
        }

        // สร้าง Method สำหรับวาดเส้นประคั่นบรรทัด
        static void DrawSeparator()
        {
            Console.WriteLine("----------------------------------------");
        }

        // ==========================================
        // จุดเริ่มต้นโปรแกรม (Entry Point)
        // ==========================================
        static void Main(string[] args)
        {
            // เรียกใช้งานครั้งที่ 1 (วาดส่วนหัวของรายงาน)
            PrintCompanyHeader();
            Console.WriteLine("รายงานสรุปยอดขายประจำเดือน");
            
            // เรียกใช้งานเส้นคั่น
            DrawSeparator();

            Console.WriteLine("ยอดขายเดือนที่ 1: 50,000 บาท");
            Console.WriteLine("ยอดขายเดือนที่ 2: 75,000 บาท");

            // เรียกใช้งานเส้นคั่นอีกครั้ง (ไม่ต้องเขียน Console.WriteLine(---) ซ้ำๆ เอง)
            DrawSeparator();

            Console.WriteLine("รายงานสรุปรายจ่ายประจำเดือน");
            
            // เรียกใช้งานครั้งที่ 2 (วาดส่วนท้ายกระดาษ ซึ่งบังเอิญใช้ Header แบบเดียวกัน)
            PrintCompanyHeader();
            
            Console.WriteLine("สิ้นสุดการทำงาน");
        }
    }
}
```
:::

**ความฉลาดทางวิศวกรรม:**
ลองจินตนาการว่า ผู้บริหารสั่งว่า "บริษัทเราเปลี่ยนชื่อเป็น C# Global Group แล้วนะ ไปแก้ทุกรายงานให้หน่อย"
ถ้าคุณไม่มี Method คุณต้องไปไล่ค้นหาทุกหน้าที่พิมพ์ชื่อบริษัทแล้วแก้ 100 รอบ 
แต่ด้วยโค้ดด้านบน... คุณแค่แก้ที่ `PrintCompanyHeader()` หน้าที่เดียว จบ! ประหยัดเวลาไปได้ 1 วันเต็มๆ!

### ตัวอย่างที่ 2: เมนูหลักของเกม (Game Menu Architecture)

ตัวอย่างนี้ผสานเอาความรู้บทที่ 4 (`do-while` และ `switch`) เข้ากับการเรียกใช้ Method เพื่อให้โค้ดใน `Main()` ดูสะอาดตา (Clean Code) มากที่สุด

::: code-group
```csharp [Program.cs]
using System;
using System.Threading;

namespace GameMenuArchitecture
{
    class Program
    {
        // ------------------------------------------
        // เมธอดเก็บรายละเอียดหน้าจอหลัก (ซ่อนความรกรุงรัง)
        // ------------------------------------------
        static void ShowMainMenu()
        {
            Console.Clear();
            Console.WriteLine("  ⚔️  THE LEGEND OF C#  ⚔️  ");
            Console.WriteLine();
            Console.WriteLine(" [1] เริ่มเกมใหม่ (New Game)");
            Console.WriteLine(" [2] โหลดเกม (Load Game)");
            Console.WriteLine(" [3] การตั้งค่า (Settings)");
            Console.WriteLine(" [9] ออกจากเกม (Exit)");
            Console.WriteLine();
            Console.Write("กรุณาเลือกเมนู: ");
        }

        // ------------------------------------------
        // เมธอดจำลองการโหลดเกม (โชว์จุดไข่ปลาวิ่งๆ)
        // ------------------------------------------
        static void PlayLoadingAnimation()
        {
            Console.Write("ระบบกำลังโหลดข้อมูล ");
            for (int i = 0; i < 5; i++)
            {
                Console.Write(".");
                Thread.Sleep(300); // ดีเลย์ 0.3 วินาที (หลอกว่ากำลังโหลดหนัก)
            }
            Console.WriteLine("\n[สำเร็จ] ข้อมูลพร้อมใช้งาน!");
        }

        // ------------------------------------------
        // หัวใจหลักของโปรแกรม
        // ------------------------------------------
        static void Main(string[] args)
        {
            string choice;

            do
            {
                // เรียกใช้หน้าจอเมนู (โค้ด Main ดูสั้นกระชับมาก!)
                ShowMainMenu();
                choice = Console.ReadLine();

                Console.WriteLine(); // เคาะเว้นบรรทัด

                switch (choice)
                {
                    case "1":
                        // เรียกใช้ Animation การโหลด
                        PlayLoadingAnimation();
                        Console.WriteLine(">>> ยินดีต้อนรับสู่โลกใบใหม่ อัศวินน้อย!");
                        break;
                    case "2":
                        PlayLoadingAnimation();
                        Console.WriteLine(">>> ยินดีต้อนรับกลับสู่จุดเซฟล่าสุด!");
                        break;
                    case "3":
                        Console.WriteLine(">>> เข้าสู่หน้าจอการตั้งค่ากราฟิกและเสียง");
                        break;
                    case "9":
                        Console.WriteLine(">>> ขอบคุณที่เล่นเกมของเรา แล้วพบกันใหม่!");
                        break;
                    default:
                        Console.WriteLine(">>> กรุณากดเฉพาะตัวเลขที่กำหนดเท่านั้น!");
                        break;
                }

                // ถ้าไม่ได้กด 9 ก็หยุดรอให้ผู้ใช้อ่านข้อความก่อนเคลียร์จอทิ้ง
                if (choice != "9")
                {
                    Console.WriteLine("\n(กด Enter เพื่อกลับไปหน้าเมนูหลัก)");
                    Console.ReadLine();
                }

            } while (choice != "9");
        }
    }
}
```
:::

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

### **โจทย์ที่ 1: การสะกดคำผิด (Method Call Syntax)**
พิจารณาโค้ดการสร้างและเรียกใช้ Method ด้านล่างนี้ มีจุดผิดพลาดทางไวยากรณ์ (Syntax Error) หรือผิดธรรมเนียม (Convention) ซ่อนอยู่อย่างน้อย 3 จุด จงค้นหาและแก้ไขให้ถูกต้องตามหลักสากลของ C#

```csharp
class Program
{
    static void draw_Line()
    {
        Console.WriteLine("==================");
    }

    static void Main(string[] args)
    {
        draw_Line;
        Console.WriteLine("ข้อมูลรายงาน");
        Draw_line();
    }
}
```

### **โจทย์ที่ 2: สถาปนิกจัดระเบียบห้อง (Code Refactoring)**
คุณได้รับโค้ดดิบๆ จากนักศึกษาฝึกงาน:
```csharp
static void Main(string[] args)
{
    Console.WriteLine("กำลังเชื่อมต่อ Database...");
    Console.WriteLine("ดึงข้อมูลสำเร็จ");
    
    Console.WriteLine("ชื่อ: John");
    Console.WriteLine("นามสกุล: Doe");
    
    Console.WriteLine("ชื่อ: Jane");
    Console.WriteLine("นามสกุล: Smith");
    
    Console.WriteLine("ปิดการเชื่อมต่อ Database.");
}
```
**ภารกิจ:** ให้คุณยุบรวมโค้ดที่มีหน้าตาเหมือนกัน (ในที่นี้คือการพิมพ์ชื่อและนามสกุล) ออกไปสร้างเป็น Method `static void PrintPersonInfo()` (ไม่ต้องรับตัวแปรใดๆ ให้ปรินท์ชื่อตายตัวไปก่อน) แล้วนำกลับมาเรียกใช้ใน `Main` ให้สั้นและสะอาดที่สุด

::: details 💡 คำใบ้และแนวทาง (Hints)
- **สำหรับโจทย์ 1:** (1) ชื่อเมธอดผิดมาตรฐาน PascalCase แถมมีขีดล่าง (2) การเรียกใช้เมธอดรอบแรกขาดอะไรไป? วงเล็บ `()` ไงล่ะ! (3) การเรียกใช้รอบสองสะกดตัวพิมพ์ใหญ่พิมพ์เล็กไม่ตรงกับต้นฉบับ C# เป็นภาษา Case-Sensitive นะ!
- **สำหรับโจทย์ 2:** ดึงเอาโค้ดพิมพ์ชื่อ John ออกมาสร้างเมธอด `PrintJohnInfo()` และ `PrintJaneInfo()` หรือจะยุบรวมเป็นเมธอดเดียวก็ยังได้ แต่เนื่องจากเรายังไม่ได้เรียนเรื่อง "การส่งข้อมูล (Parameter)" การสร้างเมธอดแยกเฉพาะคนถือเป็นวิธีแก้ปัญหาเบื้องต้นที่ถูกต้องในบริบทนี้
:::

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: คอนเสิร์ตตัวอักษร (The ASCII Art Concert)**

ศิลปะ ASCII (ASCII Art) คือการเอาสัญลักษณ์บนคีย์บอร์ดมาเรียงกันเป็นรูปภาพ
คุณได้รับคำสั่งให้สร้างโปรแกรมแสดงคอนเสิร์ตวงดนตรี
1. ให้คุณออกแบบ ASCII Art ง่ายๆ 3 รูป (ห้ามก๊อปปี้จากเน็ต พิมพ์เองเลย) เช่น รูปกีตาร์, รูปไมโครโฟน, รูปกลอง
2. ให้แยกเอาโค้ด `Console.WriteLine` สำหรับวาดรูปแต่ละรูป ไปสร้างเป็น `static void` Method เช่น `DrawGuitar()`, `DrawMic()`, `DrawDrums()`
3. ใน `Main()` ให้เขียนโปรแกรมจำลองคอนเสิร์ต
   - โชว์กีตาร์
   - หน่วงเวลา (ใช้ `Thread.Sleep(1000);`)
   - ล้างหน้าจอ `Console.Clear();`
   - โชว์ไมโครโฟน
   - วนลูปโชว์สลับกันสัก 2-3 รอบเพื่อจำลอง Animation!

การทำโจทย์ข้อนี้จะทำให้คุณเข้าใจแจ่มแจ้งว่า Method ช่วยห่อหุ้ม (Encapsulate) ความรกรุงรัง (อย่างเช่น Console.WriteLine 10 บรรทัดที่วาดรูปกีตาร์) ให้เหลือเพียงการเรียกใช้งานแค่ 1 บรรทัด `DrawGuitar();` ที่อ่านง่ายสบายตา!

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** ในมุมมองของการจัดสรรหน่วยความจำ (Memory Allocation) ทุกครั้งที่เราเรียกใช้เมธอด เช่น `SayHello()` คอมพิวเตอร์จะทำอะไรเบื้องหลัง? และมันจะกลับมาทำงานที่ `Main()` ต่อได้อย่างไร?
**แนวคำตอบ:** สิ่งที่เกิดขึ้นคือการทำงานของ **"Call Stack"** 
เมื่อรันโปรแกรม `Main()` จะเป็นฐานรากของ Stack เมื่อโค้ดเดินมาถึงคำว่า `SayHello()` CPU จะ "จดจำบรรทัดปัจจุบันของ Main ไว้ (Return Address)" แล้วนำเมธอด `SayHello()` วางซ้อนทับขึ้นไป (Push) และกระโดดไปรันโค้ดในนั้น 
เมื่อ `SayHello()` ทำงานจบบรรทัดสุดท้าย (เจอเครื่องหมาย `}`) มันจะทำลายตัวเองทิ้ง (Pop) แล้วดึงความจำที่จดไว้กลับมา เพื่อกระโดดกลับมาทำงานใน `Main()` ที่บรรทัดถัดไปอย่างแม่นยำ! 
*(นี่คือกลไกพื้นฐานที่สุดของคอมพิวเตอร์ทุกเครื่องบนโลก!)*

**คำถาม 2:** โค้ดต่อไปนี้สามารถ Compile ผ่านหรือไม่? เพราะเหตุใด?
```csharp
class Program
{
    static void Main(string[] args)
    {
        PrintLogo();
    }

    // เมธอดถูกประกาศไว้ "ด้านล่าง" ของ Main!
    static void PrintLogo()
    {
        Console.WriteLine("C#");
    }
}
```
**แนวคำตอบ:** **ผ่าน 100%!** ภาษา C# ต่างจากภาษาสมัยโบราณ (เช่น C หรือ C++ รุ่นเก่า) ที่บังคับว่าต้องประการฟังก์ชันไว้ "ก่อน (ด้านบน)" ที่จะเรียกใช้เสมอ (Forward Declaration) 
สำหรับ C# คอมไพเลอร์จะฉลาดพอที่จะกวาดสายตาอ่านทั้ง Class ให้เสร็จสิ้นก่อน 1 รอบ เพื่อจดจำชื่อ Method ทั้งหมด แล้วค่อยลงมือเชื่อมโยง (Link) การเรียกใช้งานเข้าด้วยกัน ดังนั้นคุณจะวาง Method ไว้บรรทัดแรก บรรทัดสุดท้าย หรือสลับหน้าหลังกัน มันก็เรียกหากันเจอเสมอ (ตราบใดที่อยู่ในระดับ Access Modifier เดียวกัน)
