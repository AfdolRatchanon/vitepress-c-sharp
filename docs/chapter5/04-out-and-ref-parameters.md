# `out` & `ref` Parameters (ข้ามขีดจำกัดหน่วยความจำ)

> 💡 **เป้าหมาย:** ทลายกฎกติกาความปลอดภัยสูงสุดของ C# ที่ชื่อว่า "Pass by Value (ก๊อปปี้ข้อมูลส่งต่อ)" เปลี่ยนมาใช้ "Pass by Reference (ส่งกุญแจห้องให้ไปไขถึงที่)" เพื่อให้ Method สามารถดัดแปลงแก้ไขตัวแปรต้นฉบับได้โดยตรง และสามารถคืนค่ากลับมาได้มากกว่า 1 ตัว!

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. วิกฤติของ Pass by Value (Copying Limitation)

โดยปกติ (Default) ภาษา C# จะใช้ระบบที่เรียกว่า **Pass by Value**
เมื่อคุณโยนตัวแปรเข้าไปใน Method คอมพิวเตอร์จะแอบเอาข้อมูลไปถ่ายเอกสาร แล้วส่งกระดาษใบนั้นให้ Method ไปใช้งาน 
ไม่ว่า Method จะฉีกกระดาษนั้นทิ้ง หรือเขียนค่าใหม่ทับลงไป **ตัวแปรต้นฉบับใน Main จะไม่มีวันเปลี่ยนแปลง!**

```text
แผนภาพจำลอง (Pass by Value)

[ Main Method ]                    [ SomeMethod(int x) ]
 int age = 20; ────────(copy)───────> int x = 20;
                                          │
                                          ▼
                                      x = 99; (เปลี่ยนค่า)
                                      
(ผลลัพธ์: กลับมาที่ Main ค่า age ก็ยังคงเป็น 20 อยู่ดี เพราะ x เป็นแค่ร่างโคลน)
```

แต่ถ้าเรา **"อยากให้ Method ช่วยแก้ไขตัวแปรต้นฉบับให้เราล่ะ?"** (เช่น ระบบอัปเกรดไอเทมในเกม หรือระบบสลับไพ่) เราจะทำอย่างไร? 
พระเอกที่จะมาแก้ปัญหานี้คือคีย์เวิร์ด `ref` และ `out`!

---

### 2. การเปิดประตูหลังด้วย `ref` (Reference Parameter)

เมื่อคุณวางคำว่า **`ref`** ไว้หน้าพารามิเตอร์ มันคือการบอก C# ว่า "ไม่ต้องถ่ายเอกสารนะ ให้โยน **พิกัดที่อยู่ (Memory Address)** ไปให้ Method เลย!" 
เปรียบเสมือนคุณส่ง "กุญแจบ้าน" ให้เพื่อน เพื่อนสามารถเปิดเข้าบ้านคุณและหยิบของเปลี่ยนของได้โดยตรง (Two-way communication)

```text
แผนภาพจำลอง (Pass by Reference)

[ Main Method ]                    [ SomeMethod(ref int x) ]
 int age = 20; <──────(สายยาง)──────> ref int x 
    ▲                                     │
    └─────────────────────────────────────┘
          (x กับ age มองทะลุไปที่กล่อง RAM ใบเดียวกันเป๊ะ!)
          (ดังนั้น ถ้าสั่ง x = 99; ค่าของ age จะกลายเป็น 99 ทันที!)
```

**กฎเหล็กของ `ref`:**
ตัวแปรที่จะส่งผ่าน `ref` ได้ **"ต้องมีค่าเริ่มต้นมาก่อนเสมอ"** คุณจะสร้างกล่องเปล่าๆ แล้วส่ง `ref` ไปไม่ได้เด็ดขาด เพราะ C# กลัวว่าฟังก์ชันปลายทางอาจจะเอาข้อมูลมาอ่านก่อนที่จะเขียนทับ (การอ่านของจากกล่องเปล่าทำให้แครชได้)

---

### 3. การรีเทิร์นหลายค่าด้วย `out` (Output Parameter)

ข้อจำกัดอย่างหนึ่งของคำสั่ง `return` คือมันสามารถโยนของกลับมาได้ **"แค่ 1 ก้อนเท่านั้น"** 
แล้วถ้าเราอยากให้ฟังก์ชันหนึ่งคำนวณทั้ง "ผลบวก" และ "ผลคูณ" ส่งกลับมาพร้อมกัน 2 ตัวล่ะ?

คำว่า **`out`** ทำงานเหมือน `ref` ทุกประการ (เชื่อมต่อทะลุหน่วยความจำ) แต่มีจุดเด่นคือมันออกแบบมาเพื่อ **"โยนของกลับออกมา (Output) โดยเฉพาะ"**

**ความแตกต่างสุดขั้วระหว่าง `ref` และ `out`:**
1. **ตัวแปรที่ส่ง `out` ไม่จำเป็นต้องมีค่าเริ่มต้น:** ปล่อยกล่องให้ว่างเปล่าใน Main ไว้ได้เลย 
2. **Method บังคับต้องเขียนค่าใส่กล่อง `out`:** ก่อนที่ Method จะตาย (จบปีกกา) กฎของ C# บังคับว่าคุณต้องใส่ค่าลงไปในตัวแปร `out` ให้เรียบร้อย ไม่งั้นจะไม่ยอมให้ Compile ผ่านเด็ดขาด! (เป็นการการันตีว่าคนเรียก จะไม่มีวันได้กล่องว่างๆ กลับมา)

*(ปล. นี่คือกลไกเบื้องหลังของ `int.TryParse(input, out number)` ที่คุณเคยใช้มาตลอดนั่นเอง!)*

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่างที่ 1: ระบบเติมเลือดฉุกเฉิน (The `ref` Healing System)

โค้ดนี้พิสูจน์ให้เห็นว่า `ref` สามารถทะลวงขอบเขต (Scope) กลับมาแก้ค่าใน `Main` ได้จริง

::: code-group
```csharp [Program.cs]
using System;

namespace RefDemo
{
    class Program
    {
        // สร้างเมธอดธรรมดา (Pass by value)
        static void FakeHeal(int hp)
        {
            hp = hp + 50; // รักษาเสร็จแล้ว แต่ได้ผลแค่กับร่างโคลน
        }

        // สร้างเมธอดเทพ (Pass by reference)
        static void RealHeal(ref int hp)
        {
            hp = hp + 50; // วิ่งตามสายยางกลับไปบวกเลือดที่ตัวหลักโดยตรง!
        }

        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบรักษาพยาบาล (Healing Simulator) ===");
            
            // กฎของ ref: ตัวแปรต้องมีค่าตั้งต้นเสมอ
            int playerHealth = 10; 
            Console.WriteLine($"[ก่อนรักษา] พลังชีวิตฮีโร่: {playerHealth}");

            // 1. ลองใช้การฮีลแบบปลอม (Pass by value)
            FakeHeal(playerHealth);
            Console.WriteLine($"[หลัง FakeHeal] พลังชีวิตฮีโร่: {playerHealth} (ไม่เพิ่มขึ้นเลย!)");

            // 2. ลองใช้การฮีลของจริง (Pass by reference)
            // ⚠️ ต้องมีคำว่า ref กำกับไว้ตอนเรียกใช้ด้วย เพื่อเตือนสติตัวเองว่า "ของสิ่งนี้จะโดนแก้ไขนะ!"
            RealHeal(ref playerHealth);
            Console.WriteLine($"[หลัง RealHeal] พลังชีวิตฮีโร่: {playerHealth} (เพิ่มขึ้นสำเร็จ!)");
        }
    }
}
```
:::

### ตัวอย่างที่ 2: เครื่องมือคำนวณสถิติรวบยอด (The Multiple Outputs with `out`)

โค้ดนี้สาธิตวิธีส่งข้อมูล 3 ค่ากลับออกมาจากเมธอดเดียว โดยไม่ต้องสร้าง Object หรือ Array ให้วุ่นวาย

::: code-group
```csharp [Program.cs]
using System;

namespace OutDemo
{
    class Program
    {
        // -----------------------------------------------------------
        // เมธอดนี้รับตัวเลข 2 ตัว และคืนค่ากลับมา 4 อย่างพร้อมกัน!
        // (1 ตัวคืนผ่านการ return ปกติ, อีก 3 ตัวคืนผ่านท่อ out)
        // -----------------------------------------------------------
        static bool AnalyzeNumbers(double num1, double num2, out double sum, out double diff, out double prod)
        {
            // กฎของ out: เราถูกบังคับให้ต้องกำหนดค่าให้ทั้ง 3 ตัวนี้ให้ครบก่อนถึงปีกกาปิด!
            sum = num1 + num2;
            diff = Math.Abs(num1 - num2); // หาระยะห่างแบบค่าสัมบูรณ์
            prod = num1 * num2;

            // ส่งคืนความจริงว่าการคำนวณเสร็จสิ้นด้วยดี (เป็นผลลัพธ์ตัวที่ 4)
            return true;
        }

        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบวิเคราะห์คณิตศาสตร์ขั้นสูง ===");

            double val1 = 15.5;
            double val2 = 4.5;

            // 1. ไม่ต้องกำหนดค่าเริ่มต้นให้ตัวแปร out ก็ได้ (สามารถประกาสตัวแปรแทรกกลางวงเล็บได้เลย! - ฟีเจอร์ของ C# ยุคใหม่)
            bool isSuccess = AnalyzeNumbers(val1, val2, out double finalSum, out double finalDiff, out double finalProd);

            // 2. ดึงค่าจากกล่อง out ที่ถูกเมธอดข้างบนยัดของใส่มาให้แล้ว มาใช้งาน
            if (isSuccess)
            {
                Console.WriteLine($"ข้อมูลทดสอบ: A = {val1}, B = {val2}");
                Console.WriteLine("-----------------------------");
                Console.WriteLine($"ผลบวก (Sum)       : {finalSum}");
                Console.WriteLine($"ผลต่าง (Diff)     : {finalDiff}");
                Console.WriteLine($"ผลคูณ (Product)   : {finalProd}");
            }
        }
    }
}
```
:::

**ความรู้เสริม (Pro Tip):** ใน C# 7.0 เป็นต้นมา เราไม่จำเป็นต้องประกาศกล่องตัวแปรรอไว้ข้างนอกอีกต่อไป เราสามารถพิมพ์ `out double finalSum` โผล่พรวดขึ้นมากลางวงเล็บตอนเรียกฟังก์ชันได้เลย (Inline out variable declaration) ซึ่งทำให้โค้ดสั้นและสะอาดขึ้นมาก!

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: การสลับร่างขั้นเทพ (The Ultimate Variable Swapper)**

ในบทที่ 2 เราเคยเขียนโค้ด "สลับค่าตัวแปร (Variable Swapping)" โดยต้องสร้างตัวแปรชั่วคราว (Temp) มากักเก็บข้อมูลไว้ แต่นั่นเราทำใน Main ตรงๆ

**ภารกิจของคุณ:** 
1. ให้สร้างเมธอดชื่อ `SwapText( ... )` ขึ้นมา
2. เมธอดนี้ต้องสามารถรับตัวแปร `string` 2 ตัว เข้ามาสลับค่ากันอย่างถาวร! 
3. เมื่อทำงานจบ และกลับมาที่ Main... สิ่งที่เคยอยู่ในตัวแปร A ต้องย้ายไปอยู่ B และสิ่งที่อยู่ใน B ต้องย้ายไปอยู่ A 

*(คำใบ้ที่ 1: การโยนตัวแปรเข้าไปให้ฟังก์ชันทำอะไรแบบนี้ จะใช้ Pass by Value หรือ Pass by Reference ล่ะ?)*
*(คำใบ้ที่ 2: ภายในเมธอด คุณยังคงต้องใช้เทคนิค แก้วน้ำใบที่สาม (temp) เหมือนเดิม)*

นี่คือการพิสูจน์ว่าคุณเข้าใจเรื่อง Pointer เชิงคอนเซปต์ของหน่วยความจำ (Memory Address) แล้วอย่างแท้จริง!

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** ในเมื่อ `out` ทำหน้าที่เชื่อมหน่วยความจำและส่งของกลับมาได้ ทำไมภาษา C# ถึงไม่ใช้ `ref` ตัวเดียวรวบยอดไปเลย (เพราะ `ref` ก็แก้ค่ากลับมาได้เหมือนกัน)? 
**แนวคำตอบ:** เป็นเรื่องของ **"ความมั่นใจและความปลอดภัย (Safety Guarantees)"** 
- ถ้าเป็น `ref` คนสร้างฟังก์ชันอาจจะ "ลืม" อัปเดตค่าให้ก็ได้ (ไม่บังคับ) และคนเรียกใช้ก็ต้อง "เสียเวลาเอาขยะ (ค่าเริ่มต้น) ไปยัดใส่กล่อง" ทั้งๆ ที่เดี๋ยวก็โดนฟังก์ชันทับอยู่ดี
- แต่ `out` ถูกสร้างมาเพื่อเป็น "คำมั่นสัญญาของคอมไพเลอร์" ว่า **กล่องนี้จะถูกฟังก์ชันเติมข้อมูลใหม่เอี่ยมอ่องลงไปให้เสมอ 100% ไม่มีทางได้กล่องเปล่าๆ กลับมาเด็ดขาด** การแยก `out` ออกมา ทำให้คอมไพเลอร์ช่วยด่าโปรแกรมเมอร์ได้ทันที ถ้าลืมใส่ค่า ซึ่งช่วยลดบั๊กได้มหาศาล

**คำถาม 2:** ถ้าเราโยน "อาเรย์ (Array)" หรือ "ออบเจกต์ (Object / Class)" เข้าไปใน Method ธรรมดา (โดยไม่ใส่คำว่า `ref` หรือ `out`) ค่าข้างในมันจะโดนเปลี่ยนกลับมายัง `Main` หรือไม่?
**แนวคำตอบ:** **เปลี่ยนครับ!! (เรื่องนี้สำคัญมากและเป็นกับดักของมือใหม่)** 
ชนิดข้อมูลใน C# แบ่งเป็น Value Type (เช่น int, double, bool) และ Reference Type (เช่น string, array, class) 
สำหรับกลุ่ม Value Type มันจะถูก Copy จริงๆ และไม่ส่งผลกลับมา (ตามที่เรียนในบทนี้) 
**แต่สำหรับกลุ่ม Reference Type** สิ่งที่ C# ก๊อปปี้ส่งไปให้ฟังก์ชัน มันคือ "กุญแจรีโมท (Pointer Address)" ที่ชี้ไปหากล่องบ้านเดียวกันใน Heap Memory (ถึงแม้คุณจะไม่ได้พิมพ์คำว่า ref ก็ตาม!) ดังนั้นถ้าฟังก์ชันเอาอาร์เรย์ไปสับเปลี่ยนลำดับ มันจะส่งผลกระทบไปถึงอาร์เรย์ต้นฉบับใน Main โดยตรงทันที! (จะเจาะลึกเรื่องนี้ในบทของ Arrays และ OOP)
:::
