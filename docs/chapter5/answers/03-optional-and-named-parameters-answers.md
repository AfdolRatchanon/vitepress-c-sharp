# เฉลย: Optional & Named Parameters (ยืดหยุ่นกว่าเดิม)

## 🎯 เฉลย Mini Exercise

### **โจทย์ที่ 1: กฎเหล็กแห่งพารามิเตอร์**
**โจทย์พิจารณาความผิดพลาด:**
1. `static void RegisterUser(string name, int age = 18, string role = "Guest")`
2. `static void ProcessData(int limit = 100, string filterQuery)`
3. `static void SetupDevice(string ipAddress, int port = 8080, bool enableWifi)`

**การวิเคราะห์จุดบอดจากสายตาของ Compiler:**
กฎข้อเดียวในระบบสุริยะจักรวาลของ Optional Parameter คือ: **"พารามิเตอร์ที่มีค่าเริ่มต้น (มีเครื่องหมาย =) จะต้องถูกผลักไปอยู่ด้านขวาสุดของวงเล็บเสมอ ห้ามมีพารามิเตอร์ปกติ (Required) ใดๆ มาขวางกั้นอยู่ข้างหลังมันเด็ดขาด!"**

1. **ถูกต้อง 100% (Pass):** เริ่มต้นด้วยตัวบังคับ `name` (ไม่มีค่าเริ่มต้น) จากนั้นตามด้วยตัวที่เป็น Optional ทั้งคู่ (`age` และ `role`) วางเรียงกองอยู่ด้านขวาสุด ถูกต้องตามกฎเกณฑ์ทุกประการ
2. **ผิดพลาดพินาศ (Compile Error):** ทะลึ่งเอา `limit = 100` มาวางไว้หน้าสุด แล้วตามด้วยตัวบังคับ `filterQuery` (ไม่มีค่าเริ่มต้น) 
   - ลองจินตนาการว่าถ้าคนเรียก พิมพ์คำสั่ง: `ProcessData("SELECT * FROM TABLE");` 
   - คอมไพเลอร์จะช็อก! มันไม่รู้ว่า ข้อความนั้น ควรจะเอาไปทับใส่ตัวแปรตัวแรก (limit) ซึ่งชนิดข้อมูลก็ไม่ตรงกัน หรือควรจะข้ามไปใส่ช่อง filterQuery ดี? ระบบจึงสั่งพังทันทีเพื่อความปลอดภัย วิธีแก้คือ สลับตำแหน่งเป็น `(string filterQuery, int limit = 100)`
3. **ผิดพลาดเช่นกัน (Compile Error):** มีตัวบังคับ `ipAddress` นำหน้า อันนี้ถูกต้อง แต่พอตามด้วย `port = 8080` (ที่เป็น Optional) ดันถูกปิดท้ายอีกครั้งด้วย `enableWifi` (ที่เป็น Required ตัวบังคับ) ทำให้มันผิดกฎทันที! วิธีแก้คือดึงตัวบังคับให้ไปกระจุกอยู่ข้างซ้ายให้หมด เป็น `(string ipAddress, bool enableWifi, int port = 8080)`

### **โจทย์ที่ 2: ร้านกาแฟมักง่าย (The Lazy Barista)**
**เป้าหมาย:** สร้างเมธอดรับออร์เดอร์กาแฟ ที่มีค่าดีฟอลต์เป็น "หวานปกติ" และ "เย็น"

**เฉลยและวิธีทำ:**
โจทย์ข้อนี้เป็นการออกแบบ API (Application Programming Interface) ที่ดีเยี่ยม เพราะลูกค้า 80% เข้าร้านมาสั่ง "ลาเต้" คำเดียว บาริสต้าก็ต้องรู้ใจว่ามันคือลาเต้เย็นหวานปกติ ไม่ต้องถามเซ้าซี้ให้รำคาญ

```csharp
using System;

class Program
{
    // ----------------------------------------------------
    // สร้างเมธอด: กำหนดค่า Default ไว้ที่ตัวแปรด้านขวาสองตัว
    // ----------------------------------------------------
    static void OrderCoffee(string menuName, string sweetness = "หวานปกติ", bool isHot = false)
    {
        // แปลงสถานะบูลีน (bool) ออกมาเป็นคำอ่านภาษาไทยด้วย Ternary Operator
        string temperatureText = isHot ? "ร้อน" : "เย็น";
        
        Console.WriteLine($"[ระบบหลังร้าน] รับออเดอร์: {menuName} | สูตร: {temperatureText} | ระดับความหวาน: {sweetness}");
    }

    static void Main(string[] args)
    {
        Console.WriteLine("=== ระบบบาริสต้าอัตโนมัติ ===");
        
        // 1. ลองสั่งแบบลูกค้าขี้เกียจ (โยนไปแค่ Required parameter)
        // ระบบจะแอบเอา "หวานปกติ" และ false ยัดใส่ให้เอง
        OrderCoffee("Latte"); 

        // 2. ลองสั่งแบบเจาะจงระดับความหวาน (แต่ไม่ได้บอกว่าร้อนเย็น)
        // ระบบจะเอา "หวานน้อย" ไปทับตำแหน่งที่ 2 และตำแหน่งที่ 3 ยังคงเป็น false (เย็น)
        OrderCoffee("Americano", "หวานน้อย");

        // 3. (แถม) ลองสั่งแบบลูกค้าเรื่องมาก (เจาะจงสถานะร้อน แต่ไม่ระบุความหวาน)
        // เราต้องใช้ท่าไม้ตาย "Named Argument" กระโดดข้ามพารามิเตอร์ตรงกลางไป!
        OrderCoffee("Espresso", isHot: true);
    }
}
```

## 🔥 เฉลย Challenge (ระบบสร้างโพรไฟล์ตัวละคร RPG)

**เป้าหมาย:** ใช้ Optional Parameter 4 ตัว เพื่ออำนวยความสะดวกในการสร้างตัวละคร และใช้ Named Parameter ทะลวงเข้าไปเซ็ตค่าเฉพาะบางจุด

**กระบวนการคิดของ Game Developer:**
ระบบในเกมจริงๆ มีตัวแปรมหาศาล (หน้าตา, ทรงผม, สีตา) การจะบังคับให้ผู้เล่นกรอกครบ 100 ช่องก่อนเริ่มเกมเป็นเรื่องน่าเบื่อ การทำ Default Parameter คือทางออกที่เกมระดับ AAA ทุกเกมใช้งาน

```csharp
using System;

namespace RPGCharacterBuilder
{
    class Program
    {
        // ========================================================
        // ENGINE: Character Creator (พร้อมค่าเริ่มต้นมหาศาล)
        // ========================================================
        static void CreateCharacter(
            string heroName,               // ตัวนี้ไม่มี = แปลว่า "บังคับส่ง" เสมอ
            string jobClass = "Novice",    // อาชีพ
            int hp = 100,                  // เลือดพื้นฐาน
            string weapon = "มีดสั้น",        // อาวุธโง่ๆ
            bool hasMagic = false)         // เวทมนตร์ไม่มี
        {
            Console.WriteLine("\n==================================");
            Console.WriteLine($"⚔️ PROFILE: {heroName.ToUpper()}");
            Console.WriteLine("==================================");
            Console.WriteLine($"> อาชีพ: {jobClass}");
            Console.WriteLine($"> พลังชีวิต: {hp} HP");
            Console.WriteLine($"> อาวุธประจำกาย: {weapon}");
            
            // โชว์ข้อความให้ดูอลังการขึ้น
            string magicSkill = hasMagic ? "ปลดล็อคแล้ว (ใช้งานได้)" : "ยังไม่ปลดล็อค";
            Console.WriteLine($"> พลังเวทมนตร์: {magicSkill}");
        }

        // ========================================================
        // MAIN: การเรียกใช้งานในรูปแบบต่างๆ (Client usage)
        // ========================================================
        static void Main(string[] args)
        {
            Console.WriteLine("=== THE FANTASY CHARACTER CREATOR ===");

            // ตัวละครที่ 1: Arthur คนขี้เกียจ (ให้ระบบจัดค่าเริ่มต้นให้หมด)
            // เราโยนไปแค่ Required Parameter อันเดียว
            CreateCharacter("Arthur");

            // ตัวละครที่ 2: Merlin พ่อมด (ต้องการเปลี่ยนแค่อาชีพ กับ เปิดโหมดเวทมนตร์)
            // เราใช้ Named Argument กระโดดข้าม hp และ weapon ไปเลย
            CreateCharacter("Merlin", jobClass: "Mage", hasMagic: true);

            // ตัวละครที่ 3: Leon นักดาบถึกทน (ต้องการเปลี่ยนอาวุธ กับ HP)
            // ข้อดีของ Named Argument คือ "สลับลำดับ" ได้ตามใจชอบด้วยนะ!
            CreateCharacter("Leon", hp: 500, weapon: "ดาบยักษ์");
            
            // แถม: ตัวละครที่ 4: นักธนู (เปลี่ยนทุกอย่าง ไม่ใช้ค่าดีฟอลต์เลยสักอัน)
            // กรณีนี้ไม่ต้องมีชื่อตัวแปร (Named Argument) ก็ได้ ถ้ายอมกรอกเรียงตามลำดับเป๊ะๆ
            CreateCharacter("Robin", "Archer", 150, "ธนูยาว", false);
        }
    }
}
```

**ผลลัพธ์การสร้าง Merlin:**
```text
==================================
⚔️ PROFILE: MERLIN
==================================
> อาชีพ: Mage
> พลังชีวิต: 100 HP  <--- สังเกตว่าระบบดึงค่า Default มาแปะให้เองอย่างแม่นยำ
> อาวุธประจำกาย: มีดสั้น <--- สังเกตว่าระบบดึงค่า Default มาแปะให้เองอย่างแม่นยำ
> พลังเวทมนตร์: ปลดล็อคแล้ว (ใช้งานได้)
```

**บทสรุปทางสถาปัตยกรรม:**
คุณจะเห็นว่า ตัวเมธอด `CreateCharacter` ของเราแข็งแกร่งและยืดหยุ่นมากๆ ไม่ว่าคนเรียกจะอยากได้ความละเอียดระดับไหน (ปรับทุกฟังก์ชัน หรือ ปล่อยเบลอ) เมธอดนี้ก็รองรับได้หมด นี่คือคุณสมบัติของโค้ดที่เรียกว่า **"Developer-Friendly API"** ที่บริษัทเทคโนโลยีระดับโลกคาดหวังจากโปรแกรมเมอร์ของพวกเขาครับ!
