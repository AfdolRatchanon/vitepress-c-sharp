# เฉลย: Parameters & Return Values (การรับ-ส่งข้อมูล)

## 🎯 เฉลย Mini Exercise

### **โจทย์ที่ 1: ผ่าตัดโค้ดหาค่าเฉลี่ย (The Average Fixer)**
**โจทย์ที่ให้มา:**
```csharp
static void GetAverage(int num1, num2, int num3) // บรรทัดที่ 1
{
    int total = num1 + num2 + num3; // บรรทัดที่ 3
    double avg = total / 3.0;       // บรรทัดที่ 4
    return avg;                     // บรรทัดที่ 5
}
```

**การวิเคราะห์สาเหตุและจุดผิด:**
โค้ดนี้ถูกเขียนด้วยลอจิกคณิตศาสตร์ที่ถูกต้อง แต่พังพินาศทางด้านไวยากรณ์ (Syntax) และความเข้าใจเรื่อง Data Types อย่างรุนแรง
1. **หัวเมธอดประกาศว่าจะคืน `void` แต่ดันทะลึ่ง `return` ของออกมา:** 
   เมื่อหัวเมธอดเป็น `void` (แปลว่าว่างเปล่า) บรรทัดที่ 5 `return avg;` จึงผิดกฎหมายของ C# วิธีแก้คือต้องเปลี่ยนคำว่า `void` เป็น `double` เพื่อให้สอดคล้องกับตัวแปร `avg`
2. **ขี้เกียจใส่ Data Type ให้พารามิเตอร์ `num2`:** 
   ภาษา C# ไม่ใช่ Python หรือ JavaScript! คุณจะประกาศตัวแปรลอยๆ อย่าง `num2` โดยไม่มีชนิดกำกับไม่ได้เด็ดขาด แม้ว่ามันจะอยู่ตรงกลางระหว่าง int ตัวอื่นก็ตาม กฎคือ **ทุกตัวแปรต้องมีชนิดข้อมูลของตัวเองเสมอ!** วิธีแก้คือเปลี่ยนเป็น `int num2`

**โค้ดที่ได้รับการผ่าตัด (Clean Code):**
```csharp
// 1. เปลี่ยน void เป็น double
// 2. เติมคำว่า int หน้า num2 ให้ครบถ้วนสมบูรณ์
static double GetAverage(int num1, int num2, int num3) 
{
    int total = num1 + num2 + num3;
    
    // การเอา int (total) หารด้วย double (3.0) จะได้ผลลัพธ์เป็น double ตรงตามกฎทฤษฎี!
    double avg = total / 3.0; 
    
    return avg; 
}
```

### **โจทย์ที่ 2: เครื่องตัดเกรดอัจฉริยะ (Grade Evaluator)**
**เป้าหมาย:** ทำระบบตัดเกรดเป็น A, B, C, F โดยรับเข้าเป็น `int` คืนค่าเป็น `char` และใช้เทคนิค Early Return

**เฉลยและกระบวนการคิด (Early Return Pattern):**
ถ้าเป็นโปรแกรมเมอร์มือใหม่ จะเขียนโค้ดซ้อน `else if` ลากยาวไปถึงบรรทัดที่ 20 (แถมอาจจะมีตัวแปร `char finalGrade = 'F'` มากางรอก่อน) 
แต่ในเมื่อเราเรียนเรื่อง `return` แล้ว เราต้องรู้ความลับข้อหนึ่งว่า: **เมื่อคอมพิวเตอร์ชนกับคำสั่ง return มันจะเตะโปรแกรมเด้งออกจากปีกกาทันที โดยไม่อ่านบรรทัดข้างล่างอีกเลย!**

ดังนั้น เราไม่จำเป็นต้องใช้คำว่า `else` แม้แต่คำเดียว โค้ดจะสะอาดเหมือนถูกขัดด้วยน้ำยา!

```csharp
static char EvaluateGrade(int score)
{
    // ด่านที่ 1: ถ้าได้ 80 ขึ้นไป คืนเกรด A แล้วจบการทำงานทันที (ไม่ต้องสนข้างล่าง)
    if (score >= 80)
    {
        return 'A'; 
    }
    
    // ด่านที่ 2: ถ้าหลุดรอดมาจากด่านแรกได้ แปลว่าคะแนนมัน "น้อยกว่า 80 ชัวร์ๆ" โดยที่เราไม่ต้องเขียนเช็ค (score < 80) เลย!
    // เราเลยเช็คแค่ว่า "ถึง 70 หรือเปล่า?" ก็พอ
    if (score >= 70)
    {
        return 'B';
    }

    // ด่านที่ 3: หลุดรอดมาได้แปลว่าน้อยกว่า 70 ชัวร์
    if (score >= 60)
    {
        return 'C';
    }

    // ด่านก้นหีบ (Fallback Return): 
    // ถ้าคอมพิวเตอร์ไหลร่วงลงมาถึงบรรทัดสุดท้ายนี้ได้ 
    // แปลว่ามันสอบตกทุกเงื่อนไขข้างบน! เราก็เลยสั่งคืนเกรด F ไปเลย โดยไม่ต้องใช้ else
    return 'F'; 
}
```

## 🔥 เฉลย Challenge (ระบบวิเคราะห์ความอ้วน The BMI Architect)

**เป้าหมาย:** สับย่อยโค้ดคำนวณและวิเคราะห์สุขภาพ BMI ออกเป็น 2 Methods ขาดจากกัน แล้วให้ `Main` รับบทบาทเป็นผู้บัญชาการสั่งการเท่านั้น

**สถาปัตยกรรมระดับซอฟต์แวร์ (Architectural Breakdown):**
นี่คือตัวอย่างระดับเทพของคำว่า **Single Responsibility Principle (ทำหน้าที่เดียวให้ดีที่สุด)** 
- เมธอดเครื่องคิดเลข (Calculator) ห้ามยุ่งเกี่ยวกับการประเมินโรค
- เมธอดหมอ (Doctor) ห้ามยุ่งเกี่ยวกับการคำนวณสูตร
- โค้ด Main ห้ามยุ่งกับการคำนวณ หรือตัดสินใจเอง มีหน้าที่รับส่งของเท่านั้น

```csharp
using System;

namespace BmiArchitect
{
    class Program
    {
        // ========================================================
        // Component 1: เครื่องยนต์คณิตศาสตร์ (The Calculator)
        // หน้าที่: รับน้ำหนัก ส่วนสูง(ซม) -> แปลงหน่วย -> เข้าสูตร -> คืนผลลัพธ์
        // ========================================================
        static double CalculateBMI(double weightKg, double heightCm)
        {
            // แปลงเซนติเมตร เป็นเมตรก่อน
            double heightMeters = heightCm / 100.0;
            
            // สูตร BMI = น้ำหนัก / (ส่วนสูงเมตร x ส่วนสูงเมตร)
            double bmi = weightKg / (heightMeters * heightMeters);
            
            return bmi;
        }

        // ========================================================
        // Component 2: ระบบแพทย์ประเมินผล (The Doctor)
        // หน้าที่: รับตัวเลขดิบๆ -> เข้าเงื่อนไขทางการแพทย์ -> คืนข้อความ
        // (เราใช้เทคนิค Early Return เพื่อความสะอาดตา)
        // ========================================================
        static string GetBMIStatusText(double bmiValue)
        {
            if (bmiValue < 18.5) return "น้ำหนักน้อย / ผอม";
            if (bmiValue <= 22.9) return "ปกติ (สุขภาพดี)";
            if (bmiValue <= 24.9) return "ท้วม / โรคอ้วนระดับ 1";
            
            // Fallback กรณีที่เกินกว่า 24.9 ไปแล้ว
            return "อ้วน / โรคอ้วนระดับ 2"; 
        }

        // ========================================================
        // Component 3: ศูนย์บัญชาการ (The Entry Point)
        // หน้าที่: ถาม Input -> สั่งงาน Component 1 -> สั่งงาน Component 2 -> แสดงผล UI
        // ========================================================
        static void Main(string[] args)
        {
            Console.WriteLine("=== โรงพยาบาลอัจฉริยะ (Smart BMI) ===");

            // 1. รับข้อมูลจากผู้ใช้งาน
            Console.Write("กรุณากรอกน้ำหนัก (กิโลกรัม): ");
            double userWeight = double.Parse(Console.ReadLine());

            Console.Write("กรุณากรอกส่วนสูง (เซนติเมตร): ");
            double userHeight = double.Parse(Console.ReadLine());

            // 2. [กระแสข้อมูล] ส่งวัตถุดิบเข้า Component 1 แล้วเก็บผลลัพธ์ไว้
            double userBmiResult = CalculateBMI(userWeight, userHeight);

            // 3. [กระแสข้อมูล] เอาผลลัพธ์จากกล่องแรก โยนเข้า Component 2
            string healthStatus = GetBMIStatusText(userBmiResult);

            // 4. สรุปผลรายงานให้ผู้ใช้งาน
            Console.WriteLine("\n------------------------------------");
            Console.WriteLine($"[ผลการตรวจสุขภาพของคุณ]");
            Console.WriteLine($"คะแนน BMI: {userBmiResult:N2}");
            Console.WriteLine($"สถานะแพทย์: {healthStatus}");
            Console.WriteLine("------------------------------------");
        }
    }
}
```

**ทำไมถึงต้องเขียนแยกส่วนให้ดูวุ่นวาย?**
สมมติว่าในอนาคต โรงพยาบาลต้องการทำระบบ "แอปพลิเคชันมือถือ" และ "เว็บไซต์" เพิ่มเติม 
โปรแกรมเมอร์แอปมือถือ ไม่ต้องมานั่งเขียนสูตรคำนวณ BMI ใหม่อีกรอบ! เค้าสามารถหยิบเอาฟังก์ชัน `CalculateBMI` และ `GetBMIStatusText` ที่เราเขียนแยกไว้อย่างดี ไปผูกกับปุ่มบนแอปมือถือได้เลยทันที (Reusability) 
แต่ถ้าคุณเขียนทุกอย่างรวมกันใน `Main()` คุณจะไม่สามารถดึงมันไปใช้งานที่อื่นได้เลย เพราะในแอปมือถือไม่มี Console! นี่คือความสำคัญระดับคอขาดบาดตายของ Modular Architecture ครับ!
