# เฉลย: `out` & `ref` Parameters (ข้ามขีดจำกัดหน่วยความจำ)

## 🎯 เฉลย Mini Exercise

### **โจทย์ที่ 1: จับผิดกลไก Memory**
**จุดบอดในโค้ดที่ให้มา:**
```csharp
// จุดที่ 1
int score;
UpdateScore(ref score); // 💥 Error แน่นอน!

// จุดที่ 2
static void GetCoordinates(out int x, out int y)
{
    x = 100;
    // ... ลืมอะไรไปหรือเปล่า? // 💥 Error แน่นอน!
}
```

**การวิเคราะห์สาเหตุและวิธีแก้ (The Deep Theory):**
- **จุดที่ 1:** ปัญหาเรื่อง "สายยางท่อเปล่า" (Uninitialized ref) 
  กฎสูงสุดของ `ref` คือ **"ห้ามส่งตัวแปรที่ยังไม่มีค่า (Uninitialized) เข้าไปเด็ดขาด!"** เพราะ `ref` อนุญาตให้ปลายทาง "อ่านค่า" ในกล่องได้ด้วย ถ้าส่งกล่องที่ว่างเปล่า (ไม่มีตัวเลขใน RAM) ไปให้ แล้วปลายทางพยายามดึงค่าออกมา ระบบจะแครช (Null Pointer หรือ Undefined Behavior) C# เลยป้องกันโดยการตบหน้าคุณตั้งแต่ตอน Compile เลย! 
  *วิธีแก้:* ต้องตั้งค่าเริ่มต้นก่อนเสมอ เช่น `int score = 0;`

- **จุดที่ 2:** ปัญหาเรื่อง "ผิดสัญญา out" (Unassigned out parameter)
  กฎสูงสุดของ `out` คือ **"คุณสัญญาแล้วว่าจะคายข้อมูลออกมาให้ครบทุกกล่อง!"** ในโค้ดตัวอย่าง คุณเซ็ตค่าให้กล่อง `x = 100;` เสร็จแล้วก็ปล่อยทิ้งไว้จนปีกกาปิด โดยที่ลืมยัดของลงไปในกล่อง `y`! คอมไพเลอร์มองว่านี่คือการทุจริตหลอกลวงคนเรียกใช้งาน มันจึงบังคับไม่ให้คุณรันโปรแกรมจนกว่าจะยัดค่าให้ `y` 
  *วิธีแก้:* ต้องเติมบรรทัด `y = 200;` (หรือค่าอะไรก็ได้) ให้ครบถ้วนก่อนจบปีกกา

### **โจทย์ที่ 2: เครื่องแยกเศษเงิน (The Coin Splitter)**
**เป้าหมาย:** ทำอัลกอริทึมทอนเงิน โดยคืนค่าออกมา 3 กล่อง (แบงค์ห้าสิบ, เหรียญสิบ, เหรียญห้า) ผ่านระบบ `out`

**เฉลยและวิธีคิดเชิงลึก (The Greedy Algorithm):**
การทอนเงิน เป็นโจทย์อัลกอริทึมสาย Greedy (ตะกละ) คือพยายามหักค่าที่ใหญ่ที่สุดออกไปก่อนเสมอ
เคล็ดลับคือการใช้เครื่องหมาย `/` (หารเพื่อเอาแค่จำนวนเต็ม ตัดเศษทิ้ง) ควบคู่กับ `%` (หารเพื่อเอาแค่เศษที่เหลือ ไปโยนให้คนอื่นทำต่อ)

```csharp
using System;

class Program
{
    // เมธอดฉลาดๆ ที่ไม่ต้อง return ค่าหลักเลย (เป็น void) แต่พ่นของออกทางท่อ out แทน!
    static void SplitMoney(int totalAmount, out int fiftyNotes, out int tenCoins, out int fiveCoins)
    {
        // 1. จัดการแบงค์ห้าสิบ (ใหญ่สุด)
        // สมมติ 85 บาท / 50 ได้ 1.7 (แต่ int ตัดทิ้งเหลือ 1)
        fiftyNotes = totalAmount / 50;
        
        // อัปเดตเงินที่เหลือ: 85 % 50 จะได้เศษคือ 35 บาท
        int remaining = totalAmount % 50;

        // 2. จัดการเหรียญสิบ (เอา 35 บาท มาจัดการต่อ)
        // 35 / 10 ได้ 3
        tenCoins = remaining / 10;
        
        // อัปเดตเงินที่เหลือ: 35 % 10 จะได้เศษ 5 บาท
        remaining = remaining % 10;

        // 3. จัดการเหรียญห้า (เอา 5 บาท มาจัดการต่อ)
        // 5 / 5 ได้ 1
        fiveCoins = remaining / 5;
        
        // หมายเหตุ: ตามกฎของ out เราได้บรรจุตัวเลขลงกล่องทั้ง 3 ใบครบถ้วนแล้ว! ถือว่าเสร็จสิ้นพิธี
    }

    static void Main(string[] args)
    {
        int myMoney = 85;

        // ใช้ฟีเจอร์ C# 7.0 สร้างกล่องกลางอากาศ (Inline variable declaration)
        SplitMoney(myMoney, out int notes50, out int coins10, out int coins5);

        Console.WriteLine($"แลกเงินจากยอด {myMoney} บาท:");
        Console.WriteLine($"- ได้แบงค์ 50 จำนวน: {notes50} ใบ");
        Console.WriteLine($"- ได้เหรียญ 10 จำนวน: {coins10} เหรียญ");
        Console.WriteLine($"- ได้เหรียญ 5  จำนวน: {coins5} เหรียญ");
    }
}
```

## 🔥 เฉลย Challenge (การสลับร่างขั้นเทพ The Ultimate Variable Swapper)

**เป้าหมาย:** สลับร่างสติง 2 กล่องในฟังก์ชัน แต่ให้ผลลัพธ์สะเทือนกลับมาถึง Main ตัวแม่

**กระบวนการคิดวิเคราะห์วิกฤติ (Crisis Analysis):**
ถ้าเราเขียนฟังก์ชันรับค่าปกติ `Swap(string a, string b)` สิ่งที่คอมพิวเตอร์ทำคือ แอบถ่ายเอกสาร A และ B ยื่นให้ฟังก์ชันเอาไปสลับ ฟังก์ชันก็สลับกระดาษแผ่นก๊อปปี้ไปมา พอฟังก์ชันตายกระดาษก็ถูกเผาทิ้ง... `Main` ก็มองตาปริบๆ ว่าของฉันไม่เห็นสลับเลย!
ทางออกเดียวของสมการนี้ คือการส่ง **"กุญแจรีโมต (Memory Address Reference)"** ไปให้ฟังก์ชันไขเข้ามายกกล่องสลับกันเองเลย! (นี่คือเวทมนตร์ของวิชา Pointer ในภาษาตระกูล C)

```csharp
using System;

namespace UltimateSwapper
{
    class Program
    {
        // สังเกตคีย์เวิร์ด `ref` สีทองอำไพ ที่หน้า Parameter ทั้งสองตัว
        static void SwapText(ref string firstBox, ref string secondBox)
        {
            // โค้ดส่วนนี้เหมือนบทที่ 2 เป๊ะ!
            // แก้วใบที่ 3 (ชั่วคราว) ดูดของจากกล่องแรกมาพักไว้
            string tempBox = firstBox;

            // กล่องสอง เทของทับใส่กล่องแรก (ซึ่งตอนนี้สายยางกำลังสูบไปแก้ที่กล่องใน Main โดยตรงเลยนะ!)
            firstBox = secondBox;

            // เอาของในกล่องชั่วคราว เทลงกล่องสอง
            secondBox = tempBox;
            
            // เสร็จสิ้นภารกิจ! ไม่ต้อง return อะไรทั้งนั้น เพราะเราแก้ของในห้องของ Main โดยตรงไปเรียบร้อยแล้ว
        }

        static void Main(string[] args)
        {
            Console.WriteLine("=== มายากลสลับร่าง (The Dark Magic of REF) ===\n");

            // กล่องทั้งสองต้องมีของตั้งต้นเสมอ (กฎเหล็กของ ref)
            string player1 = "อัศวินศักดิ์สิทธิ์ (Arthur)";
            string player2 = "ชาวบ้านเดินดิน (Peasant)";

            Console.WriteLine("[ก่อนใช้เวทมนตร์]");
            Console.WriteLine($"Player 1 ควบคุมร่าง: {player1}");
            Console.WriteLine($"Player 2 ควบคุมร่าง: {player2}");

            Console.WriteLine("\n...ร่ายมนตร์ดำ... ปิ๊ง! ✨\n");

            // โยน "สายยาง" (Reference) ไปให้หมอผีจัดการ 
            // ข้อควรระวัง: ตอนโยน ก็ต้องพิมพ์คำว่า ref ควบคู่ไปด้วยเสมอ เป็นการเตือนสติตัวเอง
            SwapText(ref player1, ref player2);

            Console.WriteLine("[หลังใช้เวทมนตร์]");
            Console.WriteLine($"Player 1 กลายเป็น: {player1}");
            Console.WriteLine($"Player 2 กลายเป็น: {player2}");
            
            Console.WriteLine("\nโอ้โห! ค่ามันสลับกันจริงๆ ด้วย โดยที่ไม่ต้องใช้เครื่องหมาย = รับค่าใน Main เลย!");
        }
    }
}
```

**เคล็ดวิชาสุดยอด:**
หลายคนเข้าใจผิดว่า `ref` มีประโยชน์แค่ตอนทำตัวเลข (int) แต่ความจริงแล้ว **การสลับ Data ขนาดใหญ่มหึมา (เช่น ภาพความละเอียด 4K หรือ Array ข้อมูล 1 ล้านตัว)** เป็นประโยชน์สูงสุดของ `ref`!
เพราะถ้าเราไม่มี `ref` การโยน Array 1 ล้านตัวไปให้ฟังก์ชัน มันจะต้องเสียเวลา "ก๊อปปี้ข้อมูล 1 ล้านตัว" ซึ่งทำให้คอมกระตุกชั่วขณะ (Lag) แต่การใช้ `ref` เป็นการ "ส่งแค่แผนที่" (กุญแจ) แผ่นเดียวไปให้ฟังก์ชัน ซึ่งรวดเร็วระดับแสง (Zero overhead)!
