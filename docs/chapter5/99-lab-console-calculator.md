> 💡 **เป้าหมายของ Lab นี้:** ท้าทายความสามารถในการยกระดับโค้ดแบบสปาเก็ตตี้ใน `Main()` ให้กลายเป็นระบบสถาปัตยกรรมชั้นยอดที่ถูกหั่นเป็นชิ้นๆ (Modular Architecture) อย่างสมบูรณ์แบบ โดยนำความรู้เรื่อง `out` parameters, `return` values, และสไตล์การตั้งชื่อ Method (PascalCase) มาใช้งานร่วมกับระบบ Interactive Menu ของบทก่อนๆ

## 📖 ภาพรวมของโปรแกรม (Program Overview)

ในบทที่ 4 เราเคยสร้าง `The Resilient Calculator` มาแล้ว แต่มันเป็นเพียงการเททุกอย่างลงหม้อเดียว (Monolithic) 
ตอนนี้คุณก้าวเข้าสู่สถานะวิศวกรซอฟต์แวร์เต็มตัว! งานของคุณคือการนำเครื่องคิดเลขตัวนั้น มา "รื้อและจัดระเบียบใหม่ทั้งหมด (Refactoring)" 

**คอนเซปต์ใหม่ของเรา:** 
1. `Main()` จะมีหน้าที่แค่เรียกใช้ระบบ ไม่มีการคำนวณหรือ `Console.ReadLine()` ใดๆ ยัดเยียดอยู่ในนั้นเลย
2. เราจะสร้าง "ป้อมปราการคัดกรองตัวเลข" ที่จะไม่มีวันปล่อยให้ผู้ใช้พิมพ์ตัวอักษรเข้ามาทำให้ระบบแครชได้ (โดยใช้เครื่องมือ `out`)
3. เราจะดึงหน่วยประมวลผลทางคณิตศาสตร์ (Core Logic) แยกออกไปอยู่ในห้องปิดตายที่เรียกใช้งานได้ซ้ำๆ อย่างเสถียรภาพ

### ตัวอย่างหน้าจอเมื่อรันโปรแกรมสำเร็จ (เหมือนบทที่ 4 เป๊ะ แต่ข้างในเทพกว่า 10 เท่า)

```text
======================================
    [ MODULAR CALCULATOR ENGINE ]
======================================
กรุณาป้อนตัวเลขที่ 1: ห้า
❌ ผิดพลาด: สิ่งที่คุณกรอกไม่ใช่ตัวเลข กรุณาลองใหม่!
กรุณาป้อนตัวเลขที่ 1: 5

กรุณาระบุเครื่องหมาย (+, -, *, /): ?
❌ ผิดพลาด: ไม่รองรับเครื่องหมายนี้
กรุณาระบุเครื่องหมาย (+, -, *, /): *

กรุณาป้อนตัวเลขที่ 2: 12.5

--------------------------------------
> ผลลัพธ์: 5 * 12.5 = 62.5
--------------------------------------

[กด Y เพื่อคำนวณต่อ หรือปุ่มอื่นเพื่อออก] : n
ลาก่อน!
```

## 📋 กฎกติกาและเกณฑ์การประเมิน (Business Requirements)

เพื่อเป็นการจำลองการทำงานในบริษัทจริง คุณต้องทำตาม Specification (สเปก) ของ Senior Developer ที่ร่างไว้ให้อย่างเคร่งครัด:

1. **ห้ามมีตรรกะคณิตศาสตร์หลงเหลืออยู่ใน `Main`:** 
   `Main` ต้องสะอาดและสั้นที่สุด
2. **ต้องสร้างเมธอด `GetValidNumber`:**
   - เป็นเมธอดที่บังคับผู้ใช้พิมพ์จนกว่าจะได้ตัวเลขที่เป็นชนิด `double` เท่านั้น
   - (บังคับ) ต้องใช้ `double.TryParse(..., out ...)` ควบคู่กับลูปดักจับ
   - คืนค่าออกมาเป็น `double` ที่เชื่อถือได้ 100%
3. **ต้องสร้างเมธอด `GetValidOperator`:**
   - เป็นเมธอดที่บังคับผู้ใช้พิมพ์จนกว่าจะได้สัญลักษณ์ 1 ใน 4 ตัวนี้ `+`, `-`, `*`, `/` เท่านั้น 
   - คืนค่าเป็น `string` หรือ `char`
4. **ต้องสร้างเมธอด `CalculateCore`:**
   - รับค่าพารามิเตอร์เข้า: ตัวเลขที่ 1, ตัวเลขที่ 2, และเครื่องหมาย
   - ภายในเมธอดทำหน้าที่เข้า `switch` case ประมวลผลเลข
   - **(บังคับ)** ห้ามมีการพิมพ์ `Console.WriteLine` ในเมธอดนี้เด็ดขาด! มันต้องส่งค่ากลับ (Return) เป็นผลลัพธ์คำตอบทางคณิตศาสตร์เท่านั้น!
5. **ดักบั๊กหายนะ (Division by Zero):**
   - ถ้าผู้ใช้สั่งหารด้วย 0 ในเมธอดประมวลผล ต้องมีวิธีรับมือ (อาจจะส่งค่า `double.NaN` กลับไปเพื่อเป็นสัญญาณบอกความผิดพลาด)

## 📝 ขั้นตอนการพัฒนา (Step-by-Step Implementation)

### ขั้นตอนที่ 1: โครงสร้างกระดูกสันหลัง (The Skeleton)
- [ ] เปิดคลาส `Program` แล้วสร้างโครงร่างของ Method เปล่าๆ ทิ้งไว้ (นอก `Main`):
```csharp
static double GetValidNumber(string promptMessage) { return 0; }
static string GetValidOperator() { return ""; }
static double CalculateCore(double n1, double n2, string op) { return 0; }
```

### ขั้นตอนที่ 2: ประกอบร่างเมธอด `GetValidNumber`
- [ ] ในเมธอดนี้ ใช้ลูป `while (true)` สร้างกรงขังอมตะ
- [ ] สั่ง `Console.Write(promptMessage);` (เพื่อให้ข้อความยืดหยุ่น เช่น จะบอกว่า "ป้อนเลขที่ 1" หรือ "ป้อนเลขที่ 2" ก็ได้)
- [ ] รับค่าเป็น string และโยนเข้ากระบวนการ `double.TryParse(input, out double result)`
- [ ] ถ้าผ่าน (เป็น `true`) ให้คำสั่ง `return result;` เพื่อเตะกระเด็นหลุดพ้นจากลูปและเมธอดกลับไปให้คนเรียก
- [ ] ถ้าไม่ผ่าน ให้พิมพ์ด่าเบาๆ ว่า "❌ กรุณากรอกตัวเลข" แล้วปล่อยให้ลูปหมุนรอบใหม่

### ขั้นตอนที่ 3: ประกอบร่างเมธอด `GetValidOperator`
- [ ] สร้างลูปกรงขัง `while (true)` เช่นเดียวกัน
- [ ] ขอรับค่าเครื่องหมาย 
- [ ] ตรวจเช็คว่า: `if (op == "+" || op == "-" || op == "*" || op == "/")`
- [ ] ถ้าถูกให้ `return op;` ถ้าผิดให้ด่าแล้วให้กรอกใหม่

### ขั้นตอนที่ 4: เตาปฏิกรณ์ `CalculateCore`
- [ ] นำค่า 3 ชิ้น มาเข้าโครงสร้าง `switch (op)`
- [ ] ถ้าเป็น `+`, `-`, `*` ให้คืนค่าการคำนวณออกไปตรงๆ ทันที
- [ ] **เฉพาะกรณี `/` (การหาร)**: คุณต้องแทรก `if (n2 == 0)` ดักไว้ก่อน ถ้ามันเป็น 0 จริง ให้ระบบคายค่าพิเศษของคอมพิวเตอร์คือ **`return double.NaN;`** (Not-a-Number) กลับไป 
- [ ] นอกนั้น `return n1 / n2;`

### ขั้นตอนที่ 5: บัญชาการหลัก `Main`
- [ ] สร้างลูป `do-while` คุม UI เหมือนบทที่แล้ว 
- [ ] เรียกใช้ `GetValidNumber("กรุณาป้อนตัวเลขที่ 1: ")` ไปเก็บลงกล่อง `num1`
- [ ] เรียกใช้ `GetValidOperator()` เก็บลงกล่อง `opSymbol`
- [ ] เรียกใช้ `GetValidNumber("กรุณาป้อนตัวเลขที่ 2: ")` เก็บลงกล่อง `num2`
- [ ] โยนของทั้งสาม ส่งเข้าไปในเตาปฏิกรณ์ `CalculateCore` แล้วนำผลรับมาใส่กล่อง `finalResult`
- [ ] **แสดงผล:** ตรวจสอบว่า `if (double.IsNaN(finalResult))` (เช็คว่าพังจากการหารศูนย์ไหม) ถ้าพังก็พิมพ์ว่า "คำนวณไม่ได้" ถ้าไม่พังก็พิมพ์โชว์คำตอบอย่างสวยงาม!
- [ ] ถามผู้ใช้ว่า "(Y/N) เอาอีกรอบไหม"

## 🔥 Challenge (ภารกิจท้าทายขีดจำกัด)

สำหรับสถาปนิกที่ต้องการรื้อโครงสร้างให้ไปสุดทาง:

**ภารกิจ 1: The Multiple Out Parameters**
ลองสร้างเมธอดพิเศษที่รวบขั้นตอนใน `Main` ซะใหม่ ชื่อ `static bool RunCalculation(out string report)` 
- ให้การพิมพ์หาข้อมูลและการโยนเข้าสูตรคำนวณไปอยู่ในนั้นทั้งหมด 
- เมื่อคำนวณเสร็จ ให้ประกอบประโยค (เช่น "5 + 5 = 10") โยนกลับออกมาผ่านช่องทาง `out` 
- แล้วตัวเมธอด `RunCalculation` จะทำการ return ออกมาเป็นค่า `bool` แท้ๆ เท่านั้น (เช่น `true` คือคำนวณสำเร็จ, `false` คือมีคนพยายามหารศูนย์!) 
- ให้ `Main` ทำหน้าที่แค่สั่ง "Print" ประโยคที่หลุดรอดมาจากท่อ out อย่างเดียวพอ! (นี่คือแพทเทิร์นสุดคลาสสิกของระบบ Log)

ขอให้สนุกกับประสบการณ์การแยกส่วนชิ้นส่วนประกอบร่างโค้ด (Refactoring) ครับ! มันคือเวทมนตร์ที่คุณต้องใช้ไปตลอดชีวิตการเป็นโปรแกรมเมอร์!
