# Parameters & Return Values (การรับ-ส่งข้อมูล)

> 💡 **เป้าหมาย:** สลัดข้อจำกัดของ Method แบบ "สุญญากาศ (`void` และไม่มีวงเล็บ)" ทิ้งไป! เรียนรู้วิธีเจาะช่องทางเข้า (Parameters) เพื่อส่งข้อมูลดิบเข้าไปให้ Method ประมวลผล และเปิดช่องทางออก (Return Values) เพื่อส่งคำตอบที่คำนวณเสร็จแล้วกลับคืนสู่ผู้เรียก

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. เข้าใจกระแสข้อมูล (Data Flow) 

ในบทที่แล้ว ตู้กดน้ำอัตโนมัติของเรามีป้ายชื่อ (`SayHello`) แต่มันไม่มีรูให้ยอดเหรียญและไม่มีของหล่นลงมา 
ในโลกความเป็นจริง ฟังก์ชันที่มีประโยชน์ที่สุด คือฟังก์ชันที่เป็น **"เครื่องจักรกล (Black Box Machine)"**
- **Input (ข้อมูลเข้า):** คุณโยนวัตถุดิบ (เช่น เนื้อสัตว์, ผัก) เข้าไป
- **Processing (การประมวลผล):** เครื่องจักรทำการสับ บด และปรุงสุก (โค้ดลอจิกภายใน)
- **Output (ข้อมูลออก):** เครื่องจักรคาย "ไส้กรอก" ออกมาให้คุณนำไปประกอบอาหารจานอื่นต่อ

---

### 2. ช่องทางเข้า: Parameters vs Arguments

การเจาะช่องทางเข้าทำได้โดยการประกาศ "ตัวแปร" เอาไว้ **ภายในวงเล็บ ( )** ของ Method
สิ่งที่เรามักสับสนคือ 2 คำนี้:

1. **Parameter (พารามิเตอร์):** คือตัวแปรที่ถูกประกาศไว้ "ตอนสร้าง Method" (เป็นเหมือนกรอบแม่พิมพ์ หรือช่องหยอดเหรียญที่ระบุว่า "รับเฉพาะเหรียญสิบ (int) เท่านั้นนะ")
   ```csharp
   // userName คือ "Parameter"
   static void GreetUser(string userName) 
   {
       Console.WriteLine($"ยินดีต้อนรับคุณ {userName}");
   }
   ```

2. **Argument (อาร์กิวเมนต์):** คือข้อมูล "ของจริง" ที่คุณโยนเข้าไปในช่องนั้น "ตอนเรียกใช้งาน Method"
   ```csharp
   // "John" และ "Alice" คือ "Argument"
   GreetUser("John"); 
   GreetUser("Alice");
   ```

**⚠️ กฎของ Scope Isolation (การแยกเอกเทศ):**
ตัวแปรที่โยนเข้าไปเป็น Argument จะถูก **"Copy" (คัดลอกร่างโคลน)** ไปใส่ใน Parameter การแก้ไขตัวแปรภายใน Method จะไม่มีผลกระทบใดๆ กับตัวแปรต้นฉบับข้างนอกเด็ดขาด! (เรียกว่า Pass by Value - ซึ่งเราจะเจาะลึกการทำลายกฎนี้ในหัวข้อ `ref`/`out`)

### 3. ช่องทางออก: The `Return` Keyword

ถ้าคุณต้องการให้เครื่องจักรคายไส้กรอกออกมา คุณต้องทำ 2 อย่าง:
1. **เปลี่ยน `void` เป็น Data Type ที่ต้องการส่งกลับ:** เช่น ถ้าจะคำนวณเลขแล้วส่งกลับเป็นจำนวนเต็ม ต้องเปลี่ยนหัวเมธอดเป็น `int`
2. **ใช้คำสั่ง `return` ภายในปีกกา:** เพื่อโยนข้อมูลนั้นกลับออกไป

```csharp
// เมธอดนี้สัญญาว่าจะส่ง "int" กลับมาให้ 1 ตัวแน่นอน
static int AddNumbers(int a, int b)
{
    int result = a + b;
    
    // โยนคำตอบกลับไปยังคนเรียก (และถือเป็นการ จบการทำงาน ทันที!)
    return result; 
}
```

**กฎเหล็ก 3 ข้อของการทำ Return:**
1. **เมื่อเจอ `return` เมธอดจะตายทันที:** โค้ดใดๆ ก็ตามที่ถูกเขียนไว้ "ข้างใต้" บรรทัดที่มีคำสั่ง `return;` จะไม่มีวันถูกทำงานคอมพิวเตอร์ประมวลผล (Unreachable code) 
2. **ต้องคืนค่าให้ตรงชนิด:** ถ้าสัญญากับหัวเมธอดไว้ว่าเป็น `int` คุณจะ `return "Hello";` ไม่ได้อย่างเด็ดขาด (Compile Error)
3. **ทุกเส้นทางต้องมีทางออก:** ถ้าคุณใช้ `if-else` อยู่ในเมธอด คุณต้องวาง `return` ไว้ให้ครบทุกกรณี (All code paths must return a value) คอมพิวเตอร์ไม่อนุญาตให้เกิดเหตุการณ์ที่ "บางกรณีไม่มีการส่งค่ากลับ"

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่างที่ 1: เครื่องมือคณิตศาสตร์ส่วนตัว (Math Utility)

โค้ดนี้สาธิตกระบวนการรับค่า 2 ชิ้น และส่งผลลัพธ์กลับไปให้ `Main` เอาไปจัดรูปแบบแสดงผลต่อเอง (สังเกตว่าเราไม่เขียน `Console.WriteLine` ไว้ในตัวเมธอดคณิตศาสตร์เลย เพื่อให้เมธอดนั้นทำงาน "คำนวณล้วนๆ" ไม่ยุ่งกับ UI)

::: code-group
```csharp [Program.cs]
using System;

namespace MethodDataFlow
{
    class Program
    {
        // ------------------------------------------
        // เมธอดคำนวณพื้นที่สี่เหลี่ยม (รับ int 2 ตัว, คืนกลับ int 1 ตัว)
        // ------------------------------------------
        static int CalculateArea(int width, int length)
        {
            // สูตร: กว้าง x ยาว
            int area = width * length;
            
            // ส่งค่ากลับไปให้ผู้เรียก
            return area;
        }

        // ------------------------------------------
        // เมธอดประเมินราคาบ้าน (รับ int, คืนกลับ double)
        // ------------------------------------------
        static double EvaluateHousePrice(int areaSize)
        {
            // สมมติว่าตารางเมตรละ 15,500.50 บาท
            double price = areaSize * 15500.50;
            return price;
        }

        // ------------------------------------------
        // 
        // ------------------------------------------
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบประเมินราคาอสังหาริมทรัพย์ ===");

            Console.Write("กรอกความกว้างของบ้าน (เมตร): ");
            int w = int.Parse(Console.ReadLine());

            Console.Write("กรอกความยาวของบ้าน (เมตร): ");
            int l = int.Parse(Console.ReadLine());

            // 1. โยนค่า Arguments (w, l) เข้าไปใน Parameter (width, length)
            // และสร้างตัวแปร totalArea มารอรับข้อมูลที่ Return กลับมา
            int totalArea = CalculateArea(w, l);

            // 2. นำผลลัพธ์ที่ได้ ไปเป็น Input โยนให้เครื่องจักรถัดไปทำงานต่อ!
            double estimatedPrice = EvaluateHousePrice(totalArea);

            // 3. จัดการแสดงผล UI อย่างสวยงามใน Main
            Console.WriteLine("\n[รายงานผลประเมิน]");
            Console.WriteLine($"พื้นที่ใช้สอยทั้งหมด: {totalArea} ตารางเมตร");
            Console.WriteLine($"ราคาประเมินเบื้องต้น: {estimatedPrice:N2} บาท");
        }
    }
}
```
:::

**ความฉลาดทางวิศวกรรม (Architectural Brilliance):**
แนวคิดการให้เมธอดหนึ่งทำแค่ "เรื่องเดียว (Single Responsibility Principle)" เช่น ให้เมธอดนึงคำนวณพื้นที่, อีกเมธอดตีราคา, แล้วให้ `Main` เอามาประกอบร่างกัน ถือเป็นการออกแบบชั้นยอด 
เพราะในอนาคตคุณอาจจะเอาเมธอด `CalculateArea()` ไปใช้กับระบบคำนวณกระเบื้องปูพื้น แทนระบบตีราคาบ้านก็ได้! (ถ้าคุณเผลอใส่ `Console.WriteLine("ราคาบ้าน...");` ทิ้งไว้ในเมธอด มันจะใช้กับงานกระเบื้องไม่ได้ทันที)

### ตัวอย่างที่ 2: ระบบคัดกรองพนักงาน (The Early Return Pattern)

บ่อยครั้งที่เราต้องเขียน `if-else` ซ้อนกันลึกๆ (Arrow Code) แต่ด้วยพลังของคำสั่ง `return` เราสามารถ "ตัดจบการทำงานล่วงหน้า" ทันทีที่เจอเงื่อนไขขยะได้เลย (Guard Clauses) ทำให้โค้ดสะอาดขึ้นมหาศาล

::: code-group
```csharp [Program.cs]
using System;

namespace EarlyReturnDemo
{
    class Program
    {
        // เมธอดเช็คคุณสมบัติผู้สมัครงาน คืนค่ากลับมาเป็นความจริง (bool)
        static bool IsEligibleToApply(int age, int experienceYears)
        {
            // ------------------------------------------------
            // วิธีแบบดั้งเดิม (Nested If-Else - ไม่ค่อยดีเพราะอ่านยาก)
            // ------------------------------------------------
            /*
            if (age >= 18) {
                if (experienceYears >= 2) {
                    return true;
                } else {
                    return false;
                }
            } else {
                return false;
            }
            */

            // ------------------------------------------------
            // วิธีระดับโปร (Early Return / Guard Clauses)
            // สกัดกั้นเงื่อนไขที่แย่ๆ แล้วเตะออก (return) ทันที!
            // ------------------------------------------------
            
            // ด่านที่ 1: อายุไม่ถึงเกณฑ์? เตะออกเลยไม่ต้องตรวจต่อ
            if (age < 18) 
            {
                return false; 
            }

            // ด่านที่ 2: ประสบการณ์ไม่ถึงเกณฑ์? เตะออกเช่นกัน
            if (experienceYears < 2) 
            {
                return false; 
            }

            // ถ้ารอดชีวิตผ่านมาถึงบรรทัดนี้ได้ แปลว่าผ่านทุกเงื่อนไขแน่นอน!
            // (ไม่มีทางลงมาถึงตรงนี้ได้เลยถ้าติดด่านข้างบน เพราะคำสั่ง return ได้ทำลายกระบวนการทิ้งไปแล้ว)
            return true;
        }

        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบตรวจสอบคุณสมบัติสมัครงาน ===");
            
            Console.Write("กรุณาระบุอายุของคุณ: ");
            int userAge = int.Parse(Console.ReadLine());

            Console.Write("กรุณาระบุประสบการณ์การทำงาน (ปี): ");
            int userExp = int.Parse(Console.ReadLine());

            // โยนค่าให้เมธอดวิเคราะห์ และเอาผลลัพธ์มาเช็คเงื่อนไขโดยตรงเลย (เพราะมันคืนค่าเป็น bool)
            if (IsEligibleToApply(userAge, userExp))
            {
                Console.WriteLine("✅ ยินดีด้วย! คุณมีคุณสมบัติครบถ้วน สามารถยื่นใบสมัครได้");
            }
            else
            {
                Console.WriteLine("❌ เสียใจด้วย คุณสมบัติของคุณไม่ผ่านเกณฑ์ขั้นต่ำของบริษัท");
            }
        }
    }
}
```
:::

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

### **โจทย์ที่ 1: ผ่าตัดโค้ดหาค่าเฉลี่ย (The Average Fixer)**
นักศึกษาคนหนึ่งพยายามเขียนเมธอดหาค่าเฉลี่ยของเลข 3 ตัว แต่คอมไพเลอร์ขีดเส้นใต้สีแดงโวยวายหลายจุด:
```csharp
static void GetAverage(int num1, num2, int num3)
{
    int total = num1 + num2 + num3;
    double avg = total / 3.0;
    return avg;
}
```
**ภารกิจ:** จงระบุข้อผิดพลาดทางไวยากรณ์ (Syntax) และทางตรรกะ (Logic) ในหัวเมธอดนี้ แล้วปรับปรุงโค้ดให้ถูกต้อง สามารถ Compile ผ่านและคืนค่าได้อย่างแม่นยำ

### **โจทย์ที่ 2: เครื่องตัดเกรดอัจฉริยะ (Grade Evaluator)**
**ภารกิจ:** สร้างเมธอดชื่อ `EvaluateGrade` 
- รับค่า Parameter 1 ตัว คือคะแนนสอบแบบ `int score`
- ส่งค่ากลับ (Return Type) เป็นสัญลักษณ์เกรดแบบตัวอักษร 1 ตัว (ประเภท `char`)
- โลจิกภายใน: 
  - ถ้าคะแนน >= 80 ส่งคืน `'A'`
  - ถ้าคะแนน >= 70 ส่งคืน `'B'`
  - ถ้าคะแนน >= 60 ส่งคืน `'C'`
  - ถ้าต่ำกว่านั้น ส่งคืน `'F'`
*(คำใบ้: ลองใช้แพทเทิร์น Early Return ในการดักเงื่อนไขดูสิ คุณจะไม่ต้องใช้คำว่า `else` เลยแม้แต่คำเดียว!)*

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: ระบบวิเคราะห์ความอ้วน (The BMI Architect)**

ในบทที่ 2 เราเคยเขียนโปรแกรมคำนวณ BMI ใน `Main()` รวดเดียวจบไปแล้ว แต่ในฐานะสถาปนิกซอฟต์แวร์ เราต้องแยกชิ้นส่วนมันออกมาให้ทำงานแบบโมดูลาร์ (Modular)

**ภารกิจของคุณ:** 
แยกการทำงานของระบบประเมินค่า BMI ออกเป็น **2 เมธอด** (ไม่รวม Main) ดังนี้:

1. **เมธอดคณิตศาสตร์ (The Calculator):**
   - ชื่อเมธอด: `CalculateBMI`
   - พารามิเตอร์รับเข้า: น้ำหนัก (`double`), ส่วนสูงหน่วยเซนติเมตร (`double`)
   - สิ่งที่ต้องคืนค่า: ค่า BMI ที่คำนวณเสร็จแล้ว (`double`)
   - *หมายเหตุ: ต้องมีการแปลงเซนติเมตรเป็นเมตรข้างในนี้ด้วย*

2. **เมธอดการแพทย์ (The Doctor):**
   - ชื่อเมธอด: `GetBMIStatusText`
   - พารามิเตอร์รับเข้า: ค่า BMI ที่เป็นตัวเลข (`double`)
   - สิ่งที่ต้องคืนค่า: ข้อความวิเคราะห์สุขภาพ (`string`)
   - โลจิก: 
     - BMI < 18.5 คืนข้อความ "น้ำหนักน้อย / ผอม"
     - BMI 18.5 - 22.9 คืนข้อความ "ปกติ (สุขภาพดี)"
     - BMI 23.0 - 24.9 คืนข้อความ "ท้วม / โรคอ้วนระดับ 1"
     - BMI 25.0 ขึ้นไป คืนข้อความ "อ้วน / โรคอ้วนระดับ 2"

3. **ประกอบร่าง (The Main Integration):**
   - ใน `Main()` ให้รับข้อมูลน้ำหนักส่วนสูงจากผู้ใช้
   - โยนใส่เมธอดที่ 1
   - เอาผลลัพธ์ของเมธอด 1 ไปโยนใส่เมธอด 2
   - นำผลลัพธ์จากเมธอด 2 มาแสดงออกทางหน้าจอให้สวยงาม

ถ้าคุณเขียนระบบนี้สำเร็จ โค้ดใน `Main()` ของคุณจะเหลือแค่ไม่ถึง 10 บรรทัด แต่อ่านง่ายเหมือนหนังสือจิตวิทยาเลยล่ะ!

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** ในกระบวนการ Pass by Value ถ้าเราโยนตัวแปร `int myAge = 20;` ไปเป็นอาร์กิวเมนต์ให้ฟังก์ชัน `AddTen(int age)` แล้วในฟังก์ชันนั้นสั่ง `age = age + 10;` ถามว่าเมื่อหลุดออกมาจากฟังก์ชันแล้ว ตัวแปรต้นฉบับ `myAge` ใน Main จะมีค่าเท่าใด? (20 หรือ 30?)
**แนวคำตอบ:** **มีค่าเท่ากับ 20 เหมือนเดิมไม่เปลี่ยนแปลง!** 
กลไกของ Pass by Value คือการ "ถ่ายเอกสาร (Copy)" ชิ้นข้อมูลไปวางแปะในมือของฟังก์ชัน ฟังก์ชันจะเอาเอกสารก๊อปปี้ไปขยำหรือขีดเขียนแก้เป็นเลข 30 อย่างไร เอกสารต้นฉบับในมือของ `Main` ก็จะไม่มีวันถูกสัมผัสหรือเปลี่ยนแปลงใดๆ ทั้งสิ้น! (เป็นระบบรักษาความปลอดภัยเบื้องต้นของ C# ป้องกันไม่ให้ฟังก์ชันดัดแปลงข้อมูลของเราโดยพละการ)

**คำถาม 2:** โค้ดต่อไปนี้เกิด Error ในการ Compile ทันทีที่บรรทัดปีกกาปิด ทำไมคอมพิวเตอร์ถึงโวยวายว่า "Not all code paths return a value"?
```csharp
static string CheckPositive(int number)
{
    if (number >= 0)
    {
        return "ค่าเป็นบวกครับ";
    }
}
```
**แนวคำตอบ:** เพราะมี **"รูรั่วของข้อมูล (Data Leak)"**!
หัวเมธอดประกาศไว้ชัดเจนว่า สัญญาว่าจะต้องส่งออก `string` กลับไปให้เสมอ 
แต่ถ้าเกิดคนเรียก โยนเลขติดลบ (เช่น -5) เข้ามาล่ะ? 
เงื่อนไข `if` จะกลายเป็นเท็จ โปรแกรมจะกระโดดข้ามบล็อก `if` มาตกที่บรรทัดปีกกาปิดโดยที่ **"ไม่มีการ return อะไรกลับไปเลย"** (ผิดสัญญา!)
วิธีแก้ไขคือ คุณต้องมีท่าไม้ตายก้นหีบ (Fallback return) ใส่ไว้ด้านล่างสุดของฟังก์ชันเสมอ หรือเติมกระบวนการ `else` ให้ครบถ้วน เช่น เติม `return "ค่าติดลบครับ";` ต่อท้ายด้านล่างสุด เป็นอันจบปัญหา
