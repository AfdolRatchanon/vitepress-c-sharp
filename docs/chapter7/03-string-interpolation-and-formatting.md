# String Interpolation & Formatting (การปั้นและจัดรูปแบบ)

> 💡 **เป้าหมาย:** หยุดการใช้เครื่องหมาย `+` เพื่อต่อข้อความที่ดูรกรุงรัง และก้าวเข้าสู่ศาสตร์แห่งการ **"ฉีดตัวแปร (Interpolation)"** และ **"การจัดหน้ากระดาษ (Formatting)"** เพื่อสร้างใบเสร็จรับเงิน รายงาน หรือตารางข้อมูลที่เป๊ะทุกบรรทัด!

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. วิวัฒนาการของการต่อข้อความ (The Evolution of Concat)

ลองจินตนาการว่าคุณต้องสร้างประโยค: `"สวัสดี [ชื่อ], คุณมีเงิน [เงิน] บาท"`
ในยุคหิน (C# ยุคเก่า) โปรแกรมเมอร์ต้องเขียนแบบนี้: 
`string msg = "สวัสดี " + name + ", คุณมีเงิน " + money + " บาท";`
*(ผลคือ: โค้ดอ่านยากมาก ลืมเว้นวรรคพัง และเปลืองแรมจาก Immutability มหาศาล)*

ยุคกลาง (C# 2.0) ไมโครซอฟต์เปิดตัว `string.Format`:
`string msg = string.Format("สวัสดี {0}, คุณมีเงิน {1} บาท", name, money);`
*(ผลคือ: โค้ดสะอาดขึ้น แต่ถ้าตัวแปรมีเป็นสิบตัว จะสับสนหมายเลข {0}, {1} จนตาลาย)*

**ยุคแสงสว่าง (C# 6.0+) เกิดนวัตกรรม String Interpolation (`$`)**
เพียงแค่คุณแปะเครื่องหมายดอลลาร์ **`$`** ไว้หน้าฟันหนู... ข้อความนั้นจะกลายเป็น "ข้อความเวทมนตร์" ที่ยอมให้คุณเอาตัวแปรยัดใส่ปีกกา `{ }` ลงไปได้ตรงๆ เลย!
`string msg = $"สวัสดี {name}, คุณมีเงิน {money} บาท";`
*(ผลคือ: อ่านง่ายที่สุดในสามโลก และเป็นมาตรฐานที่วิศวกรทุกคนบนโลกใช้ในปัจจุบัน!)*

### 2. เวทมนตร์แต่งหน้าตัวเลข (Formatting Specifiers)

เวลาคุณทำระบบการเงิน ถ้าค่าในคอมคือ `12500.5` คุณอยากโชว์ผู้ใช้ว่า `12,500.50 ฿` 
คุณไม่ต้องไปเขียนโค้ดหาลูกน้ำมาแทรกเอง! C# มีรหัสลับ (Format Specifier) ที่แค่คุณเขียนต่อท้ายตัวแปรในปีกกาด้วยเครื่องหมาย **โคลอน `:`** มันจะแต่งหน้าตัวเลขให้ทันที!

- **`:N0` (Number):** ใส่ลูกน้ำให้ และ **ไม่เอา** ทศนิยม (เช่น `1,000`)
- **`:N2` (Number):** ใส่ลูกน้ำให้ และ **ล็อกทศนิยม 2 ตำแหน่ง** (เช่น `1,000.50`)
- **`:C` (Currency):** เหมือน N2 แต่แอบ **"แถมสัญลักษณ์สกุลเงิน"** ของเครื่องคอมพิวเตอร์นั้นๆ ให้ด้วย (เช่น ฿ หรือ $)
- **`:P` (Percent):** เอาเลขไปคูณ 100 แล้วแปะเครื่องหมาย `%` ให้ (เช่น `0.5:P` จะกลายเป็น `50.00%`)

---

### 3. ศิลปะการตีกรอบตาราง (Alignment & Padding)

ความลับสุดยอดที่มือใหม่ไม่รู้: ปีกกาของ Interpolation `{ }` สามารถจัดหน้ากระดาษ (ชิดซ้าย / ชิดขวา) ได้ด้วยการใช้ **จุลภาค `,`**
- **`{name, 10}`**: บังคับให้ช่องนี้กินพื้นที่ **10 ตัวอักษร** และผลักข้อความ **ชิดขวา** (ถ้าข้อความสั้นกว่า 10 มันจะเติมช่องว่างด้านซ้ายให้เอง)
- **`{name, -10}`**: เหมือนเดิม แต่ค่าติดลบแปลว่าผลักข้อความ **ชิดซ้าย**

*(หมายเหตุ: กฎการใช้ปีกกาแบบ Full Option คือ `{ตัวแปร, การจัดหน้า : การแต่งหน้า}` เช่น `{price, 10 :N2}`)*

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่างที่ 1: ระบบออกใบเสร็จรับเงิน (Smart Receipt Generator)

โค้ดนี้สาธิตพลังของการแปลงข้อมูลดิบระดับ Decimal ให้กลายเป็นรายงานการเงินที่สวยงามและดูแพง

::: code-group
```csharp [Program.cs]
using System;

namespace ReceiptGenerator
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบออกใบเสร็จร้านกาแฟ (Cafe POS) ===\n");

            // ข้อมูลดิบจากระบบ
            string customer = "John Doe";
            string itemName = "อเมริกาโน่เย็น";
            int quantity = 2;
            double pricePerUnit = 65.5; // เลขแปลกๆ มีทศนิยม 1 ตำแหน่ง
            
            double total = quantity * pricePerUnit; // เป็น 131.0
            double tax = total * 0.07;              // ภาษี 7% เลขทศนิยมมาเต็มแน่นอน

            // -----------------------------------------------------------
            // การใช้ String Interpolation แบบผสมผสานทุกฟีเจอร์
            // -----------------------------------------------------------
            
            // ใช้ @ ผสมกับ $ เพื่อสร้างข้อความแบบรักษาย่อหน้าและฝังตัวแปรได้พร้อมกัน!
            string receipt = $@"
===========================================
             CAFE ZERO TO HERO
===========================================
ลูกค้า: {customer}
วันที่: {DateTime.Now}
-------------------------------------------
รายการสินค้า: {itemName}
จำนวน: {quantity} แก้ว
ราคาต่อแก้ว: {pricePerUnit:C}  <-- ใช้ :C แปลงเป็นเงินบาทให้เอง!
-------------------------------------------
รวมเงิน:       {total:N2} บาท  <-- บังคับทศนิยม 2 ตำแหน่งเสมอ
ภาษี (7%):      {tax:N2} บาท
-------------------------------------------
ยอดสุทธิ:      {total + tax:C}
===========================================
";
            Console.WriteLine(receipt);
        }
    }
}
```
:::

### ตัวอย่างที่ 2: เครื่องจักรวาดตาราง (The Table Architect)

ปัญหาคลาสสิกของ Console Application คือการปริ้นข้อมูลหลายบรรทัดให้ "คอลัมน์ตรงกันเป๊ะ"
วิศวกรที่ไม่มีความรู้ จะใช้ `\t` เคาะแท็บเดาๆ เอา ซึ่งถ้าชื่อคนนึงยาว คนนึงสั้น ตารางจะเบี้ยวพินาศทันที!
มาดูวิธีของสถาปนิกที่ใช้ Alignment `,` ในการบังคับความกว้างคอลัมน์กันครับ

::: code-group
```csharp [Program.cs]
using System;

namespace ConsoleTables
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบตารางรายงานพนักงาน (Employee Roster) ===\n");

            // ข้อมูลพนักงาน (ชื่อ, แผนก, เงินเดือน)
            string[] names = { "Arthur", "Merlin", "LancelotTheGreat" };
            string[] depts = { "Warrior", "Mage", "Knight" };
            double[] salaries = { 15000, 45500.5, 22000 };

            // -----------------------------------------------------------
            // การสร้างส่วนหัวของตาราง (Header)
            // -----------------------------------------------------------
            // สเปคคอลัมน์: 
            // 1. ชื่อ (-20) = ชิดซ้าย กว้าง 20 ช่อง
            // 2. แผนก (-15) = ชิดซ้าย กว้าง 15 ช่อง
            // 3. เงินเดือน (15) = ชิดขวา กว้าง 15 ช่อง (เพราะตัวเลขควรชิดขวาเสมอ หลักหน่วยจะได้ตรงกัน)
            
            Console.WriteLine($"{"Name", -20} | {"Department", -15} | {"Salary (THB)", 15}");
            Console.WriteLine(new string('-', 56)); // ขีดเส้นคั่น 56 ตัว

            // -----------------------------------------------------------
            // การวนลูปวาดข้อมูล (Data Rows)
            // -----------------------------------------------------------
            for (int i = 0; i < names.Length; i++)
            {
                // ผสม Alignment เข้ากับ Format Specifier (:N2) ในปีกกาสุดท้าย!
                string row = $"{names[i], -20} | {depts[i], -15} | {salaries[i], 15:N2}";
                Console.WriteLine(row);
            }
        }
    }
}
```
:::
**ผลลัพธ์บนหน้าจอ (เป๊ะทุกมิลลิเมตร):**
```text
Name                 | Department      |    Salary (THB)
--------------------------------------------------------
Arthur               | Warrior         |       15,000.00
Merlin               | Mage            |       45,500.50
LancelotTheGreat     | Knight          |       22,000.00
```

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

### **โจทย์ที่ 1: เครื่องมือแต่งหน้าสถิติ (The Data Formatter)**
กำหนดตัวแปร: `double successRate = 0.87654;`
จงใช้คำสั่ง `Console.WriteLine` ร่วมกับ String Interpolation `$` เพื่อเสกข้อความนี้ออกหน้าจอ:
`"อัตราความสำเร็จในภารกิจนี้คือ: 87.65%"` 
*(คำใบ้: อย่าเอาไปคูณ 100 เองนะ! ให้ใช้พลังของ Formatting ตัวอักษร `P` พร้อมล็อกเลขท้าย 2 ตัวมาจัดการแทน)*

### **โจทย์ที่ 2: ระบบล็อกอินปลอมตัว (The PadLeft Hacker)**
**ภารกิจ:** ในบางแอปพลิเคชัน รหัสพนักงานจะต้องมี 5 หลักเสมอ (เช่น พนักงานรหัส `42` ต้องถูกโชว์เป็น `00042`)
จงรับ Input ตัวเลขจากผู้ใช้มา 1 ก้อน (เช่น `"7"`) แล้วใช้ String Method `.PadLeft(ความยาวรวม, 'ตัวอักษรที่จะเติม')` เพื่อเสกให้มันกลายเป็นก้อนข้อความความยาว 5 หลัก ที่ถูกอุดด้วยเลข `'0'` ด้านหน้าให้เต็ม!

::: details 💡 คำใบ้และแนวทาง (Hints)
- **สำหรับโจทย์ 1:** คำตอบคือ `Console.WriteLine($"อัตราความสำเร็จในภารกิจนี้คือ: {successRate:P2}");` ง่ายจนน่าตกใจใช่ไหมล่ะครับ!
- **สำหรับโจทย์ 2:** ใช้ `string paddedId = input.PadLeft(5, '0');` คลาส String มีอาวุธซ่อนรูปเยอะมาก PadLeft กับ PadRight เอาไว้ชดเชยช่องว่างด้วยตัวอักษรอะไรก็ได้ตามใจเรา ซึ่งทำงานคล้ายๆ Alignment ของ Interpolation นั่นแหละครับ
:::

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: มหกรรมบิลเงินสดอัจฉริยะ (The Dynamic Invoice Builder)**

นี่คือข้อสอบประเมินผลการจัดหน้ากระดาษของจริง!
คุณได้รับ Array ของสินค้า (ชื่อยาวสั้นไม่เท่ากัน) และ Array ของราคา
```csharp
string[] products = { "Apple", "Watermelon", "Gum" };
double[] prices = { 15.5, 120.0, 5.0 };
```

**ภารกิจของคุณ:** 
จงเขียนลูปเพื่อสร้างบิลเงินสด ที่มีสเปคเข้มงวดระดับองค์กรดังนี้:
1. คอลัมน์ที่ 1 (ชื่อสินค้า) ต้องกว้าง 15 ช่องชิดซ้าย
2. คอลัมน์ที่ 2 (ราคา) ต้องกว้าง 10 ช่องชิดขวา และต้องแสดงผลแบบสกุลเงิน (`:N2` ก็ได้ถ้าไม่มีสกุลบาท)
3. ระหว่างที่วิ่งลูปโชว์สินค้าแต่ละตัว... คุณต้องคำนวณ "ยอดรวมสุทธิ (Grand Total)" สะสมไปด้วย
4. พอจบตาราง ให้ขีดเส้นคั่น และพิมพ์ยอดรวมสุทธิชิดขวา โดยตัวเลขยอดรวมต้องตรงกับหลักหน่วยของราคาสินค้าด้านบนเป๊ะๆ!

ผลลัพธ์ที่คุณควรทำได้:
```text
ITEM            |      PRICE
----------------------------
Apple           |      15.50
Watermelon      |     120.00
Gum             |       5.00
----------------------------
TOTAL           |     140.50
```

ถ้าคุณทำได้เป๊ะขนาดนี้ คุณก็พร้อมไปรับจ้างเขียนระบบออกใบเสร็จให้เซเว่นอีเลฟเว่นได้แล้วครับ!

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** ในเอกสารคู่มือระบุว่า `$"Hello {name}"` จะถูก Compiler แอบแปลงร่าง (Compile-time translation) กลับไปเป็นคำสั่ง `string.Format("Hello {0}", name)` เสมอก่อนรันจริง... ถ้าเป็นแบบนั้น แปลว่า Interpolation `$` ก็ทำงานได้ช้าพอๆ กับ string.Format ใช่หรือไม่? แล้วเราจะได้ประโยชน์อะไรนอกจากพิมพ์สั้นลง?
**แนวคำตอบ:** **ถูกต้องครับ! เบื้องหลังมันคือสิ่งเดียวกันเป๊ะๆ (Performance เท่ากัน)** 
แต่วิศวกรซอฟต์แวร์ยอมรับกันทั่วโลกว่า "เวลาของโปรแกรมเมอร์ (Developer Time) มีค่ามากกว่าเวลาของ CPU (Execution Time)" การใช้ `$` ช่วยลดบั๊กจากมนุษย์ (Human Error) ที่เกิดจากการใส่เลข `{0}, {1}, {2}` สลับตำแหน่งกัน หรือลืมส่งตัวแปรเข้าไปใน Method ให้ครบ ทำให้โค้ด Clean และ Maintain ง่ายขึ้น 1000% ครับ! (แต่ถ้าต้องต่อข้อความในลูปแสนรอบ ทั้งคู่ก็พาโปรแกรมแครชได้พอๆ กันอยู่ดีนะ!)

**คำถาม 2:** โค้ดด้านล่างนี้ เกิดข้อผิดพลาดร้ายแรง (Format Exception) ตอนรันโปรแกรม เพราะอะไร?
```csharp
int money = 500;
string msg = $"คุณได้รับเงิน {money:N2} บาท {ขอขอบคุณ}";
```
**แนวคำตอบ:** **เพราะคุณทะลึ่งเอาปีกกาไปใช้เขียนข้อความธรรมดาครับ!**
เมื่อคุณแปะสัญลักษณ์ `$` ไว้หน้าประโยคแล้ว คอมพิวเตอร์จะถือว่า "ปีกกาทุกตัวคือเขตแดนของตัวแปรหรือโค้ด C# เท่านั้น!" การที่คุณเขียน `{ขอขอบคุณ}` คอมไพเลอร์จะพยายามไปควานหาตัวแปรชื่อ "ขอขอบคุณ" ในระบบ ซึ่งมันไม่มีอยู่จริง! 
**วิธีแก้ระดับโปร:** ถ้าในประโยคที่มี `$` ดันมีคำที่ต้องใช้ปีกกาครอบจริงๆ ให้คุณเบิ้ลปีกกา 2 ตัวติดกันครับ เช่น `{{ขอขอบคุณ}}` คอมไพเลอร์จะรู้ทันทีว่าคุณต้องการปริ้นปีกกาออกมาโชว์เฉยๆ ไม่ได้จะฝังตัวแปร!
:::
