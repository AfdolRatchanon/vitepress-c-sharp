# String Methods (เครื่องมือผ่าตัดข้อความ)

> 💡 **เป้าหมาย:** สวมบทบาท Data Engineer เรียนรู้การหยิบใช้ "มีดผ่าตัดอเนกประสงค์" ที่ฝังมากับคลาส `string` เพื่อทำความสะอาด (Cleanse), ค้นหา (Search), สับ (Split), และสกัดข้อมูล (Extract) ที่สกปรกให้พร้อมใช้งาน

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. หมวดทำความสะอาด (Cleansing Methods)

ข้อมูลที่ได้จากผู้ใช้งาน (User Input) มักจะสกปรกเสมอ! ผู้ใช้มักจะเผลอกด Spacebar ทิ้งไว้ หรือพิมพ์ตัวพิมพ์ใหญ่พิมพ์เล็กสลับกันมั่วซั่ว
- **`.Trim()`**: กรรไกรตัดหญ้า! ใช้ตัด "ช่องว่าง (Space)" ที่อยู่หัวและท้ายข้อความทิ้งไปให้หมด (แต่ช่องว่างตรงกลางประโยคยังคงอยู่นะ)
- **`.ToUpper()` & `.ToLower()`**: เครื่องปรับโทนเสียง! แปลงเป็นตัวพิมพ์ใหญ่ล้วน หรือพิมพ์เล็กล้วน (มีประโยชน์สุดๆ เวลาเอาไป `if` เปรียบเทียบ เพื่อหลีกเลี่ยงปัญหาตัวพิมพ์ไม่ตรงกัน Case-sensitive)

---

### 2. หมวดค้นหาและตรวจสอบ (Inspection Methods)

บ่อยครั้งที่เราต้องทำระบบ AI เพื่อสแกนหาคำหยาบ หรือตรวจสอบว่านี่ใช่ลิงก์เว็บไซต์หรือไม่
- **`.Contains("คำค้น")`**: ถามว่า "มีคำนี้ซ่อนอยู่ในประโยคไหม?" (คืนค่าเป็น `bool`)
- **`.StartsWith("http")`**: ถามว่า "ข้อความนี้ **ขึ้นต้นด้วย** คำว่า http ใช่หรือไม่?" (มักใช้ตรวจ URL)
- **`.EndsWith(".png")`**: ถามว่า "ข้อความนี้ **ลงท้ายด้วย** นามสกุล .png ใช่หรือไม่?" (มักใช้ตรวจไฟล์รูปภาพ)
- **`.IndexOf("A")`**: ถามหาพิกัด! "ตัว A ปรากฏตัวครั้งแรกที่ห้อง (Index) เบอร์อะไร?" (ถ้าหาไม่เจอ จะคืนค่า `-1` เสมอ)

---

### 3. หมวดสับและประกอบร่าง (Split & Join) - ดาบซามูไรขั้นเทพ

นี่คืออาวุธที่ถูกใช้งานบ่อยที่สุดในกระบวนการ Data Analysis!
- **`.Split('สัญลักษณ์')`**: การสับหมู! มันจะวิ่งหาเครื่องหมายที่เรากำหนด (เช่น จุลภาค `,` หรือ ช่องว่าง `' '`) แล้วสับข้อความตรงจุดนั้นให้แตกกระจาย กลายเป็น **"Array ของข้อความ (string[])"** กระเด็นออกมา! (เช่น ใช้สับข้อมูลไฟล์ CSV ออกเป็นคอลัมน์ๆ)
- **`string.Join("กาว", Array)`**: เวทมนตร์ประกอบร่าง! ตรงข้ามกับ Split อันนี้คือการเอา Array ของข้อความที่กระจัดกระจาย มาต่อกันใหม่เป็นข้อความยาวๆ 1 ก้อน โดยเอา "กาว (เช่น เครื่องหมายลูกน้ำ หรือ ช่องว่าง)" ไปทาเชื่อมตรงรอยต่อให้สวยงาม

---

### 4. หมวดศัลยกรรมตกแต่ง (Extraction & Replacement)

- **`.Substring(จุดเริ่ม, จำนวนกี่ตัว)`**: การหั่นสเต็ก! เอาไว้ "ดึงเฉพาะท่อนกลาง" ของข้อความออกมา เช่น ถ้าข้อความเป็นเบอร์โทร `"081-123-4567"` เราสามารถระบุตำแหน่งเพื่อหั่นเอาเฉพาะเบอร์ 4 ตัวหลังออกมาโชว์ได้
- **`.Replace("เก่า", "ใหม่")`**: เวทมนตร์เปลี่ยนผ้าปูโต๊ะ! สั่งค้นหาคำเก่าทั้งหมดที่โผล่ในประโยค แล้วเอาคำใหม่ไปเสียบทับแทนที่ (มักใช้ในการแบนคำหยาบแบบเซ็นเซอร์ให้เป็น `***`)

*(⚠️ คำเตือนสำคัญจากกฎ Immutability: เมธอดทุกตัวที่กล่าวมาข้างต้น จะ **"ไม่เคยแตะต้องหรือเปลี่ยนค่ากล่องต้นฉบับเลย"** แต่มันจะ "คืนค่า (Return) กล่องใบใหม่ที่ถูกปรับปรุงแล้ว" กลับมาให้ ดังนั้นคุณต้องใช้ตัวแปรมารับค่ามันเสมอ!)*

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่างที่ 1: ระบบล้างข้อมูลลงทะเบียน (Registration Cleanser)

โค้ดนี้จำลองสถานการณ์จริงเวลาผู้ใช้กรอกข้อมูลฟอร์มสมัครสมาชิกที่สกปรกสุดๆ

::: code-group
```csharp [Program.cs]
using System;

namespace DataCleansing
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบตรวจสอบและล้างข้อมูลอีเมล ===");

            // ข้อมูลดิบที่ผู้ใช้กรอก (เว้นวรรคหน้าหลังบานเบอะ แถมพิมพ์ใหญ่สลับพิมพ์เล็ก)
            string dirtyEmail = "   JoHn.DoE@GmaiL.com   ";
            Console.WriteLine($"[ข้อมูลดิบ] : '{dirtyEmail}'");

            // ----------------------------------------------------
            // กระบวนการ Pipeline ล้างข้อมูล (Chain Methods)
            // ----------------------------------------------------
            
            // 1. ใช้ Trim ตัดช่องว่างทิ้ง
            string step1 = dirtyEmail.Trim();
            
            // 2. แปลงเป็นพิมพ์เล็กให้หมด จะได้ง่ายต่อระบบฐานข้อมูล
            string cleanEmail = step1.ToLower();

            // 💡 ทริคระดับโปร: เราสามารถเอา Method มา "ต่อคิว (Chain)" กันในบรรทัดเดียวได้เลย!
            // string cleanEmail = dirtyEmail.Trim().ToLower();

            Console.WriteLine($"[ล้างสำเร็จ] : '{cleanEmail}'");

            // ----------------------------------------------------
            // กระบวนการตรวจสอบความถูกต้อง (Validation)
            // ----------------------------------------------------
            if (cleanEmail.EndsWith("@gmail.com"))
            {
                Console.ForegroundColor = ConsoleColor.Green;
                Console.WriteLine("✅ สมัครสำเร็จ: นี่คือบัญชี Gmail ที่ถูกต้อง");
            }
            else
            {
                Console.ForegroundColor = ConsoleColor.Red;
                Console.WriteLine("❌ ล้มเหลว: ระบบรองรับเฉพาะ @gmail.com เท่านั้น");
            }
            Console.ResetColor();
        }
    }
}
```
:::

### ตัวอย่างที่ 2: ผ่าตัดข้อมูลลับด้วย Substring และ Replace

ถ้าคุณจะทำระบบโชว์เบอร์โทรศัพท์ลูกค้า แต่ต้องเซ็นเซอร์ตัวเลขตรงกลางให้เป็นความลับ (เช่น `081-xxx-4567`) 

::: code-group
```csharp [Program.cs]
using System;

namespace SecretExtraction
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบศัลยกรรมข้อความ ===");

            string idCard = "1-1234-56789-01-2"; // สมมติว่านี่คือเลขบัตร ปชช. ที่มีขีดกั้น

            // 1. ตัดขีดทิ้งด้วย Replace (แปลงเครื่องหมายขีด ให้กลายเป็น "ความว่างเปล่า")
            string pureNumbers = idCard.Replace("-", "");
            Console.WriteLine($"[หลังตัดขีด] {pureNumbers} (เหลือ {pureNumbers.Length} หลัก)");

            // 2. ดึงรหัส 4 ตัวสุดท้ายมาโชว์ (Substring)
            // ลอจิก: ตัดจากตำแหน่ง Index = ความยาวทั้งหมด - 4, เอามา 4 ตัว
            int startIndex = pureNumbers.Length - 4;
            string lastFour = pureNumbers.Substring(startIndex, 4);
            
            Console.WriteLine($"[รหัสความปลอดภัย] xxxxxxxx{lastFour}");

            // -----------------------------------------------------------
            // ตัวอย่างที่ 3: ดาบซามูไร Split (แยกข้อมูล CSV)
            // -----------------------------------------------------------
            Console.WriteLine("\n[กระบวนการสับข้อมูล (Split)]");
            string csvData = "อาร์เธอร์,นักรบ,ระดับ99,มีชีวิต";
            
            // สับด้วยเครื่องหมาย จุลภาค ',' สิ่งที่ได้ออกมาคือ Array ของข้อความ!
            string[] columns = csvData.Split(',');

            Console.WriteLine($"สับได้ทั้งหมด {columns.Length} ช่องข้อมูล:");
            Console.WriteLine($"> ชื่อ  : {columns[0]}");
            Console.WriteLine($"> อาชีพ : {columns[1]}");
            Console.WriteLine($"> เลเวล : {columns[2]}");
        }
    }
}
```
:::

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

### **โจทย์ที่ 1: การเปลี่ยนนามสกุลไฟล์ (File Extractor)**
กำหนดตัวแปร `string filename = "profile_picture.JPG";` 
จงเขียนลอจิกบรรทัดเดียว หรือสองบรรทัด ที่ทำการดึงเอาข้อมูลมาตรวจสอบแบบรัดกุม โดยไม่สนใจตัวพิมพ์เล็กใหญ่ เพื่อให้คอมพิวเตอร์ตอบให้ได้ว่า ไฟล์นี้เป็นรูป `.jpg` ใช่หรือไม่? (ระวัง: ไฟล์ต้นฉบับมันพิมพ์ใหญ่ `.JPG`)

### **โจทย์ที่ 2: ระบบเซ็นเซอร์คำหยาบ 18+ (Censorship)**
**ภารกิจ:** รับ Input ประโยคจากผู้ใช้ เช่น `"ไอ้คนบ้า แกมันบ้าที่สุดเลยว่ะ"`
จงใช้พลังของ `.Replace()` เปลี่ยนคำว่า `"บ้า"` ทุกคำที่เจอ ให้กลายเป็นสัญลักษณ์ `"***"` และปริ้นออกมาโชว์
*(คำใบ้: อย่าลืมกฎ Immutability นะ! `.Replace` มันไม่แก้ตัวแปรเดิมให้ คุณต้องสร้างตัวแปรใหม่มารับค่ามันด้วย!)*

::: details 💡 คำใบ้และแนวทาง (Hints)
- **สำหรับโจทย์ 1:** คุณมี 2 ทางเลือก! ท่ามาตรฐานคือ `filename.ToLower().EndsWith(".jpg")` หรือท่าระดับโปรคือใช้ `filename.EndsWith(".jpg", StringComparison.OrdinalIgnoreCase)` ซึ่งเป็นฟีเจอร์แฝงที่ช่วยเปรียบเทียบข้อความแบบละทิ้งอคติเรื่องตัวพิมพ์เล็ก/ใหญ่ได้เลย!
- **สำหรับโจทย์ 2:** โค้ดง่ายมากครับ `string cleanMsg = inputMsg.Replace("บ้า", "***");` จบเลย! เวทมนตร์ของไมโครซอฟต์จะกวาดหาคำว่าบ้าทุกจุดในประโยคและสับเปลี่ยนให้เองอัตโนมัติ
:::

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: นักประมวลผลโดเมน (The Email Domain Extractor)**

บริษัทของคุณกำลังทำระบบรวบรวมรายชื่อลูกค้าเพื่อวิเคราะห์การตลาด (ว่าลูกค้าใช้เมลค่ายไหนเยอะสุด)
แต่ลูกค้าชอบกรอกข้อมูลมาแปลกๆ 

**ข้อมูลต้นฉบับ (Raw Data):**
```csharp
string[] rawEmails = {
    "   admin@google.com  ",
    "HELLO@Yahoo.com",
    "ceo@MICROSOFT.com",
    "invalid_email_format" // ⚠️ ตัวกวนประสาท (ไม่มี @)
};
```

**ภารกิจของคุณ:** จงใช้ลูป `foreach` วิ่งกวาด Array นี้ทีละอัน แล้วดึง **"เฉพาะชื่อค่ายอีเมล (Domain)"** ออกมาให้เป็นตัวพิมพ์เล็กทั้งหมด!
ผลลัพธ์ที่คุณต้องปริ้นลงจอคือ:
```text
> google.com
> yahoo.com
> microsoft.com
> [Error] รูปแบบอีเมลไม่ถูกต้อง!
```

**Algorithm Guide (วิชาระดับเซียน):**
1. ทุกรอบที่ดึงเมลออกมา ต้องเอามา `.Trim()` เคลียร์ช่องว่างผีสิงก่อน
2. เอามา `.ToLower()` ให้เป็นพิมพ์เล็กล้วน
3. เช็ค `Contains("@")` ก่อนเลย! ถ้าไม่มีให้ด่าว่า Error แล้วใช้ `continue;` ข้ามไปคนต่อไป
4. **จุดวัดกึ๋น (Extraction):** การดึงเฉพาะโดเมนหลัง `@` ทำได้ 2 วิธี:
   - **วิธีวิศวกรถึก (Substring):** ใช้ `.IndexOf("@")` เพื่อหาพิกัดว่า @ อยู่เบอร์อะไร แล้วบวก 1 ขยับไป 1 ก้าว เพื่อหั่น `.Substring(ตำแหน่งที่บวกแล้ว)` ดึงยาวไปจนสุดประโยค
   - **วิธีสถาปนิก (Split):** เอาประโยคมา `.Split('@')` มันจะหั่นเมลขาดครึ่งกลางอากาศกลายเป็น Array 2 ช่อง (ช่อง 0 คือชื่อหน้า, ช่อง 1 คือโดเมน) เราก็แค่หยิบช่อง `[1]` มาตอบ! ง่ายกว่าเยอะ!

จงเลือกใช้วิธีสถาปนิก (Split) แล้วคุณจะรู้ว่าเครื่องมือ C# ออกแบบมาเพื่อลบความปวดหัวของโปรแกรมเมอร์ได้ดีแค่ไหน!

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** โค้ดด้านล่างนี้ เกิดข้อผิดพลาดทางสถาปัตยกรรม (Logical Bug) ขึ้น ทำไมคำที่ปริ้นออกมาถึงยังคงตัวเล็กเหมือนเดิม ไม่กลายเป็นตัวใหญ่?
```csharp
string myName = "john";
myName.ToUpper();
Console.WriteLine(myName);
```
**แนวคำตอบ:** **เพราะคุณแหกกฎ Immutability ครับ!**
`myName.ToUpper()` คอมพิวเตอร์ทำตามคำสั่งคุณอย่างดี สร้างกล่องคำว่า `"JOHN"` พิมพ์ใหญ่สวยงามลอยเคว้งอยู่ในแรม... แต่คุณดัน **"ไม่ยอมเอาตัวแปรไปรับค่านั้นกลับมา"** (มันคืนค่ากลับมาแล้วคุณปล่อยทิ้งลงพื้น) พอบรรทัดต่อมาคุณไปสั่งปริ้นตัวแปร `myName` ตัวเดิม มันก็ย่อมชี้ไปที่กล่อง `"john"` ต้นฉบับใบเดิมที่ไม่มีวันถูกแก้ไขได้นั่นเองครับ! 
วิธีแก้คือต้องรับค่าทับไปเลย: `myName = myName.ToUpper();`

**คำถาม 2:** สมมติคุณมีประโยค `"กล้วย,ส้ม,แอปเปิ้ล"` และคุณต้องการเปลี่ยนเครื่องหมายจุลภาค `,` ให้กลายเป็นเครื่องหมายขีดคั่น ` - ` แทน (ผลลัพธ์เป็น `"กล้วย - ส้ม - แอปเปิ้ล"`) คุณมี 2 เครื่องมือให้เลือกใช้ คือ `Replace` และการนำ `Split` ไปตีบวกกับ `Join` คุณคิดว่ากรณีไหนเร็วกว่ากัน และควรใช้แบบไหน?
**แนวคำตอบ:** **ใช้ `Replace` ชนะขาดลอยด้านความเร็ว (Performance) และความสะอาดของโค้ด!**
ถ้าเราใช้ `string.Join(" - ", words.Split(','))` มันเท่ก็จริงครับ แต่มันเป็นความสิ้นเปลืองระดับจักรวาล! เพราะ `Split` จะต้องจองแรมเพื่อสร้าง Array ลอยขึ้นมาในอากาศ ยัดผลไม้ลงไป 3 ช่อง แล้ว `Join` ก็ต้องลากมันกลับมาประกอบร่างใหม่... ในขณะที่ `.Replace(",", " - ")` สามารถวิ่งลูปสแกนและทะลวงเปลี่ยนข้อความได้โดยตรงในระดับหน่วยความจำ (สร้าง String ก้อนใหม่แค่ใบเดียว) ดังนั้นจงเลือกเครื่องมือให้เหมาะสมกับงานเสมอครับ!
