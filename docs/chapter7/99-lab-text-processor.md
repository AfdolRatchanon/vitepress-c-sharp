# 💻 Lab: Text Processor (ระบบล้างข้อมูลดิบอัจฉริยะ)

> 💡 **เป้าหมายของ Lab นี้:** จำลองการเป็นนักพัฒนาซอฟต์แวร์สายจัดการฐานข้อมูล (Data Cleanser) คุณจะต้องประยุกต์ใช้อาวุธทุกอย่างในหมวดข้อความ: เครื่องมือล้างข้อมูล (`Trim`, `Replace`, `Split`), พลังเวทมนตร์จัดรูปแบบ (`Interpolation`), และคลังแสงทำลายล้างความตายตัว (`StringBuilder`) เพื่อสร้างโปรแกรมรายงานสรุปข้อความขนาดยาว!

## 📖 ภาพรวมของโปรแกรม (Program Overview)

ในฐานะนักวิเคราะห์ข้อมูลของบริษัท คุณได้รับ **"บทความรีวิวสินค้าขนาดยาว (Raw Text Block)"** ที่ลูกค้าพิมพ์มาแบบไม่ได้ตั้งใจ (มีช่องว่างเกิน มีคำหยาบ และพิมพ์ใหญ่พิมพ์เล็กมั่วไปหมด)
คุณต้องสร้างเครื่องจักร (Console App) ที่สามารถดูดข้อความดิบนี้เข้าไป แล้วบ้วน "รายงานเชิงสถิติที่สวยงาม" ออกมาให้ทีมการตลาดอ่านได้ง่ายๆ

### ตัวอย่างหน้าจอเมื่อรันโปรแกรมสำเร็จ

```text
================================================
          [ TEXT PROCESSOR ENGINE ]
================================================
[ข้อมูลดิบ]:
"   Hello world! this product is VERY bad.  i hate it so much. but the delivery was fast.   "

[กำลังทำงาน... ชำแหละและทำความสะอาด...]

================================================
              [ DATA REPORT ]
================================================
> จำนวนประโยคทั้งหมด: 3 ประโยค
> จำนวนคำทั้งหมด: 15 คำ

[ เนื้อหาที่ผ่านการทำความสะอาด (Cleaned Text) ]
Hello world! This product is VERY ***. I hate it so much. But the delivery was fast.

[ คำค้นหาสำคัญ (Keywords Found) ]
- พบคำว่า 'delivery': ใช่
================================================
```

## 📋 กฎกติกาและเกณฑ์การประเมิน (Business Requirements)

คุณต้องเขียนโปรแกรมให้ทำงานเป็นสัดส่วน (Step-by-step Pipeline) โดยปฏิบัติตามกฎเหล็กดังนี้:

1. **ระบบป้อนข้อมูล (Input Pipeline):**
   - ให้ประกาศตัวแปร `string rawText;` แล้วกำหนดค่าให้เป็นข้อความที่แสนจะสกปรก (ตามตัวอย่างด้านบน)
   - *คำแนะนำ:* ไม่ต้องให้ผู้ใช้พิมพ์เองหรอก มันยาวไป! ให้ Hardcode ค่าลงไปในโค้ดเลย

2. **ระบบนับสถิติ (Statistics Engine):**
   - **นับประโยค (Sentence Count):** ในภาษาอังกฤษ 1 ประโยคจบด้วยเครื่องหมายจุด `.` (ให้ใช้ฟีเจอร์ `.Split('.')` เพื่อแยกประโยค และนับจำนวน Array ที่ได้มา)
   - **นับคำ (Word Count):** คำแต่ละคำถูกคั่นด้วยเครื่องหมายช่องว่าง `' '` (ให้สับมันด้วย `.Split(' ')` เช่นกัน) 
   - *ข้อควรระวัง:* ข้อมูลดิบมีช่องว่างซ้อนกันหลายตัว (`"   "`) ถ้าสับตรงๆ มันจะได้ช่องว่างผีสิงแถมมาด้วย! คุณต้องหาทาง `.Trim()` ทิ้งก่อนสับ หรือศึกษาวิชา `StringSplitOptions.RemoveEmptyEntries` เพื่อล้างขยะออก!

3. **ระบบเซ็นเซอร์ (Censorship):**
   - เจ้านายสั่งให้ลบคำหยาบ! ให้ใช้พลังของ `.Replace()` ตรวจหาคำว่า `"bad"` (และคำหยาบอื่นๆ ที่คุณอยากเติม) แล้วแทนที่ด้วย `"***"` ซะ

4. **ระบบสร้างรายงาน (Report Generator):**
   - **ห้ามใช้** `Console.WriteLine()` ค่อยๆ ปริ้นทีละบรรทัดเด็ดขาด!
   - ภารกิจบังคับ: คุณต้องสร้าง `StringBuilder report = new StringBuilder();`
   - แล้วใช้ท่า `.AppendLine()` เพื่อก่ออิฐสร้างรายงานทั้งหมด (หัวตาราง, สถิติ, ข้อความที่คลีนแล้ว) ให้ครบถ้วนในโกดัง
   - ตอนสุดท้ายค่อยสั่ง `Console.WriteLine(report.ToString());` เพื่อพ่นรายงานออกจอในทีเดียว! (หลักการ Buffer Rendering)

## 📝 ขั้นตอนการพัฒนา (Step-by-Step Implementation)

### ขั้นตอนที่ 1: เตรียมเตียงผ่าตัด (Data Preparation)
- [ ] เปิดคลาส `Program` และ import `using System.Text;` ไว้บนสุด
- [ ] เตรียมข้อมูลดิบทดสอบ: 
```csharp
string rawText = "   Hello world! this product is VERY bad.  i hate it so much. but the delivery was fast.   ";
```
- [ ] ขั้นแรก ล้างขยะหัวท้ายทิ้งซะ: `string cleanText = rawText.Trim();`

### ขั้นตอนที่ 2: ผ่าตัดเปลี่ยนอะไหล่ (Censorship)
- [ ] นำ `cleanText` มาสแกนหาคำหยาบ
- [ ] สมมติเราจะแบนคำว่า "bad" และ "hate"
```csharp
// ใช้ Method Chaining เพื่อความเท่!
cleanText = cleanText.Replace("bad", "***").Replace("hate", "love");
```

### ขั้นตอนที่ 3: เครื่องจักรสับละเอียด (Analytics Splitter)
- [ ] สับหา "ประโยค" โดยใช้เครื่องหมายจุด `.`
```csharp
string[] sentences = cleanText.Split('.');
// คำเตือน: ประโยคสุดท้ายมีจุดจบประโยค พอมันสับ มันอาจจะได้ช่องว่างเปล่าๆ มา 1 กล่องเป็นของแถม!
// คุณต้องลบออก 1 หรือใช้วิธีเช็คความว่างเปล่า
```
- [ ] สับหา "คำศัพท์" โดยใช้ช่องว่าง `' '`
```csharp
string[] words = cleanText.Split(new char[] { ' ' }, StringSplitOptions.RemoveEmptyEntries);
```

### ขั้นตอนที่ 4: เตาอบรายงาน (The StringBuilder Oven)
- [ ] สร้างเตาอบ: `StringBuilder sb = new StringBuilder();`
- [ ] เริ่มโยนอิฐใส่เตา: `sb.AppendLine("=== DATA REPORT ===");`
- [ ] โยนตัวแปรสถิติลงไป: `sb.AppendLine($"> จำนวนคำ: {words.Length} คำ");`
- [ ] โยนข้อความที่คลีนแล้วลงไป
- [ ] รีดผลลัพธ์และโชว์ออกจอ: `Console.WriteLine(sb.ToString());`

## 🔥 Challenge (ภารกิจท้าทายขีดจำกัด)

สำหรับวิศวกรที่ต้องการแสดงแสนยานุภาพด้าน Algorithm:

**ภารกิจ The Auto-Capitalizer (เครื่องแปลงตัวใหญ่หัวประโยคอัตโนมัติ):**
สังเกตไหมว่าในข้อมูลดิบ มีประโยคที่ขึ้นต้นด้วยตัวพิมพ์เล็ก (`"i hate it..."`, `"but the delivery..."`) ซึ่งผิดหลักไวยากรณ์ภาษาอังกฤษอย่างรุนแรง!
*เครื่องมืออย่าง `.ToUpper()` ธรรมดาๆ ช่วยคุณไม่ได้ เพราะมันจะแปลงใหญ่หมดทั้งประโยค!*

**ภารกิจของคุณ:** 
1. เมื่อคุณใช้ `.Split('.')` สับประโยคแยกออกมาเป็น Array แล้ว...
2. ให้ใช้ลูป `for` วิ่งกวาดทีละประโยคใน Array
3. ในแต่ละประโยค คุณต้องไป **"หั่น (Substring) ดึงเอาเฉพาะตัวอักษรแรกสุด"** (ที่ Index 0) ออกมาทำการ `.ToUpper()`
4. แล้วค่อยเอาส่วนหัวที่ตัวใหญ่แล้ว ไปประกบติด (Concat) กับ **"ท่อนที่เหลือทั้งหมด (Substring ตั้งแต่ Index 1 เป็นต้นไป)"**
5. สุดท้าย นำประโยคที่ซ่อมเสร็จแล้ว มาประกอบร่างกลับคืน (ใช้ `string.Join` หรือโยนใส่ `StringBuilder`) ให้กลายเป็นย่อหน้าที่สมบูรณ์แบบ!

ใครเขียนลอจิกบรรทัดนี้ได้สำเร็จ คุณคือปรมาจารย์แห่งการเล่นแร่แปรธาตุข้อความ (String Manipulator) อย่างแท้จริงครับ!
