# บทที่ 7: Strings in Depth (เจาะลึกศิลปะแห่งข้อความ)

> 💡 **ภาพรวม:** ในโลกของการเขียนโปรแกรมจริง 90% ของข้อมูลที่วิ่งเข้าออกระบบ (เช่น ข้อมูลจาก Web API, ข้อมูลจาก Database, หรือข้อมูลที่ผู้ใช้พิมพ์ในช่อง Login) ล้วนอยู่ในรูปแบบของ **ข้อความ (String)** ทั้งสิ้น! บทนี้จะพาคุณเจาะลึกเข้าไปในวิศวกรรมการจัดการข้อความ เพื่อเปลี่ยนคุณจากคนที่แค่พิมพ์ `Console.WriteLine("Hello")` ให้กลายเป็นผู้เชี่ยวชาญด้านการดัดแปลงและทำความสะอาดข้อมูลระดับ Data Engineer

## 🎯 สิ่งที่จะได้เรียนรู้ในบทนี้

- ✅ **String Immutability:** ความลับระดับสถาปัตยกรรมที่ว่า "ข้อความใน C# เปลี่ยนแปลงค่าไม่ได้!" และอันตรายจากการต่อสตริงมั่วๆ
- ✅ **Escape Sequences & Verbatim Strings:** ศิลปะการจัดหน้ากระดาษ และการจัดการ Path ไฟล์ที่น่าปวดหัว
- ✅ **String Methods (เครื่องมือผ่าตัดข้อความ):** การสับ (`Split`), การต่อ (`Join`), การค้นหา (`Contains`), และการแทนที่ (`Replace`)
- ✅ **String Interpolation & Formatting:** การปั้นตัวเลขและข้อความให้เป็น Report ที่สวยงาม
- ✅ **`StringBuilder`:** อาวุธหนักสำหรับการสร้างข้อความขนาดยักษ์ (ที่ช่วยชีวิต Server ไม่ให้แรมเต็ม)

## 🔗 Prerequisite (ความรู้ที่ต้องมีก่อน)

> ⚠️ เพื่อความราบรื่นในการผ่าตัดข้อความ คุณควรแม่นยำในเรื่องเหล่านี้:
- **Array 1 มิติ (บทที่ 6):** เพราะโดยเนื้อแท้แล้ว `string` คือ "โกดังเก็บตัวอักษร (`char[]`)"
- **Methods & Parameters (บทที่ 5):** เราจะใช้ Method สำเร็จรูปที่มากับ String อย่างหนักหน่วง
- **Loops (บทที่ 4):** การวิ่งกวาดสายตาทีละอักษรในประโยค

## 🚀 ทำไมเราต้อง "เจาะลึก" เรื่องแค่นี้?

หลายคนคิดว่าข้อความก็แค่ตัวหนังสือ จะมีอะไรซับซ้อน? ลองนึกถึงสถานการณ์เหล่านี้ครับ:
1. **ระบบ Login:** ผู้ใช้เผลอกด Spacebar ทิ้งไว้ตอนพิมพ์ Email (เช่น `"  john@gmail.com "`) ระบบคุณจะพังไหมถ้าไม่ลบช่องว่างทิ้งก่อน?
2. **ระบบฐานข้อมูล:** ข้อมูลวันเกิดถูกส่งมาเป็น `"1990-12-31"` คุณจะเอามีดสับข้อความนี้ยังไงให้ได้แค่ ปีเกิด (`1990`) ออกมาคำนวณอายุ?
3. **ระบบ Log ของ Server:** ถ้าคุณเอาเครื่องหมาย `+` มาต่อข้อความเพื่อเขียน Log ลงไฟล์วินาทีละ 10,000 บรรทัด Server ของคุณอาจจะแรมเต็มและดับไปเลยภายใน 5 นาที (เพราะกฎ Immutability)

เตรียมมีดผ่าตัดของคุณให้พร้อม แล้วไปทำความสะอาดโลกแห่งข้อความกันครับ!
