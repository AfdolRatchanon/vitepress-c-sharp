# String Basics & Immutability (พื้นฐานและความตายตัว)

> 💡 **เป้าหมาย:** ทำความเข้าใจ "ธาตุแท้" ของตัวแปร `string` ในภาษา C# เรียนรู้วิธีการจัดการอักขระพิเศษ (Escape Characters) และเจาะลึกความลับระดับแรมที่ชื่อว่า **Immutability (ความไม่สามารถเปลี่ยนแปลงได้)** ซึ่งเป็นกุญแจสำคัญสู่การเขียนแอปพลิเคชันที่กินทรัพยากรน้อย

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. ธาตุแท้ของ String (The Array of Chars)

แม้เราจะคุ้นเคยกับการประกาศตัวแปร `string name = "John";` และมองมันเหมือนก้อนข้อมูล 1 ก้อน 
แต่ในทางสถาปัตยกรรม (Architecture) **`string` คือ `Array` ของ `char` (ตัวอักษร)** ที่เอามาเรียงต่อกัน!

ดังนั้น กฎทุกอย่างของ Array (จากบทที่ 6) สามารถนำมาใช้กับ String ได้ทั้งหมด!
- มันมี `Length` เพื่อบอกจำนวนตัวอักษร
- มันเป็น Zero-based Indexing (ตัวอักษรแรกคือช่องเบอร์ 0)
- เราสามารถใช้ `name[0]` เพื่อดึงตัว 'J' ออกมาได้เลย
- เราสามารถใช้ `foreach (char c in name)` เพื่อวิ่งอ่านทีละตัวอักษรได้

---

### 2. อักขระหลบหนี (Escape Sequences)

บ่อยครั้งที่เราต้องพิมพ์ข้อความที่ "ขัดแย้งกับไวยากรณ์โค้ด" เช่น ถ้าเราอยากพิมพ์เครื่องหมายฟันหนู `"` ลงไปบนหน้าจอ เราจะพิมพ์แบบนี้ไม่ได้: `Console.WriteLine("เขาพูดว่า "สวัสดี" ครับ");` (เพราะคอมไพเลอร์จะสับสนว่าฟันหนูตัวไหนคือตัวปิดประโยค)

เราจึงต้องใช้ **Backslash `\`** (ตัวหลบหนี) เพื่อบอกคอมพิวเตอร์ว่า "อักษรตัวถัดไปคืออักษรพิเศษนะ ไม่ใช่โค้ด!"
- `\"` : พิมพ์เครื่องหมายฟันหนู
- `\n` : ขึ้นบรรทัดใหม่ (New Line)
- `\t` : กดแท็บ (Tab) ให้ข้อความเว้นวรรคยาวๆ
- `\\` : พิมพ์ตัว Backslash ออกมาจริงๆ (มักใช้กับการพิมพ์ Path ของ Windows เช่น `C:\\Windows\\System32`)

---

### 3. Verbatim Strings (`@`) (เวทมนตร์ปิดผนึก)

การพิมพ์ `\\` ซ้อนกันเพื่อทำ File Path เป็นเรื่องน่าปวดหัวมาก Microsoft จึงให้เครื่องหมาย **`@` (At-sign)** มาวางหน้าข้อความ 
มันคือคำสั่ง **"ปิดผนึก (Verbatim)"** ที่บอกคอมพิวเตอร์ว่า "ไม่ต้องสนใจเครื่องหมาย `\` ใดๆ ทั้งสิ้น! ฉันพิมพ์อะไรลงไป ให้โชว์ออกมาแบบนั้นเป๊ะๆ เลย!" (สามารถเคาะ Enter ขึ้นบรรทัดใหม่กลางโค้ดได้ด้วย)

```csharp
// แบบปกติ (ต้องใช้ \\)
string path1 = "C:\\Users\\Admin\\Documents\\file.txt";

// แบบ Verbatim (ใช้ @ วางข้างหน้า พิมพ์ \ ได้ตรงๆ เลย)
string path2 = @"C:\Users\Admin\Documents\file.txt";

// พิมพ์ขึ้นบรรทัดใหม่ตรงๆ กลางโค้ดได้เลย!
string poem = @"Roses are red,
Violets are blue,
C# is awesome,
And so are you!";
```

### 4. ⚠️ กฎแห่งความตายตัว (String Immutability) - ระวังแรมพินาศ!

นี่คือหัวใจสำคัญที่สุดของวิศวกรรม C#! 
**"เมื่อใดก็ตามที่ String ถูกสร้างขึ้นมาในหน่วยความจำแล้ว... มันจะไม่สามารถถูกแก้ไขได้อีกตลอดกาล (Immutable)!"**

สมมติคุณเขียนโค้ด:
```csharp
string msg = "Hello";
msg = msg + " World";
```
คุณอาจคิดว่าคอมพิวเตอร์แค่เอาคำว่า " World" ไปต่อท้ายคำว่า "Hello" ในกล่องใบเดิมใช่ไหม? **ผิดถนัดครับ!**
สิ่งที่คอมพิวเตอร์ทำเบื้องหลังคือ:
1. สร้างกล่องข้อความ `"Hello"` ในแรม
2. สร้างก้อนข้อความ `" World"` ในแรม
3. **สร้างกล่องข้อความใบใหม่เอี่ยมเลย** ชื่อว่า `"Hello World"` 
4. ทิ้งกล่อง `"Hello"` ใบเก่าให้กลายเป็นขยะ (Garbage) ล่องลอยอยู่ในหน่วยความจำ!

ถ้าคุณเอาเครื่องหมาย `+=` ไปใช้บวกข้อความในลูป 10,000 รอบ คอมพิวเตอร์จะต้องสร้างกล่องข้อความใหม่แล้วทิ้งเป็นขยะ 10,000 ใบ! นี่คือสาเหตุอันดับ 1 ที่ทำให้โปรแกรมของมือใหม่กินแรมมหาศาลและทำงานกระตุก (ซึ่งเราจะไปเรียนวิธีแก้ปัญหานี้ในเรื่อง StringBuilder ครับ)

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่างที่ 1: ศิลปะการจัดหน้ากระดาษ (ASCII Art & Pathing)

โค้ดนี้สาธิตความแตกต่างของการใช้อักขระพิเศษ `\n`, `\t` เทียบกับการใช้ Verbatim String `@` 

::: code-group
```csharp [Program.cs]
using System;

namespace StringBasics
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบรายงานตัวละคร ===");

            // 1. การใช้ Escape Sequences จัดรูปแบบ
            string report1 = "ชื่อ:\tอาร์เธอร์\nอาชีพ:\tอัศวิน\nพลัง:\t100";
            Console.WriteLine("[แบบดั้งเดิม]");
            Console.WriteLine(report1);
            Console.WriteLine();

            // 2. การใช้ Verbatim String (จัดบรรทัดตามที่ตาเห็นในโค้ดเป๊ะๆ)
            string report2 = @"[แบบ Verbatim]
ชื่อ:   เมอร์ลิน
อาชีพ:  พ่อมด
พลัง:   999";
            Console.WriteLine(report2);
            Console.WriteLine();

            // 3. การผสมผสาน: ใส่ Path ระบบ และคำพูดคน (ฟันหนู)
            // ถ้าใช้ @ เราไม่ต้องเบิ้ล \\ แต่ถ้าอยากพิมพ์ฟันหนูใน @ เราต้องเบิ้ลฟันหนู "" สองตัวติดกัน!
            string systemLog = @"C:\Game\Logs\error.txt : บันทึกไว้ว่า ""Server Disconnected"" เมื่อวานนี้";
            Console.WriteLine(systemLog);
        }
    }
}
```
:::

### ตัวอย่างที่ 2: ผ่าตัด String ประหนึ่งมันคือ Array

การมอง String ให้เป็นก้อน Array ถือเป็นสกิลพื้นฐานที่สำคัญมาก

::: code-group
```csharp [Program.cs]
using System;

namespace StringAsArray
{
    class Program
    {
        static void Main(string[] args)
        {
            string secretWord = "C-SHARP";

            Console.WriteLine($"คำรหัสลับคือ: {secretWord}");
            Console.WriteLine($"มีความยาว: {secretWord.Length} ตัวอักษร");

            // ดึงตัวอักษรแรก (Index 0) และตัวอักษรสุดท้าย (Length - 1)
            Console.WriteLine($"ตัวแรกสุด: {secretWord[0]}");
            Console.WriteLine($"ตัวสุดท้าย: {secretWord[secretWord.Length - 1]}");

            // 💥 พยายามแก้ไขอักษรตัวแรกให้เป็น 'D'
            // secretWord[0] = 'D'; 
            // ^ ถ้าคุณเอาคอมเมนต์บรรทัดนี้ออก โปรแกรมจะพังตอน Compile ทันที! 
            // เพราะกฏ Immutability ไม่อนุญาตให้แก้ของในกล่อง String หน้าด้านๆ แบบนี้

            Console.WriteLine("\n[ชำแหละอักขระทีละตัว]");
            // ใช้ลูปกวาดสายตาแบบเดียวกับ Array เป๊ะ
            for (int i = 0; i < secretWord.Length; i++)
            {
                Console.WriteLine($"Index {i} = {secretWord[i]}");
            }
        }
    }
}
```
:::

### **โจทย์ที่ 1: การหลบหนีในเขาวงกต (Escape The Path)**
จงประกาศตัวแปร `string sqlCommand` เพื่อเก็บประโยคคำสั่งนี้ (ต้องพิมพ์ให้ได้หน้าตาแบบนี้เป๊ะๆ บน Console):
```text
SELECT * FROM Users WHERE Name = "John" AND FilePath = "C:\Data\images"
```
**คำถาม:** คุณจะเขียนโค้ดกำหนดค่าให้ตัวแปรนี้อย่างไร (ลองเขียนให้ดูทั้งแบบปกติ และแบบใช้ `@`)

### **โจทย์ที่ 2: เครื่องนับพื้นที่ว่าง (The Space Counter)**
**ภารกิจ:** จงรับข้อความยาวๆ จากผู้ใช้ (เช่น `"Hello World Programming in C# "`) 
แล้วจงใช้ความรู้เรื่อง String คือ `Array` ของ `char` ในการวิ่งลูปกวาดหาว่า ข้อความที่ผู้ใช้พิมพ์มา มีการเคาะ Spacebar (`' '`) แฝงอยู่ทั้งหมดกี่ช่อง?

::: details 💡 คำใบ้และแนวทาง (Hints)
- **สำหรับโจทย์ 1:** 
  แบบปกติ: `"SELECT * FROM Users WHERE Name = \"John\" AND FilePath = \"C:\\Data\\images\""`
  แบบ `@`: `@"SELECT * FROM Users WHERE Name = ""John"" AND FilePath = ""C:\Data\images"""` (จำไว้ว่า ในกล่อง `@` ถ้าอยากพิมพ์ `"` 1 ตัว ต้องพิมพ์ `""` ติดกัน)
- **สำหรับโจทย์ 2:** ใช้ `foreach (char c in input)` แล้วเอา `c == ' '` (อย่าลืมว่าเครื่องหมายซิงเกิ้ลโควทของ char ต้องมีช่องว่างตรงกลาง) ไปตั้งเงื่อนไขนับเลขครับ!
:::

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: การกลับหลังหันข้อความที่แท้จริง (The True String Reversal)**

มีคนบอกว่า ถ้าอยากกลับหลังหันข้อความ `"HELLO"` ให้เป็น `"OLLEH"` เราใช้แค่ลูปถอยหลังมาบวกข้อความเอาก็ได้:
```csharp
string result = "";
string word = "HELLO";
for (int i = word.Length - 1; i >= 0; i--) {
    result += word[i];
}
```
**คำเตือนจากสถาปนิก:** โค้ดด้านบน **"เป็นขยะ (Bad Practice)"** ครับ! เพราะด้วยกฎ Immutability คอมพิวเตอร์จะต้องสร้างกล่องข้อความใหม่ทิ้งเป็นขยะถึง 5 ใบ (`"O"`, `"OL"`, `"OLL"`, `"OLLE"`, `"OLLEH"`) ถ้าคุณสลับประโยคที่มี 10,000 อักษร แรมเครื่องคุณจะเต็มไปด้วยขยะ!

**ภารกิจของคุณ:** จงสลับคำว่า `"HELLO"` ให้เป็น `"OLLEH"` **โดยห้ามใช้เครื่องหมาย `+` หรือ `+=` ต่อข้อความเด็ดขาด!**

**Algorithm Guide (วิชาระดับเซียน):**
1. ในเมื่อแก้ `string` ตรงๆ ไม่ได้ เราก็ทุบมันให้แตกกลายเป็น Array ดิบๆ สิ! `char[] charArray = word.ToCharArray();`
2. อ๊ะ! คุ้นๆ ไหม? ตอนนี้มันกลายเป็น Array 1 มิติธรรมดาแล้ว! ดังนั้นจงงัด **"วิชาสลับร่าง Two-Pointer หุบเข้าหากึ่งกลาง"** (ที่เพิ่งเรียนใน Challenge ของบทที่ 6) มาใช้สลับ `charArray` ให้กลับหลังหันซะ (ใช้กล่อง Temp)
3. พอสลับเสร็จ มันยังเป็นตัวหนังสือแตกๆ อยู่ ให้เรารวบมันกลับเป็น String ก้อนเดียวด้วยคำสั่ง `string finalWord = new string(charArray);`

ใครทำข้อนี้ได้ แสดงว่าคุณเข้าใจการแก้ปัญหาเรื่อง Memory Management (การบริหารแรม) แบบทะลุปรุโปร่งแล้วครับ!

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** ในเอกสารไมโครซอฟต์ระบุว่า "String ใน C# ถูกบริหารจัดการด้วยระบบที่เรียกว่า String Intern Pool" มันคืออะไร และช่วยประหยัดแรมยังไง?
**แนวคำตอบ:** เนื่องจาก String มันแก้ไขไม่ได้ (Immutable) ไมโครซอฟต์เลยสร้างความฉลาดแฝงไว้ครับ: ถ้าคุณสร้าง `string a = "Cat";` และสร้าง `string b = "Cat";` ในภาษาอื่นมันอาจจะจองแรมสร้างกล่องคำว่า Cat ซ้ำซ้อน 2 ใบ แต่ใน C# ระบบจะเห็นว่าอักษรมันเหมือนกันเป๊ะ! มันเลยให้ตัวแปร `a` และ `b` ชี้แผนที่ (Pointer) ไปหากล่องข้อความ `"Cat"` **ใบเดียวกันในแรม (Intern Pool)!** นี่คือสาเหตุที่โปรแกรม C# ประหยัดแรมมากเวลาเราประกาศข้อความซ้ำๆ กันเต็มระบบครับ!

**คำถาม 2:** โค้ดด้านล่างนี้ เกิดขยะในแรม (Garbage) กี่ก้อน อะไรบ้าง?
```csharp
string name = "A";
name += "B";
name += "C";
```
**แนวคำตอบ:** เกิดขยะ 2 ก้อนครับ!
1. รอบแรกสร้างก้อน "A"
2. บรรทัดสอง สร้างก้อน "B", แล้วเอาไปปั้นเป็น "AB" กล่องใหม่... กล่อง "A" และ "B" กลายเป็นขยะ!
3. บรรทัดสาม สร้างก้อน "C", แล้วเอาไปปั้นเป็น "ABC" กล่องใหม่... กล่อง "AB" และ "C" กลายเป็นขยะ!
จบโปรแกรม เราได้คำว่า "ABC" ใช้งาน แต่มีขยะลอยฟ่องในแรมให้ Garbage Collector ตามเก็บกวาดอื้อซ่าครับ!
:::
