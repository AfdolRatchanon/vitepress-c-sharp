# 02 - เหตุการณ์ (Event) และโค้ดเบื้องหลัง (Code-Behind)

> 💡 **เป้าหมาย:** เข้าใจกระบวนทัศน์แบบ Event-Driven, รู้จักพารามิเตอร์ `sender` และ `EventArgs`, และเจาะลึกการใช้งาน `MessageBox` เพื่อโต้ตอบกับผู้ใช้ขั้นสูง

## 📖 ทฤษฎีและแนวคิด UI (UI Concept & Theory)

### จาก Sequential สู่ Event-Driven
ในการเขียนโปรแกรมแบบ Console โค้ดของเราจะทำงานจากบรรทัดบนสุดลงมาล่างสุดทันที (Sequential) โปรแกรมจะเป็นฝ่าย "สั่ง" ให้ผู้ใช้กรอกข้อมูล (เช่น พิมพ์ `Console.ReadLine()`) แล้วโปรแกรมจึงจะทำงานต่อ

แต่ในโลกของ Windows Forms โปรแกรมจะไม่บังคับผู้ใช้อีกต่อไป เมื่อโปรแกรมเปิดขึ้นมา มันจะสร้างหน้าต่างแล้ว **"หยุดนิ่งเพื่อรอคอย"** การกระทำบางอย่างจากผู้ใช้ ไม่ว่าผู้ใช้จะคลิกปุ่มไหนก่อน พิมพ์กล่องข้อความใดก่อน หรือเลื่อนเมาส์ไปมา โปรแกรมก็พร้อมจะตอบสนองเสมอ รูปแบบการเขียนโปรแกรมเช่นนี้เรียกว่า **Event-Driven Programming (การเขียนโปรแกรมแบบตอบสนองต่อเหตุการณ์)**

### วงจรชีวิตของ Event (The Event Loop)

```text
[กระบวนการทำงานของ Event-Driven]

1. 💻 ผู้ใช้กระทำกับ UI (เช่น เอาเมาส์ไป "คลิก" ที่ปุ่ม Button1)
           |
           v
2. ⚙️ ระบบปฏิบัติการ Windows ตรวจจับการคลิก และส่งข้อความไปหาแอปของเรา
           |
           v
3. ⚡ แอปพลิเคชันเช็คว่ามีโค้ด (Event Handler) ผูกไว้กับเหตุการณ์ "Click" หรือไม่?
           |
   +-------+-------+
   | (มีโค้ดผูกไว้)  | (ไม่มีโค้ด)
   v               v
4. รันโค้ด C#       เมินเฉยและรอเหตุการณ์ต่อไป...
```

### การตั้งชื่อ Control (Naming Convention)
ก่อนที่เราจะไปเขียนโค้ดผูกกับปุ่ม กฎเหล็กข้อสำคัญที่สุดในการเขียนโปรแกรม GUI คือ **ต้องเปลี่ยน Property `Name` ของ Control เสมอ**
ถ้าเราไม่เปลี่ยน Visual Studio จะตั้งชื่อให้ว่า `button1`, `button2`, `textBox1` ซึ่งเวลาเราสลับหน้าจอไปเขียนโค้ด C# เราจะสับสนมากว่าตัวแปรไหนคือปุ่มอะไร ดังนั้นโปรแกรมเมอร์จึงนิยมใช้ "คำนำหน้า (Prefix)" ดังนี้:

- **Button (ปุ่มกด):** ขึ้นต้นด้วย `btn` เช่น `btnLogin`, `btnSubmit`
- **TextBox (ช่องกรอก):** ขึ้นต้นด้วย `txt` เช่น `txtUsername`, `txtPassword`
- **Label (ป้ายข้อความ):** ขึ้นต้นด้วย `lbl` เช่น `lblStatus`, `lblWarning`

ให้นักเรียนเตรียมหน้าต่าง Form สำหรับบทเรียนนี้ โดยนำ Button จาก Toolbox มาวางตรงกลาง และตั้งค่าดังต่อไปนี้:

| Control | Property | ตั้งค่าเป็น | คำอธิบาย |

| `Button` | `Name` | `btnClickMe` | **สำคัญมาก:** เปลี่ยนชื่อตัวแปรของปุ่มนี้ให้สื่อความหมาย |
| `Button` | `Text` | `คลิกฉันสิ!` | ข้อความที่จะโชว์ให้ผู้ใช้เห็นบนปุ่ม |

## 💻 การสร้าง Event และเขียนโค้ด (Code Implementation)

วิธีการสร้าง Event Handler (ฟังก์ชันที่จะทำงานเมื่อเกิดเหตุการณ์) ใน Visual Studio ที่ง่ายและเป็นที่นิยมที่สุดคือ **การดับเบิลคลิก (Double-Click)** ที่ตัว Control นั้นๆ บนหน้า Design 

เมื่อนักเรียนดับเบิลคลิกที่ปุ่ม `btnClickMe` ตัว Visual Studio จะทำการ:
1. สร้างโค้ดเชื่อมโยง (Wire-up) ปุ่มเข้ากับ Event โดยอัตโนมัติ (ซ่อนอยู่ในไฟล์ Designer.cs)
2. สร้างเมธอด C# เปล่าๆ ให้เราเติมโค้ดลงไป (ในไฟล์ Form1.cs)

::: code-group
```csharp [Form1.cs]
using System;
using System.Windows.Forms;

namespace MyWinApp
{
    public partial class Form1 : Form
    {
        public Form1()
        {
            InitializeComponent(); // ฟังก์ชันนี้วาดหน้าจอ UI ห้ามลบเด็ดขาด!
        }

        // [1] เมธอดนี้ถูกสร้างขึ้นมาอัตโนมัติเมื่อเราดับเบิลคลิกที่ปุ่ม btnClickMe
        // สังเกตว่ามีพารามิเตอร์ sender และ e ส่งเข้ามาด้วย
        private void btnClickMe_Click(object sender, EventArgs e)
        {
            // [2] การเรียกใช้ MessageBox พื้นฐาน
            // พารามิเตอร์แรก: ข้อความข้างในกล่อง
            // พารามิเตอร์สอง: ข้อความบนหัวกล่อง (Caption)
            MessageBox.Show("สวัสดี นี่คือ Event แรกของคุณ!", "ข้อความจากระบบ");
        }
    }
}
```
:::

**พารามิเตอร์ลึกลับ: `object sender` และ `EventArgs e` คืออะไร?**
- `sender`: คือ **ผู้ที่ส่งเหตุการณ์นี้มา** ในกรณีนี้ `sender` ก็คือตัวปุ่ม `btnClickMe` นั่นเอง
- `e` (EventArgs): คือ **ข้อมูลเพิ่มเติมเกี่ยวกับเหตุการณ์** เช่น ถ้านี่เป็น Event การคลิกเมาส์ ตัว `e` จะบอกได้ว่าผู้ใช้คลิกเมาส์ซ้ายหรือเมาส์ขวา หรือเมาส์อยู่ที่พิกัด X, Y เท่าไหร่

**Expected Output:**
เมื่อกดรัน (F5) และคลิกปุ่ม จะมีกล่องข้อความเล็กๆ เด้งขึ้นมาตรงกลางหน้าจอ

## 🚀 เจาะลึก MessageBox (Advanced MessageBox)

`MessageBox.Show()` ใน C# เป็นเครื่องมือที่ทรงพลังมาก มันไม่ได้มีไว้แค่แสดงข้อความแจ้งเตือนโง่ๆ แต่สามารถมีไอคอน มีปุ่มกดหลายปุ่ม และดักจับได้ว่าผู้ใช้กดปุ่มไหน ลองลบโค้ดข้างใน Event เดิมแล้วใส่โค้ดนี้แทน:

```csharp
private void btnClickMe_Click(object sender, EventArgs e)
{
    // [1] การใช้ MessageBox แบบสมบูรณ์ (Overload ที่มี 4 พารามิเตอร์)
    // - ข้อความที่ต้องการสื่อ
    // - หัวข้อหน้าต่าง
    // - ชุดปุ่มกด (Yes/No)
    // - ไอคอน (เครื่องหมายคำถาม)
    DialogResult result = MessageBox.Show(
        "คุณต้องการเรียนรู้ C# ต่อไปหรือไม่?", // ข้อความ
        "ยืนยันการเรียน",                     // หัวข้อ
        MessageBoxButtons.YesNo,             // เลือกปุ่ม Yes และ No
        MessageBoxIcon.Question              // ใส่ไอคอนคำถาม
    );

    // [2] เช็คว่าผู้ใช้กดปุ่มอะไรตอบกลับมา
    if (result == DialogResult.Yes)
    {
        MessageBox.Show("ยอดเยี่ยม! ลุยกันต่อเลย!", "ระบบ", MessageBoxButtons.OK, MessageBoxIcon.Information);
    }
    else
    {
        MessageBox.Show("เสียใจจัง แต่อย่ายอมแพ้นะ!", "ระบบ", MessageBoxButtons.OK, MessageBoxIcon.Error);
    }
}
```

ด้วยการใช้ตัวแปรประเภท `DialogResult` มารับค่า เราจึงสามารถสร้างโปรแกรมโต้ตอบ (Interactive) ที่ฉลาดขึ้นได้ เช่น การถามย้ำก่อนผู้ใช้จะกดปุ่มลบข้อมูลจริง

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

**โจทย์ที่ 1: การสร้างปุ่มออกโปรแกรมพร้อมระบบยืนยัน**
1. ให้นักเรียนลาก Button มาวางเพิ่มอีก 1 ปุ่ม
2. ตั้งชื่อ Name ว่า `btnExit` และปรับ Text เป็น `ปิดโปรแกรม (Exit)`
3. ดับเบิลคลิกเพื่อสร้าง Event `Click`
4. **โจทย์:** ให้เขียนโค้ดเรียก `MessageBox` เพื่อถามผู้ใช้ว่า "คุณต้องการออกจากโปรแกรมจริงๆ หรือไม่?" โดยต้องมีปุ่ม Yes และ No และไอคอนเครื่องหมายคำถาม
5. ถ้าผู้ใช้กด Yes ให้ใช้คำสั่ง `this.Close();` เพื่อปิดโปรแกรม แต่ถ้ากด No ก็ไม่ต้องทำอะไรเลยให้หน้าต่างปิดไปเอง

::: details 💡 คำใบ้ (Hint)
สร้างตัวแปร `DialogResult` มารับค่า `MessageBox.Show(...)` แล้วใช้คำสั่ง `if` เช็คว่าตัวแปรนั้นมีค่าเท่ากับ `DialogResult.Yes` หรือไม่ ถ้าใช่ ค่อยเรียก `this.Close();`
:::

**โจทย์ที่ 2: Event แบบอื่นๆ ที่ไม่ใช่การคลิก (Hover Event)**
1. ที่หน้าต่าง Design ให้คลิกเลือกปุ่ม `btnClickMe`
2. มองไปที่หน้าต่าง Properties ตรงแถบเครื่องมือด้านบน จะมีไอคอนรูปสายฟ้า (⚡) ชื่อว่า **Events** ให้คลิกที่นั่น (นี่คือวิธีดู Event ทั้งหมดที่ปุ่มนี้ทำได้)
3. เลื่อนหา Event ที่ชื่อว่า **`MouseEnter`** (ทำงานเมื่อเอาเมาส์มาวางทับปุ่ม โดยยังไม่ได้คลิก)
4. ดับเบิลคลิกที่ช่องว่างด้านขวาของคำว่า `MouseEnter`
5. ในช่องโค้ด ให้ใส่คำสั่ง `btnClickMe.BackColor = Color.LightGreen;` 
6. รันโปรแกรมแล้วสังเกตผลลัพธ์เวลาเอาเมาส์ไปชี้ที่ปุ่ม

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจ 1:
**คำถาม:** เหตุใดการเขียนโค้ด GUI จึงมักไม่ใช้คำสั่ง Loop (`while` หรือ `for`) เพื่อรอรับคำสั่งจากผู้ใช้ แบบที่ทำใน Console App?
**แนวคำตอบ:** เพราะเทคโนโลยี GUI มี **Event Loop** อยู่เบื้องหลังให้แล้ว โปรแกรมเมอร์แค่เขียนฟังก์ชัน (Event Handler) ไปรอรับเหตุการณ์เมื่อเกิด Action จากผู้ใช้ก็พอ หากเราเขียน `while` loop ครอบไว้เอง จะทำให้ UI Thread ถูกบล็อก และส่งผลให้ "โปรแกรมค้าง (Not Responding)"
:::

::: details ❓ คำถามทบทวนความเข้าใจ 2:
**คำถาม:** โค้ดที่ว่า `if (result == DialogResult.Yes)` ทำงานร่วมกับ `MessageBox` อย่างไร?
**แนวคำตอบ:** ฟังก์ชัน `MessageBox.Show()` ไม่ได้ทำหน้าที่แค่แสดงผล แต่มันสามารถ "คืนค่า (Return)" กลับมาเป็นประเภท `DialogResult` ได้ ทำให้เรารู้ได้แน่ชัดว่าผู้ใช้ตัดสินใจกดปุ่มใดบนหน้าต่างนั้น (เช่น Yes, No, OK, Cancel)
:::
