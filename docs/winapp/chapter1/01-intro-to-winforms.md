# 01 - รู้จักกับ Windows Forms และสร้างโปรเจกต์แรก

> 💡 **เป้าหมาย:** ทำความเข้าใจภาพรวมของ Windows Forms, ความแตกต่างระหว่างหน้าจอแบบ CLI กับ GUI, ส่วนประกอบที่ซ่อนอยู่ในโปรเจกต์ (Designer.cs) และการตั้งค่าคุณสมบัติ (Properties) เบื้องต้นอย่างละเอียด

## 📖 ทฤษฎีและแนวคิด UI (UI Concept & Theory)

ตลอดบทเรียนที่ผ่านมา เราได้เขียนโปรแกรมแบบ **Console Application** ซึ่งแสดงผลลัพธ์เป็นตัวหนังสือสีขาวบนพื้นดำ (Command Line Interface - CLI) การทำงานของ Console จะเป็นแบบ "บนลงล่าง" (Top-to-Bottom) หรือทำงานตามลำดับคำสั่งอย่างเคร่งครัด

แต่ในโลกความเป็นจริง ผู้ใช้งานคอมพิวเตอร์ทั่วไปไม่ได้คุ้นเคยกับการพิมพ์คำสั่งลงในหน้าจอสีดำ พวกเขาต้องการ **Graphic User Interface (GUI)** หรือระบบที่มีหน้าต่าง มีปุ่มให้คลิก มีกล่องข้อความให้กรอกข้อมูล มีเมนูให้เลือก ซึ่งเป็นมาตรฐานของแอปพลิเคชันบนระบบปฏิบัติการ Windows

ในภาษา C# และ .NET Framework มีเครื่องมือหลายตัวสำหรับสร้าง GUI แต่เครื่องมือที่ง่าย คลาสสิก และเหมาะสำหรับการเริ่มต้นเรียนรู้มากที่สุดคือ **Windows Forms (WinForms)**

### 🖥️ Windows Forms คืออะไร?
Windows Forms เป็นเทคโนโลยีในการสร้างหน้าต่างโปรแกรม (Window) บนระบบปฏิบัติการ Windows โดยมีจุดเด่นคือการ **"ลากแล้ววาง" (Drag & Drop)** นักพัฒนาสามารถนำเครื่องมือ (Control) เช่น ปุ่ม, กล่องข้อความ มาวางบนหน้าจอจำลอง (Canvas) ได้ทันที ทำให้การออกแบบหน้าตารวดเร็วมาก

### 📂 โครงสร้างของโปรเจกต์ Windows Forms
เมื่อเราสร้างโปรเจกต์ WinForms ใน Visual Studio ระบบไม่ได้สร้างแค่ไฟล์เดียวเหมือนตอนทำ Console แต่จะสร้างโครงสร้างที่ซับซ้อนกว่าเพื่อรองรับการแสดงผลหน้าต่าง:

```text
[โครงสร้างไฟล์ของโปรเจกต์ Windows Forms ใน Solution Explorer]

MyWinApp
 ├── Dependencies            (ไลบรารีที่จำเป็น)
 ├── Properties
 │    └── launchSettings.json
 │
 ├── Form1.cs                <-- ไฟล์ที่เราใช้เขียนโค้ด C# (Code-Behind)
 │    ├── Form1.Designer.cs  <-- ไฟล์ที่ Visual Studio เขียนโค้ด UI ให้อัตโนมัติ (ห้ามแก้เอง!)
 │    └── Form1.resx         <-- ไฟล์เก็บทรัพยากร เช่น รูปภาพหรือข้อความหลายภาษา
 │
 └── Program.cs              <-- จุดเริ่มต้นการรันโปรแกรม (Main Method)
```

**ความลับของ `partial class`:**
ใน C# มีคีย์เวิร์ดหนึ่งที่ชื่อว่า `partial class` ซึ่งอนุญาตให้เราแบ่งคลาส 1 คลาส ออกเป็น 2 ไฟล์ได้ 
ใน WinForms คลาส `Form1` ถูกแบ่งออกเป็น 2 ส่วน:
1. `Form1.Designer.cs`: จัดการเรื่องขนาดหน้าต่าง ตำแหน่งของปุ่ม (เครื่องเป็นคนเขียน)
2. `Form1.cs`: จัดการเรื่องลอจิก เช่น กดปุ่มแล้วทำอะไร (เราเป็นคนเขียน)

การแบ่งแบบนี้ทำให้โค้ดของเราสะอาดและไม่ต้องไปยุ่งกับโค้ดตัวเลขแกน X, Y ที่ยุ่งยาก!

## 🛠️ ส่วนประกอบของ Visual Studio IDE

ก่อนที่เราจะเริ่มเขียนโปรแกรม เราต้องรู้จักเครื่องมือรอบๆ ตัวใน Visual Studio ก่อน เพราะหน้าต่างเหล่านี้คืออาวุธหลักของเรา:

```text
[องค์ประกอบหน้าจอ Visual Studio IDE]
+-----------------------------------------------------------------------+
| 🛠️ Toolbox (ซ้าย)   |  🗔 Form1.cs [Design] (กลาง) | 📂 Solution (ขวาบน)|
|                    |                           | - Program.cs          |
| 🔍 Search Controls |  +---------------------+  | - Form1.cs            |
| -----------------  |  |                     |  |                       |
| [ ] Button         |  | (Canvas พื้นที่ออกแบบ) |  +-----------------------+
| [ ] Label          |  |                     |  | ⚙️ Properties (ขวาล่าง)|
| [ ] TextBox        |  |                     |  |                       |
| [ ] CheckBox       |  +---------------------+  | [A-Z] เรียงตามอักษร      |
| [ ] RadioButton    |                           | Text: Form1           |
|                    |                           | Size: 800, 450        |
+-----------------------------------------------------------------------+
```

1. **Toolbox (กล่องเครื่องมือ):** เปรียบเสมือนกล่องเก็บชิ้นส่วนเลโก้ หากไม่เห็นหน้าต่างนี้ ให้ไปที่เมนู `View > Toolbox` (หรือกด Ctrl+Alt+X)
2. **Design View (พื้นที่ออกแบบ):** หน้าต่างตรงกลางที่ให้เราจำลองหน้าจอ สามารถใช้เมาส์ย่อขยายขนาดฟอร์มได้
3. **Properties Window (หน้าต่างคุณสมบัติ):** สำคัญที่สุด! ใช้สำหรับกำหนดค่าต่างๆ ให้กับสิ่งที่คลิกอยู่ เช่น เปลี่ยนสี เปลี่ยนชื่อตัวแปร เปลี่ยนขนาดฟอนต์ หากหน้าต่างนี้หายไป ให้คลิกขวาที่ฟอร์มแล้วเลือก `Properties` (หรือกด F4)

## ⚙️ การสร้างโปรเจกต์และการตั้งค่า Properties เบื้องต้น

**Step 1: การสร้างโปรเจกต์**
1. เปิด Visual Studio เลือก `Create a new project`
2. ค้นหาคำว่า `Windows Forms App` (แนะนำให้เลือกเวอร์ชัน .NET ไม่ใช่ .NET Framework เก่า)
3. ตั้งชื่อโปรเจกต์ว่า `MyFirstWinApp`

**Step 2: การทำความเข้าใจ Properties ของ Form**
ให้คลิกที่พื้นที่ว่างบน Form1 แล้วมองไปที่ **Properties Window** ด้านขวาล่าง เราจะมาตั้งค่าคุณสมบัติพื้นฐานที่แอปพลิเคชันที่ดีควรต้องมี:

| Property | ความหมายและการทำงาน | ค่าที่แนะนำให้ลองเปลี่ยน |
| :--- | :--- | :--- |
| `Text` | ข้อความบนหัวหน้าต่าง (Title Bar) | เปลี่ยนจาก `Form1` เป็น `ระบบจัดการข้อมูลนักเรียน` |
| `BackColor` | สีพื้นหลังของหน้าต่าง | เลือกแท็บ `Web` แล้วเลือกสี `AliceBlue` หรือ `LightGray` |
| `StartPosition` | ตำแหน่งที่หน้าต่างจะโผล่ขึ้นมาตอนเปิดโปรแกรม | แนะนำให้ตั้งเป็น `CenterScreen` เสมอ |
| `FormBorderStyle`| รูปแบบของกรอบหน้าต่าง | หากตั้งเป็น `FixedSingle` ผู้ใช้จะยืด/หดหน้าต่างไม่ได้ |
| `MaximizeBox` | ปุ่มขยายเต็มจอ (มุมขวาบน) | หากตั้งเป็น `False` ผู้ใช้จะไม่สามารถขยายหน้าต่างได้ |
| `Opacity` | ความโปร่งใสของหน้าต่าง (100% คือทึบสนิท) | ลองปรับเป็น `80%` (หน้าต่างจะใสทะลุเห็นพื้นหลัง) |

> ⚠️ **Best Practice:** การสร้างโปรแกรมที่ดี เราไม่ควรปล่อยให้ชื่อหน้าต่างเป็นคำว่า `Form1` ควรเปลี่ยน Property `Text` เสมอเพื่อให้แอปพลิเคชันดูเป็นมืออาชีพ

## 💻 ตัวอย่างโค้ดเบื้องหลัง (Program.cs)

ในโปรเจกต์ Windows Forms ไฟล์ที่ทำหน้าที่เป็นจุดเริ่มต้นของโปรแกรม (Entry Point) ยังคงเป็น `Program.cs` เหมือนใน Console ลองเปิดไฟล์นี้ดู เราจะเห็นโค้ดที่รันเพื่อปลุกหน้าต่างฟอร์มขึ้นมา:

::: code-group
```csharp [Program.cs]
using System;
using System.Windows.Forms;

namespace MyFirstWinApp
{
    internal static class Program
    {
        // [STAThread] คือ Attribute ที่บอกว่าแอปพลิเคชันนี้ทำงานในโหมด Single-Threaded Apartment
        // ซึ่งเป็นข้อบังคับสำหรับการทำงานของระบบ GUI บน Windows
        [STAThread]
        static void Main()
        {
            // คำสั่งเตรียมความพร้อมให้หน้าต่างรองรับดีไซน์สมัยใหม่ (Visual Styles)
            Application.EnableVisualStyles();
            Application.SetCompatibleTextRenderingDefault(false);
            
            // [หัวใจสำคัญ] คำสั่งนี้คือการสร้างออบเจกต์ Form1 ขึ้นมา และสั่งให้มันแสดงค้างไว้
            // โปรแกรมจะไม่ไปบรรทัดถัดไป จนกว่าผู้ใช้จะกดกากบาทปิดหน้าต่าง Form1
            Application.Run(new Form1());
        }
    }
}
```
:::

**เกิดอะไรขึ้นเมื่อกดรัน (F5) ?**
เมื่อคุณกด F5 ฟังก์ชัน `Main()` จะทำงาน มันจะเตรียมระบบภาพกราฟิกให้พร้อม จากนั้นคำสั่ง `Application.Run(new Form1())` จะสร้างหน้าต่างฟอร์มขึ้นมา และเข้าสู่สภาวะที่เรียกว่า **"Message Loop"** ซึ่งจะคอยดักจับว่าผู้ใช้ขยับเมาส์ไหม คลิกปุ่มไหนบ้าง โปรแกรมจะวนลูปรอคอยการกระทำไปเรื่อยๆ จนกว่าหน้าต่างจะถูกปิด

**Expected Output:**
```text
(จะปรากฏหน้าต่างแอปพลิเคชันที่มีสีพื้นหลังตามที่เลือก โผล่ขึ้นมาตรงกลางหน้าจอพอดี และข้อความบนหัวหน้าต่างถูกเปลี่ยนชื่อแล้ว)
```

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

**โจทย์ที่ 1: จัดการขนาดและการแสดงผล**
ให้นักเรียนตั้งค่า Properties ของ `Form1` ดังต่อไปนี้ผ่านหน้าต่าง Properties:
1. เปลี่ยนชื่อหน้าต่าง (Text) เป็น "My Personal Diary"
2. เปลี่ยนสีพื้นหลัง (BackColor) เป็นสีที่สบายตา (เช่น สีพาสเทล)
3. เปลี่ยนขนาดหน้าต่าง (Size) เป็น `Width: 1024, Height: 768` (ความละเอียดระดับ HD)
4. ตั้งค่าให้เปิดโปรแกรมมาแล้วหน้าต่างไปอยู่ตรงกลางจอพอดี
5. **กดรัน (F5) เพื่อตรวจสอบผลลัพธ์**

**โจทย์ที่ 2: ป้องกันผู้ใช้ทำหน้าต่างพัง (UI Constraints)**
บางครั้งเราออกแบบ UI ไว้สวยงามแล้ว แต่ผู้ใช้งานเอาเมาส์ไปลากขอบหน้าต่างเพื่อย่อหน้าต่างให้เล็กลงจนปุ่มหายไปหมด ให้นักเรียนแก้ปัญหานี้โดย:
1. ไปที่ Properties มองหาหมวดหมู่ Layout
2. ตั้งค่า `MinimumSize` เป็น `800, 600`
3. กดรันโปรแกรม แล้วลองพยายามย่อขอบหน้าต่างให้เล็กมากๆ สังเกตว่าจะย่อให้เล็กกว่า 800x600 ไม่ได้แล้ว

::: details 💡 คำใบ้ (Hint)
Property `MinimumSize` จะบังคับให้ฟอร์มไม่สามารถถูกย่อลงเล็กกว่าค่าที่กำหนดได้ เป็นเทคนิคที่ใช้บ่อยมากในการป้องกันผู้ใช้ย่อหน้าจอจน Control ภายในเบียดกันพัง
:::

## 🔥 Challenge (โจทย์ท้าทาย!)

**โจทย์:** ให้นักเรียนทำให้หน้าต่างโปรแกรมของเรา **"ไร้กรอบ" (Borderless)**
ในแอปพลิเคชันสมัยใหม่ เช่น Spotify หรือ Discord มักจะไม่มีกรอบ Windows สีขาวๆ หนาๆ แบบดั้งเดิม ให้นักเรียนค้นหา Property ที่ใช้จัดการกับ "สไตล์ของเส้นขอบหน้าต่าง" แล้วเปลี่ยนค่าให้หน้าต่างนี้แสดงผลแบบไม่มีกรอบใดๆ เลย (ไม่มีปุ่มกากบาท ไม่มีหัวหน้าต่าง)

*(คำใบ้: ลองมองหา Property ที่ชื่อขึ้นต้นด้วยคำว่า `FormBorder...` และเลือกค่าเป็น `None` จากนั้นลองรันโปรแกรมดู... อ๊ะ! แล้วจะปิดโปรแกรมยังไงล่ะ? ในเมื่อไม่มีปุ่มกากบาทแล้ว คุณต้องกดปุ่ม `Stop` สีแดงใน Visual Studio เพื่อหยุดโปรแกรมครับ!)*

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจ 1:
**คำถาม:** เหตุใด Visual Studio จึงแบ่งไฟล์ Form1 ออกเป็น `Form1.cs` และ `Form1.Designer.cs`?
**แนวคำตอบ:** เพื่อแยกส่วนรับผิดชอบ (Separation of Concerns) ไฟล์ `Designer.cs` มีไว้ให้ Visual Studio สร้างโค้ดวาด UI โดยอัตโนมัติ เราไม่ควรเข้าไปแก้ ส่วน `Form1.cs` มีไว้ให้โปรแกรมเมอร์เขียนโค้ดลอจิก ทำให้โค้ดของเราสะอาดและไม่งง
:::

::: details ❓ คำถามทบทวนความเข้าใจ 2:
**คำถาม:** `Application.Run(new Form1())` ทำหน้าที่อะไร?
**แนวคำตอบ:** เป็นคำสั่งที่ปลุกหน้าต่าง Form1 ขึ้นมา และทำให้โปรแกรมเข้าสู่โหมดวนลูปรอรับคำสั่งจากผู้ใช้ (Message Loop) หากไม่มีบรรทัดนี้ โปรแกรมจะรันเสร็จและปิดตัวเองทันทีในเสี้ยววินาที
:::
