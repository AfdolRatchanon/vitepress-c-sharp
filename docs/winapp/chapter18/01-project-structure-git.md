# 01 - Project Structure & Git

> 💡 **เป้าหมาย:** เรียนรู้การจัดระเบียบโฟลเดอร์สำหรับโปรเจกต์ขนาดใหญ่ (Layered Architecture) และการใช้ Git เบื้องต้นเพื่อจัดการ Source Code ก่อนขึ้น Production

## 📖 ทฤษฎีและแนวคิด UI (UI Concept & Theory)

**1. Project Structure (โครงสร้างโปรเจกต์)**
เมื่อแอปพลิเคชันใหญ่ขึ้น การนำโค้ดทั้งหมด (ต่อ Database, ตรวจสอบข้อมูล, โค้ด UI) ไปกองรวมกันใน `Form1.cs` จะทำให้โปรแกรมเมอร์หัวหมุน (เราเรียกโค้ดแบบนี้ว่า Spaghetti Code)
เราจึงควรแบ่งโฟลเดอร์ตาม **Layer**:
- `/Models` : เก็บ Class ก้อนข้อมูล เช่น `Product.cs`, `User.cs`
- `/DataAccess` : เก็บ Class ที่ต่อ DB เช่น `ProductRepository.cs`
- `/Services` : เก็บ Class ประมวลผลลอจิก เช่น `PaymentService.cs`
- `/Views` หรือ `/Forms` : เก็บหน้าจอ UI ทั้งหมด
- `/Utils` หรือ `/Helpers` : เก็บฟังก์ชันช่วยเหลือ เช่น `HashHelper.cs`

**2. Git Version Control**
Git คือระบบเก็บประวัติการแก้ไขโค้ด ทำให้เรารู้ว่าใครแก้ไฟล์ไหน เมื่อไหร่ และสามารถ "ย้อนเวลา" กลับไปโค้ดเวอร์ชันเก่าได้หากโปรแกรมพัง

## 💻 การใช้งาน Git พื้นฐาน (Git Commands)

ใน Visual Studio คุณสามารถใช้หน้าต่าง **Git Changes** แบบคลิกได้เลย แต่เบื้องหลังมันทำงานด้วยคำสั่งเหล่านี้:

```bash
# 1. เริ่มต้นระบบ Git ในโฟลเดอร์ปัจจุบัน (ทำครั้งเดียว)
git init

# 2. เพิ่มไฟล์ทั้งหมดเข้าสู่ Staging Area (เตรียมบันทึก)
git add .

# 3. บันทึกประวัติพร้อมข้อความอธิบาย (Commit)
git commit -m "Initial commit - สร้างหน้า Login และเชื่อมฐานข้อมูล"

# 4. ดูสถานะปัจจุบันว่ามีไฟล์ไหนแก้ไปบ้าง
git status

# 5. ดูประวัติการ Commit
git log
```

> ⚠️ **ข้อควรระวัง (.gitignore):** เราไม่ควรนำโฟลเดอร์ `/bin/` และ `/obj/` เข้าสู่ Git เพราะเป็นไฟล์ที่ถูก Compile (ไฟล์ .exe, .dll) ซึ่งมีขนาดใหญ่และเครื่องใครเครื่องมัน Visual Studio จะสร้างไฟล์ `.gitignore` ให้อัตโนมัติเพื่อบอกให้ Git มองข้ามโฟลเดอร์เหล่านี้

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวน 1:
**คำถาม:** เหตุใดการแยก Layer (เช่น แยก Repository ออกจาก Form) จึงสำคัญในการทำงานร่วมกันเป็นทีม?

**แนวคำตอบ:** หากไม่แยก Layer โปรแกรมเมอร์สองคนแก้ไข Form เดียวกันพร้อมกัน (คนนึงแก้ UI อีกคนแก้คำสั่ง SQL) เมื่อนำโค้ดมารวมกัน (Merge) มักเกิด Code Conflict อย่างรุนแรง การแยก Layer ทำให้คนนึงสามารถทำงานที่ไฟล์ Repository และอีกคนออกแบบหน้า UI ได้โดยไม่ทับซ้อนกัน และยังนำโค้ดไปใช้ซ้ำได้ง่าย
:::
