# Jump Statements (`break` & `continue`)

> 💡 **เป้าหมาย:** ทำความรู้จักกับ "เบรกฉุกเฉิน" และ "ปุ่มข้าม" เครื่องมือสุดคลาสสิกที่โปรแกรมเมอร์ใช้สำหรับแทรกแซงการทำงานของ Loop เพื่อให้ระบบสามารถหนีจากสถานการณ์ที่ไม่คาดฝัน หรือข้ามกระบวนการที่ไม่จำเป็นไปได้อย่างรวดเร็ว

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. อำนาจในการแทรกแซงลูป (Loop Intervention)

โดยปกติแล้ว เมื่อเราเขียนคำสั่ง `for`, `while`, หรือ `do-while` โปรแกรมจะหมุนวนทำตามรอบหรือตามเงื่อนไขอย่างซื่อสัตย์ แต่ในโลกของความเป็นจริง (Real-world scenarios) เราอาจจะเจอกับเงื่อนไขบางอย่างที่ทำให้เรา **"ไม่อยากทำรอบที่เหลืออีกแล้ว"** หรือ **"ไม่อยากทำโค้ดครึ่งหลังของรอบนี้แล้ว"**

C# จึงได้เตรียมคีย์เวิร์ดพิเศษ 2 ตัว (Jump Statements) เอาไว้ให้เราใช้ "กระโดด" ออกจากกระแสการทำงานปกติของลูป:
1. **`break;`** (ทุบกระจกหนีไฟ)
2. **`continue;`** (กดปุ่ม Skip ข้ามไปรอบถัดไป)

---

### 2. คำสั่ง `break` (การทำลายล้างลูป)

คำสั่ง `break;` แปลตรงตัวคือ "พังมันทิ้งซะ" 
เมื่อโปรแกรมคอมพิวเตอร์ทำงานมาเจอคำสั่งนี้ ไม่ว่ามันจะอยู่ในลูปที่หมุนไปแล้วกี่รอบ หรือเหลืออีกกี่แสนรอบ มันจะ **"กระโดดทะลุปีกกาของลูป (Loop's scope) เพื่อจบการทำงานของลูปนั้นทันที!"**

```text
แผนภาพจำลองการเกิด break:

  for (รอบที่ 1 ถึง 100)
  {
      [ โค้ดส่วนที่ 1 ]
      
      if (เจอข้อผิดพลาดร้ายแรง)
      {
           break;  ═════════════════════════╗ (กระโดดทะลุปีกกาทันที!)
      }                                     ║
                                            ║
      [ โค้ดส่วนที่ 2 ] (จะไม่ได้ทำเลย)     ║
  }                                         ║
                                            ║
  [ โค้ดบรรทัดล่างสุด นอกลูป ] <════════════╝
```

**ตัวอย่างการใช้งานจริง:**
- **การค้นหาข้อมูล (Searching):** สมมติว่ามีกล่อง 1,000 ใบ เราตามหาแหวน 1 วง ถ้าเปิดเจอกล่องที่ 5 แล้วเจอแหวน เราก็ควรสั่ง `break;` ทันที ไม่จำเป็นต้องบ้าจี้เปิดกล่องที่เหลือให้ครบ 1,000 ใบ (ประหยัดเวลา CPU ไปได้ 995 รอบ!)
- **เกมโอเวอร์ (Game Over):** ลูปรันเกมอยู่เรื่อยๆ แต่ถ้าเลือดผู้เล่นเหลือ 0 ก็สั่ง `break;` เพื่อออกจากลูปเกม

---

### 3. คำสั่ง `continue` (การกดยอมแพ้ในรอบนี้)

คำสั่ง `continue;` ไม่ได้ทำลายลูปทิ้ง แต่มันทำหน้าที่เหมือนปุ่ม Skip Ad ใน YouTube 
เมื่อโปรแกรมเจอคำสั่งนี้ มันจะ **"ยกเลิกการทำโค้ดบรรทัดที่เหลือในรอบปัจจุบัน (Iteration) แล้วกระโดดกลับขึ้นไปที่หัวลูป เพื่อเริ่มนับรอบถัดไปทันที!"**

```text
แผนภาพจำลองการเกิด continue:

  for (รอบที่ 1 ถึง 10) <═══════════════════╗ (กระโดดกลับขึ้นมาเตรียมขึ้นรอบใหม่)
  {                                         ║
      [ โค้ดส่วนที่ 1 ]                     ║
                                            ║
      if (ข้อมูลนี้เป็นค่าขยะ)              ║
      {                                     ║
           continue; ═══════════════════════╝
      }
      
      [ โค้ดส่วนที่ 2 การประมวลผลซับซ้อน ] (ถูกข้าม!)
  }
```

**ตัวอย่างการใช้งานจริง:**
- **การคัดกรองข้อมูล (Data Filtering):** สมมติลูปกำลังดึงรายชื่อลูกค้ามาคำนวณภาษี แต่ถ้าเจอลูกค้าคนไหนที่ "ถูกแบน" เราก็สั่ง `continue;` ข้ามคนนั้นไปเลย ไม่ต้องเอาไปเข้าสูตรคำนวณ แล้วระบบก็จะวิ่งไปดึงลูกค้าคนต่อไปมาทำต่อตามปกติ

---

### 4. แพทเทิร์น `while (true)` (ลูปอนันต์แบบตั้งใจ)

นักพัฒนาซอฟต์แวร์หลายคน นิยมสร้าง "ลูปอนันต์ (Infinite Loop)" ขึ้นมาอย่างจงใจ ด้วยคำสั่ง `while (true)` (หมุนตลอดกาลเพราะเงื่อนไขจริงเสมอ) 
แล้วค่อยเอาคำสั่ง `if` และ `break` ไปฝังไว้ข้างใน เพื่อกำหนดจุดออกเองตามใจชอบ 
เทคนิคนี้ช่วยให้เราไม่ต้องไปตั้งตัวแปรเงื่อนไข (Flag variables) นอกลูปให้ยุ่งยาก

---

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่างที่ 1: ระบบค้นหาเพชรด้วย `break` (The Early Exit Pattern)

โค้ดนี้พิสูจน์ให้เห็นว่า `break` สามารถประหยัดทรัพยากรการประมวลผล (Performance Optimization) ได้อย่างมหาศาล

::: code-group
```csharp [Program.cs]
using System;

namespace BreakStatementDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบหุ่นยนต์ค้นหากล่องสมบัติ ===");
            
            // สมมติมีกล่องอยู่ 100,000 ใบ (เยอะมาก)
            // เราซ่อนเพชรไว้ที่กล่องหมายเลข 42
            int targetBox = 42;
            int totalBoxes = 100000;

            Console.WriteLine($"[ระบบ] เริ่มต้นการค้นหาในกล่องทั้ง {totalBoxes} ใบ...");

            // หุ่นยนต์จะไล่เปิดทีละกล่อง
            for (int boxNumber = 1; boxNumber <= totalBoxes; boxNumber++)
            {
                // ตรวจสอบว่ากล่องนี้มีเพชรหรือไม่
                if (boxNumber == targetBox)
                {
                    Console.WriteLine($"\n🎉 แจ็คพอต! หุ่นยนต์ค้นพบเพชรที่กล่องหมายเลข {boxNumber} แล้ว!");
                    Console.WriteLine("ไม่มีความจำเป็นต้องหากล่องที่เหลือ สั่งหยุดระบบทันที!");
                    
                    // สั่งหยุดลูปทันที! กระโดดทะลุปีกกาทิ้ง 99,958 รอบที่เหลือไปเลย
                    break; 
                }

                // โค้ดนี้จะทำงานเฉพาะตอนที่ยังไม่เจอเพชร
                if (boxNumber % 10 == 0) // พิมพ์รายงานทุกๆ 10 กล่องเพื่อไม่ให้หน้าจอรก
                {
                    Console.WriteLine($"> สแกนกล่องที่ {boxNumber} ผ่านไป... ไม่พบอะไร");
                }
            }

            // เมื่อ break ทำงาน โปรแกรมจะกระโดดมารันบรรทัดนี้ทันที
            Console.WriteLine("\n[ระบบ] หุ่นยนต์เดินทางกลับฐานทัพเรียบร้อย");
        }
    }
}
```
:::

### ตัวอย่างที่ 2: ระบบคัดกรองขยะด้วย `continue` (The Filter Pattern)

โค้ดนี้แสดงให้เห็นการใช้ `continue` เพื่อลดความซ้อนทับของปีกกา (Reduce Nesting) และทำให้อ่านโค้ดได้ง่ายขึ้น

::: code-group
```csharp [Program.cs]
using System;

namespace ContinueStatementDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== โปรแกรมรวมยอดขาย (ข้ามวันหยุดเสาร์-อาทิตย์) ===");
            
            // สมมติเรากำลังรันข้อมูล 7 วัน (1 = จันทร์, ..., 6 = เสาร์, 7 = อาทิตย์)
            int totalSales = 0;

            for (int day = 1; day <= 7; day++)
            {
                // 1. กระบวนการคัดกรอง (Filtering)
                // ถ้าเป็นวันเสาร์ (6) หรือ วันอาทิตย์ (7) บริษัทหยุดทำการ ไม่มีรันระบบ
                if (day == 6 || day == 7)
                {
                    Console.WriteLine($"[ข้าม] วันที่ {day} เป็นวันหยุดสุดสัปดาห์ ไม่มีการขาย");
                    
                    // กระโดดข้ามโค้ดครึ่งหลังทั้งหมด แล้วกลับขึ้นไปขึ้นวันใหม่ (รอบถัดไป) ทันที
                    continue; 
                }

                // 2. กระบวนการประมวลผล (Processing)
                // โค้ดตรงนี้จะทำงานเฉพาะวันจันทร์-ศุกร์เท่านั้น
                // สมมติว่าขายได้วันละ 1,000 บาท
                Console.WriteLine($"[บันทึก] วันที่ {day} ทำยอดขายได้ 1,000 บาท");
                totalSales += 1000;
            }

            Console.WriteLine("------------------------------------------");
            Console.WriteLine($"💰 สรุปยอดขายตลอด 7 วันรวมทั้งสิ้น: {totalSales:N0} บาท");
        }
    }
}
```
:::

**เปรียบเทียบกับโค้ดแบบไม่มี `continue` (มือใหม่มักจะเขียนแบบนี้):**
```csharp
// แบบมี if ซ้อนปีกกาลึกๆ (ไม่แนะนำ)
for (int day = 1; day <= 7; day++)
{
    if (day != 6 && day != 7)
    {
        Console.WriteLine($"[บันทึก] วันที่ {day} ทำยอดขายได้ 1,000 บาท");
        totalSales += 1000;
    }
}
```
*การใช้ `continue` เป็นการ "ตัดจบเร็ว (Early Return/Guard Clause)" ทำให้โค้ดหลักไม่ต้องถูกดันเยื้องเข้าไปในปีกกาที่ลึกเกินไป (Arrow Code)*

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

### **โจทย์ที่ 1: นักทำลายลูป (The Breaker)**
จงเขียนโปรแกรมที่มีลูปอนันต์ `while (true)` ที่ให้ผู้ใช้ป้อนคำหรือข้อความเข้าไปเรื่อยๆ
- ถ้ายูสเซอร์ป้อนอะไรมาก็ตาม ให้โปรแกรมพิมพ์ทวนกลับไปว่า `"คุณพูดว่า: [ข้อความนั้น]"`
- แต่ถ้าผู้ใช้พิมพ์คำว่า `"quit"` โปรแกรมจะต้องพิมพ์ว่า `"ลาก่อน!"` และสั่ง `break;` ออกจากลูปอนันต์ทันที 

### **โจทย์ที่ 2: ระบบข้ามเลขอาถรรพ์ (The Superstitious Counter)**
เจ้าของตึกแห่งหนึ่งมีความเชื่อเรื่องเลข 13 และเลข 4 
**ภารกิจ:** จงใช้ `for` loop นับเลขตั้งแต่ชั้น 1 ถึงชั้น 20 และพิมพ์เลขชั้นออกมา (เช่น "ชั้น 1", "ชั้น 2")
แต่ถ้าถึงชั้น 4 หรือ ชั้น 13 คุณต้องใช้คำสั่ง `continue;` ข้ามมันไป ห้ามพิมพ์ออกมาเด็ดขาด!

::: details 💡 คำใบ้และแนวทาง (Hints)
- **สำหรับโจทย์ 1:** โครงสร้างหลักคือ `while (true) { string input = Console.ReadLine(); if (input == "quit") { break; } ... }`
- **สำหรับโจทย์ 2:** สร้างเงื่อนไข `if (floor == 4 || floor == 13)` แล้วข้างในปีกกาใส่แค่คำสั่ง `continue;` คำเดียว
:::

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: ตามหาจำนวนเฉพาะ (Prime Number Finder)**

จำนวนเฉพาะ (Prime Number) คือ ตัวเลขที่มีค่ามากกว่า 1 และมีตัวเลขที่สามารถหารมันลงตัว (เศษเป็น 0) ได้เพียง 2 ตัวเท่านั้น คือ "เลข 1" และ "ตัวมันเอง" (เช่น 2, 3, 5, 7, 11, 13...)

**ภารกิจของคุณ:** 
ให้รับค่า Input ตัวเลขจำนวนเต็มจากผู้ใช้ (สมมติว่าเป็นตัวแปร `target`) แล้วตรวจสอบว่าเลขนั้นเป็นจำนวนเฉพาะหรือไม่?

**กระบวนการคิด (Algorithm):**
1. คุณต้องสร้าง `for` loop เพื่อนับตัวเลขหาร (สมมติตัวแปร `i`) เริ่มจาก `2` ไปจนถึง `target - 1` (เช่น ถ้าเช็คเลข 7 ก็จะเอา 7 ไปลองหาร 2, 3, 4, 5, 6 ดู)
2. ถ้าในระหว่างทาง คุณเจอรอบไหนก็ตามที่ `target % i == 0` (หารลงตัวเป๊ะ) นั่นหมายความว่า **มันไม่ใช่จำนวนเฉพาะอีกต่อไปแล้ว!**
3. ให้คุณใช้คำสั่ง **`break;`** หนีออกจากลูปทันที เพราะคุณเจอผู้ต้องหาแล้ว ไม่จำเป็นต้องเสียเวลาหารตัวเลขที่เหลืออีกต่อไป!
4. เมื่อหลุดลูป ให้นำตัวแปรที่คุณสร้างไว้มาเช็คว่า ตกลงมันหาผู้ต้องหาเจอไหม? ถ้าหาไม่เจอเลย แปลว่ามันคือ "จำนวนเฉพาะ"

*(ทริคระดับโปร: ในความเป็นจริง คุณไม่ต้องวนลูปจนถึง `target - 1` หรอก แค่วนไปจนถึง ค่ารากที่สอง (Square Root) ของ `target` ก็เพียงพอแล้ว ถือเป็นการทำ Optimization แบบสุดยอด!)*

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** สมมติว่าคุณมีโปรแกรมที่มี Nested Loops (การเขียนลูปซ้อนกัน 2 ชั้น ซึ่งจะเรียนในบทถัดไป) ถ้าคุณสั่ง `break;` ในลูปชั้นในสุด มันจะเกิดอะไรขึ้น? ลูปชั้นนอกจะถูกทำลายไปด้วยหรือไม่?
**แนวคำตอบ:** **ไม่!** คำสั่ง `break;` มีอาณาเขตการทำลายล้างจำกัดอยู่แค่ **"ปีกกาของลูปชั้นที่ห่อหุ้มมันอยู่โดยตรง (Inner-most enclosing loop)"** เท่านั้น มันจะทำลายลูปชั้นในทิ้งและกระโดดออกไป แต่ลูปชั้นนอกจะยังคงดำเนินต่อไปอย่างไม่รู้ร้อนรู้หนาว (นี่คือเรื่องสำคัญที่โปรแกรมเมอร์หลายคนตกม้าตายตอนเขียนระบบ 2 มิติ)

**คำถาม 2:** คำสั่ง `continue` ในวงเล็บ `while` loop มีความเสี่ยงที่จะทำให้เกิด Infinite Loop ได้อย่างไร?
**แนวคำตอบ:** นี่คือบั๊กสุดคลาสสิก!
ใน `for` loop ถ้าสั่ง `continue` มันจะวิ่งไปอัปเดตค่า (เช่น `i++`) ที่ส่วนหัวของ for อัตโนมัติ ทำให้ไม่เกิดปัญหา
แต่ใน `while` loop การอัปเดตค่า (เช่น `i++`) เรามักจะเขียนไว้ "บรรทัดล่างสุดของปีกกา" ถูกไหม? 
ถ้าคุณเผลอสั่ง `continue;` **ก่อน** บรรทัดที่สั่งอัปเดตค่า โปรแกรมจะกระโดดกลับขึ้นไปหัวลูปทันที ส่งผลให้ `i` มีค่าเท่าเดิมตลอดกาล และเงื่อนไขก็จะเป็น True ตลอดกาล กลายเป็นบั๊ก "ลูปอนันต์" ทันที! (วิธีแก้คือต้องจำไว้ว่า ถ้าจะ continue ใน while ต้องบวกค่าอัปเดตก่อนเสมอ)
:::
