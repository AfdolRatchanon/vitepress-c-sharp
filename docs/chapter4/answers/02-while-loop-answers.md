# เฉลย: The `while` loop (การวนซ้ำแบบอิงเงื่อนไข)

## 🎯 เฉลย Mini Exercise

### **โจทย์ที่ 1: กลไกหลบหนี (Escape the Loop)**
**โจทย์ที่ให้มา:**
```csharp
int counter = 1;
int sum = 0;
while (counter < 5)
{
    sum += counter;
}
Console.WriteLine(sum);
```

**การวิเคราะห์และการแก้ปัญหา:**
1. **จะเกิดอะไรขึ้นเมื่อรัน?** โค้ดนี้คือ "มหันตภัยลูปอนันต์ (Infinite Loop)"! โปรแกรมจะค้าง นิ่งงัน (Freeze) และหน้าจอจะไม่แสดงผลอะไรออกมาเลย เพราะ CPU 100% ถูกดึงไปทำงานอยู่ในปีกกาอันดำมืดที่ไม่สิ้นสุด เนื่องจากค่า `counter` เป็น 1 ตลอดกาล เงื่อนไข `1 < 5` จึงเป็นจริงตลอดกาล
2. **วิธีแก้ไข:** เราต้องสร้าง "ทางออก" ให้กับลูป ด้วยการนำ `counter` มาบวกเพิ่ม 1 (`counter++`) ในแต่ละรอบ เพื่อให้สักวันหนึ่งมันเดินทางไปแตะเลข 5 และทำลายเงื่อนไขให้กลายเป็นเท็จให้จงได้

**โค้ดที่ได้รับการผ่าตัดแล้ว:**
```csharp
int counter = 1;
int sum = 0;
while (counter < 5)
{
    sum += counter;  // สะสมค่ารอบแรก (sum + 1)
    
    // เติมบรรทัดนี้ลงไป! (เครื่องยนต์ขับเคลื่อนให้ลูปมีวันสิ้นสุด)
    counter++; 
}
Console.WriteLine(sum);
// ผลลัพธ์: จะได้ 10 (เกิดจาก 1+2+3+4)
```

### **โจทย์ที่ 2: ระบบสั่งซื้อน้ำดื่ม (Water Vending Machine)**
**เป้าหมาย:** สร้างระบบตู้หยอดเหรียญที่รับเงินไปเรื่อยๆ จนกว่าจะครบ 15 บาท (Sentinel Value Pattern แบบบวกสะสม)

**เฉลยและวิธีทำ:**
โจทย์ข้อนี้เป็นการทดสอบ "การรักษา State (สถานะ)" ของตัวแปรนอกลูป 
ข้อผิดพลาดยอดฮิตของมือใหม่คือการไปสร้างตัวแปรรับเงิน **ไว้ในลูป** ทำให้ยอดเงินถูกลบหายไปเริ่ม 0 ใหม่ทุกรอบ

```csharp
Console.WriteLine("===================================");
Console.WriteLine("  💧 ตู้จำหน่ายน้ำดื่มอัตโนมัติ 💧  ");
Console.WriteLine("    [ น้ำเปล่าเย็นชื่นใจ ราคา 15 บาท ]");
Console.WriteLine("===================================");

// 1. กำหนดค่าคงที่ (ราคาของ)
const int WATER_PRICE = 15;

// 2. สร้างกระเป๋าตังค์ (State) ไว้ "นอกลูป" เพื่อให้ยอดเงินสะสมได้
int totalInserted = 0;

// 3. เริ่มวงลูป ขณะที่ยอดเงิน "ยังน้อยกว่า" 15
while (totalInserted < WATER_PRICE)
{
    // บอกผู้ใช้ว่ายังขาดเงินอีกเท่าไหร่
    int remaining = WATER_PRICE - totalInserted;
    Console.Write($"(ยังขาดอีก {remaining} บาท) กรุณาหยอดเหรียญเพิ่ม: ");
    
    // รับค่าและนำไป "บวกทบ" เข้ากับกระเป๋าตังค์
    int coin = int.Parse(Console.ReadLine());
    totalInserted += coin;
    
    Console.WriteLine($">> รับเหรียญ {coin} บาท. (ยอดรวม: {totalInserted} บาท)");
}

// 4. เมื่อหลุดออกจากลูป แปลว่าเงื่อนไข < 15 พังทลายแล้ว (แปลว่าเงินครบ หรือ เกิน แล้ว!)
Console.WriteLine("\n-----------------------------------");
Console.WriteLine("🎉 เครื่องกำลังจ่ายน้ำของคุณ... กริ๊ก!");

// คำนวณเงินทอนเผื่อไว้ในกรณีที่คนหยอดเหรียญ 10 มาสองเหรียญ (20 บาท)
int change = totalInserted - WATER_PRICE;
if (change > 0)
{
    Console.WriteLine($"💰 โปรดรอรับเงินทอน: {change} บาท");
}
Console.WriteLine("ขอบคุณที่ใช้บริการครับ!");
```

## 🔥 เฉลย Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: มหาสงครามแบคทีเรีย (The Bacteria Population Simulator)**
เป้าหมายคือจำลองการแบ่งตัวทวีคูณ (Exponential Growth) และหาว่าใช้เวลากี่รอบ (กี่ชั่วโมง) จึงจะทะลุล้าน

**กระบวนการคิดและเฉลยเชิงลึก (Algorithmic Approach):**
นี่คือตัวอย่างสมบูรณ์แบบที่สุดที่อธิบายว่า **ทำไมเราถึงต้องมี `while` loop ในโลกนี้** 
ถ้าโจทย์สั่งให้จำลองเวลาไปเรื่อยๆ 24 ชั่วโมง เราอาจใช้ `for (int i=1; i<=24)` ได้
แต่โจทย์ข้อนี้ **เราไม่รู้เลยว่าต้องใช้เวลากี่ชั่วโมง (ไม่รู้อนาคต)** เรามีเป้าหมายแค่ว่า "พุ่งไปจนกว่าประชากรจะทะลุ 1,000,000 แล้วหยุดทันที!" ลูปแบบนี้ `for` ไม่สามารถทำได้ง่ายๆ

```csharp
Console.WriteLine("=====================================");
Console.WriteLine(" 🔬 BACTERIA POPULATION SIMULATOR 🔬 ");
Console.WriteLine("=====================================");

// 1. ตั้งค่าสถานะเริ่มต้น (Initial States)
int targetPopulation = 1000000; // 1 ล้านตัว
long currentPopulation = 1;   // ใช้ long เผื่อแบคทีเรียทะลุล้านไปไกลมาก
int hoursElapsed = 0;         // นาฬิกาจับเวลา (เริ่มต้น 0 ชม.)

Console.WriteLine($"[ชั่วโมงที่ 0] เริ่มต้นหยอดแบคทีเรียลงในจานเพาะเชื้อ: {currentPopulation} ตัว");

// 2. สร้างวงลูปหมุนตามเงื่อนไขเป้าหมาย
// ตราบใดที่จำนวนประชากร ยังคง "น้อยกว่าหรือเท่ากับ" 1 ล้าน
while (currentPopulation <= targetPopulation)
{
    // ในทุกๆ การหมุน 1 รอบ (1 ชั่วโมง) จะเกิดเหตุการณ์ 2 อย่าง:
    
    // เหตุการณ์ที่ 1: แบคทีเรียแบ่งตัวเพิ่ม 2 เท่า
    currentPopulation *= 2; 
    
    // เหตุการณ์ที่ 2: เวลาผ่านไป 1 ชั่วโมง
    hoursElapsed++;

    // แสดงสถิติเพื่อให้เห็นภาพการเติบโตแบบ Exponential
    Console.WriteLine($"> ชั่วโมงที่ {hoursElapsed:D2}: ประชากร = {currentPopulation:N0} ตัว");
}

// 3. หลุดพ้นจากลูปเมื่อประชากรทะลุ 1 ล้านไปแล้ว
Console.WriteLine("\n-------------------------------------");
Console.WriteLine("🚨 สัญญาณเตือนภัยทำงาน! ประชากรแบคทีเรียเกินขีดจำกัดแล้ว!");
Console.WriteLine($"สรุป: ใช้เวลาทั้งหมด [ {hoursElapsed} ชั่วโมง ]");
Console.WriteLine($"จำนวนแบคทีเรียสุดท้ายหยุดที่: {currentPopulation:N0} ตัว");
Console.WriteLine("=====================================");
```

**บทวิเคราะห์จากมุมมองสถาปนิกซอฟต์แวร์:**
- หากนำโค้ดไปรัน จะพบว่าในชั่วโมงแรกๆ ตัวเลขจะเพิ่มช้ามาก (2->4->8->16)
- แต่พอผ่านไปถึงชั่วโมงที่ 15 ตัวเลขจะกระโดดไป 32,768 
- และทะลุหลักล้าน (1,048,576) ได้สำเร็จในชั่วโมงที่ **20** พอดี!
- พลังของการทำซ้ำ (Iteration) บวกกับคณิตศาสตร์เลขยกกำลัง (Exponential) เป็นกระบวนการที่คอมพิวเตอร์เก่งกาจกว่ามนุษย์นับล้านเท่า (ถ้าให้คนมานั่งคูณเลขทีละบรรทัด คงใช้เวลาหลายนาที แต่ CPU คำนวณรันโค้ดทั้งหมดนี้ภายในเวลาไม่ถึง `0.001` วินาที)
