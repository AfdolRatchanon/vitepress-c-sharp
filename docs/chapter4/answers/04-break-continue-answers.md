# เฉลย: Jump Statements (`break` & `continue`)

## 🎯 เฉลย Mini Exercise

### **โจทย์ที่ 1: นักทำลายลูป (The Breaker)**
**เป้าหมาย:** สร้างระบบลูปอนันต์ `while(true)` ที่นกแก้วพูดตาม (Echo) และจะหยุดเมื่อเจอคำสาป "quit"

**เฉลยและวิธีทำ:**
นี่คือแพทเทิร์นสุดคลาสสิกที่ถูกใช้ในเบื้องหลังโปรแกรม Terminal หรือ Command Prompt ของระบบปฏิบัติการ (OS Shell) ทุกชนิดบนโลก! (เช่น เวลารอรับคำสั่ง `git pull` ถ้าพิมพ์ `exit` หน้าต่างดำๆ ก็จะปิดลงไป)

```csharp
Console.WriteLine("=== บอทนกแก้วจอมกวน (พิมพ์ 'quit' เพื่อไล่บอทไป) ===");

// 1. สร้างลูปอนันต์อย่างจงใจ
while (true)
{
    Console.Write("ผู้ใช้: ");
    string input = Console.ReadLine();

    // 2. ประตูทางออก (Exit Condition Guard)
    // เช็คก่อนเลยว่าคำที่พิมพ์มาคือรหัสทำลายล้างหรือไม่
    // แปลงข้อความเป็นตัวเล็กเสมอด้วย .ToLower() เผื่อผู้ใช้พิมพ์ 'Quit' หรือ 'QUIT' จะได้ออกได้
    if (input.ToLower() == "quit")
    {
        Console.WriteLine("นกแก้ว: นกแก้วยอมแพ้แล้ว ลาก่อนนน! 🦜💨");
        
        // ดึงคันโยกฉุกเฉิน ทำลายลูปอนันต์ทิ้งทันที!
        break; 
    }

    // 3. ถ้าไม่ได้พิมพ์ quit โค้ดส่วนนี้ถึงจะได้ทำงาน
    Console.WriteLine($"นกแก้ว: คุณพูดว่า '{input}' !! 🦜");
}

Console.WriteLine("--- จบโปรแกรม ---");
```

### **โจทย์ที่ 2: ระบบข้ามเลขอาถรรพ์ (The Superstitious Counter)**
**เป้าหมาย:** พิมพ์เลขชั้น 1 ถึง 20 แต่ห้ามมีชั้น 4 และชั้น 13 (กระโดดข้าม)

**เฉลยและวิธีทำ:**
ความผิดพลาดของมือใหม่ในข้อนี้คือ มักจะไปเขียนโค้ด `if` สองอันซ้อนกันให้ยุ่งเหยิง หรือเผลอเอา `continue` ไปสลับที่กับ Console.Write ทำให้พิมพ์ชื่อชั้นออกมาก่อน แล้วค่อยกระโดดข้าม (ซึ่งไม่ได้ประโยชน์อะไร) กฎเหล็กของ `continue` คือ **ต้องตั้งด่านสกัดไว้ข้างบนสุดเสมอ (Guard Clause)**

```csharp
Console.WriteLine("=== ระบบลิฟต์คอนโด (ไม่มีชั้น 4 และ 13) ===");

for (int floor = 1; floor <= 20; floor++)
{
    // 1. ด่านสกัดกั้น (Guard Filter)
    // ถ้าชั้นปัจจุบัน ตรงกับเลขอาถรรพ์
    if (floor == 4 || floor == 13)
    {
        // ยกเลิกการประมวลผลรอบนี้ และกระโดดกลับขึ้นไปหัว for เพื่อดึงชั้นต่อไปมาทำทันที
        continue; 
    }

    // 2. กระบวนการปกติ (ทำงานเฉพาะเมื่อหลุดจากด่านสกัดมาได้)
    Console.WriteLine($"กำลังเดินทางไปที่... ชั้น {floor}");
}

// ผลลัพธ์: ชั้น 3 แล้วจะกระโดดไป ชั้น 5 ทันที!
```

## 🔥 เฉลย Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: ตามหาจำนวนเฉพาะ (Prime Number Finder)**
เป้าหมายคือการสร้าง Algorithm ระดับสูง เพื่อพิสูจน์ว่าตัวเลขที่กรอกมา เป็นจำนวนเฉพาะหรือไม่ โดยใช้การหา "หลักฐาน" (ตัวหาร) ถ้าเจอหลักฐานแค่ชิ้นเดียว ถือว่าผิดทันที และสั่ง `break` ศาลสั่งปิดคดีโดยไม่ต้องสืบต่อ!

**กระบวนการคิดและเฉลยเชิงลึก (Algorithmic Approach):**
นี่คือโจทย์คัดกรองโปรแกรมเมอร์เข้าทำงาน (Whiteboard Interview) ระดับตำนาน

```csharp
Console.WriteLine("======================================");
Console.WriteLine("     🔬 PRIME NUMBER DETECTOR 🔬      ");
Console.WriteLine("======================================");

Console.Write("กรุณาป้อนตัวเลขเพื่อทำการตรวจสอบ: ");
int target = int.Parse(Console.ReadLine());

// ดักจับกรณีพิเศษ (Edge cases): เลข 1 หรือเลขติดลบ ไม่ใช่จำนวนเฉพาะตามหลักคณิตศาสตร์
if (target <= 1)
{
    Console.WriteLine($"[ผลการวิเคราะห์] {target} ❌ ไม่ใช่จำนวนเฉพาะ");
    return; // คำสั่ง return ใช้หยุดการทำงานของ Method Main() ทั้งหมดทันที (แรงกว่า break)
}

// 1. สร้างธง (Flag Variable) สมมติว่ามันบริสุทธิ์ (เป็นจำนวนเฉพาะ) ไปก่อน จนกว่าจะหาหลักฐานเจอ
bool isPrime = true;

// 2. สร้างทีมค้นหา (For loop) ไล่ตั้งแต่เลข 2 ไปจนถึง ค่าก่อนหน้าตัวมันเอง (target - 1)
Console.WriteLine($"[ระบบ] เริ่มต้นกระบวนการหารตั้งแต่ 2 ถึง {target - 1}...");

for (int divisor = 2; divisor < target; divisor++)
{
    // 3. กระบวนการพิสูจน์หลักฐาน
    // ลองเอาเลขเป้าหมาย มาหารเอาเศษ (Modulo) ด้วยทีมค้นหาแต่ละเบอร์
    if (target % divisor == 0)
    {
        // 💥 เจอแล้ว! มันหารลงตัวเป๊ะ (เศษเป็น 0)
        // แสดงว่ามันมีตัวประกอบอื่นนอกจาก 1 กับตัวมันเอง!
        Console.WriteLine($"> ตรวจพบหลักฐาน: {target} สามารถหารด้วย {divisor} ลงตัวเป๊ะ!");
        
        // พลิกธงให้เป็นเท็จทันที (ทำลายความบริสุทธิ์)
        isPrime = false;
        
        // ดึงคันโยกฉุกเฉิน! ศาลสั่งปิดคดี! ไม่จำเป็นต้องลองเอาเลขอื่นมาหารต่อให้เสียเวลา
        // (เช่น สมมติเป้าหมายคือ 1,000,000 พอเราเอา 2 ไปหารลงตัว เราก็สั่ง break เลย 
        // ช่วยประหยัดการวนลูปเปล่าๆ ไปได้ถึง 999,997 รอบ! นี่คือพลังของ Optimization)
        break; 
    }
}

// 4. บทสรุปผลวิเคราะห์ (อ่านค่าจากธง)
Console.WriteLine("\n--------------------------------------");
if (isPrime)
{
    Console.WriteLine($"✅ ยืนยัน! {target} คือ 'จำนวนเฉพาะ' (Prime Number)");
}
else
{
    Console.WriteLine($"❌ ฟันธง! {target} 'ไม่ใช่' จำนวนเฉพาะ");
}
Console.WriteLine("======================================");
```

**เคล็ดวิชาความเร็วแสง (Performance Trick O(sqrt(N))):**
ในโลกของการแข่งขันโอลิมปิกวิชาการ ลูป `for (int divisor = 2; divisor < target; divisor++)` จะถูกมองว่าเป็นโค้ดที่ช้ามาก!
ความจริงทางคณิตศาสตร์คือ "ตัวประกอบคู่ที่ใหญ่ที่สุดของตัวเลข จะไม่มีทางมีค่าเกิน รากที่สอง (Square Root) ของตัวมันเอง" 
ดังนั้น โปรแกรมเมอร์ระดับเซียนจะเขียนเงื่อนไขเป็น:
`for (int divisor = 2; divisor <= Math.Sqrt(target); divisor++)`
ถ้าคุณเอาโค้ดนี้ไปตรวจเลข **1,000,000** แทนที่จะต้องวิ่งลูปแสนรอบ มันจะวิ่งแค่ **1,000 รอบ** เท่านั้น! นี่คือศิลปะของการผสมผสานคณิตศาสตร์เข้ากับคอมพิวเตอร์ศาสตร์ครับ!
