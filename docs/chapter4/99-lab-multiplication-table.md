## 📖 ภาพรวมของโปรแกรม (Program Overview)

ในฐานะโปรแกรมเมอร์ประจำสถาบันกวดวิชาคณิตศาสตร์ คุณได้รับโจทย์ให้พัฒนา "โปรแกรมสร้างตารางสูตรคูณแบบคัสตอม (Customizable Multiplication Matrix)" ที่อนุญาตให้นักเรียนเลือกได้ว่าจะดูสูตรคูณแม่ไหนถึงแม่ไหน (เช่น เริ่มที่แม่ 2 ไปจบที่แม่ 5) 

แต่ความท้าทายคือ โปรแกรมจะต้องป้องกันข้อผิดพลาดจากการพิมพ์ของเด็กๆ อย่างรัดกุม (ห้ามพิมพ์ตัวอักษร, ห้ามพิมพ์แม่ติดลบ) และที่สำคัญคือ **มันต้องแสดงผลออกมาในรูปแบบตารางที่เรียงเป็นคอลัมน์สวยงาม ไม่ใช่การปรินท์ลงมาเป็นเส้นก๋วยเตี๋ยวยาวๆ แบบโปรแกรมเมอร์มือใหม่!**

## 📋 กฎกติกาและเกณฑ์การประเมิน (Business Requirements)

1. **ระบบเมนูอมตะ (The Main Loop):**
   - โปรแกรมทั้งหมดต้องถูกหุ้มด้วย `do-while` loop เพื่อให้เมื่อคำนวณสูตรคูณเสร็จ มันจะถามผู้ใช้ว่า "ต้องการสร้างตารางใหม่หรือไม่?" ถ้าตอบ Y ให้วนกลับไปเคลียร์หน้าจอและรันใหม่
2. **ระบบกักขังผู้ใช้ดื้อรั้น (Input Validation Trap):**
   - ในขั้นตอนการรับเลข "แม่สูตรคูณเริ่มต้น" และ "สิ้นสุด" ให้ใช้ `while` (หรือ `do-while`) คู่กับ `int.TryParse` ดักไว้เลยว่า:
     - กฎข้อที่ 1: ต้องกรอกเป็นตัวเลขเท่านั้น (ห้ามแครช)
     - กฎข้อที่ 2: ห้ามกรอกเลขติดลบ และห้ามกรอกเลข 0 (สูตรคูณแม่ 0 ไม่มีประโยชน์)
     - กฎข้อที่ 3: แม่สิ้นสุด ต้องมีค่ามากกว่า หรือเท่ากับ แม่เริ่มต้นเสมอ! (เช่น เริ่ม 2 จบ 5 ได้ แต่ เริ่ม 5 จบ 2 ไม่ได้)
     - *ถ้าผู้ใช้แหกกฎข้อใดข้อหนึ่ง โปรแกรมต้องเตือนสีแดง และวนลูปบังคับให้กรอกใหม่จนกว่าจะถูกต้อง*
3. **มิติแห่งตารางสูตรคูณ (Nested Loops Architecture):**
   - การจะพิมพ์ตารางออกมาให้เป็นหน้ากระดานกว้างๆ แบบในตัวอย่าง คุณต้องใช้ **Nested Loop แบบกลับด้าน (Inverted Thinking)** 
   - ลูปชั้นนอก ต้องวิ่งรอบการคูณ (แม่ x 1 จนถึง แม่ x 12)
   - ลูปชั้นใน ต้องวิ่งรายชื่อแม่ (เช่น แม่ 2 ไป แม่ 5) 
4. **ความสวยงามของตาราง (Padding & Formatting):**
   - คุณต้องหาวิธีจัดช่องว่างให้ตรงกัน! สังเกตว่า `2x2=4` มันสั้นกว่า `2x10=20` ถ้าพิมพ์ต่อกันทื่อๆ ตารางจะเบี้ยว
   - *คำใบ้:* ค้นหาวิธีใช้เครื่องมือ `.PadRight(10)` ในภาษา C# มันคือท่าไม้ตายที่จะเติมช่องว่างให้ครบ 10 ตัวอักษรเสมอไม่ว่าข้อความจะสั้นแค่ไหน!

## 📝 ขั้นตอนการพัฒนา (Step-by-Step Implementation)

### ขั้นตอนที่ 1: วางโครงข่ายเมนูหลัก
- [ ] สร้างลูป `do-while` ขนาดใหญ่ครอบโปรแกรมทั้งหมด
- [ ] วางคำสั่งถามผู้ใช้ตอนท้ายสุดว่า "ต้องการสร้างตารางใหม่หรือไม่? (Y/N)" รับค่าใส่ตัวแปร แล้วเอาไปตั้งเงื่อนไขเช็คใน `while` ด้านล่างสุด

### ขั้นตอนที่ 2: ดักจับ Input ขั้นเทพ (The Validator)
- [ ] เขียนลูปรับเลข **แม่เริ่มต้น (startMultiplier)** 
  - สร้างตัวแปร bool มาเป็นธง (Flag) เช่น `bool isValidStart = false;`
  - ใช้ `while (!isValidStart)` ครอบคำสั่งรับค่า ถ้า TryParse ผ่านและตัวเลข > 0 ให้สับสวิตช์ธงเป็น `true` เพื่อออกจากลูป
- [ ] เขียนลูปรับเลข **แม่สิ้นสุด (endMultiplier)**
  - ใช้เทคนิคธงแบบเดียวกัน แต่เพิ่มเงื่อนไขว่าตัวเลขนี้ ต้อง **`>= startMultiplier`** ด้วยถึงจะยอมให้ผ่าน

### ขั้นตอนที่ 3: วาดหัวตาราง (Table Header)
- [ ] ใช้ `for` loop (แค่ชั้นเดียว) วิ่งจากเลข `startMultiplier` ไปจนถึง `endMultiplier`
- [ ] สั่ง `Console.Write($"แม่ {i}".PadRight(10));`
- [ ] จบลูปแล้วอย่าลืมเคาะ `Console.WriteLine()` ขึ้นบรรทัดใหม่ พร้อมขีดเส้นใต้ -----------------

### ขั้นตอนที่ 4: สงคราม Nested Loop (The Core Logic)
นี่คือจุดที่ยากที่สุด!
- [ ] สร้างลูปชั้นนอก (Outer Loop) ให้วิ่งตัวแปร `row` ตั้งแต่ 1 ไปจนถึง 12 (นี่คือตัวแทนของ x1, x2, x3... ถึง x12)
- [ ] **ข้างในลูปชั้นนอก** ให้สร้างลูปชั้นใน (Inner Loop) วิ่งตัวแปร `col` ตั้งแต่ `startMultiplier` ไปจนถึง `endMultiplier`
- [ ] **ข้างในลูปชั้นใน** ทำการคำนวณผลลัพธ์: `int result = col * row;`
- [ ] สร้างข้อความ: `string text = $"{col}x{row}={result}";`
- [ ] สั่งพิมพ์แบบไม่ขึ้นบรรทัดใหม่: `Console.Write(text.PadRight(10));`
- [ ] **เมื่อหลุดออกจากลูปชั้นใน (แต่ยังอยู่ในลูปนอกสุด):** สั่ง `Console.WriteLine();` เพื่อกระโดดลงไปปรินท์ x2 ในบรรทัดล่างถัดไป!

### ขั้นตอนที่ 5: ตรวจสอบความถูกต้อง (QA Testing)
- [ ] ลองกรอกตัวอักษรแทนตัวเลข (โปรแกรมต้องไม่พัง)
- [ ] ลองกรอกแม่เริ่มต้น: 5, แม่สิ้นสุด: 2 (โปรแกรมต้องด่าแล้วให้กรอกใหม่)
- [ ] ลองกรอกแม่เริ่มต้น: 2, แม่สิ้นสุด: 5 (เช็คว่าตารางหน้าตาเหมือนภาพตัวอย่างเป๊ะๆ หรือไม่)

## 🔥 Challenge (ภารกิจท้าทายขีดจำกัด)

สำหรับโปรแกรมเมอร์ที่ไม่ยอมแพ้อะไรง่ายๆ ลองเพิ่มออพชั่นเหล่านี้เข้าไปในโปรแกรมของคุณ:

**ภารกิจที่ 1: ข้ามการคูณเลขอาถรรพ์ (The Continue Filter)**
สมมติว่าสถาบันนี้มีความเชื่อแปลกๆ ห้ามไม่ให้นักเรียนท่องสูตรคูณที่คูณด้วยเลข **13** 
- ปรับลูปชั้นนอกของคุณให้วิ่งไปถึง x24 (คูณ 24)
- จงใช้คำสั่ง **`continue;`** แทรกเข้าไปในจุดที่ถูกต้อง เพื่อดักไว้ว่า ถ้ารอบการคูณ (row) เป็นเลข 13 ให้กระโดดข้ามการพิมพ์ตารางบรรทัดนั้นไปเลย! (บรรทัด x12 แล้วต้องกระโดดไป x14 ทันที)

**ภารกิจที่ 2: ไฮไลต์สีให้เลขเบิ้ล (Conditional Coloring)**
หากผลลัพธ์ของการคูณ เป็นการเอาเลขตัวเดียวกันมาคูณกัน (เช่น แม่ 2 x 2, แม่ 5 x 5, แม่ 12 x 12) ให้เปลี่ยนสีตัวหนังสือเฉพาะสมการนั้นให้เป็น **สีเขียวสะท้อนแสง (ConsoleColor.Green)** เพื่อให้นักเรียนสังเกตเห็นตัวเลขยกกำลังสองได้ง่ายขึ้น
*(คำใบ้: ใช้การเช็ค if `col == row` ก่อนทำการ Console.Write แล้วสลับกลับเป็น ConsoleColor.Gray เมื่อพิมพ์เสร็จ)*

ขอให้โชคดีกับการสร้างสรรค์ตารางสูตรคูณระดับเทพของคุณ!
