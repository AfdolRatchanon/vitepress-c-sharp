# The `do-while` loop (ทำก่อน ค่อยถามทีหลัง)

> 💡 **เป้าหมาย:** ทำความรู้จักกับลูปสายพันธุ์คนจริง ที่มีคติประจำใจว่า **"ลงมือทำไปก่อน 1 ครั้งเสมอ แล้วค่อยเช็คทีหลังว่าทำผิดไหม"** ซึ่งเป็นเครื่องมือที่ดีที่สุดในโลกสำหรับการสร้างระบบสารบัญ (Menu System) ให้กับ Console Application ของคุณ!

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. ปัญหาของ While Loop ธรรมดา

ในบทที่แล้ว เราเรียนรู้ว่า `while` เป็นแบบ **Pre-test (ตรวจสอบก่อนทำ)** ซึ่งถ้าเงื่อนไขเป็นเท็จตั้งแต่แรก มันจะไม่ทำงานเลยแม้แต่รอบเดียว
แต่ในบางสถาปัตยกรรมของซอฟต์แวร์ เช่น "การสร้างหน้าจอเมนู (Main Menu)" ให้ผู้ใช้กดเลือก 
เรา **ต้องการให้เมนูโชว์ขึ้นมาให้ผู้ใช้เห็นก่อนอย่างน้อย 1 ครั้งเสมอ!** ไม่งั้นผู้ใช้จะรู้ได้อย่างไรว่ามีเมนูอะไรให้กดบ้าง?

ถ้าใช้ `while` ธรรมดาเขียนเมนู คุณจะต้องเจอสภาพที่น่าเกลียดแบบนี้:
```csharp
int choice = 0; // ต้องตั้งค่าหลอกๆ ให้ผ่านเงื่อนไข while ตอนแรกให้ได้
while (choice != 9)
{
    Console.WriteLine("1. เริ่มเกม | 9. ออก");
    choice = int.Parse(Console.ReadLine());
}
```
การต้องมานั่งตั้งตัวแปร "หลอกๆ (Dummy value)" เพื่อหลอกรปภ. ให้เปิดประตูลูปในครั้งแรก ถือเป็นโค้ดที่ไม่สวยงามและเกิดบั๊กได้ง่าย (Code Smell)

นี่คือจุดที่ **`do-while`** ขี่ม้าขาวมาช่วยแก้ปัญหา!

---

### 2. สถาปัตยกรรมของ `do-while` (The Post-test Loop)

`do-while` มีการทำงานแบบกลับหัวกลับหาง (Inverted) คือเอาบล็อกการกระทำ (`do`) ขึ้นก่อน แล้วเอาเงื่อนไขการตรวจบัตร (`while`) ไปไว้ล่างสุด

```text
ไวยากรณ์ (Syntax):

  do
  {
      // โค้ดตรงนี้จะถูก "การันตี" (Guaranteed) ว่าจะทำงานล้านเปอร์เซ็นต์ อย่างน้อย 1 ครั้งแน่ๆ 
      // โดยไม่มีการตรวจสอบเงื่อนไขใดๆ ขัดขวาง
      
  } while ( เงื่อนไขที่เป็นจริง ); // ⚠️ สังเกตว่าบรรทัดนี้ "ต้อง" จบด้วยเซมิโคลอน (;) เสมอ ซึ่งต่างจากลูปแบบอื่น!
```

**กฎเหล็ก 3 ประการของ `do-while`:**
1. **ทำงานก่อนเสมอ:** เหมาะสมอย่างยิ่งกับงานที่ต้อง "ขอรับค่าจากผู้ใช้ก่อน แล้วค่อยนำค่านั้นมาเช็คทีหลัง"
2. **ขอบเขตตัวแปรสุดอันตราย (Scope Warning):** ถ้าคุณประกาศตัวแปร *ข้างใน* ปีกกาของ `do` ตัว `while` ที่อยู่ข้างล่างสุดจะมองไม่เห็นตัวแปรนั้น! คุณต้องสร้างตัวแปรไว้ *ข้างบน* หัวของ `do` เสมอ
3. **ห้ามลืมเซมิโคลอน:** จุดจบของ `while(condition);` ต้องมีเซมิโคลอนปิดประโยค ไม่งั้นจะ Compile ไม่ผ่าน

### 3. วงจรการทำงานเบื้องหลัง (Execution Flow)

การไล่สเต็ปการทำงาน (Tracing) ของ `do-while` นั้นตรงไปตรงมามาก:

```text
แผนภาพวงจรชีวิตของ Do-While Loop

  ┌────────────────────────────────────┐
  │                                    │  <-- (วนกลับมาที่นี่ถ้าผ่านเงื่อนไข)
  │  [1. เข้าไปทำงานในปีกกา ทันที! ]   │  <-- (ไม่มีประตู ไม่มี รปภ. ขวาง)
  │        │                           │
  │        ▼                           │
  │  [2. ประมวลผลและอัปเดตตัวแปร ]     │
  │        │                           │
  │        ▼                           │
  │  [3. เช็คเงื่อนไข (Condition) ] ──(True)
  │
  └───────(False)
             │
             ▼
     [ จบการทำงาน กระโดดออก ]
```

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่างที่ 1: ระบบเมนูหลักของตู้ ATM (Interactive Console Menu)

นี่คือโครงสร้างที่โปรแกรมเมอร์ทุกคน "ต้อง" เคยเขียนอย่างน้อยหนึ่งครั้งในชีวิต มันคือแพทเทิร์นสุดคลาสสิกที่ถูกใช้ในโปรแกรมระดับองค์กร

::: code-group
```csharp [Program.cs]
using System;

namespace ATMSystem
{
    class Program
    {
        static void Main(string[] args)
        {
            // 1. ประกาศตัวแปร "นอกลูป" เพื่อให้บล็อก while() ด้านล่างสุดมองเห็น
            int userChoice;

            // 2. ใช้ do-while เพื่อบังคับโชว์เมนูอย่างน้อย 1 ครั้ง
            do
            {
                // -- วาดหน้าจอเมนูให้สวยงาม --
                Console.WriteLine("\n==============================");
                Console.WriteLine("    ยินดีต้อนรับสู่ C# Bank    ");
                Console.WriteLine("==============================");
                Console.WriteLine(" 1. เช็คยอดเงินคงเหลือ");
                Console.WriteLine(" 2. ถอนเงิน");
                Console.WriteLine(" 3. ฝากเงิน");
                Console.WriteLine(" 9. ออกจากระบบ (คืนบัตร)");
                Console.WriteLine("==============================");
                Console.Write("กรุณาเลือกทำรายการ (1-9): ");
                
                // รับค่าจากผู้ใช้ (เรายังคงใช้ Parse ทื่อๆ เพื่อให้โฟกัสที่ Loop)
                userChoice = int.Parse(Console.ReadLine());

                // ประมวลผลตามที่ผู้ใช้เลือก (เอาไปผสมกับ switch แจ่มมาก!)
                switch (userChoice)
                {
                    case 1:
                        Console.WriteLine(">> ยอดเงินของคุณคือ: 5,000,000 บาท");
                        break;
                    case 2:
                        Console.WriteLine(">> ระบบกำลังจ่ายธนบัตร โปรดรอสักครู่...");
                        break;
                    case 3:
                        Console.WriteLine(">> กรุณาใส่ธนบัตรลงในช่องรับฝากเงิน");
                        break;
                    case 9:
                        Console.WriteLine(">> ขอบคุณที่ใช้บริการ โปรดอย่าลืมรับบัตรคืนครับ");
                        break;
                    default:
                        Console.WriteLine("❌ ข้อผิดพลาด: ไม่พบเมนูที่คุณเลือก กรุณาลองใหม่");
                        break;
                }

                // ถ่วงเวลาให้ผู้ใช้อ่านผลลัพธ์สักนิด (เทคนิค UX)
                Console.WriteLine("\n(กด Enter เพื่อดำเนินการต่อ...)");
                Console.ReadLine(); // หยุดรอให้กดปุ่มเคลียร์หน้าจอ

                Console.Clear(); // ล้างหน้าจอคอนโซลให้สะอาด ก่อนวนลูปโชว์เมนูใหม่

            } while (userChoice != 9); // ตรวจสอบเงื่อนไขหลังสุด: หมุนไปเรื่อยๆ จนกว่าจะกดเลข 9
            
            // เมื่อหลุดออกจากลูป
            Console.WriteLine("--- จบการทำงานของโปรแกรม ---");
        }
    }
}
```
:::

**ความฉลาดของโครงสร้างนี้:**
ลองจินตนาการว่าคุณเขียนด้วย `while` ธรรมดา คุณต้องมีตัวแปร `choice = 0;` ดักไว้ก่อนเข้าลูป และถ้าข้างในมีอะไรซับซ้อน โค้ดจะอ่านยากขึ้น แต่ด้วย `do-while` โฟลว์มันเป็นธรรมชาติ (Natural Flow) มาก คือ โชว์หน้าจอ -> รับค่า -> ตรวจค่า -> วนใหม่ถ้าไม่ใช่ออก

### ตัวอย่างที่ 2: ระบบสุ่มรหัสผ่านจนกว่าจะถูกต้อง (Brute-Force Simulation)

ในยุค Cyber Security แฮกเกอร์ใช้วิธีสุ่มเดารหัสผ่าน (Brute-force attack) โค้ดนี้จำลองให้คอมพิวเตอร์เดารหัสไปเรื่อยๆ จนกว่าจะเจอ

::: code-group
```csharp [Program.cs]
using System;

namespace PasswordHacker
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบจำลองการแฮกรหัสผ่าน (Brute-Force Demo) ===");
            
            // รหัสผ่านที่เป็นความลับ (เป้าหมาย)
            int targetPin = 845;
            
            // สร้างเครื่องสุ่มตัวเลขแบบ C#
            Random rng = new Random();
            
            int currentGuess;
            int attemptCount = 0;

            // บังคับให้เริ่มเดาก่อน 1 ครั้งเสมอ
            do
            {
                attemptCount++;
                
                // สุ่มตัวเลข 3 หลัก (ตั้งแต่ 000 ถึง 999)
                currentGuess = rng.Next(0, 1000); 

                // เพื่อไม่ให้หน้าจอรกเกินไป เราจะสั่งปรินท์เฉพาะทุกๆ 100 รอบ
                if (attemptCount % 100 == 0)
                {
                    Console.WriteLine($"[รอบที่ {attemptCount}] กำลังทดสอบรหัส: {currentGuess:D3}..."); // :D3 คือใส่ 0 ข้างหน้าให้ครบ 3 หลักเสมอ
                }

            } while (currentGuess != targetPin); // ตรวจสอบเงื่อนไข: "ให้ทำต่อตราบใดที่ยังเดาไม่ตรง"

            // หลุดลูปมาได้ แปลว่าเดาถูกแล้ว (currentGuess == targetPin กลายเป็นจริง)
            Console.WriteLine("\n====================================");
            Console.WriteLine($"🔓 แฮกสำเร็จ! รหัสผ่านคือ [ {currentGuess:D3} ]");
            Console.WriteLine($"🤖 คอมพิวเตอร์เดาสุ่มไปทั้งหมด {attemptCount} ครั้ง");
            Console.WriteLine("====================================");
        }
    }
}
```
:::

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

### **โจทย์ที่ 1: แกะรอยบั๊กสุดคลาสสิก (Scope Trap)**
โปรแกรมเมอร์ฝึกหัดคนหนึ่ง ต้องการเขียนตู้เกมทายหัวก้อย (รับคำว่า "Y" เพื่อเล่นต่อ หรือ "N" เพื่อเลิก) เค้าเขียนโครงสร้างมาแบบนี้:
```csharp
do
{
    Console.WriteLine("โยนเหรียญได้... หัว!");
    Console.Write("ต้องการเล่นอีกรอบหรือไม่? (Y/N): ");
    string answer = Console.ReadLine();

} while (answer == "Y");
```
**คำถาม:** โค้ดนี้เขียนถูกหลักการ แต่ทำไมพอกด Build (รัน) ถึงขึ้น Error แดงเถือกที่บรรทัดสุดท้าย (`while (answer == "Y")`) ว่า "The name 'answer' does not exist in the current context"? จงอธิบายเหตุผลจากทฤษฎี Scope และแก้ไขโค้ดให้ถูกต้อง

### **โจทย์ที่ 2: ระบบยืนยันตัวตนคนดื้อ (Stubborn Captcha)**
**ภารกิจ:** ให้เขียนโปรแกรมจำลอง CAPTCHA อัตโนมัติด้วย `do-while`
- ตัวแปรตั้งต้น: `string secretWord = "C#Rocks";`
- วงจรการทำงาน:
  1. ให้โปรแกรมสั่งให้ผู้ใช้: `"กรุณาพิมพ์คำว่า C#Rocks เพื่อพิสูจน์ว่าคุณไม่ใช่บอท: "`
  2. รับค่าที่ผู้ใช้พิมพ์มา 
  3. ตรวจสอบว่าตรงกับรหัสลับหรือไม่ (พิจารณาตัวพิมพ์เล็กพิมพ์ใหญ่ด้วย)
  4. ถ้าย้อนกลับไปข้อ 3 แล้วพบว่าผู้ใช้พิมพ์ผิด (ไม่เหมือนเป๊ะ) ให้วนกลับไปทำข้อ 1 ใหม่เรื่อยๆ อย่างไม่มีจุดสิ้นสุด
- เมื่อกรอกถูกเป๊ะ ให้หลุดลูปและโชว์คำว่า `"ยินดีต้อนรับ มนุษย์!"`

::: details 💡 คำใบ้และแนวทาง (Hints)
- **สำหรับโจทย์ 1:** ปัญหาคือตัวแปร `answer` ถูกประกาศ **ภายใน** ปีกกา `{}` ของคำสั่ง `do` ทำให้พอหลุดปีกกาออกมาแล้ว มันถูกคอมพิวเตอร์สั่งลบความจำทิ้งทันที ตัว `while` ที่อยู่นอกปีกกาจึงมองไม่เห็นมัน! วิธีแก้คือต้องย้ายคำว่า `string answer = "";` ขึ้นไปไว้ด้านบนคำว่า `do`
- **สำหรับโจทย์ 2:** โครงสร้างหลักคือประกาศตัวแปร `string input;` ไว้นอกลูป แล้วในลูปให้รับค่าใส่ตัวแปรนั้น และเงื่อนไข `while` คือตราบใดที่ข้อมูลที่รับมา "ไม่เท่ากับ" (!=) รหัสลับ ก็ให้หมุนต่อไป
:::

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** ในเอกสารระดับโลกของ Microsoft นิยามว่า "The `do` statement executes a statement or a block of statements while a specified Boolean expression evaluates to true." สรุปแบบภาษาคนคือ ทำไมการใช้ `while` กับ `do-while` จึงให้ผลลัพธ์เหมือนกันเป๊ะในแทบทุกกรณีที่การวนรอบตั้งแต่รอบที่ 2 เป็นต้นไป?
**แนวคำตอบ:** เพราะความแตกต่างเพียงจุดเดียวของลูปทั้งสองชนิด อยู่ที่ **"เสี้ยววินาทีแรกสุดที่โปรแกรมวิ่งมาถึง (First Iteration)"** เท่านั้น 
- `while` จะเช็คเงื่อนไขก่อนก้าวเท้าเข้าวงลูป
- `do-while` จะก้าวเท้าเข้าไปทำก่อนเลย 1 ครั้งหน้าด้านๆ แล้วค่อยมาเช็คตอนกำลังจะก้าวรอบที่สอง
แต่พอผ่านรอบแรกไปได้แล้ว พฤติกรรมของมันคือ "ทำ-เช็ค-ทำ-เช็ค" เหมือนกันเป๊ะร้อยเปอร์เซ็นต์ (นี่คือเหตุผลที่โปรแกรมเมอร์หลายคนใช้วิธีแปลงโค้ดไปมาตามความถนัด)

**คำถาม 2:** โค้ดนี้จะก่อให้เกิดลูปอนันต์ (Infinite Loop) หรือไม่?
```csharp
bool keepRunning = false;
do
{
    Console.WriteLine("Test");
} while (keepRunning);
```
**แนวคำตอบ:** **ไม่เกิดลูปอนันต์เด็ดขาด** 
ลองไล่สเต็ปการทำงาน (Trace) ดู:
1. โปรแกรมวิ่งเข้า `do` และพิมพ์คำว่า "Test" 1 ครั้ง
2. วิ่งมาเจอ `while (keepRunning)` ซึ่งพบว่าค่าของมันคือ `false`
3. ประตูการวนรอบใหม่ (Iteration) ปิดตาย โปรแกรมจึงกระโดดหลุดออกจากลูปไปบรรทัดถัดไปทันที
ผลลัพธ์: จะพิมพ์คำว่า Test แค่ครั้งเดียวถ้วน!
