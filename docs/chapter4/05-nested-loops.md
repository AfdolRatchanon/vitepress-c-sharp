# Nested Loops (การซ้อนลูป)

> 💡 **เป้าหมาย:** ทำความเข้าใจคอนเซปต์ขั้นสูงของการ "เอาลูปไปใส่ในลูป" ซึ่งเป็นกระบวนการหัวใจหลักในการสร้างระบบตาราง 2 มิติ (Row & Column), ระบบประมวลผลพิกเซลรูปภาพ (X,Y Coordinates), และระบบตรวจสอบแบบจับคู่ทุกกรณี (Brute-Force Pairings)

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. โครงสร้างมิติแห่งเวลา (The Dimensions of Loops)

ในบทที่ผ่านมา เราใช้งานลูปแค่ 1 ชั้นเปรียบเสมือนการเดินเส้นตรง (1 มิติ) แต่ในความเป็นจริง ข้อมูลในคอมพิวเตอร์มักมาในรูปแบบ "ตาราง" (2 มิติ มีแกน X และแกน Y) หรือแม้กระทั่ง "ลูกบาศก์" (3 มิติ)
การจัดการข้อมูลหลายมิติ จำเป็นต้องใช้โครงสร้าง **Nested Loops (ลูปซ้อนลูป)**

```text
ไวยากรณ์ (Syntax) - การนำ for ซ้อน for:

  for (int row = 1; row <= 3; row++)  <--- [ลูปชั้นนอก (Outer Loop)]
  {
      // โค้ดตรงนี้ทำงาน 3 ครั้ง
      
      for (int col = 1; col <= 5; col++) <--- [ลูปชั้นใน (Inner Loop)]
      {
           // ⚠️ โค้ดตรงนี้ทำงาน 3 x 5 = 15 ครั้ง !!
      }
  }
```

---

### 2. กลไกการหมุนของเฟือง (The Gear Mechanics)

ให้จินตนาการถึงลูปซ้อนลูปว่าเหมือน **"เฟืองนาฬิกา"** 
- **ลูปชั้นในสุด (เข็มวินาที):** จะหมุนเร็วที่สุด มันต้องหมุนให้ครบทุกรอบ (เช่น 60 รอบ) จนจบเงื่อนไขของตัวมันเอง
- **ลูปชั้นนอก (เข็มนาที):** จะขยับก้าวเดินไปเพียงแค่ 1 ก้าวเท่านั้น หลังจากที่ลูปชั้นในหมุนจบไปแล้ว 1 เซ็ตเต็มๆ

**การไล่รอบ (Tracing) อย่างละเอียด:**
(อ้างอิงจากตัวอย่างโค้ด `row=3`, `col=5` ด้านบน)
- ลูปชั้นนอก `row` เริ่มต้นที่ 1
  - มันสั่งให้ลูปชั้นใน `col` เริ่มทำงาน...
  - `col` หมุนรอบที่ 1, 2, 3, 4, 5 (จบเงื่อนไข <= 5)
- ลูปชั้นนอก `row` ขยับเป็น 2
  - ลูปชั้นใน `col` **ถูกสร้างใหม่ตั้งแต่ต้น!** และหมุน 1, 2, 3, 4, 5 อีกครั้ง
- ลูปชั้นนอก `row` ขยับเป็น 3
  - ลูปชั้นใน `col` ทำงาน 1, 2, 3, 4, 5 เป็นครั้งสุดท้าย
- ปิดฉากการทำงานทั้งหมดอย่างสมบูรณ์แบบ

### 3. ต้นทุนแห่งความซับซ้อน (Time Complexity & Big O)

*นี่คือเนื้อหาพื้นฐานของวิชาวิทยาการคอมพิวเตอร์ (Computer Science)*
การใช้ลูป 1 ชั้น ถ้ามีข้อมูล N ตัว โปรแกรมจะทำงาน N รอบ (เรียกว่าใช้เวลา $O(N)$)
แต่ถ้าคุณใช้ Nested Loop (2 ชั้น) ที่ทำงาน N รอบทั้งคู่ โปรแกรมจะทำงาน **$N \times N = N^2$ รอบ** (เรียกว่าใช้เวลา $O(N^2)$)

**ตัวอย่างความน่ากลัว:**
ถ้าคุณมีข้อมูล 1,000 ชิ้น:
- ลูป 1 ชั้น ทำงาน `1,000` ครั้ง (ไวมาก ระดับเสี้ยวของเสี้ยววินาที)
- ลูปซ้อนลูป 2 ชั้น ทำงาน `1,000 x 1,000 = 1,000,000` ครั้ง (เริ่มช้าลง)
- ลูปซ้อนลูป 3 ชั้น ทำงาน `1,000 x 1,000 x 1,000 = 1 พันล้าน` ครั้ง (โปรแกรมอาจจะค้างไปหลายวินาที!)

ดังนั้น **กฎเหล็กของการเขียนโค้ดคือ: "จงหลีกเลี่ยงการซ้อนลูปเกิน 3 ชั้นขึ้นไป ถ้าไม่จำเป็นจริงๆ"** เพราะมันจะสูบพลังประมวลผล (CPU) ของเครื่องพังพินาศได้เลย

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่างที่ 1: การวาดตารางและพิกัด 2 มิติ (Coordinate Grid)

การฝึกพื้นฐาน Nested Loop ที่ดีที่สุดคือการสั่งพรินต์ข้อความใน Console ให้เป็นรูปสี่เหลี่ยม

::: code-group
```csharp [Program.cs]
using System;

namespace MatrixGridDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== การวาดแผนที่ 2 มิติ (3 แถว x 4 คอลัมน์) ===");

            int maxRow = 3;
            int maxCol = 4;

            // ลูปชั้นนอก (ตัวแทนของ "แกน Y" หรือการเลื่อนลงทีละบรรทัด)
            for (int y = 1; y <= maxRow; y++)
            {
                // ลูปชั้นใน (ตัวแทนของ "แกน X" หรือการพิมพ์จากซ้ายไปขวา)
                for (int x = 1; x <= maxCol; x++)
                {
                    // ใช้ Console.Write (ไม่มี Line) เพื่อให้มันพิมพ์ต่อกันในบรรทัดเดียว
                    // พิมพ์พิกัด (X,Y) ออกมาให้เห็นชัดๆ
                    Console.Write($"[{x},{y}] "); 
                }

                // ⚠️ สำคัญมาก: เมื่อจบการวิ่งของลูปชั้นใน (จบ 1 แถว) 
                // เราต้องสั่ง "ขึ้นบรรทัดใหม่" เสมอ เพื่อให้แถวต่อไปลงมาอยู่ข้างล่าง
                Console.WriteLine();
            }

            Console.WriteLine("--- จบการวาดตาราง ---");
        }
    }
}
```
:::

**Expected Output:**
```
=== การวาดแผนที่ 2 มิติ (3 แถว x 4 คอลัมน์) ===
[1,1] [2,1] [3,1] [4,1] 
[1,2] [2,2] [3,2] [4,2] 
[1,3] [2,3] [3,3] [4,3] 
--- จบการวาดตาราง ---
```
*(ถ้าสังเกตดีๆ แกน Y จะขยับช้ามาก (1->2->3) ในขณะที่แกน X หมุนเร็วจี๋ (1->2->3->4) วนไปซ้ำๆ)*

### ตัวอย่างที่ 2: ระบบตรวจจับผู้ต้องสงสัยด้วยการจับคู่ (Brute-Force Pair Matching)

โค้ดนี้จำลองระบบการหาคนที่อายุซ้ำกันในห้องเรียน ซึ่งจะต้องใช้การเปรียบเทียบ "ทุกคน จับคู่กับ ทุกคน" (ซึ่งเกิดจาก Nested Loop)

::: code-group
```csharp [Program.cs]
using System;

namespace PairMatchingDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบจับคู่รหัสผ่าน (ผสมตัวอักษร) ===");

            // สมมติแฮกเกอร์ต้องการเจาะรหัสผ่าน 2 ตัวอักษร โดยมีแค่ A, B, C ให้เดา
            char[] letters = { 'A', 'B', 'C' }; // (อาร์เรย์ จะเรียนอย่างละเอียดในบทหลัง)
            int attempt = 1;

            // ลูปชั้นนอก: ล็อคตัวอักษรตัวแรกไว้ก่อน
            for (int i = 0; i < 3; i++)
            {
                char firstChar = letters[i];

                // ลูปชั้นใน: นำตัวอักษรตัวที่สองมาประกบ
                for (int j = 0; j < 3; j++)
                {
                    char secondChar = letters[j];

                    // สร้างคำผสม
                    string passwordGuess = firstChar.ToString() + secondChar.ToString();

                    Console.WriteLine($"รอบที่ {attempt}: ลองเดารหัส -> {passwordGuess}");
                    attempt++;
                }
            }

            Console.WriteLine("================================");
            Console.WriteLine("จำลองการเดารหัส (Brute Force) เสร็จสิ้นทุกกรณี (3x3 = 9 รูปแบบ)");
        }
    }
}
```
:::

**Expected Output:**
```
=== ระบบจับคู่รหัสผ่าน (ผสมตัวอักษร) ===
รอบที่ 1: ลองเดารหัส -> AA
รอบที่ 2: ลองเดารหัส -> AB
รอบที่ 3: ลองเดารหัส -> AC
รอบที่ 4: ลองเดารหัส -> BA
รอบที่ 5: ลองเดารหัส -> BB
รอบที่ 6: ลองเดารหัส -> BC
รอบที่ 7: ลองเดารหัส -> CA
รอบที่ 8: ลองเดารหัส -> CB
รอบที่ 9: ลองเดารหัส -> CC
================================
จำลองการเดารหัส (Brute Force) เสร็จสิ้นทุกกรณี (3x3 = 9 รูปแบบ)
```

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

### **โจทย์ที่ 1: ตารางหมากรุก (Checkerboard Pattern)**
จงเขียนโปรแกรมวาดตารางขนาด 5 แถว 5 คอลัมน์ โดยใช้อักขระ `X` แทนช่องสีดำ และ `O` (ตัวโอ) แทนช่องสีขาว สลับกันไปมาเหมือนกระดานหมากรุก:
```text
XOXOX
OXOXO
XOXOX
OXOXO
XOXOX
```

### **โจทย์ที่ 2: พิระมิดดาวครึ่งซีก (Half-Pyramid Stars)**
นี่คือโจทย์สอบสัมภาษณ์ยอดฮิตตลอดกาลของเด็กสายไอที 
ให้คุณใช้ Nested Loop วาดรูปสามเหลี่ยมดาว 5 ชั้น ดังนี้:
```text
*
**
***
****
*****
```

::: details 💡 คำใบ้และแนวทาง (Hints)
- **สำหรับโจทย์ 1:** คุณจะต้องมีลูป 2 ชั้น (row 1-5, col 1-5) แล้วเอาเทคนิคหาเลขคู่เลขคี่ (Modulo `%`) มาจับคู่กัน ถ้า `(row + col) % 2 == 0` ให้พิมพ์ X ถ้าไม่ใช่ให้พิมพ์ O
- **สำหรับโจทย์ 2:** ลูปชั้นนอกนับ 1 ถึง 5 ปกติ (สมมติชื่อตัวแปร `row`) จุดสับขาหลอกคือลูปชั้นใน! แทนที่คุณจะให้ลูปชั้นในวิ่งไปถึง 5 ตลอด ให้แก้เงื่อนไขเป็น `for (int col = 1; col <= row; col++)` (นั่นคือ ให้คอลัมน์วิ่งเท่ากับจำนวนแถวปัจจุบันเท่านั้น!)
:::

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: พิระมิดอียิปต์ (The Perfect Egyptian Pyramid)**

ถ้าสามเหลี่ยมครึ่งซีกมันง่ายไป ลองทำสามเหลี่ยมแบบเต็มใบให้สมมาตร (Symmetrical) ดูสิ!
ให้คุณเขียนโปรแกรมวาดรูปดาว 5 ชั้น (หรือให้ผู้ใช้กรอกจำนวนชั้นเองก็ได้) โดยมีหน้าตาแบบนี้:
```text
    *
   ***
  *****
 *******
*********
```

**คำแนะนำจากสถาปนิก:**
คุณไม่สามารถใช้ลูปซ้อนกันแค่ 2 ตัวได้อีกแล้ว! โครงสร้างที่ถูกต้องคือ:
- มี 1 ลูปชั้นนอก (คุมจำนวนแถว 5 แถว)
- ข้างในปีกกาลูปนอก จะต้องมี **ลูปชั้นใน 2 ตัว ที่ไม่ได้ซ้อนกันเอง แต่วางเรียงต่อกัน**
  - ลูปชั้นในตัวที่ 1: วิ่งเพื่อพิมพ์ "ช่องว่าง (Spacebar)" ให้ลดลงเรื่อยๆ (ผลักดาวไปตรงกลาง)
  - ลูปชั้นในตัวที่ 2: วิ่งเพื่อพิมพ์ "ดาว (*)" ให้เพิ่มขึ้นเรื่อยๆ ด้วยสมการเลขคี่ `(2 * row) - 1`
ลองท้าทายความสามารถทาง Logic ของสมองคุณดู!

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** ในวงเล็บของลูปชั้นใน เราสามารถตั้งชื่อตัวแปรนับรอบ (เช่น `int i = 0`) ให้ **ซ้ำซ้อน** กับชื่อตัวแปรนับรอบในลูปชั้นนอก (ซึ่งใช้ชื่อ `i` อยู่ก่อนแล้ว) ได้หรือไม่? จะเกิดอะไรขึ้น?
**แนวคำตอบ:** **ไม่ได้เด็ดขาด (Compile Error)!** 
ตามกฎของ Scope ตัวแปรที่สร้างในปีกกานอก ย่อมมีชีวิตทะลุลงไปถึงปีกกาข้างในด้วย ดังนั้นการพยายามประกาศตัวแปร `i` ทับซ้อนอีกรอบ จะทำให้ C# โวยวายว่า "A local variable named 'i' is already defined in this scope" 
*(เคล็ดลับ: โปรแกรมเมอร์จึงตกลงกันทั่วโลกว่า ลูปชั้นนอกใช้ตัว `i` ลูปชั้นในใช้ตัว `j` และถ้ามีซ้อนอีกจะใช้ตัว `k` ไล่ตามตัวอักษร)*

**คำถาม 2:** โค้ดด้านล่างนี้ จะทำงานคำสั่ง `Console.Write("A");` รวมทั้งหมดกี่รอบ?
```csharp
for (int i = 0; i < 10; i++)
{
    for (int j = 0; j < 5; j++)
    {
        for (int k = 0; k < 2; k++)
        {
            Console.Write("A");
        }
    }
}
```
**แนวคำตอบ:** หลักการคำนวณรอบของ Nested Loop คือให้เอาจำนวนของทุกลูปมา "คูณกัน" เสมอ
ลูปนอก = 10, ลูปกลาง = 5, ลูปในสุด = 2
รวมทั้งหมด 10 x 5 x 2 = **100 รอบ (พิมพ์คำว่า A 100 ครั้ง)** 
และนี่คือภาพสะท้อนของสมการ Time Complexity ที่เรียกว่า $O(N^3)$ (Cubic Time) ซึ่งจะพุ่งทะยานเป็นกราฟชันอย่างรวดเร็วมากเมื่อตัวเลขเพิ่มขึ้นแค่นิดเดียว!
:::
