# The `while` loop (การวนซ้ำแบบอิงเงื่อนไข)

> 💡 **เป้าหมาย:** ทำความเข้าใจกับโครงสร้างลูปประเภทที่สอง ที่ไม่ได้ถูกสร้างมาเพื่อนับตัวเลข แต่ถูกสร้างมาเพื่อ **"ทำงานไปเรื่อยๆ อย่างไร้จุดหมาย จนกว่าเงื่อนไขบางอย่างจะถูกทำลายลง"** ซึ่งเป็นหัวใจหลักของการสร้างระบบรอรับคำสั่ง (Menu Loop) หรือการดักจับข้อผิดพลาด (Validation Loop)

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. ความแตกต่างระหว่าง `for` และ `while`

ถ้าเปรียบเทียบในชีวิตจริง:
- **`for` loop** เหมือนการสั่งให้วิ่งรอบสนาม 10 รอบ (รู้เป้าหมายแน่นอนนับตั้งแต่ก้าวแรก)
- **`while` loop** เหมือนการสั่งให้วิ่งต่อไปเรื่อยๆ **"จนกว่าจะเหนื่อยหอบ"** (ไม่รู้เลยว่าจะวิ่งไปกี่ก้าว อาจจะ 5 รอบ หรือ 100 รอบก็ได้ ขึ้นอยู่กับสภาพร่างกาย)

ในแง่ของโครงสร้างโค้ด คำสั่ง `while` จะมีไวยากรณ์ที่มินิมอล (Minimalist) มากที่สุด มันมีแค่ส่วน **"เงื่อนไข (Condition)"** ล้วนๆ ไม่มีจุดเริ่มต้น ไม่มีการอัปเดตใดๆ ในวงเล็บ

```text
ไวยากรณ์ (Syntax):

  // ก่อนเข้าลูป คุณต้องจัดเตรียมตัวแปร (State) ไว้ล่วงหน้า
  
  while ( เงื่อนไขที่ยังคงเป็น "จริง" )
  {
      // โค้ดที่จะถูกทำซ้ำไปเรื่อยๆ ตราบใดที่เงื่อนไขด้านบนยังเป็น True
      
      // ⚠️ ข้อควรระวังระดับอันตราย:
      // ต้องมีโค้ดสักบรรทัดข้างในนี้ ที่สามารถเปลี่ยนเงื่อนไขให้กลายเป็น False ได้ในสักวันหนึ่ง
      // มิฉะนั้นโปรแกรมจะติดอยู่ในหลุมดำ (Infinite Loop) ตลอดกาล!
  }
```

---

### 2. วงจรการทำงาน (Execution Flow) ของ While Loop

`while` เป็นลูปประเภท **Pre-test Loop (ตรวจสอบก่อนทำ)** 
หมายความว่า มันจะทำตัวเป็น รปภ. ตรวจบัตรหน้าประตู ถ้าเงื่อนไขเป็นเท็จตั้งแต่แรก มันจะ **ปิดประตูใส่หน้าทันทีโดยไม่ให้เข้าไปทำโค้ดข้างในเลยแม้แต่รอบเดียว!**

```text
แผนภาพวงจรชีวิตของ While Loop

       │
  ┌────▼─────────────────────────┐
  │                              │
  │  [ เช็คเงื่อนไข (Condition) ] ──(False)──> [จบการทำงาน! กระโดดออก]
  │        │
  │     (True)
  │        │
  │  [ เข้าไปทำงานในปีกกา ]
  │        │
  │  [ เปลี่ยนแปลงค่าตัวแปร ] <--- (สำคัญมาก เพื่อไม่ให้เกิด Infinite Loop)
  │        │
  └────────┘ (วนกลับไปหา รปภ. หน้าประตูใหม่)
```

### 3. มหันตภัยลูปอนันต์ (The Infinite Loop Danger)

บั๊กที่โปรแกรมเมอร์มือใหม่ (หรือแม้แต่ระดับซีเนียร์บางคนที่เบลอจัด) มักจะเจอบ่อยที่สุด คือการลืมอัปเดตตัวแปรภายในปีกกาของ `while`

**โค้ดตัวอย่างมฤตยู (ห้ามนำไปรันจริง ถ้าไม่อยากให้พัดลมคอมพิวเตอร์ดังกระหึ่ม):**
```csharp
int hp = 100;
while (hp > 0)
{
    Console.WriteLine("ฉันยังมีชีวิตอยู่!");
    // ลืมเขียนโค้ดลดค่า hp ลง!!
}
```
สิ่งที่เกิดขึ้นคือ คอมพิวเตอร์จะพิมพ์ประโยคนั้นนับล้านครั้งต่อวินาที เพราะ `hp` มีค่าเป็น 100 ตลอดกาล ไม่มีวันน้อยกว่าหรือเท่ากับ 0 ได้เลย โปรแกรมจะค้าง (Freeze) และคุณต้องบังคับปิดโปรแกรมลูกเดียว

### 4. สถาปัตยกรรม Sentinel Value (การใช้ค่าพิเศษเพื่อหยุดลูป)

เทคนิคยอดฮิตในการใช้งาน `while` คือการรอรับค่าจากผู้ใช้ (Input) ไปเรื่อยๆ จนกว่าผู้ใช้จะพิมพ์ **"รหัสผ่านลับ"** หรือ **"คำสั่งยุติ"** ที่เราตกลงกันไว้ (เรียกค่านี้ว่า Sentinel Value) เช่น "พิมพ์ exit เพื่อออก"

เทคนิคนี้ทำให้เราสร้าง "แชทบอท" หรือ "ระบบเมนูหลัก" ที่รอรับคำสั่งผู้ใช้ได้ตลอดเวลาเหมือนโปรแกรมจริง

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่างที่ 1: ระบบดักจับการกรอกข้อมูล (Input Validation Loop)

หนึ่งในประโยชน์สูงสุดของ `while` คือการใช้มันเป็น **"เครื่องมือทรมานผู้ใช้"** คือถ้าผู้ใช้กรอกข้อมูลผิด เราจะไม่ยอมให้โปรแกรมเดินหน้าต่อ และบังคับให้กรอกใหม่ไปเรื่อยๆ (ซึ่งช่วยป้องกันระบบแครชได้ดีเยี่ยม)

::: code-group
```csharp [Program.cs]
using System;

namespace ValidationLoopDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบตั้งรหัสผ่านที่ปลอดภัย (Secure Password Setup) ===");

            // 1. กำหนดสถานะเริ่มต้นที่ทำให้ลูปทำงานได้ (ยังไม่มีรหัสผ่าน)
            string password = "";

            // 2. ลูปจะทำงาน "ตราบใดที่ความยาวรหัสผ่านน้อยกว่า 8 ตัวอักษร"
            // ครั้งแรกสุด password.Length คือ 0 (เพราะเป็นสริงว่าง "") เงื่อนไขจึงเป็น True ลูปเริ่มทำงาน
            while (password.Length < 8)
            {
                Console.Write("\nกรุณาตั้งรหัสผ่าน (ต้องมีอย่างน้อย 8 ตัวอักษร): ");
                password = Console.ReadLine(); // อัปเดตค่า

                // ถ้าผู้ใช้กรอกมาแค่ 3 ตัวอักษร...
                if (password.Length < 8)
                {
                    Console.WriteLine("❌ ผิดพลาด: รหัสผ่านของคุณสั้นเกินไป ไม่ปลอดภัยเลย");
                    Console.WriteLine("โปรแกรมจะบังคับให้คุณกรอกใหม่จนกว่าจะผ่าน!");
                }
            }

            // 3. ถ้าโปรแกรมหลุดรอดมาถึงบรรทัดนี้ได้ แปลว่าเงื่อนไขใน while กลายเป็น False แล้ว
            // (นั่นคือ รหัสผ่านยาว >= 8 ตัวอักษรสำเร็จ!)
            Console.WriteLine("\n-------------------------------------------");
            Console.WriteLine("✅ ยอดเยี่ยม! รหัสผ่านของคุณผ่านเกณฑ์ความปลอดภัยแล้ว");
            Console.WriteLine("ระบบกำลังทำการบันทึกข้อมูล...");
        }
    }
}
```
:::

### ตัวอย่างที่ 2: เครื่องนับยอดบริจาค (Sentinel Value Pattern)

โค้ดนี้สาธิตเทคนิคการปล่อยให้ลูปหมุนไปเรื่อยๆ จนกว่าผู้ใช้จะพิมพ์คำสั่งเฉพาะเจาะจง (ในที่นี้คือ -1)

::: code-group
```csharp [Program.cs]
using System;

namespace DonationCounter
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== โปรแกรมนับยอดเงินบริจาคการกุศล ===");
            Console.WriteLine("กติกา: กรอกยอดเงินไปเรื่อยๆ ทีละคน");
            Console.WriteLine("พิมพ์ '-1' เมื่องานจบ เพื่อให้ระบบรวมยอดทั้งหมด\n");

            // กล่องสะสมยอดรวม และกล่องนับจำนวนคน
            int totalDonation = 0;
            int donorCount = 0;

            // ตัวแปรสำหรับรับค่าแต่ละรอบ
            int currentInput = 0;

            // ลูปจะหมุนไปเรื่อยๆ ตราบใดที่ยังไม่พิมพ์ -1
            while (currentInput != -1)
            {
                Console.Write($"รับเงินจากคนที่ {donorCount + 1} (พิมพ์ -1 เพื่อจบ): ");
                
                // สำหรับมือใหม่ เราละเว้นการใช้ TryParse ไปก่อนเพื่อลดความซับซ้อนของลูป 
                // แต่ในงานจริงต้องใส่เสมอ!
                currentInput = int.Parse(Console.ReadLine());

                // ป้องกันไม่ให้เอาค่าคำสั่งปิดระบบ (-1) ไปสะสมในยอดเงิน
                if (currentInput != -1)
                {
                    totalDonation += currentInput;
                    donorCount++; // นับเพิ่มอีก 1 คน
                }
            }

            // แสดงสรุปผล
            Console.WriteLine("\n=========================================");
            Console.WriteLine("              สรุปผลงานบุญ             ");
            Console.WriteLine("=========================================");
            Console.WriteLine($"มีผู้ใจบุญร่วมบริจาคทั้งหมด : {donorCount} คน");
            Console.WriteLine($"ยอดเงินที่เรี่ยไรได้รวมทั้งสิ้น : {totalDonation:N0} บาท");
        }
    }
}
```
:::

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

### **โจทย์ที่ 1: กลไกหลบหนี (Escape the Loop)**
พิจารณาโค้ดการบวกเลขแบบคลาสสิกต่อไปนี้:
```csharp
int counter = 1;
int sum = 0;
while (counter < 5)
{
    sum += counter;
}
Console.WriteLine(sum);
```
**คำถาม:** 
1. เมื่อคุณนำโค้ดนี้ไปรัน จะเกิดอะไรขึ้นบนหน้าจอ?
2. ข้อผิดพลาด (Bug) อยู่ที่ไหน และคุณจะเติมโค้ด 1 บรรทัดตรงไหน เพื่อให้โปรแกรมแสดงผลลัพธ์เป็น `10` ออกมาได้สำเร็จ (เพราะ `1+2+3+4 = 10`)

### **โจทย์ที่ 2: ระบบสั่งซื้อน้ำดื่ม (Water Vending Machine)**
**ภารกิจ:** จงเขียนโปรแกรมจำลองตู้กดน้ำอัตโนมัติ 
- ตู้มีน้ำเปล่าราคา 15 บาท (ตั้งเป็นค่า `const`)
- ผู้ใช้ต้องหยอดเหรียญ (กรอกตัวเลขทีละครั้ง) เข้ามาในระบบ
- ใช้ `while` loop ในการเช็คว่า **"ยอดเงินที่หยอดเข้ามาสะสมรวมกัน ยังน้อยกว่า 15 บาทใช่หรือไม่?"**
- ถ้าใช่น้อยกว่า ให้วนถามผู้ใช้ไปเรื่อยๆ ว่า "ยอดเงินยังไม่พอ กรุณาหยอดเพิ่ม:"
- เมื่อผู้ใช้หยอดจนครบหรือเกิน 15 บาท ลูปจะจบลง ให้พิมพ์ว่า "นี่คือน้ำของคุณ! (เงินทอน: xx บาท)"

::: details 💡 คำใบ้และแนวทาง (Hints)
- **สำหรับโจทย์ 1:** โค้ดนี้คือฆาตกรที่ทำให้คอมพิวเตอร์ค้าง (Infinite Loop) เพราะตัวแปร `counter` มีค่าเป็น 1 ตลอดกาล ไม่เคยถูกบวกเพิ่มเลย วิธีแก้คือเติม `counter++;` ไว้ในปีกกาล่างสุด
- **สำหรับโจทย์ 2:** คุณต้องมีตัวแปร `int currentMoney = 0;` ไว้นอกลูป แล้วสร้างเงื่อนไข `while (currentMoney < 15)` จากนั้นข้างในลูป ค่อยรับค่ามาบวกทบใส่ `currentMoney` ไปเรื่อยๆ
:::

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: มหาสงครามแบคทีเรีย (The Bacteria Population Simulator)**

ในห้องทดลองทางชีววิทยา แบคทีเรียสายพันธุ์ X มีพฤติกรรมการแบ่งตัวสุดสยองดังนี้:
- เริ่มต้นมีแบคทีเรีย 1 ตัว
- **ทุกๆ 1 ชั่วโมง แบคทีเรียจะแบ่งตัวเพิ่มจำนวนขึ้นเป็น 2 เท่า (x2) ของชั่วโมงที่แล้ว**

**ภารกิจของคุณ:** 
ให้เขียนโปรแกรมใช้ `while` loop หาคำตอบว่า... **ต้องใช้เวลากี่ชั่วโมง ประชากรแบคทีเรียถึงจะข้ามผ่านหลัก "1 ล้านตัว" ?**

*(คำแนะนำ: สร้างตัวแปร `int population = 1;` และ `int hours = 0;` ไว้ด้านนอกลูป จากนั้นให้เงื่อนไขลูปหมุนไปตราบใดที่ `population` ยังน้อยกว่าหรือเท่ากับ 1,000,000 ในแต่ละรอบให้ทำสมการคูณสอง และนับชั่วโมงเพิ่มขึ้น)*

ถ้าเขียนถูกต้อง โปรแกรมควรจะตอบกลับมาว่าใช้เวลา "20 ชั่วโมง" ลองดูสิว่าคุณทำได้ไหม!

**แนวคำตอบ:** เป็นเรื่องของ "ความหมายทางการออกแบบ (Semantics & Clean Code)" 
ถ้างานนั้นเรา **รู้จุดสิ้นสุดแน่นอน (Deterministic)** เช่น ทำ 5 รอบ, ตรวจแถว 10 แถว, ไล่ตัวอักษร 20 ตัว การใช้ `for` จะรวบรวม "จุดเริ่ม-เงื่อนไข-ก้าวเดิน" ไว้ในบรรทัดเดียว ทำให้อ่านง่ายและไม่ลืมอัปเดตตัวแปร 
แต่ถ้าระบบนั้นทำงานบน **ความไม่แน่นอน (Non-deterministic)** เช่น ผู้ใช้จะพิมพ์รหัสผ่านผิดกี่รอบถึงจะถูก?, ระบบเซ็นเซอร์จะจับความร้อนถึง 100 องศาเมื่อไหร่? การใช้ `while` จะสื่อความหมาย (Intent) ได้ชัดเจนที่สุดว่า "เราไม่รู้รอบนะ เราแค่รอคอยให้เหตุการณ์นี้เกิดขึ้นก็พอ"

**คำถาม 2:** โค้ดต่อไปนี้ จะพิมพ์คำว่า "Hello" ออกมากี่รอบ? ทำไม?
```csharp
bool isRaining = false;
while (isRaining)
{
    Console.WriteLine("Hello");
}
```
**แนวคำตอบ:** **0 รอบ (ไม่ถูกทำงานเลยแม่แต่ครั้งเดียว)** 
เนื่องจากกลไกของ `while` คือแบบทดสอบ Pre-test มันจะให้ รปภ. ตรวจบัตรก่อนเข้าประตูลูป เมื่อพบว่า `isRaining` เป็น `false` ตั้งแต่จุดสตาร์ท เงื่อนไขก็ไม่ผ่าน ประตูจึงถูกปิดตายและโปรแกรมจะกระโดดข้ามบล็อกลูปนี้ไปทำงานบรรทัดล่างต่อไปทันที
