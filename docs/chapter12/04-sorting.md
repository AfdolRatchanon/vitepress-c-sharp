# Sorting (เวทมนตร์จัดระเบียบข้อมูล)

> 💡 **เป้าหมาย:** สัมผัสความสะดวกสบายขั้นสุดยอดของการจัดเรียงข้อมูล (Sorting)! ลืมการเขียนอัลกอริทึมสลับค่าหน้ามืด (Bubble Sort, Quick Sort) หรือการสร้างคลาสสลับซับซ้อนทิ้งไปได้เลย LINQ มอบพลังให้คุณจัดเรียงข้อมูลแบบกี่ระดับชั้นก็ได้ ด้วยโค้ดที่สั้นและอ่านง่ายราวกับภาษาอังกฤษ

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. `OrderBy` และ `OrderByDescending` (เสาหลักแห่งการจัดเรียง)

การจะสั่งให้กล่องข้อมูลจัดเรียงตัวเอง คุณแค่ต้อง **"จิ้มบอกมันว่า จะให้เรียงตามตัวแปรไหน"**
ฟังก์ชันจัดเรียงจะรับ Lambda เพื่อโยงไปหาเป้าหมาย (`Func<TSource, TKey>`)

- **`.OrderBy(x => ...)`** : จัดเรียงจาก **น้อยไปมาก** (A-Z, 0-9)
- **`.OrderByDescending(x => ...)`** : จัดเรียงจาก **มากไปน้อย** (Z-A, 9-0)

```csharp
List<Employee> staff = ...;

// เรียงรายชื่อพนักงานตามลำดับตัวอักษร A-Z
var sortedByName = staff.OrderBy(e => e.Name);

// เรียงตามเงินเดือน จากคนรวยสุด ไปหาคนจนสุด
var sortedByWealth = staff.OrderByDescending(e => e.Salary);
```

---

### 2. `ThenBy` (การจัดเรียงซ้อนระดับขั้น)

ปัญหาคลาสสิกของชาวโลก: ถ้าจัดเรียงตามเงินเดือนแล้ว... "แล้วถ้าเงินเดือนเท่ากันล่ะ จะให้ใครขึ้นก่อน!?"
ในอดีตเราต้องเขียนสูตรคณิตศาสตร์หักล้างกันวุ่นวาย แต่ใน LINQ ไมโครซอฟต์เสกฟังก์ชันตีบวกมาให้เราใช้ต่อท่อได้ทันที นั่นคือ **`.ThenBy` (แล้วตามด้วย...)**

> ⚠️ **กฎเหล็ก:** ห้ามใช้ OrderBy ซ้อน OrderBy เด็ดขาด! เพราะตัวที่สองจะไประเบิดล้างบางการเรียงของตัวแรกทิ้งทั้งหมด! ต้องใช้ OrderBy เริ่มต้น 1 ครั้ง แล้วตามด้วย ThenBy เสมอ!

```csharp
// เรียงตามแผนก A-Z... ถ้าอยู่แผนกเดียวกัน ให้เรียงตามอายุจากมากไปน้อย!
var sortedStaff = staff
    .OrderBy(e => e.Department)               // เสาหลัก (Major Sort)
    .ThenByDescending(e => e.Age);            // เสารอง (Minor Sort)
    // สามารถ .ThenBy(...) ซ้อนกี่ร้อยระดับก็ได้!
```

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่าง: บอร์ดจัดอันดับผู้เล่น (The Leaderboard System)

เกมเมอร์ทุกคนคุ้นเคยกับตารางคะแนน (Leaderboard) เราจะมาจำลองระบบบอร์ดที่ต้องรับมือกับสถานการณ์ "คะแนนเท่ากัน" กันครับ!

::: code-group
```csharp [Program.cs]
using System;
using System.Collections.Generic;
using System.Linq;

namespace LinqSortingConcept
{
    class Player
    {
        public string Username { get; set; }
        public int Score { get; set; }
        public double PlayTimeHours { get; set; } // ชั่วโมงบิน
    }

    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== บอร์ดจัดอันดับเซิร์ฟเวอร์ (Hall of Fame) ===\n");

            List<Player> players = new List<Player>
            {
                new Player { Username = "NoobSlayer", Score = 500, PlayTimeHours = 12.5 },
                new Player { Username = "ProGamer",   Score = 800, PlayTimeHours = 50.0 },
                new Player { Username = "LazyBoy",    Score = 500, PlayTimeHours = 100.5 },
                new Player { Username = "SpeedRun",   Score = 800, PlayTimeHours = 2.5 },
                new Player { Username = "Arthur",     Score = 200, PlayTimeHours = 1.0 }
            };

            // ==========================================================
            // ความต้องการของระบบ:
            // 1. เรียงคนคะแนนสูงที่สุดขึ้นก่อน (มากไปน้อย)
            // 2. ถ้าคะแนนเท่ากัน! ให้ดูที่เวลาเล่น ใครใช้เวลาเล่นน้อยกว่า ถือว่าเทพกว่า (น้อยไปมาก)
            // 3. ถ้าเทพเท่ากันอีก! ให้เรียงตามชื่อตัวอักษร A-Z 
            // ==========================================================

            var leaderboard = players
                .OrderByDescending(p => p.Score)       // ขั้นที่ 1: คะแนน มาก -> น้อย
                .ThenBy(p => p.PlayTimeHours)          // ขั้นที่ 2: เวลา น้อย -> มาก
                .ThenBy(p => p.Username);              // ขั้นที่ 3: ชื่อ A -> Z

            Console.WriteLine("ลำดับ | ชื่อผู้เล่น       | คะแนน  | เวลาเล่น");
            Console.WriteLine("------------------------------------------");
            
            int rank = 1;
            foreach (var p in leaderboard)
            {
                Console.WriteLine($"#{rank,-4}| {p.Username,-15}| {p.Score,-6}| {p.PlayTimeHours} ชม.");
                rank++;
            }

            /*
               ผลลัพธ์ที่จะออกมา:
               #1   | SpeedRun       | 800   | 2.5 ชม.   (คะแนน 800 เท่า ProGamer แต่เวลาไวกว่า)
               #2   | ProGamer       | 800   | 50 ชม.
               #3   | NoobSlayer     | 500   | 12.5 ชม.  (คะแนน 500 เท่า LazyBoy แต่เวลาไวกว่า)
               #4   | LazyBoy        | 500   | 100.5 ชม.
               #5   | Arthur         | 200   | 1 ชม.
            */
        }
    }
}
```
:::

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

### **โจทย์ที่ 1: ความทรงจำปลาทอง (The Amnesia Sort)**
มีเด็กฝึกงานเขียนโค้ดจัดเรียงลำดับพนักงาน โดยอยากให้เรียงแผนกก่อน แล้วค่อยเรียงเงินเดือน เขาเขียนมาดังนี้:
```csharp
var sorted = staff.OrderBy(e => e.Department).OrderBy(e => e.Salary);
```
**คำถาม:** ผลลัพธ์สุดท้ายที่เด้งออกมา พนักงานจะถูกเรียงลำดับแบบไหน? (เรียงแผนกไหม? เรียงเงินเดือนไหม?) และจงอธิบายความพังพินาศของโค้ดนี้

### **โจทย์ที่ 2: ของแถม (The String Length Sort)**
ปกติเราใช้ `OrderBy` กับตัวเลข หรือ ข้อความ A-Z
**คำถาม:** ถ้าผมอยากเรียงลำดับ `List<string> words = { "Apple", "Banana", "Cat" }` โดยให้คำที่ **"สั้นที่สุด"** ขึ้นก่อน (Cat -> Apple -> Banana) ผมจะยัด Lambda แบบไหนลงไปใน `.OrderBy()` ดีครับ?

::: details 💡 คำใบ้และแนวทาง (Hints)
- **สำหรับโจทย์ 1:** 
  **โค้ดนี้พังพินาศครับ! มันจะเรียงแค่ "เงินเดือน" อย่างเดียวเลย!**
  เหตุผล: LINQ มองคำสั่ง `.OrderBy` เป็นการ "ล้างไพ่จัดกระดานใหม่ทั้งหมด 100%" 
  พอมันท่อที่ 1 (เรียงแผนกเสร็จ) พอมาเจอท่อที่ 2 ดันเป็น `OrderBy` ซ้ำ... มันก็เตะกระดานแรกทิ้งกระจาย แล้วจัดเรียงใหม่ตามเงินเดือนล้วนๆ ครับ! (ความพยายามแรกสูญเปล่า)
  วิธีแก้คือต้องเปลี่ยนท่อที่สองเป็น **`.ThenBy(e => e.Salary)`** เท่านั้นครับ! มันถึงจะรู้ว่า "อ๋อ ให้จำกระดานเก่าไว้ด้วยนะ"
- **สำหรับโจทย์ 2:** **`words.OrderBy(w => w.Length)`** ครับ!
  จำไว้เสมอว่า Lambda ฝั่งขวา คือการบอกคอมพิวเตอร์ว่า "แกจงเอาค่านี้ ไปใช้เป็นไม้บรรทัดวัดความสูงนะ!" ดังนั้นเราโยนความยาว `.Length` ให้มันเป็นไม้บรรทัดได้เลยครับ โคตรอิสระ!
:::

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: มหกรรมกีฬาสี (The Sports Day Leaderboard)**

**ภารกิจของคุณ:** จงเขียนท่อประกอบร่าง LINQ มหัศจรรย์แบบ 1 Statement (รวดเดียวจบ)!

**ข้อมูลตั้งต้น:**
```csharp
class Team {
    public string Color { get; set; }
    public int GoldMedals { get; set; }
    public int SilverMedals { get; set; }
    public bool IsDisqualified { get; set; } // ถูกตัดสิทธิ์ทุจริตไหม?
}
```

**เป้าหมายการกรองและจัดเรียง (รวดเดียว):**
1. กรองเอาเฉพาะทีมที่ **ยังไม่ถูกตัดสิทธิ์ (`IsDisqualified` เป็น `false`)** เท่านั้น
2. จัดเรียงอันดับตาม **"เหรียญทอง"** จากมากไปน้อย
3. ถ้าเหรียญทองเท่ากัน ให้ดู **"เหรียญเงิน"** จากมากไปน้อย
4. ถ้าเหรียญยังเท่ากันอีก ให้เรียง **"ชื่อสี"** ตามตัวอักษร A-Z
5. แปรรูปผลลัพธ์ (`Select`) ให้กลายเป็น String สวยๆ หน้าตาแบบนี้: `"[ชื่อสี] ทอง: 5, เงิน: 2"`

จงร้อยเรียงท่อทั้ง 5 ท่อนให้งดงามครับ! นี่คือโจทย์ที่ใกล้เคียงกับงานหลังบ้าน (Backend API) ของบริษัทจริงมากที่สุด!

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** เวลาผมใช้ `staff.OrderBy(e => e.Name)` มันเปลี่ยนลำดับพนักงานในตัวแปรกล่อง `staff` ตัวดั้งเดิมของผมเลยไหมครับ? เหมือนฟังก์ชัน `.Sort()` ของ List ปกติไหม?
**แนวคำตอบ:** **ไม่เปลี่ยนเลยแม้แต่เส้นขนเดียวครับ! (Non-destructive)**
นี่คือปรัชญาความปลอดภัยขั้นสูงสุดของ Functional Programming ครับ (เรียกว่า Immutability)
ฟังก์ชันของ LINQ ทุกตัว (รวมถึง Where, Select, OrderBy) **"จะไม่มีวันแตะต้องทำลายข้อมูลในกล่องดั้งเดิมเด็ดขาด!"** มันจะสร้างร่างโคลน (กล่องใบใหม่) ขึ้นมาเรียงลำดับให้เสร็จสรรพ แล้วโยนมาให้คุณเก็บลงตัวแปร `var sorted = ...` แทนครับ! (ปลอดภัยมาก ไม่ต้องกลัวเพื่อนในทีมมาด่าว่าแอบสลับข้อมูล)

**คำถาม 2:** แล้วถ้าผมอยากจะทำสุ่มข้อมูล (Shuffle / Random Order) ใน LINQ มีฟังก์ชัน `.OrderByRandom()` ให้ใช้ไหมครับ?
**แนวคำตอบ:** **ไม่มีแบบสำเร็จรูปครับ! แต่มีทริคสถาปนิกแฝงอยู่!**
ถ้าอยากสับไพ่มั่วๆ ด้วย LINQ คุณต้องใช้สติปัญญานิดนึงครับ ด้วยการใช้คลาส `Random` มาช่วยเป็นไม้บรรทัด:
`Random rng = new Random();`
`var shuffled = cards.OrderBy(c => rng.Next());`
แปลว่า: "คอมจ๋า ช่วยเอาตัวเลขมั่วซั่ว (Next) ไปเป็นไม้บรรทัดในการเรียงลำดับไพ่แต่ละใบให้หน่อยนะ!" ผลลัพธ์คือไพ่คุณจะถูกสับเละเทะกระจายเต็มกระดานแบบโคตรงามเลยครับ!
:::
