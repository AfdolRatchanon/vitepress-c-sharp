# Filtering with Where (ตะแกรงคัดกรองข้อมูล)

> 💡 **เป้าหมาย:** เจาะลึกท่าไม้ตายที่ถูกใช้งานบ่อยที่สุดในโลกของ LINQ! `.Where()` คือตะแกรงร่อนขนาดมหึมา ที่จะช่วยคุณคัดกรองข้อมูลนับล้านตัวให้เหลือเพียงสิ่งที่คุณต้องการ ภายในพริบตาเดียว!

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. โครงสร้างวิญญาณของ `.Where()`

ฟังก์ชัน `.Where()` มีกฎหมายศักดิ์สิทธิ์ประจำตัวคือ: **"ต้องรับฟังก์ชันผู้พิพากษา (Predicate) เท่านั้น!"**
- **ฝั่งซ้ายของ Lambda:** คือออบเจกต์ **1 ตัว** ที่กำลังถูกวนลูปมาตรวจ
- **ฝั่งขวาของ Lambda:** คือสมการตรรกะ (Boolean Expression) ที่ต้องตอบว่า `true` (ให้ผ่าน) หรือ `false` (โยนทิ้ง) เสมอ!

```csharp
// ตัวอย่าง: ร่อนเอาเฉพาะเลขที่มากกว่า 50
// x คือตัวแทนของเลข 1 ตัวในกล่อง... ให้ผ่านได้ก็ต่อเมื่อ x > 50
var bigNums = numbers.Where(x => x > 50);
```

---

### 2. วิชาต่อท่อประปา (Method Chaining)

ความมหัศจรรย์ของ LINQ คือ ทุกครั้งที่คุณใช้ `.Where()` มันจะไม่ไปทำลายกล่องข้อมูลต้นฉบับ แต่มันจะเสก "กล่องใบใหม่ที่บรรจุผู้รอดชีวิต" ออกมาให้คุณเสมอ! 
แปลว่าอะไร? แปลว่าคุณสามารถเอา `.Where()` **หลายๆ ตัวมาต่อคิวเรียงกันเป็นท่อน้ำได้เลย!** (เรียกว่า Method Chaining หรือ Fluent API)

```csharp
// สเปค: อยากได้มือถือยี่ห้อ Apple, ที่ราคาถูกกว่า 30,000 บาท, และยังมีของในสต็อก
var targetPhones = phones
    .Where(p => p.Brand == "Apple")       // ท่อที่ 1: ร่อนกรองยี่ห้อ (Apple รอด)
    .Where(p => p.Price < 30000)          // ท่อที่ 2: ร่อนราคาต่อ (พวกที่แพงกว่า 3หมื่น ร่วง!)
    .Where(p => p.StockQuantity > 0);     // ท่อที่ 3: เช็คว่ามีของไหม (ของหมด ร่วง!)
```
*การเขียนโค้ดแบบนี้ ทำให้คนในทีมสามารถเพิ่ม/ลด ท่อกรองข้อมูลได้อย่างอิสระ โค้ดโคตรจะสะอาดและเป็นระบบ!*

---

### 3. ตรรกะทับซ้อน (Complex Conditions)

ถ้าไม่อยากต่อท่อให้ยาวเฟื้อย คุณสามารถรวบตรรกะทั้งหมดไว้ใน Lambda ตัวเดียวได้เลย โดยใช้ `&&` (และ), `||` (หรือ), `!` (นิเสธ) เหมือนการเขียน `if` ปกติ!

```csharp
var targetPhones = phones.Where(p => p.Brand == "Apple" && p.Price < 30000 && p.StockQuantity > 0);
```

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่าง: สถาปัตยกรรมระบบคัดกรองพนักงาน HR (HR Filtering System)

เราจะยกเอาระบบที่เคยเขียนโค้ดลอจิกตาแตกใน Lab บทที่ 11 มาเขียนใหม่ด้วยท่ามาตรฐานของ LINQ ครับ! คุณจะเห็นเลยว่า ทำไมไมโครซอฟต์ถึงไม่ต้องให้เราสร้าง `EmployeeFilter` เองอีกต่อไป!

::: code-group
```csharp [Program.cs]
using System;
using System.Collections.Generic;
using System.Linq; // 🌟 ขาดไม่ได้เด็ดขาด!

namespace LinqWhereConcept
{
    class Employee
    {
        public string Name { get; set; }
        public string Department { get; set; }
        public decimal Salary { get; set; }
        public int Age { get; set; }
    }

    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบสืบค้นข้อมูลพนักงาน HR ขั้นสูง (LINQ) ===\n");

            // ฐานข้อมูล Database (จำลอง)
            List<Employee> staff = new List<Employee>
            {
                new Employee { Name = "Arthur", Department = "IT", Salary = 60000, Age = 28 },
                new Employee { Name = "Sarah", Department = "HR", Salary = 45000, Age = 35 },
                new Employee { Name = "John", Department = "IT", Salary = 30000, Age = 22 },
                new Employee { Name = "Bob", Department = "Sale", Salary = 80000, Age = 40 },
                new Employee { Name = "Alice", Department = "IT", Salary = 55000, Age = 29 }
            };

            // ==========================================================
            // เคสที่ 1: ค้นหาเป้าหมายเดียว (Single Condition)
            // ==========================================================
            Console.WriteLine(">> 1. พนักงานแผนก IT ทั้งหมด:");
            
            // อ่านว่า: "จาก staff ขอพนักงาน e ที่ e.Department เท่ากับ IT"
            var itTeam = staff.Where(e => e.Department == "IT");
            
            foreach(var e in itTeam) Console.WriteLine($"- {e.Name} (เงินเดือน {e.Salary})");
            Console.WriteLine();

            // ==========================================================
            // เคสที่ 2: ค้นหาหลายเงื่อนไข (Multiple Conditions - Chaining vs AND)
            // ==========================================================
            Console.WriteLine(">> 2. พนักงานแผนก IT ที่เงินเดือนมากกว่า 50,000:");

            // ท่าที่ 1: ต่อท่อชิดกัน (Chaining) 
            var richIt1 = staff.Where(e => e.Department == "IT")
                               .Where(e => e.Salary > 50000);

            // ท่าที่ 2: ยุบรวมด้วย && (Logical AND) - ผลลัพธ์เหมือนกันเป๊ะ!
            var richIt2 = staff.Where(e => e.Department == "IT" && e.Salary > 50000);

            foreach(var e in richIt2) Console.WriteLine($"- {e.Name} (รวย!)");
            Console.WriteLine();

            // ==========================================================
            // เคสที่ 3: ตรรกะบรรทัดเดียวสุดโหด (OR Logic)
            // ==========================================================
            Console.WriteLine(">> 3. พนักงานที่ 'อายุน้อยกว่า 25' หรือ 'เงินเดือนเกิน 70,000':");

            var specialTargets = staff.Where(e => e.Age < 25 || e.Salary > 70000);

            foreach(var e in specialTargets) Console.WriteLine($"- {e.Name} (อายุ {e.Age}, เงิน {e.Salary})");
        }
    }
}
```
:::

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

### **โจทย์ที่ 1: มือปืนค้นหาชื่อ (Substring Search)**
ถ้าเรามีลิสต์รายชื่อ `List<string> names = new List<string> { "Tom", "Jerry", "Mickey", "Donald" };`
จงเขียนโค้ด LINQ บรรทัดเดียว เพื่อหาตัวละครทั้งหมดที่ **"ในชื่อมีตัวอักษร 'o' ผสมอยู่"** (ใบ้: ใน string มีฟังก์ชัน `.Contains()` อยู่!)

### **โจทย์ที่ 2: การดักจับ Index (The Hidden Index)**
ในการวนลูป `for (int i=0; i<10; i++)` เราจะมีเลข `i` ไว้บอกว่ากำลังอยู่รอบที่เท่าไหร่ 
แล้วใน LINQ `Where` ล่ะครับ มีวิธียัดลอจิกแปลกๆ เช่น "เอาเฉพาะข้อมูลที่อยู่ในตำแหน่ง Index เลขคู่ (0, 2, 4...)" ได้ไหม?

::: details 💡 คำใบ้และแนวทาง (Hints)
- **สำหรับโจทย์ 1:** 
  **`var result = names.Where(n => n.Contains("o"));`**
  นี่คือท่ามาตรฐานของการทำ "ช่อง Search พิมพ์ค้นหา" ในเว็บไซต์และแอปพลิเคชันทั่วโลกครับ!
- **สำหรับโจทย์ 2:** **ทำได้ครับ! ฟีเจอร์ลับ!**
  ฟังก์ชัน `.Where` ถูก Overload ร่างที่ 2 เอาไว้ มันสามารถรับ Lambda ที่มี "2 พารามิเตอร์" ได้ครับ! 
  โดยตัวแรกคือข้อมูล และตัวที่สองคือ **ลำดับ Index** ครับ!
  เช่น: `var evenIndexes = names.Where((n, index) => index % 2 == 0);`
  (โหดไหมล่ะครับ ไม่ต้องใช้ลูป for อีกต่อไปเลยในชีวิตนี้!)
:::

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: อาจารย์จอมโหด (The Brutal Teacher Filter)**

**ภารกิจของคุณ:** คุณเป็นอาจารย์ที่มีรายชื่อนักเรียนอยู่ในมือ จงใช้ความโหดของ `.Where()` คัดเลือกนักเรียนให้ตายกันไปข้าง!

**ข้อมูลตั้งต้นใน Main:**
```csharp
List<int> scores = new List<int> { 85, 42, 99, 12, 55, 78, 30, 60 };
```

**เป้าหมาย:** สร้างตัวแปร 1 ตัว โยงท่อ (Chaining) การกรองให้ครบ 3 ด่านต่อไปนี้ (ห้ามใช้ `&&` ให้ใช้ท่อ `.Where()` ต่อกัน 3 ท่อน!):
1. **ด่านที่ 1:** ร่อนเอาเฉพาะคนที่สอบผ่าน (คะแนนมากกว่าหรือเท่ากับ 50)
2. **ด่านที่ 2:** ในกลุ่มคนสอบผ่าน... ร่อนคนที่ "สอบได้คะแนนเต็ม 99" ทิ้งไป! (เพราะอาจารย์หมั่นไส้หาว่าลอกข้อสอบ -> คะแนนต้องไม่เท่ากับ 99)
3. **ด่านที่ 3:** ในกลุ่มคนที่เหลือ... ร่อนเอาเฉพาะคนที่มีคะแนนเป็น "เลขคู่" เท่านั้น! (เพราะอาจารย์ชอบเลขคู่)

ให้เขียนให้เสร็จภายใน 1 แถวคำสั่ง (1 Statement จบด้วย `;`) แล้วใช้ `foreach` วนปริ้นคนที่เหลือรอดชีวิตมาดูครับ! (คำตอบควรเหลือ 60 และ 78)

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** ต่อท่อ `.Where(A).Where(B)` กับ รวบยอด `.Where(A && B)` อันไหนทำงานเร็วกว่ากันครับ? หรือดีกว่ากันครับ?
**แนวคำตอบ:** 
- **ในเชิง Performance:** แบบรวบยอด `&&` จะทำงานเร็วกว่านิดหน่อยครับ เพราะมันทำเสร็จใน Lambda ก้อนเดียว (ไม่ต้องโยนข้ามท่อสองรอบ)
- **ในเชิง Architecture (สถาปัตยกรรม):** การต่อท่อมักจะถูกนำมาใช้เมื่อ "เงื่อนไขมันไม่ได้ตายตัว" ครับ!
สมมติคุณทำหน้าเว็บขายของที่มี "Filter สินค้า"
```csharp
var query = products.Where(p => p.IsActive); // ดึงของที่มีขายมาเป็นแกนหลัก
if (userSearchName != null) {
    query = query.Where(p => p.Name.Contains(userSearchName)); // ถ้าพิมพ์ชื่อมา ก็ต่อท่อที่ 2 เสริมเข้าไป!
}
if (userMaxPrice > 0) {
    query = query.Where(p => p.Price <= userMaxPrice); // ถ้ากรอกราคามา ก็ต่อท่อที่ 3 เข้าไปอีก!
}
```
นี่แหละครับคือสุดยอดศิลปะแห่งการประกอบร่าง Query (Dynamic Query Building) ที่ใช้กันในระบบ Enterprise ทั่วโลก! ท่อ Where แยกชิ้นส่วนได้แบบนี้มีประโยชน์มหาศาลครับ!
:::
