# Aggregation (คณิตศาสตร์เวทมนตร์)

> 💡 **เป้าหมาย:** สัมผัสพลังของการสรุปยอด! ฟังก์ชันที่เราเรียนมา (`Where`, `Select`, `OrderBy`) มันคือการรีเทิร์นค่าออกมาเป็น "กล่องคอลเลกชัน (IEnumerable)" เสมอ... แต่ในบทนี้ เราจะมาดูฟังก์ชันกลุ่ม **Aggregation (การยุบรวม)** ที่จะเขมือบข้อมูลทั้งกล่อง แล้วคายออกมาเป็น "ตัวเลขตัวเดียว (Scalar Value)" ไม่ว่าจะเป็นผลรวม, จำนวนนับ, หรือค่าสูงสุด!

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. ฟังก์ชันตัวเลขพื้นฐาน (The Big Five)

เมื่อคุณต้องทำตารางสรุป Report ให้บอสดู คุณไม่ต้องวนลูป `foreach` บรรทัดแล้วบรรทัดเล่าอีกต่อไป 
สมมติมีกล่อง `List<int> numbers = { 10, 20, 30, 40, 50 }`

- **`.Count()`** -> นับว่ามีกี่ชิ้น (ผลลัพธ์ `5`)
- **`.Sum()`** -> จับบวกกันให้หมด (ผลลัพธ์ `150`)
- **`.Average()`** -> หาค่าเฉลี่ย (ผลลัพธ์ `30.0`)
- **`.Max()`** -> หาตัวที่มากที่สุด (ผลลัพธ์ `50`)
- **`.Min()`** -> หาตัวที่น้อยที่สุด (ผลลัพธ์ `10`)

---

### 2. อาวุธลับ: การยัด Lambda ใส่ข้างใน (Inline Overloads)

ถ้ากล่องของคุณไม่ได้เก็บตัวเลขโง่ๆ แต่เก็บเป็นคลาสลึกๆ (เช่น ออบเจกต์พนักงาน `Employee` หรือสินค้า `Product`) คุณสั่ง `.Sum()` ดื้อๆ คอมพิวเตอร์จะงงทันทีว่า "จะให้ฉันเอาพนักงานมาบวกกันยังไงวะ!?"

ความอัจฉริยะของ LINQ คือ มันให้คุณ **"ยัด Lambda ทะลวงเข้าไปบอกไม้บรรทัด"** ได้เลยครับ!
*(มันเหมือนคุณควบรวม `.Select()` กับ `.Sum()` เข้าด้วยกันในคลิกเดียว)*

```csharp
List<Employee> staff = ...;

// นับจำนวนคน ที่อายุมากกว่า 30
int seniorCount = staff.Count(e => e.Age > 30); 

// เอาเงินเดือนทุกคนมารวมกัน 
decimal totalExpense = staff.Sum(e => e.Salary); 

// หาอายุของคนที่แก่ที่สุด
int oldestAge = staff.Max(e => e.Age);
```

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่าง: สถาปัตยกรรม Dashboard อัจฉริยะ (The Executive Dashboard)

เวลาที่คุณเปิดหน้าเว็บผู้ดูแลระบบ (Admin Panel) แล้วเห็นการ์ดสรุปยอดขายสวยๆ 4-5 ใบ... เบื้องหลังมันคือคลังแสงของ LINQ Aggregation ทั้งนั้นครับ!

::: code-group
```csharp [Program.cs]
using System;
using System.Collections.Generic;
using System.Linq;

namespace LinqAggregationConcept
{
    class Order
    {
        public string OrderId { get; set; }
        public decimal TotalAmount { get; set; }
        public bool IsPaid { get; set; }
        public string Status { get; set; } // "Pending", "Completed", "Cancelled"
    }

    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== บอร์ดสรุปยอดขายสำหรับผู้บริหาร (Executive Dashboard) ===\n");

            List<Order> dbOrders = new List<Order>
            {
                new Order { OrderId = "A01", TotalAmount = 1500, IsPaid = true,  Status = "Completed" },
                new Order { OrderId = "A02", TotalAmount = 4200, IsPaid = true,  Status = "Completed" },
                new Order { OrderId = "A03", TotalAmount = 800,  IsPaid = false, Status = "Pending" },
                new Order { OrderId = "A04", TotalAmount = 5500, IsPaid = true,  Status = "Completed" },
                new Order { OrderId = "A05", TotalAmount = 300,  IsPaid = false, Status = "Cancelled" }
            };

            // ==========================================================
            // ข้อมูลสรุปรายวัน (The Aggregations)
            // ==========================================================
            
            // 1. นับจำนวนออเดอร์ "ทั้งหมด"
            int totalOrders = dbOrders.Count();

            // 2. นับออเดอร์ที่ "จ่ายเงินแล้ว" (ใช้ Lambda ข้างใน Count เลย โคตรงาม)
            int paidOrders = dbOrders.Count(o => o.IsPaid);

            // 3. ยอดขายรวมทั้งหมด (Sum) แต่เอาเฉพาะออเดอร์ที่ Completed นะ!
            // ท่านี้ใช้การต่อท่อ: กรองของเสียออกก่อน แล้วค่อยเอาที่เหลือไปรวมยอด
            decimal revenue = dbOrders.Where(o => o.Status == "Completed")
                                      .Sum(o => o.TotalAmount);

            // 4. บิลที่แพงที่สุด (Max) 
            decimal topTicket = dbOrders.Max(o => o.TotalAmount);

            // 5. ค่าเฉลี่ยการซื้อต่อบิล (Average) เฉพาะอันที่จ่ายแล้ว
            decimal avgPurchase = dbOrders.Where(o => o.IsPaid)
                                          .Average(o => o.TotalAmount);

            // ==========================================================
            // แสดงผลหน้าเว็บ (UI Render)
            // ==========================================================
            Console.WriteLine("📊 [สรุปผลประกอบการวันนี้]");
            Console.WriteLine($"📦 ออเดอร์ทั้งหมด: {totalOrders} รายการ");
            Console.WriteLine($"✅ จ่ายเงินแล้ว: {paidOrders} รายการ");
            Console.WriteLine($"💰 ยอดรับสุทธิ: {revenue:N0} บาท");
            Console.WriteLine($"👑 บิลยอดสูงสุด: {topTicket:N0} บาท");
            Console.WriteLine($"📈 เฉลี่ยต่อบิล: {avgPurchase:N2} บาท");
            Console.WriteLine("=============================================");
            
            // หมายเหตุ: โค้ดทั้งหมดนี้ ถ้าเขียนแบบไม่มี LINQ คุณต้องประกาศตัวแปรสะสมผล 5 ตัว
            // และวนลูป foreach ยักษ์ 1 รอบพร้อม if ซ้อนยุ่บยั่บ! LINQ ทำให้ชีวิตดีขึ้น 1000%
        }
    }
}
```
:::

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

### **โจทย์ที่ 1: นับแบบซ้ำซ้อน (The Count vs Where)**
มีโปรแกรมเมอร์เถียงกันว่าโค้ดบรรทัดไหน "ถูกหลักสถาปัตยกรรม (Best Practice)" มากกว่ากัน:
- แบบ A: `int amount = staff.Where(e => e.Age > 30).Count();`
- แบบ B: `int amount = staff.Count(e => e.Age > 30);`
**คำถาม:** โค้ดไหนเขียนได้ดีกว่ากัน? ทำไมถึงเป็นเช่นนั้น? (มีผลกับ Performance เล็กน้อยด้วยนะ)

### **โจทย์ที่ 2: วิกฤติกระเป๋าว่างเปล่า (The Empty Collection Trap)**
สมมติว่ารายชื่อพนักงานแผนก "แม่บ้าน" ไม่มีใครอยู่เลยสักคน (กล่องว่างเปล่า Count = 0)
ถ้าคุณพยายามสั่ง `decimal highest = maids.Max(m => m.Salary);`
**คำถาม:** จะเกิดอะไรขึ้น? (มันจะคืนค่า 0 ให้ไหม หรือจะพังพินาศ?) 

::: details 💡 คำใบ้และแนวทาง (Hints)
- **สำหรับโจทย์ 1:** **แบบ B ชนะเลิศ 100% ครับ!**
  ไมโครซอฟต์จงใจสร้างช่องเสียบ (Overload) ให้ `Count` รับ Lambda ได้ ก็เพื่อการณ์นี้แหละครับ! 
  แบบ A: มันจะเสียเวลาสร้าง "กล่องใหม่ (Array สำรอง)" มารองรับคนที่ผ่าน Where แล้วค่อยนับ ซึ่งเปลือง RAM ฟรีๆ 
  แบบ B: มันวิ่งเข้าไปนับตัวเลขสดๆ ในกล่องเก่าให้เลย ประหยัดทั้งเวลาและ RAM ครับ!
- **สำหรับโจทย์ 2:** **พังพินาศ ระเบิด Error จอแดงครับ! (InvalidOperationException: Sequence contains no elements)**
  นี่คือจุดตายของ Max, Min, Average ครับ! ถ้ากล่องว่างเปล่า คอมพิวเตอร์มันหาคำตอบให้ไม่ได้ (มันคืนค่า 0 ไม่ได้ เพราะ 0 อาจจะเป็นค่าที่แท้จริงก็ได้) 
  วิธีแก้ปัญหา: สถาปนิกมักจะดักเช็คก่อนด้วย `.Any()` (เดี๋ยวได้เรียนหัวข้อถัดไป) หรือใช้ท่าพิเศษ `.DefaultIfEmpty()` ครับ!
:::

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: การชดใช้กรรมของนักวิเคราะห์ข้อมูล (The Data Analyst)**

**ภารกิจของคุณ:** จงดึงข้อมูลที่ซับซ้อนออกมาจากบิลใบเสร็จ 1 ใบให้ได้!

**ข้อมูลตั้งต้น:**
```csharp
class Invoice {
    public string Number { get; set; }
    public List<decimal> ItemPrices { get; set; } // รายการราคาสินค้าแต่ละชิ้นในบิล
}

Invoice myBill = new Invoice { 
    Number = "INV-999", 
    ItemPrices = new List<decimal> { 150.5m, 20.0m, 500.0m, 35.0m } 
};
```

**เป้าหมาย:** จงเขียนโค้ดเพื่อคำนวณและตอบคำถาม 3 ข้อนี้ให้ได้ (ข้อละ 1 บรรทัดเท่านั้น):
1. **ยอดรวมทั้งหมด (Grand Total):** บิลนี้ราคาเท่าไหร่?
2. **ของแพงระยับ (Luxury Items):** นับซิว่าในบิลนี้ มีสินค้าที่ "ราคาเกิน 100 บาท" อยู่กี่ชิ้น?
3. **ภาษีรีดเลือด (Tax Calculation):** หา "ค่าเฉลี่ย" ของราคาสินค้าทั้งหมด แล้วนำค่าเฉลี่ยนั้นมา "คูณ 7%" ให้โชว์เป็นค่าภาษี! (ใบ้: LINQ คืนค่าเป็นตัวเลข ดังนั้นเอาไปคูณต่อได้เลย)

นี่คือชีวิตจริงของเด็ก Data Science และ Backend Developer ที่ต้องจัดการโครงสร้าง JSON ซ้อนทับกันครับ!

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** ผมเห็นบางทีมีฟังก์ชัน `.Aggregate()` ด้วย มันคืออะไรครับ อยู่ในกลุ่มนี้ไหม?
**แนวคำตอบ:** **มันคือ "ตัวพ่อ" ของกลุ่มนี้เลยครับ! (The Ultimate Aggregator)**
ความจริงแล้ว `.Sum()`, `.Max()`, `.Count()` ล้วนถูกสร้างขึ้นมาจากวิชา `.Aggregate()` ทั้งนั้นเลยครับ!
ฟังก์ชัน Aggregate ให้คุณเขียน "ลอจิกสะสมค่า (Accumulator)" เองได้ทั้งหมด เช่น "เอาเลขทุกตัวมาคูณกัน (Factorial)" หรือ "เอาข้อความทุกตัวมาร้อยต่อกันคั่นด้วยลูกน้ำ" 
แต่เนื่องจากมันเขียนค่อนข้างยากและอ่านยาก สถาปนิกจึงใช้มันเฉพาะในกรณีที่ฟังก์ชันพื้นฐาน (Sum/Max) เอาไม่อยู่เท่านั้นครับ! (แต่ถ้ารู้ไว้ จะดูโคตรโปรในสายตาซีเนียร์เลยครับ)

**คำถาม 2:** โค้ดของผมคือ `int x = numbers.Sum();` แต่คอมไพล์ไม่ผ่าน มันด่าเรื่อง Cast แปลกๆ เกิดจากอะไรครับ?
**แนวคำตอบ:** **ระวังชนิดข้อมูลของปลายทางให้ดีครับ!**
ถ้าในกล่องคุณเก็บเลขยอดเงิน `decimal`... ฟังก์ชัน `.Sum()` ก็จะคายเลขก้อนโตออกมาเป็น `decimal` ด้วย!
ดังนั้นถ้าคุณเอาตัวแปร `int x` ไปยืนรอรับ... กล่องมันยัดไม่เข้าครับ! (Type Mismatch) 
ทริคของสถาปนิกเวลาขี้เกียจจำคือ... จงใช้คำว่า **`var`** เป็นคนยืนรอรับค่าเสมอครับ แล้วคอมไพเลอร์จะปรับขนาดกล่องให้พอดีกับของที่คายออกมาอัตโนมัติเองครับ! ปลอดภัย 100%!
:::
