# เฉลย: Aggregation (คณิตศาสตร์เวทมนตร์)

## 🎯 เฉลย Mini Exercise

### **โจทย์ที่ 1: นับแบบซ้ำซ้อน (The Count vs Where)**
**เป้าหมาย:** ทำความเข้าใจเรื่อง Performance ระหว่างการสร้างกล่องใหม่ (Allocation) กับการแทรกแซงตรงๆ (Inline)

**เฉลยคำตอบระดับสถาปนิก:**
**แบบ B ชนะเลิศ 100% ครับ! `staff.Count(e => e.Age > 30)`**
- แบบ A: ท่อ Where ต้องไปสร้าง "กล่องลิสต์สำรองชั่วคราว" ในแรม เพื่อยัดคนที่อายุเกิน 30 เข้าไปให้ครบทุกคนก่อน แล้วค่อยโยนให้ Count นับ (เปลืองทั้งเวลา เปลืองทั้งพื้นที่หน่วยความจำ)
- แบบ B: ไมโครซอฟต์ฉลาดครับ! ฟังก์ชัน Count เมื่อเห็น Lambda มันจะบอกว่า "อ้อ ไม่ต้องสร้างกล่องใหม่นะเว้ย เดี๋ยวฉันวิ่งไล่ชี้ทีละคนเลย ใครอายุเกินฉันก็แค่กระดิกนิ้วบวกเลข 1 ในหัวใจพอ" ผลลัพธ์คือได้คำตอบไวกว่าและไม่เปลืองแรมเลยแม้แต่นิดเดียว!

### **โจทย์ที่ 2: วิกฤติกระเป๋าว่างเปล่า (The Empty Collection Trap)**
**เป้าหมาย:** สัมผัสถึงความน่ากลัวของ Exception ยอดฮิตที่ซีเนียร์ต้องเคยเจอทุกคน

**เฉลย:**
มันจะ **ระเบิดตู้ม แครชจอแดงครับ!** (Error ชื่อ `InvalidOperationException: Sequence contains no elements`)
เหตุผลเพราะตรรกะคณิตศาสตร์ครับ... การหาค่าสูงสุด (`Max`) ของห้องที่ไม่มีคนอยู่ มันหาไม่ได้ครับ!
คอมพิวเตอร์จะคืนค่าเป็น `0` ดื้อๆ ก็ไม่ได้ เพราะถ้าคุณกำลังหา "อุณหภูมิที่ติดลบน้อยที่สุด" ค่า 0 มันอาจจะเป็นอุณหภูมิที่ถูกต้องจริงๆ ก็ได้! คอมพิวเตอร์จึงเลือกที่จะระเบิดตายเพื่อเตือนโปรแกรมเมอร์ให้รู้ตัวครับว่า "เห้ย ข้อมูลเอ็งโบ๋นะ ไปดัก Error มาก่อนเลย!"

## 🔥 เฉลย Challenge (การชดใช้กรรมของนักวิเคราะห์ข้อมูล The Data Analyst)

**เป้าหมาย:** ฝึกฝนการใช้ 3 ทหารเสือทางคณิตศาสตร์ (`Sum`, `Count`, `Average`) ให้ประยุกต์เข้ากับโลกธุรกิจจริงๆ (Business Logic)

**กระบวนการคิดระดับสถาปนิก (Scalar Aggregation):**
จำไว้เสมอว่าฟังก์ชันแก๊งนี้ มันคายผลลัพธ์เป็น "ตัวเลขก้อนเดียว" เสมอครับ! ดังนั้นคุณสามารถเอาตัวแปรพวก `int`, `decimal` มารับมันตรงๆ หรือจะเอาสมการคณิตศาสตร์ (บวก ลบ คูณ หาร) ไปต่อท้ายมันดื้อๆ เลยก็ได้ครับ!

```csharp
using System;
using System.Collections.Generic;
using System.Linq;

namespace LinqDataAnalyst
{
    class Invoice 
    {
        public string Number { get; set; }
        public List<decimal> ItemPrices { get; set; } 
    }

    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== บอทนักวิเคราะห์บัญชี (Data Analyst Bot) ===\n");

            // ดาต้าเบสบิล 1 ใบ (มีราคาสินค้า 4 ชิ้น)
            Invoice myBill = new Invoice 
            { 
                Number = "INV-999", 
                ItemPrices = new List<decimal> { 150.5m, 20.0m, 500.0m, 35.0m } 
            };

            Console.WriteLine($">> กำลังวิเคราะห์บิลหมายเลข: {myBill.Number}...");

            // ==========================================================
            // 🌟 เฉลย 3 ภารกิจ! (1 บรรทัดจบทุกข้อ)
            // ==========================================================
            
            // ภารกิจ 1: ยอดรวมทั้งหมด (ใช้ Sum ปกติเลย)
            decimal grandTotal = myBill.ItemPrices.Sum();

            // ภารกิจ 2: นับของแพงระยับ (ใช้ Count แบบอัด Lambda เข้าไป)
            int luxuryCount = myBill.ItemPrices.Count(price => price > 100m);

            // ภารกิจ 3: คำนวณภาษีรีดเลือด (ใช้ Average แล้วคูณ 0.07 หรือ 7% นอกฟังก์ชัน)
            // ความเจ๋งคือ Average() มันคายเลขทศนิยมออกมาให้เรียบร้อยแล้ว เราเลยจับคูณต่อได้เลย!
            decimal bloodTax = myBill.ItemPrices.Average() * 0.07m;

            // ==========================================================
            // พิมพ์รายงาน
            // ==========================================================
            Console.WriteLine("------------------------------------------");
            Console.WriteLine($"💰 ยอดเงินรวมทั้งสิ้น: {grandTotal:N2} บาท");
            Console.WriteLine($"💎 มีสินค้าหรูหรา (เกิน 100 บาท): จำนวน {luxuryCount} ชิ้น");
            Console.WriteLine($"⚖️ ค่าเฉลี่ยสินค้าคือ {myBill.ItemPrices.Average():N2} บาท -> โดนหักภาษี 7% คิดเป็นเงิน: {bloodTax:N2} บาท");
            Console.WriteLine("------------------------------------------");

            /*
               ผลลัพธ์:
               💰 ยอดเงินรวมทั้งสิ้น: 705.50 บาท
               💎 มีสินค้าหรูหรา (เกิน 100 บาท): จำนวน 2 ชิ้น
               ⚖️ ค่าเฉลี่ยสินค้าคือ 176.38 บาท -> โดนหักภาษี 7% คิดเป็นเงิน: 12.35 บาท
            */
        }
    }
}
```

**สิ่งที่ได้เรียนรู้ (Architecture Lesson):**
ถ้าคุณทำรายงานการเงิน (Financial Report)... การใช้ LINQ พวกนี้คือสวรรค์เลยครับ! แต่จุดที่ซีเนียร์ต้องเตือนคือ **ระวังชนิดข้อมูล (Data Type) ให้ดี!** 
เรื่องเกี่ยวกับเงินตรา (Money) ใน C# "ห้าม" ใช้ชนิดข้อมูลแบบ `float` หรือ `double` เด็ดขาด! เพราะมันคำนวณจุดทศนิยมพลาดได้ (Floating-point precision issue)! ต้องใช้ชนิดข้อมูล **`decimal`** (ลงท้ายด้วย m เสมอ) ในการทำ `.Sum()` และ `.Average()` ในระดับสากลครับ! (ซึ่งเราก็ใช้ m ในข้อนี้พอดี รอดไป!)
