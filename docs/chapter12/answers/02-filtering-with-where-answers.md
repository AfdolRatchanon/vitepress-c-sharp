# เฉลย: Filtering with Where (ตะแกรงคัดกรองข้อมูล)

## 🎯 เฉลย Mini Exercise

### **โจทย์ที่ 1: มือปืนค้นหาชื่อ (Substring Search)**
**เป้าหมาย:** สัมผัสท่ามาตรฐานของการทำ "ระบบช่อง Search" ที่ใช้กันในโลก Enterprise

**เฉลยคำตอบระดับสถาปนิก:**
**`var result = names.Where(n => n.Contains("o"));`**
ฟังก์ชัน `Where` นั้นยืดหยุ่นมากครับ! ตัวแปร `n` ในจังหวะนั้นคือออบเจกต์ String เต็มรูปแบบ คุณจึงสามารถงัดฟังก์ชันทุกตัวของ String ออกมาใช้ร่อนตะแกรงได้หมดเลย ไม่ว่าจะเป็น `.StartsWith()`, `.EndsWith()`, `.Contains()`, หรือแม้กระทั่งความยาว `n.Length` ครับ!

---

### **โจทย์ที่ 2: การดักจับ Index (The Hidden Index)**
**เป้าหมาย:** งัดฟีเจอร์ลับ (Overload) ของ LINQ ออกมาใช้เวลาจำเป็นต้องพึ่งพาลำดับแถว

**เฉลย:**
สามารถทำได้อย่างสวยงามและโหดเหี้ยมครับ!
**`var result = names.Where((n, index) => index % 2 == 0);`**
ไมโครซอฟต์รู้ว่าบางครั้งโปรแกรมเมอร์ก็ยังตัดใจจาก `for (int i=0;...)` ไม่ขาด เลยสร้างช่องเสียบแบบ 2 พารามิเตอร์มาให้ซะเลย!
โดยคอมไพเลอร์จะรู้กันเองว่า:
- ตัวแปรซ้ายมือ (`n`) คือก้อนข้อมูลในกล่อง
- ตัวแปรขวามือ (`index`) คือ "ตัวเลขนับรอบ" อัตโนมัติ (เริ่มจาก 0, 1, 2 วิ่งไปเรื่อยๆ)
ดังนั้นการเอา `index % 2 == 0` มาเช็ค ก็คือการร่อนเอาเฉพาะข้อมูลที่อยู่ในช่อง คู่ (0, 2, 4...) นั่นเองครับ!

## 🔥 เฉลย Challenge (อาจารย์จอมโหด The Brutal Teacher Filter)

**เป้าหมาย:** ฝึกวิชาต่อท่อประปา (Method Chaining) การแยกสัดส่วนลอจิกเป็นท่อนๆ จะทำให้โค้ดอ่านง่ายและสามารถถอดเข้า-ถอดออกเพื่อเทสแยกส่วนได้ง่ายขึ้นมหาศาลครับ!

**กระบวนการคิดระดับสถาปนิก (Pipeline Construction):**
ถ้าเรายัดลอจิกทั้งหมดด้วย `&&` บรรทัดมันจะยาวทะลุขอบจอไปยันเชียงใหม่ (อ่านยากมาก)
เราจะสับมันเป็น "ท่อนๆ" และเรียงต่อกันไปเรื่อยๆ เหมือนน้ำไหลผ่านท่อกรองน้ำ 3 ชั้น! ของที่หลุดมาถึงปลายท่อได้ คือของบริสุทธิ์ที่สุด!

```csharp
using System;
using System.Collections.Generic;
using System.Linq;

namespace LinqWhereChaining
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบตะแกรงร่อนนักเรียนของอาจารย์จอมโหด ===\n");

            // ดาต้าเบสคะแนนสอบ
            List<int> scores = new List<int> { 85, 42, 99, 12, 55, 78, 30, 60 };

            // 🌟 เฉลย: การต่อท่อ 3 ชั้น (Method Chaining) สังเกตว่าสถาปนิกมักจะเคาะ Enter ขึ้นบรรทัดใหม่ตรงจุดให้ดูคลีนๆ
            var survivors = scores
                .Where(s => s >= 50)     // ด่าน 1: คนสอบตก (42, 12, 30) โดนถีบตกท่อไปตรงนี้เลย!
                .Where(s => s != 99)     // ด่าน 2: ในกลุ่มคนรอด (85, 99, 55, 78, 60) โดนถีบ 99 ทิ้ง!
                .Where(s => s % 2 == 0); // ด่าน 3: ในกลุ่มคนที่เหลือ (85, 55, 78, 60) โดนถีบเลขคี่ทิ้ง! (85, 55 ร่วง!)

            Console.WriteLine(">> ประกาศผล ผู้รอดชีวิตจากด่านหฤโหด:");
            foreach (var s in survivors)
            {
                Console.WriteLine($"- คะแนนที่ผ่านเกณฑ์: {s}");
            }

            /*
               ผลลัพธ์ที่ได้:
               - คะแนนที่ผ่านเกณฑ์: 78
               - คะแนนที่ผ่านเกณฑ์: 60
            */
        }
    }
}
```

**สิ่งที่ได้เรียนรู้ (Architecture Lesson):**
การเขียนแยกท่อแบบนี้ มีข้อดีมหาศาลในแง่ของ "การอ่าน (Readability)" ครับ 
เพราะเราสามารถเขียน Comment แปะไว้ท้ายคำสั่ง `.Where()` แต่ละบรรทัดได้เลยว่า ด่านนี้กรองเรื่องอะไร! 
และถ้าวันนึงอาจารย์บอกว่า "เออๆ ด่านคะแนน 99 ฉันยกโทษให้ละกัน ปลดล็อกให้หน่อย" 
เราก็แค่ทำไฮไลต์บรรทัดท่อกลาง แล้วกด Comment (`//`) ปิดมันทิ้งบรรทัดเดียว โค้ดก็ทำงานต่อได้ทันทีโดยไม่ต้องไปนั่งแก้ลอจิก `&&` แบบเก่าให้วุ่นวายเลยครับ โคตรพลิกแพลง!
