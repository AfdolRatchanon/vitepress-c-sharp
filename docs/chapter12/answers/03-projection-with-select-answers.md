# เฉลย: Projection with Select (การแปรรูปและการฉายภาพ)

## 🎯 เฉลย Mini Exercise

### **โจทย์ที่ 1: หน้าที่ของ Where ปะทะ Select**
**เป้าหมาย:** สลักความจำแยกหน้าที่ระหว่างการ "ร่อนออก" กับการ "ลอกคราบ" ให้ชัดเจน 100%

**เฉลยคำตอบระดับสถาปนิก:**
- **`.Where()` -> ขนาดกล่องจะ "ลดลง" หรือ "เท่าเดิม" เสมอครับ!**
  เพราะหน้าที่ของมันคือการ "เตะคนออกจากกล่อง" ถ้ารอดทุกคนขนาดก็เท่าเดิม ถ้าตายเรียบขนาดก็เหลือ 0 (แต่ไม่มีวันเพิ่มของเข้ามาใหม่แน่ๆ)
- **`.Select()` -> ขนาดกล่องจะ "เท่าเดิมเป๊ะ 100%" เสมอครับ!**
  เพราะหน้าที่ของมันคือสายพานโรงงาน "1 ต่อ 1" (เข้า 1 ออก 1) โยนออบเจกต์หน้าตาแบบ A เข้าไป... มันจะแปรรูปพ่นหน้าตาแบบ B ออกมาให้ 
  ถ้าพนักงานมี 10 คน คุณใช้ Select ดึงชื่อ... คุณก็จะได้ชื่อ 10 ชื่อเป๊ะๆ ครับ ไม่มีทางหล่นหายกลางทางแน่นอน!

### **โจทย์ที่ 2: สลับท่อประปา**
**เป้าหมาย:** เข้าใจทิศทางการไหลของข้อมูล (Flow of Data) ในท่อ LINQ

**เฉลย:**
**แบบ A รอด... แต่แบบ B คอมไพล์พัง แดงเถือกครับ!**
เหตุผลเพราะข้อมูลใน LINQ มันไหลจาก **ซ้าย ไป ขวา** ครับ!
- **แบบ A (รอด):** ก้อนแรกสุดคือ `Employee` เต็มตัว โดนส่งไปร่อน `.Where(Age>30)` (รอดเพราะมี Property Age)... จากนั้นโดนส่งไปลอกคราบ `.Select(Name)` (รอดเพราะมี Property Name ให้ดึง) ผลลัพธ์สุดท้ายกลายเป็น `IEnumerable<string>` บริสุทธิ์
- **แบบ B (พัง):** ก้อนแรกคือ `Employee` ส่งไปลอกคราบเลย `.Select(Name)` (รอด) แต่... ณ ตอนนี้ข้อมูลมันกลายสภาพเป็นก้อน **`String`** ไปเรียบร้อยแล้ว! พอไหลต่อไปเจอ `.Where(Age>30)` ปุ๊บ... คอมไพเลอร์แหกปากร้องทันทีว่า *"เห้ย! สตริงบ้าอะไรมีอายุวะ! ข้อความ string ไม่มี Property Age โว้ย!"* -> พังพินาศครับ!

*(จำไว้ให้ขึ้นใจ: การคัดกรอง Where ต้องถูกวางไว้ซ้ายสุดเสมอ ก่อนที่จะถูก Select ลอกคราบทำลายร่างต้นทิ้ง!)*

## 🔥 เฉลย Challenge (เครื่องประมวลผลตะกร้าสินค้า The Cart Processor)

**เป้าหมาย:** นี่คือหัวใจของการทำ **Backend API Payload Mapping!** เราจำลองการแปลงข้อมูลดิบ (Database Model) ให้กลายเป็นข้อมูลสวยๆ พร้อมแสดงผลบนหน้าจอ (ViewModel / DTO) ด้วยการสาดคอมโบ Where + Select ร่างจำแลง

**กระบวนการคิดระดับสถาปนิก (Data Projection):**
ท่อที่ 1 (Where) จะจัดการลบพวกของแถมหรือบั๊กขยะ (จำนวน 0) ทิ้งไปให้หมดก่อน
ท่อที่ 2 (Select) จะสวมร่างเป็นนักคณิตศาสตร์! คำนวณราคาคูณจำนวนให้เบ็ดเสร็จ สร้าง Property จำแลงขึ้นมา 2 ตัว เพื่อพร้อมเอาไปปริ้นในลูปต่อไป!

```csharp
using System;
using System.Collections.Generic;
using System.Linq;

namespace LinqSelectCart
{
    class CartItem
    {
        public string ProductName { get; set; }
        public decimal Price { get; set; }
        public int Quantity { get; set; }
    }

    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== เครื่องประมวลผลตะกร้าสินค้า (Checkout System) ===\n");

            // ดาต้าเบสตะกร้าจำลอง (มีแผ่นรองเมาส์ที่จำนวน=0 แอบเนียนอยู่ด้วย)
            List<CartItem> cart = new List<CartItem> 
            {
                new CartItem { ProductName = "เมาส์", Price = 500, Quantity = 2 },
                new CartItem { ProductName = "คีย์บอร์ด", Price = 1500, Quantity = 1 },
                new CartItem { ProductName = "แผ่นรองเมาส์", Price = 100, Quantity = 0 } 
            };

            // ==========================================================
            // 🌟 ภารกิจ LINQ Pipeline (กรอง -> แปรรูป)
            // ==========================================================
            
            var receiptData = cart
                // 1. ตะแกรงร่อนทิ้ง: ของชิ้นไหนไม่ได้ซื้อจริง (Quantity <= 0) ถีบทิ้งตกท่อไปเลย!
                .Where(item => item.Quantity > 0)
                // 2. เครื่องแปรรูปลอกคราบ: แปลงโฉมออบเจกต์เดิม ให้กลายเป็นคลาสไร้ชื่อตัวใหม่!
                .Select(item => new 
                {
                    // ปั้นชื่อสินค้า ผสมกับข้อความ
                    ItemName = $"{item.ProductName} (x {item.Quantity})",
                    // บังคับให้คอมพิวเตอร์คำนวณคูณเลขให้เสร็จสรรพ ยัดเป็นข้อมูลก้อนใหม่
                    TotalPrice = item.Price * item.Quantity
                });

            // ==========================================================
            // พิมพ์ใบเสร็จ
            // ==========================================================
            Console.WriteLine("----------------------------------");
            Console.WriteLine("         บิลใบเสร็จรับเงิน          ");
            Console.WriteLine("----------------------------------");

            // วนลูปของจำแลง ที่หลุดออกมาจากท่อ Select
            foreach (var data in receiptData)
            {
                // สังเกตว่าเราสามารถเรียกใช้ Property ชื่อใหม่ (ItemName, TotalPrice) ได้อย่างอิสระ!
                Console.WriteLine($"- {data.ItemName,-20} | ราคา: {data.TotalPrice,6:N0} ฿");
            }

            Console.WriteLine("----------------------------------");

            /*
               ผลลัพธ์ที่ได้ (แผ่นรองเมาส์หายไปอย่างหมดจด):
               ----------------------------------
                        บิลใบเสร็จรับเงิน          
               ----------------------------------
               - เมาส์ (x 2)          | ราคา:  1,000 ฿
               - คีย์บอร์ด (x 1)       | ราคา:  1,500 ฿
               ----------------------------------
            */
        }
    }
}
```

**สิ่งที่ได้เรียนรู้ (Architecture Lesson):**
สถาปัตยกรรมร่างจำแลง (`Anonymous Type`) นี้ เป็นไม้ตายก้นหีบเวลาสถาปนิกทำ Web API ครับ! 
สมมติคุณมีตาราง `User` ในดาต้าเบส ที่มีคอลัมน์ `PasswordHash`, `CreditCardInfo` ซ่อนอยู่... 
ถ้าคุณโยนก้อน `User` คืนให้หน้าบ้านตรงๆ... ข้อมูลลับก็หลุดกระจายหมดสิครับ! 
สถาปนิกจึงใช้ท่า `.Select(u => new { u.Id, u.Name })` สกัดเอามาเฉพาะเปลือกนอก แล้วค่อยโยน JSON กลับไปหาหน้าเว็บครับ ปลอดภัยระดับ Enterprise!
