# 💻 Lab: ระบบจัดการสถานะงานอัจฉริยะ (Task Status Manager)

> 💡 **เป้าหมายของ Lab นี้:** ทลายกำแพงการเขียน Procedural Code ธรรมดา! คุณจะต้องจำลองตัวเป็นสถาปนิกซอฟต์แวร์ สร้างระบบกระดานงาน (Agile Kanban Board ย่อส่วน) โดยการหลอมรวม **Enum** (คุมสถานะ), **Struct** (แพ็คเกจงาน), และการจัดการ Array ให้อยู่หมัด พร้อมโชว์สกิลการเปลี่ยนแปลงข้อมูลดิบผ่าน `ref`

---

## 📖 ภาพรวมของโปรแกรม (Program Overview)

คุณได้รับมอบหมายให้สร้างระบบ Console สำหรับติดตามงาน (Ticket Tracking) ของทีมพัฒนาซอฟต์แวร์ 
โปรแกรมจะสร้างโควต้าสำหรับเก็บตั๋วงาน (Ticket) จำนวน 3 ใบ โดยแต่ละใบจะมี ชื่อเรื่อง (Title) และ สถานะ (Status) 
เมื่อเริ่มรัน โปรแกรมจะแสดงตารางงานปัจจุบัน จากนั้นจะทำกระบวนการ "เลื่อนสถานะงาน" ทีละใบ แล้วแสดงสรุปตารางงานหลังอัปเดตให้เจ้านายดู

### ตัวอย่างหน้าจอเมื่อรันโปรแกรมสำเร็จ

```text
================================================
          [ AGILE TASK MANAGER ]
================================================
[ 📋 กระดานงานประจำวัน (ก่อนอัปเดต) ]
Ticket 1: ออกแบบ Database         [สถานะ: Todo]
Ticket 2: เขียนหน้าเว็บ Login      [สถานะ: Doing]
Ticket 3: ทดสอบระบบเซิร์ฟเวอร์     [สถานะ: Todo]

[ ⚙️ ระบบกำลังรันสคริปต์อัปเดตสถานะงาน... ]
>> ดันงานที่ 1 ไปข้างหน้า...
>> ดันงานที่ 2 ไปข้างหน้า...
>> ดันงานที่ 3 ไปข้างหน้า...

================================================
[ 📈 กระดานงานอัปเดตล่าสุด (หลังการประชุม) ]
Ticket 1: ออกแบบ Database         [สถานะ: Doing]
Ticket 2: เขียนหน้าเว็บ Login      [สถานะ: Done]
Ticket 3: ทดสอบระบบเซิร์ฟเวอร์     [สถานะ: Doing]
================================================
```

## 📋 กฎกติกาและเกณฑ์การประเมิน (Business Requirements)

คุณต้องเขียนโปรแกรมโดยวางโครงสร้างสถาปัตยกรรม (Architecture) ให้เป๊ะตามพิมพ์เขียวนี้เท่านั้น:

1. **หมวดพิมพ์เขียวโครงสร้าง (Blueprint Definition):**
   - ต้องประกาศ `enum TaskState { Todo, Doing, Done }` (ห้ามใช้ตัวเลขคุมสเตตัสเด็ดขาด)
   - ต้องประกาศ `struct Ticket { public string Title; public TaskState State; }`

2. **หมวดการจัดการข้อมูล (Data Population):**
   - ในเมธอด Main ให้สร้างตัวแปร Array ของ Ticket ขึ้นมา 3 ช่อง: `Ticket[] board = new Ticket[3];`
   - กำหนดค่าเริ่มต้น (Hardcode) ให้ตั๋วทั้ง 3 ใบ ตามตัวอย่างหน้าจอด้านบน

3. **หมวดลอจิกเปลี่ยนสถานะ (State Transition Logic):**
   - ต้องสร้างเมธอดแยกชื่อ `static void AdvanceTask(ref Ticket t)`
   - ⚠️ กฎเหล็ก: เมธอดนี้ต้องรับพารามิเตอร์แบบ **`ref`** เพื่อทะลวงเข้าไปแก้ร่างต้นฉบับ!
   - ภายในเมธอด ให้ใช้ `switch` หรือ `if` ตรวจสอบสถานะเดิมของ `t`:
     - ถ้าเป็น `Todo` ให้บังคับเปลี่ยนเป็น `Doing`
     - ถ้าเป็น `Doing` ให้บังคับเปลี่ยนเป็น `Done`
     - ถ้าเป็น `Done` อยู่แล้ว ให้ทำเฉยๆ (ไม่ต้องเปลี่ยน)

4. **หมวดการวาด UI (Rendering):**
   - ให้เขียนเมธอดแยกชื่อ `static void PrintBoard(Ticket[] tasks)` เพื่อใช้วนลูป `for` วาดตารางงานออกหน้าจอ
   - (ทริค: การแยก Method ทำให้เราสั่งพิมพ์กระดานก่อนและหลังอัปเดตได้ โดยไม่ต้องเขียนลูปปริ้นซ้ำสองรอบ!)

## ⏱️ เวลาที่ใช้ในการทำ Lab: 45 นาที

**เคล็ดลับ:** ความยากระดับบอสของ Lab นี้ ไม่ใช่วิธีเขียน Enum... แต่คือจังหวะที่คุณต้องวิ่งลูป `for` เพื่อเอาตั๋วใน Array `board[i]` ส่งเข้าไปใน Method `AdvanceTask`
คุณต้องเขียนแบบนี้: `AdvanceTask(ref board[i]);` เพื่อโยน "ร่างต้นฉบับใน Array" ส่งตรงเข้าเตาเผา!

## 📝 ขั้นตอนการพัฒนา (Step-by-Step Implementation)

### ขั้นตอนที่ 1: วางเสาเข็มโครงสร้าง (The Architect)
- [ ] ไปที่โซนเหนือสุดของไฟล์ (ก่อนคลาส Program)
- [ ] ประกาศ `enum TaskState`
- [ ] ประกาศ `struct Ticket` พร้อม Field ย่อย 2 ตัว

### ขั้นตอนที่ 2: วาดกระดาน (The Printer)
- [ ] ภายในคลาส Program (แต่กะโหลกไว้ล่าง Main) ให้สร้าง `static void PrintBoard(Ticket[] tasks)`
- [ ] เขียนลูป `for (int i=0; ...)` วิ่งวนตามความยาวของ Array `tasks`
- [ ] ปริ้นบรรทัดสวยๆ ด้วย String Interpolation: `Console.WriteLine($"Ticket {i+1}: {tasks[i].Title} [สถานะ: {tasks[i].State}]");`

### ขั้นตอนที่ 3: ระบบปัญญาประดิษฐ์เปลี่ยนสถานะ (The Transition Engine)
- [ ] สร้างเมธอด `static void AdvanceTask(ref Ticket t)` 
- [ ] จัดลอจิก `switch(t.State)` เลื่อนลำดับขั้นให้มัน

### ขั้นตอนที่ 4: ประกอบร่างใน Main (The Orchestrator)
- [ ] สร้างโกดังเก็บตั๋ว: `Ticket[] board = new Ticket[3];`
- [ ] ยัดงานใส่กล่อง:
```csharp
board[0] = new Ticket { Title = "ออกแบบ Database", State = TaskState.Todo };
board[1] = new Ticket { Title = "เขียนหน้าเว็บ Login", State = TaskState.Doing };
board[2] = new Ticket { Title = "ทดสอบระบบเซิร์ฟเวอร์", State = TaskState.Todo };
```
- [ ] สั่งปริ้นกระดานครั้งที่ 1 (อวดเจ้านาย)
- [ ] รันลูป `for` ขนาดย่อม วิ่งกวาด 3 รอบ เพื่อเรียก `AdvanceTask(ref board[i])` ดันงานไปข้างหน้า
- [ ] สั่งปริ้นกระดานครั้งที่ 2 (โชว์ผลลัพธ์)

## 🔥 Challenge (ภารกิจท้าทายขีดจำกัด)

สำหรับวิศวกรที่ต้องการแสดงแสนยานุภาพ:

**ภารกิจ The Auto-Complete Parsing (ระบบพิมพ์สั่งปิดงานอัจฉริยะ):**
เราจะอัปเกรดระบบให้มีความเป็น Interactive มากขึ้น!
ตอนท้ายโปรแกรม หลังจากปริ้นกระดานอัปเดตแล้ว... 
1. ให้ปริ้นบอกผู้ใช้ว่า: `"ต้องการปิดงาน (Done) รหัสไหน? พิมพ์ชื่อสถานะเป้าหมาย (เช่น done) หรือกด Enter เพื่อข้าม: "`
2. รับข้อความ `Console.ReadLine()`
3. **ความท้าทาย:** ให้ใช้เวทมนตร์ `Enum.TryParse` ที่เรียนมาจากหัวข้อที่ 2 ผสมกับโหมดละทิ้งอคติพิมพ์เล็กพิมพ์ใหญ่ (`true`)
4. ลองตรวจสอบว่า ข้อความที่ผู้ใช้พิมพ์มา (เช่น `"done"` หรือ `"DONE"`) สามารถแปลงร่างกลับเป็น `TaskState.Done` ได้สำเร็จหรือไม่?
5. ถ้าแปลงสำเร็จ: ให้เอา `board[0]` (สมมติว่าเป็นตั๋วใบแรก) เปลี่ยน Status เป็นสเตตัสที่ Parse มาได้ แล้วปริ้นกระดานรอบที่ 3 สรุปยอดโชว์เลย!

ใครทำข้อนี้ได้ แปลว่าคุณสามารถเชื่อมมิติระหว่าง "ข้อความมั่วๆ ของผู้ใช้" เข้าสู่ "สถาปัตยกรรมสุดเป๊ะระดับองค์กร (Enum)" ได้อย่างไร้รอยต่อครับ!
