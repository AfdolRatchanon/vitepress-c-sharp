# บทที่ 10: Inheritance & Polymorphism (พันธุกรรมและการแปลงกาย)

> 💡 **ภาพรวม:** ขอต้อนรับเข้าสู่ช่วงครึ่งหลังของสถาปัตยกรรม OOP! ในบทที่แล้วเราเรียนรู้การป้องกันข้อมูล (Encapsulation) ไปแล้ว บทนี้เราจะมาเรียนรู้สุดยอดคัมภีร์ที่ทำให้สถาปนิกซอฟต์แวร์สามารถ **"ลดการเขียนโค้ดซ้ำซ้อนได้ 100%"** นั่นคือการสืบทอดสายเลือดจากคลาสแม่สู่คลาสลูก (Inheritance) และการเปลี่ยนพฤติกรรมคำสั่งเดียวกันให้แสดงผลต่างกันตามสายพันธุ์ (Polymorphism)

## 🎯 สิ่งที่จะได้เรียนรู้ในบทนี้

- ✅ **Inheritance:** การถ่ายทอดพันธุกรรม (โค้ดและตัวแปร) จากคลาสบรรพบุรุษ (Base Class) ไปสู่คลาสลูกหลาน (Derived Class)
- ✅ **Protected Modifier:** ยามรักษาความปลอดภัยระดับครอบครัว (สิทธิพิเศษเฉพาะคนในสายเลือดเท่านั้น)
- ✅ **The `base` Keyword:** การกระโดดข้ามมิติกลับไปเรียกใช้ความสามารถของบรรพบุรุษ
- ✅ **Polymorphism & Virtual:** เวทมนตร์การแปลงกาย! วิธีอนุญาตให้ลูกหลาน "กบฏ" และเขียนพฤติกรรมทับคำสั่งของบรรพบุรุษ (`override`)
- ✅ **Abstract Classes:** แม่พิมพ์ผีสิบแปดมงกุฎ (คลาสที่มีรูปร่างแต่สร้างจริงไม่ได้ บังคับให้ลูกต้องเอาไปสานต่อเท่านั้น)

## 🚀 ทำไมเราต้องเรียนเรื่องการสืบทอด (Inheritance)?

ลองจินตนาการว่าคุณต้องเขียนโปรแกรมเกมที่มีตัวละคร 3 ตัว: นักดาบ (Swordsman), นักธนู (Archer), และนักเวทย์ (Mage)

- **วิถีคนขยัน (ไม่มี Inheritance):** คุณต้องเขียน Property ชื่อ (`Name`), เลือด (`HP`), เดิน (`Walk()`) ใส่ลงไปในคลาส `Swordsman`... แล้วคุณก็ต้องก๊อปปี้ 3 บรรทัดนี้ไปวางใน `Archer`... แล้วก็ก๊อปปี้ไปวางใน `Mage` อีก! ถ้าวันนึงคุณอยากเพิ่มระบบวิ่ง `Run()` คุณต้องตามไปแก้โค้ดถึง 3 ที่! (นรกแห่งการแก้โค้ด)
- **วิถีสถาปนิก (ใช้ Inheritance):** คุณจะสร้างคลาสแม่ชื่อ `Character` ใส่ชื่อ, เลือด, และเดิน ไว้ที่นี่แค่ที่เดียว! แล้วคุณสั่งให้ นักดาบ, นักธนู, นักเวทย์ "สืบทอดสายเลือด" มาจากคลาส `Character` 
เวทมนตร์จะบังเกิด... ลูกๆ ทั้ง 3 จะมีชื่อ เลือด และเดินได้ทันทีโดยที่คุณไม่ต้องเขียนโค้ดซ้ำเลยแม้แต่บรรทัดเดียว! แถมลูกแต่ละคนยังไปเพิ่มความสามารถพิเศษของตัวเองได้อีก (เช่น นักเวทย์เพิ่มหลอด `Mana`) 

นี่คือปรัชญา **DRY (Don't Repeat Yourself)** ขั้นสูงสุดของการเขียนโปรแกรมครับ! ลุยกันเลย!
