# Polymorphism & Virtual (เวทมนตร์การแปลงกายและกบฏ)

> 💡 **เป้าหมาย:** สัมผัสเสาหลักต้นที่ 4 ของ OOP (Polymorphism)! เรียนรู้วิชาวิญญาณร่ายรำ ที่ช่วยให้ "คำสั่งเดียว" สามารถแสดงพฤติกรรมที่ "แตกต่างกัน" ได้ พร้อมปลดล็อกคีย์เวิร์ดศักดิ์สิทธิ์ `virtual` (ประตูที่เปิดไว้ให้แก้ไข) และ `override` (การก่อกบฏเขียนทับกฎของบรรพบุรุษ)

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. ความหมายของ Polymorphism (พหุสัณฐาน)

คำนี้มาจากภาษากรีกโบราณ: **Poly (มากมาย)** + **Morph (รูปร่าง/รูปแบบ)**
หมายความว่า: "วัตถุต่างชนิดกัน สามารถตอบสนองต่อคำสั่งเดียวกัน ด้วยพฤติกรรมที่ไม่เหมือนกันได้"

ลองจินตนาการถึงวงดนตรีออร์เคสตรา: 
วาทยกร (คุณ/Programmer) สั่งเพียงคำเดียวว่า **"จงบรรเลงดนตรี! (PlayMusic)"**
- ไวโอลิน -> ก็ใช้วิธี "สีสาย"
- กลอง -> ก็ใช้วิธี "ตีหน้ากลอง"
- ทรัมเป็ต -> ก็ใช้วิธี "เป่าลม"
ทุกคนได้รับคำสั่งเดียวกันเป๊ะ! แต่ผลลัพธ์และวิธีการลงมือทำของแต่ละออบเจกต์แตกต่างกันสิ้นเชิง... นี่แหละครับคือคอนเซปต์ของ Polymorphism!

### 2. วิถีแห่งกบฏ (`virtual` ปะทะ `override`)

ในโลกของการสืบทอด (Inheritance) บางครั้งลูกก็ไม่อยากทำตามกฎของแม่ 100% 
เช่น คลาส `Animal` มีเมธอด `Speak()` ที่ร้องว่า "(เสียงสัตว์)"
ถ้าเราสร้าง `Dog` เราก็อยากให้มันร้อง "โฮ่ง!" แทนเสียงดั้งเดิมของแม่... เราจะทำยังไง?

**กฎหมายของ C# (The Virtual Contract):**
1. **คลาสแม่ ต้องเป็นฝ่าย "อนุญาต" ก่อน:** แม่ต้องใส่คีย์เวิร์ด **`virtual`** ไว้ที่เมธอด เพื่อบอกว่า *"ลูกเอ๋ย ถ้าแกไม่พอใจไส้ในของฉัน แกเอาไปเขียนทับได้เลยนะ ฉันเปิดประตูไว้ให้"*
2. **คลาสลูก ต้องแสดงตัวก่อกบฏ:** ลูกต้องใช้คีย์เวิร์ด **`override`** (แปลว่า: เหยียบทับ) เพื่อประกาศว่า *"ผมขอเอาพฤติกรรมดั้งเดิมของแม่ไปโยนทิ้ง แล้วเขียนพฤติกรรมใหม่ของผมใส่เข้าไปแทนนะ!"*

**ไวยากรณ์ (Syntax):**
```csharp
class Animal 
{
    // แม่เปิดประตูให้แก้ไข (Virtual)
    public virtual void Speak() { Console.WriteLine("..."); }
}

class Dog : Animal 
{
    // ลูกเขียนทับแบบ 100% (Override)
    public override void Speak() { Console.WriteLine("โฮ่งๆ!"); }
}
```

### 3. พลังที่แท้จริง: การยัดออบเจกต์ใส่กล่องแม่ (Upcasting)

ความมหัศจรรย์ของ Polymorphism จะเฉิดฉายที่สุด เมื่อเราเอาออบเจกต์ลูกหลายๆ ชนิด มารวมกันใน Array ของคลาสแม่!

```csharp
// สังเกตดีๆ: ชนิดข้อมูลของ Array คือคลาสแม่ (Animal)
// แต่ไส้ในที่เรา new ใส่เข้าไป ดันเป็นคลาสลูก! (Dog, Cat)
Animal[] zoo = new Animal[2];
zoo[0] = new Dog();
zoo[1] = new Cat();

// วาทยกร สั่งการด้วยคำสั่งเดียว!
foreach (Animal a in zoo)
{
    // C# ฉลาดพอที่จะแสกนลงไปไขกุญแจหาพฤติกรรมที่โดน override ของลูกแต่ละตัว!
    a.Speak(); // ตัวแรกจะร้อง "โฮ่ง", ตัวสองจะร้อง "เหมียว"
}
```
นี่คือทักษะขั้นสูงสุด (The Holy Grail) ที่ Game Engine อย่าง Unity นำไปใช้ครับ! (อัปเดตตัวละครทุกตัวบนหน้าจอด้วยลูปเดียว)

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่าง: สมรภูมิฮีโร่หลากสายพันธุ์ (The RPG Combat System)

ลบภาพการเขียน `if/else` เช็คอาชีพตัวละครทิ้งไปได้เลยครับ! Polymorphism จะทำให้โค้ดคุณสะอาดสะอ้านและทรงพลังขึ้นอีก 10 เท่า!

::: code-group
```csharp [Program.cs]
using System;

namespace PolymorphismConcept
{
    // =======================================================
    // BASE CLASS (ตัวละครพื้นฐาน)
    // =======================================================
    class Hero
    {
        public string Name { get; set; }

        public Hero(string name) { this.Name = name; }

        // 🌟 ประทับตรา VIRTUAL เปิดโอกาสให้ลูกศิษย์เปลี่ยนกระบวนท่าโจมตีได้
        public virtual void Attack()
        {
            Console.WriteLine($"{Name} โจมตีด้วยหมัดเปล่า! (ดาเมจ: 5)");
        }
    }

    // =======================================================
    // DERIVED CLASSES (ผู้ท้าชิงหลากอาชีพ)
    // =======================================================
    class Warrior : Hero
    {
        public Warrior(string n) : base(n) { }

        // 🌟 ก่อกบฏ OVERRIDE เขียนทับกระบวนท่าของคลาสแม่
        public override void Attack()
        {
            Console.ForegroundColor = ConsoleColor.Red;
            Console.WriteLine($"⚔️ {Name} ฟันดาบยักษ์เป็นวงกว้าง! (ดาเมจ: 50)");
            Console.ResetColor();
        }
    }

    class Mage : Hero
    {
        public Mage(string n) : base(n) { }

        // 🌟 ก่อกบฏ OVERRIDE ร่ายเวทมนตร์
        public override void Attack()
        {
            Console.ForegroundColor = ConsoleColor.Cyan;
            Console.WriteLine($"🔥 {Name} ร่ายเวทมังกรเพลิง! (ดาเมจ: 120 อาณาเขต 5 เมตร)");
            Console.ResetColor();
        }
    }

    class Novice : Hero
    {
        public Novice(string n) : base(n) { }
        // โนวิส เป็นเด็กดี ไม่กล้า Override... ดังนั้นมันจะสืบทอดหมัดเปล่าของคลาสแม่มาใช้โดยอัตโนมัติ!
    }

    // =======================================================
    // EXECUTION (วาทยกรสั่งการ)
    // =======================================================
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== การประลองชิงเจ้ายุทธภพ ===\n");

            // 1. สร้างปาร์ตี้ (Array ชนิดคลาสแม่ Hero แต่ยัดไส้ด้วยอาชีพต่างๆ)
            // ทริค: นี่เรียกว่า Upcasting (ลูกปลอมตัวเป็นแม่เพื่อให้เข้ากล่องได้)
            Hero[] party = new Hero[3];
            party[0] = new Warrior("นักดาบกัซ");
            party[1] = new Mage("เมกุมิน");
            party[2] = new Novice("มือใหม่หัดเล่น");

            // 2. ปล่อยออโต้โจมตี (Auto-Attack Loop)
            Console.WriteLine(">> บอสปรากฏตัว! ทุกคนเข้าโจมตีพร้อมกัน!");
            
            foreach (Hero member in party)
            {
                // บรรทัดเดียวสะเทือนจักรวาล:
                // C# Runtime จะดิ่งลงไปค้นหาว่า แท้จริงแล้ววิญญาณข้างในคือใคร 
                // แล้วหยิบกระบวนท่าที่ตรงเผงที่สุดของคนนั้นออกมาใช้! (Late Binding)
                member.Attack();
            }
        }
    }
}
```
:::

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

### **โจทย์ที่ 1: อันตรายของคำว่า `new` Method (The Shadow Hiding)**
สมมติว่าคลาสแม่ **ลืมใส่** คำว่า `virtual` (แม่ห้ามแก้) แต่อีลูกตัวแสบ ดันอยากจะเปลี่ยนท่าโจมตี มันเลยใช้คำว่า `new` แทรกเข้าไป (เรียกว่าวิชา Method Hiding ปิดบังสายตา) 
```csharp
class Parent { public void Speak() { Console.WriteLine("I am your father."); } }
class Child : Parent { public new void Speak() { Console.WriteLine("Nooo!"); } }
```
**คำถาม:** ถ้าใน Main เราสั่ง `Parent p = new Child();` (ปลอมตัวเป็นพ่อ) แล้วเรียก `p.Speak();` ใครจะรับคำสั่งนี้ไปทำงาน? พ่อหรือลูก? 

### **โจทย์ที่ 2: ผ่าตัดกระบวนท่าเสริม (The Base Expansion)**
เมื่อเราใช้ `override` มันจะทำลายลอจิกเก่าของคลาสแม่ทิ้งทั้งหมด 
แต่ถ้าเราไม่อยากทำลายทิ้งล่ะ? เราอยากให้ "ลอจิกแม่ทำงานก่อน แล้วลอจิกลูกทำงานเสริมต่อท้าย" (เช่น แม่อัปเดตฐานข้อมูล ลูกค่อยสั่งปริ้น UI)
**คำถาม:** เราจะต้องผสมผสานคีย์เวิร์ดอะไร ที่เรียนมาจากหัวข้อก่อน เพื่อมุดกลับไปเรียกฟังก์ชันเดิมของคลาสแม่ภายในบล็อก `override` ของเรา?

::: details 💡 คำใบ้และแนวทาง (Hints)
- **สำหรับโจทย์ 1:** ผลลัพธ์คือ **`I am your father.`** (พ่อทำงาน) ครับ! แตกต่างจาก override อย่างสิ้นเชิง!
  วิชา `new Method` เป็นวิชาจอมปลอม (ขยะ)! ถ้าลูกปลอมตัวเป็นแม่ใส่ในกล่อง Polymorphism... C# จะไม่แคร์ไส้ในเลย มันจะเรียกกระบวนท่าของแม่มาทำงานทันที! (Early Binding) ดังนั้น ในฐานะสถาปนิก **จงหลีกเลี่ยงการใส่คำว่า `new` หน้า Method อย่างเด็ดขาด และจงใช้วิถี `virtual` + `override` เสมอครับ!**
- **สำหรับโจทย์ 2:** ใช้คำว่า **`base.`** ครับ! โคตรทรงพลัง!
  ```csharp
  public override void Attack() {
      base.Attack(); // กระโดดไปใช้หมัดเปล่าของแม่ให้เสร็จก่อน 1 ที
      Console.WriteLine("แถมด้วยการฟันดาบยักษ์ซ้ำ!"); // ค่อยทำงานเสริมของลูกต่อ
  }
  ```
  ทริคนี้ เฟรมเวิร์คของ Unity Engine บังคับให้คุณใช้ตลอดเวลาในการเซฟเกมหรือรีเซ็ตด่านครับ! (เรียกให้ลอจิกแกนกลางของเอนจิ้นมันทำงานเคลียร์ขยะก่อน แล้วค่อยทำของแถมของเรา)

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: ระบบจ่ายเงินพนักงาน 3 ระดับ (The Polymorphic Payroll)**

**ภารกิจของคุณ:** จงเขียนระบบคำนวณเงินเดือนที่ใช้ Polymorphism แทนการใช้ `if/else` วุ่นวาย!

1. **คลาสแม่สุด: `Employee`**
   - มี Constructor รับ `(string name, decimal salary)`
   - มี **`virtual`** Method: `public virtual decimal CalculatePay()` 
     - หน้าที่: คืนค่าเงินเดือนพื้นฐานเพียวๆ กลับไปเฉยๆ (`return Salary;`)

2. **คลาสลูกสายที่ 1: `Salesman` (เซลล์แมน)** สืบทอดจาก `Employee`
   - มี Field เสริม `public decimal Commission { get; set; }` (ค่าคอมมิชชั่น)
   - มี Constructor รับ 3 ค่า (ชื่อ, เงินเดือน, ค่าคอม) แล้วใช้ `: base` โยน 2 ค่าแรกให้แม่
   - 🌟 ภารกิจ: **`override`** เมธอด `CalculatePay()` 
     - หน้าที่: เอาเงินเดือนพื้นฐานมาบวกค่าคอม แล้วค่อยคืนค่ากลับไป!

3. **คลาสลูกสายที่ 2: `Manager` (ผู้จัดการ)** สืบทอดจาก `Employee`
   - มี Field เสริม `public decimal Bonus { get; set; }` (โบนัสรายปี)
   - มี Constructor รับ 3 ค่าเหมือนเซลล์แมน
   - 🌟 ภารกิจ: **`override`** เมธอด `CalculatePay()` 
     - หน้าที่: เอาเงินเดือนพื้นฐานมาบวกโบนัส แล้วคืนค่ากลับไป!

4. **การทดสอบความเทพใน Main (The Orchestration):**
   - สร้าง Array ของพนักงาน 3 คน (ยัดไส้ คนธรรมดา, เซลล์แมน, ผู้จัดการ)
   - ตั้งค่าตัวแปร `decimal totalCompanyExpense = 0;`
   - วิ่งลูป `foreach` สั่งเรียก `CalculatePay()` ของพนักงานแต่ละคน แล้วเอาเลขมาบวกสะสมใน `totalCompanyExpense`!
   - ปริ้นสรุปว่าเดือนนี้บริษัทต้องจ่ายเงินรวมกี่บาท!

ถ้าคุณสามารถทำโจทย์ข้อนี้ได้ โดยที่ในลูป `foreach` ไม่มีคำสั่ง `if (emp is Salesman)` เลยแม้แต่บรรทัดเดียว... ขอคารวะเลยครับ! คุณบรรลุวิชา Polymorphism ระดับโลกแล้ว!

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** ใน C# ถ้าคลาสแม่ลืมใส่คำว่า `virtual` เอาไว้... ลูกๆ ก็สามารถใช้พลังของ `override` ก่อกบฏทับมันอยู่ดีใช่ไหมครับ? เหมือนในภาษา Java เค้าก็ไม่ได้บังคับนะ
**แนวคำตอบ:** **ไม่ได้เด็ดขาดครับ! แดงเถือกพินาศเลย!**
นี่คือสิ่งที่สถาปนิก C# (Anders Hejlsberg) ภูมิใจนำเสนอมาก เขาบอกว่า "ภาษา Java มันปล่อยปละละเลยเกินไป ทุกเมธอดเปิดให้ทับได้หมดจนก่อให้เกิดบั๊กระบบล่ม" (เรียกว่า Fragile Base Class Problem)
C# จึงตั้งกฎเหล็กว่า: **"ทุก Method บนโลก จะถูกปิดตายล็อคกุญแจแน่นหนา (sealed) เป็นค่า Default!"** การที่ลูกจะทับได้ แม่จะต้องเป็นคนเอากุญแจมาเปิดประตูด้วยคำว่า `virtual` ให้เท่านั้นครับ! นี่คือการดีไซน์ระดับโคตรปลอดภัย (Design by Contract)!

**คำถาม 2:** แล้วถ้าผมสร้างลูก (Dog) มา Override เสียงเห่าเรียบร้อยแล้ว... วันนึงผมเอาคลาส `Chihuahua` (ชิวาว่า) ไปสืบทอดมาจากคลาส `Dog` อีกที... ผมจะสามารถ Override ซ้อน Override ได้ไหมครับ?
**แนวคำตอบ:** **ได้ครับ! ไหลเป็นทอดๆ ไร้ขีดจำกัด! (Cascading Overrides)**
เพราะคำว่า `override` มันมีเวทมนตร์แฝงอยู่นิดนึงคือ... ทันทีที่มันเหยียบทับแม่สำเร็จ ตัวมันเองก็จะทำหน้าที่เป็น `virtual` (ประตูบานเปิด) ส่งต่อไปให้หลานๆ รุ่นถัดไปเหยียบทับต่อได้โดยอัตโนมัติครับ! 
(ชิวาว่า สามารถร้อง "แอ๋งๆ" แทน "โฮ่ง" ของพ่อได้เลยครับ โคตรอิสระ!)
:::
