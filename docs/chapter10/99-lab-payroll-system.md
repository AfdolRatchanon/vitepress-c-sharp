## 📖 ภาพรวมของโปรแกรม (Program Overview)

บริษัท **Tech Inno จำกัด** มีนโยบายจ้างพนักงาน 3 รูปแบบ ซึ่งแต่ละรูปแบบมีสูตรการคำนวณรายได้ต่างกันฟ้ากับเหว:
1. **พนักงานประจำ (Full-Time):** รับเงินเดือนคงที่ทุกเดือน
2. **พนักงานรายชั่วโมง (Part-Time / Contract):** รับเงินตามชั่วโมงที่ทำ x เรทค่าจ้าง
3. **พนักงานขาย (Salesman):** รับเงินเดือนพื้นฐาน + (ยอดขาย x เปอร์เซ็นต์คอมมิชชัน)

ถ้าคุณใช้ `if/else` เช็คประเภทพนักงาน ระบบจะพังพินาศในอนาคตแน่นอน! หน้าที่ของคุณคือการใช้ **Abstract Class** และ **Polymorphism** เพื่อให้ลูปบัญชีทำงานได้ด้วยคำสั่งเดียว!

[รหัส E-001] คุณสมชาย (พนักงานประจำ)
 -> เงินเดือนสุทธิ: 30,000.00 บาท

[รหัส C-002] น้องสมหญิง (พนักงานพาร์ทไทม์)

[รหัส S-003] คุณสมศักดิ์ (พนักงานขาย)

------------------------------------------------

## 📋 กฎกติกาและเกณฑ์การประเมิน (Business Requirements)

คุณต้องวางโครงสร้างสถาปัตยกรรม (Architecture) ของคลาสให้เป๊ะตามนี้เท่านั้น:

1. **คลาสแม่สุด (พิมพ์เขียววิญญาณ): `Employee`**
   - ต้องแปะยันต์ `abstract`
   - มี Properties: `public string Name { get; set; }` และ `public string EmployeeId { get; set; }`
   - มี Constructor รับค่า `(string id, string name)`
   - 🌟 คัมภีร์บังคับลูก: มีเมธอด `public abstract decimal CalculateSalary();` (ไม่มีไส้ใน)
   - 🌟 ประตูบานเปิด (แถมฟรี): มีเมธอด `public virtual void ShowDetails()` ที่มีไส้ในปริ้นว่า `[รหัส {Id}] คุณ {Name}`

2. **คลาสลูกสายที่ 1: `FullTimeEmployee`**
   - สืบทอดจาก `Employee`
   - มี Property เฉพาะตัว: `public decimal MonthlySalary { get; set; }`
   - มี Constructor รับ 3 ค่า `(id, name, salary)` 🌟 อย่าลืมใช้ `: base` โยน 2 ค่าแรกให้แม่!
   - บังคับ Override: `CalculateSalary()` ให้คืนค่า `MonthlySalary` ตรงๆ เลย

3. **คลาสลูกสายที่ 2: `ContractEmployee`**
   - สืบทอดจาก `Employee`
   - มี Property เฉพาะตัว: `public decimal HourlyRate` (เรทต่อชั่วโมง) และ `public int HoursWorked` (ชั่วโมงที่ทำ)
   - มี Constructor รับ 4 ค่า โยนเข้า `base` ให้ถูกต้อง
   - บังคับ Override: `CalculateSalary()` ให้คืนค่า `HourlyRate * HoursWorked` 

4. **คลาสลูกสายที่ 3: `SalesEmployee`**
   - สืบทอดจาก `Employee`
   - มี Property: `BaseSalary` (ฐานเงินเดือน), `SalesVolume` (ยอดขายรวม), `CommissionRate` (เปอร์เซ็นต์ส่วนแบ่ง เช่น 0.05 คือ 5%)
   - มี Constructor รับ 5 ค่า โยนเข้า `base` ให้ถูกต้อง
   - บังคับ Override: `CalculateSalary()` ให้คืนค่า `BaseSalary + (SalesVolume * CommissionRate)`
   - 🌟 การก่อกบฏซ้อน: ให้ `override void ShowDetails()` โดยการมุดกลับไปเรียก `base.ShowDetails()` ของแม่ก่อน 1 ที แล้วค่อยปริ้นเสริมต่อท้ายว่า `-> ยอดขายเดือนนี้: {SalesVolume}`

5. **หมวดการรันระบบ (The Orchestration in Main):**
   - สร้าง Array ของคลาสแม่ `Employee[] company = new Employee[3];`
   - ยัดไส้ออบเจกต์ลูกทั้ง 3 อาชีพลงไปใน Array (โชว์วิชา Upcasting)
   - สร้างตัวแปร `decimal totalPayroll = 0;` เอาไว้สะสมยอดรวม
   - ใช้ลูป `foreach` กวาดพนักงานทีละคน 
     - สั่ง `.ShowDetails()` ให้ปริ้นชื่อ
     - สั่ง `.CalculateSalary()` เอาเงินไปบวกสะสมในบัญชีรวม และปริ้นยอดเงินหน้าจอ (จัดฟอร์แมตให้สวยงาม)

## 📝 ขั้นตอนการพัฒนา (Step-by-Step Implementation)

### ขั้นตอนที่ 1: สร้างวิญญาณแม่ (Abstract Base Class)
- [ ] สร้าง `abstract class Employee`
- [ ] สร้าง Properties 2 ตัว (Id, Name) และ Constructor ลับสำหรับรับค่า
- [ ] สร้างเมธอด `abstract CalculateSalary` (ห้ามมีปีกกา ไร้ไส้ใน!) และเมธอด `virtual ShowDetails` (มีปีกกาปกติ)

### ขั้นตอนที่ 2: สร้างพนักงานประจำ (Full-Time)
- [ ] สร้างคลาสลูกตัวแรก แล้วสร้าง Property ตัวเงิน
- [ ] จัดการ Constructor ซิ่งส่งต่อ `: base(id, name)`
- [ ] พิมพ์ `override CalculateSalary()` แล้วกด Tab ให้มันเติมโครงให้ เสร็จแล้วใส่สมการคืนเงิน

### ขั้นตอนที่ 3: สร้างพนักงานรายชั่วโมงและพนักงานขาย
- [ ] ทำซ้ำขั้นตอนที่ 2 แต่เปลี่ยนตัวแปรตามแต่ละอาชีพ (ระวัง Constructor มันจะเริ่มยาวขึ้น)
- [ ] ในคลาสพนักงานขาย อย่าลืม `override ShowDetails` เพิ่มเป็นพิเศษ!

### ขั้นตอนที่ 4: สวมบทบาทแผนกบัญชี (Main Program)
- [ ] สร้างกล่อง Array เก็บพนักงาน 3 คน
- [ ] ลุยลูป `foreach` ปริ้นท์สลิปเงินเดือนแบบรวดเดียวจบ ไร้รอยต่อ!

## 🔥 Challenge (ภารกิจท้าทายขีดจำกัด)

สำหรับสถาปนิกที่ต้องการเลื่อนขั้นสู่ตำแหน่ง Senior:

**ภารกิจระบบหักภาษีหฤโหด (The Polymorphic Tax Deductor):**
ในโลกแห่งความจริง ไม่ใช่ทุกคนที่โดนหักภาษีเรทเดียวกัน! 
**อัปเกรดระบบของคุณดังนี้:**
1. ในคลาสแม่ `Employee` ให้เพิ่มเมธอด `public virtual decimal CalculateTax(decimal grossSalary)` 
   - ฐานแม่กำหนดให้ "ไม่มีการหักภาษีใดๆ" คืนค่า 0 ไปเลย (เผื่อฟรีแลนซ์หลบภาษี)
2. สำหรับ `FullTimeEmployee` ให้ก่อกบฏ `override CalculateTax()`:
   - ถ้าเงินเดือน > 20,000 ให้หัก 10% ของเงินเดือนทั้งหมด คืนค่าตัวเลขที่หักออกมา!
   - ถ้า <= 20,000 หักแค่ 5% 
3. ใน `Main` (ลูปบัญชี) หลังจากเรียก `CalculateSalary` (สมมติได้ตัวแปร Gross) 
   - ให้เรียก `.CalculateTax(Gross)` เพิ่มเติมเพื่อดูว่าโดนหักกี่บาท
   - จากนั้นเอา Gross - Tax = เงินเดือนสุทธิที่แท้จริง! (Net Salary)
   - ปริ้นโชว์ในสลิปด้วยว่าโดนสรรพากรหุบไปกี่บาท!

ถ้าคุณสามารถทำระบบคำนวณหักลบรายได้ด้วยการเอาหน้าต่าง `virtual` มาซ้อนกัน 2 ชั้นได้... แปลว่ากระบวนทัศน์ OOP ของคุณนั้นสมบูรณ์แบบไร้ที่ติ และโค้ดของคุณจะไม่มีวันเกิดสปาเก็ตตี้โค้ดในอนาคตแน่นอนครับ!
