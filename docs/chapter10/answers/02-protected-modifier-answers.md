# เฉลย: Protected Modifier (ยามประจำตระกูล)

## 🎯 เฉลย Mini Exercise

### **โจทย์ที่ 1: นักสืบสายเลือด (Bloodline Detective)**
**เป้าหมาย:** ทดสอบความเข้าใจเรื่องระยะการมองเห็นของคีย์เวิร์ด Protected ปะทะโลกภายนอก

**เฉลย:**
**ไม่ได้เด็ดขาดครับ! แดงเถือก!**
**เหตุผล:** คลาส `Mechanic` (ช่างซ่อม) เป็น "คนนอกตระกูล" ครับ! มันไม่ได้มีเครื่องหมาย `: Machine` ต่อท้ายชื่อคลาสเลย! 
ดังนั้นต่อให้คุณส่งออบเจกต์รถยนต์ (`Car`) เข้ามาให้ช่างซ่อม... แต่ตัวแปร `Fuel` ของรถยนต์มันถูกตั้งเป็น `protected` 
กฎของ `protected` คือ: "ถ้าไม่ใช่คนในสายเลือด (คนนอก) ฉันจะแปลงร่างเป็น `private` ปิดตู้เซฟใส่หน้าแกทันที!"
**วิธีแก้:** ช่างต้องโทรเรียกเมธอด `public` ของรถ เช่น `c.Refuel();` แทนการเอื้อมไปแก้ตัวแปรตรงๆ ครับ!

### **โจทย์ที่ 2: รหัสลับสี่ตัว (The Access Modifiers)**
**เป้าหมาย:** จับคู่สรุปรวมอาวุธ Access Modifiers ทั้งหมดใน C#

**เฉลย:**
1. **[B] `private`:** มองเห็นแค่ในคลาสตัวเองเท่านั้น ลูกก็มองไม่เห็น (ความลับส่วนตัวสุดๆ)
2. **[A] `public`:** มองเห็นได้ทุกคนในจักรวาล สดใสซาบซ่า (หน้าต่างบริการประชาชน)
3. **[D] ไม่ใส่คำนำหน้า:** คอมไพเลอร์จะมองว่ามันคือ `private` อัตโนมัติ (C# ปิดประตูตีแมว เซฟไว้ก่อน)
4. **[C] `protected`:** มองเห็นในคลาสตัวเอง และเปิดข้อยกเว้นให้ลูกหลานที่สืบทอดสายเลือดไปมองเห็นได้ด้วย!

## 🔥 เฉลย Challenge (ระบบบัญชีเงินเดือนฉบับผู้จัดการ The Payroll Manager)

**เป้าหมาย:** ประยุกต์ใช้ Protected เพื่อสร้างตัวแปร "มรดกประจำตระกูล" ที่ให้สิทธิพิเศษเฉพาะคลาสลูกสามารถดึงไปคำนวณต่อยอดได้ แต่คนนอกห้ามแตะ!

**กระบวนการคิดระดับสถาปนิก (Encapsulation Balance):**
เงินเดือนดิบ (`_baseSalary`) คือข้อมูลอ่อนไหว ถ้าปล่อยเป็น `public` ใน Main อาจจะเกิดการแก้ตัวเลขมั่วๆ ได้... แต่ถ้าตั้งเป็น `private` ผู้จัดการก็จะเอาเงินนี้ไปคำนวณโบนัสต่อไม่ได้! `protected` จึงเกิดมาเป็นพระเอกในเรื่องนี้ครับ (ปกป้องจากโลกภายนอก แต่เปิดให้คนในครอบครัวทำงานร่วมกันได้)

```csharp
using System;

namespace ProtectedPayroll
{
    // =======================================================
    // BASE CLASS (คลาสพนักงานทั่วไป)
    // =======================================================
    class Employee
    {
        public string Name { get; set; }

        // 🌟 ไฮไลต์: protected field ยามเฝ้าประตูระดับครอบครัว!
        // Main เอื้อมมือมาเปลี่ยนเงินก้อนนี้ตรงๆ ไม่ได้ แต่ลูก (Manager) หยิบไปใช้ได้สบาย!
        protected decimal _baseSalary;

        public void SetSalary(decimal money)
        {
            // เมธอดนี้เป็น public เพื่อเป็นช่องทางเดียวให้คนนอก (Main) โอนเงินเข้ามาเก็บได้
            if (money > 0)
            {
                _baseSalary = money;
            }
        }

        public void PrintSlip()
        {
            Console.WriteLine($"📄 [สลิปเงินเดือน] {Name} รับเงินเดือนสุทธิ: {_baseSalary:N2} บาท");
        }
    }

    // =======================================================
    // DERIVED CLASS (คลาสผู้จัดการ)
    // =======================================================
    class Manager : Employee
    {
        // ผู้จัดการมีความสามารถพิเศษเฉพาะตัว คือการเสกโบนัสให้ตัวเอง!
        public void CalculateBonus()
        {
            decimal bonusAmount = 50000;
            
            // 🌟 เวทมนตร์: Manager สามารถเข้าถึง _baseSalary ของคลาสแม่ได้โดยตรง!
            // เพราะมันสืบทอดสายเลือดมา และตัวแปรถูกเปิดช่องทาง (protected) ไว้ให้แล้ว
            _baseSalary += bonusAmount; 

            Console.ForegroundColor = ConsoleColor.Green;
            Console.WriteLine($"✨ [อนุมัติพิเศษ] ผู้จัดการ {Name} ได้รับโบนัส! ปรับยอดสุทธิเป็น: {_baseSalary:N2} บาท");
            Console.ResetColor();
        }
    }

    // =======================================================
    // EXECUTION
    // =======================================================
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบคำนวณโบนัสผู้จัดการ (Payroll System) ===\n");

            Manager m1 = new Manager();
            m1.Name = "Boss อลิซ";

            // 💥 พยายามแฮก: m1._baseSalary = 999999;
            // ลองเอาคอมเมนต์ออกดูครับ คอมไพเลอร์จะตบหน้าหัน! 
            // เพราะยาม protected ถือว่า Main คือ "คนนอกตระกูล" ไม่ยอมให้ล้วงเซฟตรงๆ เด็ดขาด!

            // 1. ตั้งเงินเดือนตั้งต้น (ผ่านช่องทางที่อนุญาต)
            m1.SetSalary(30000);
            
            // 2. ปริ้นสลิปก่อนได้โบนัส (ใช้เมธอดมรดกจากแม่)
            m1.PrintSlip(); // ได้ 30,000

            // 3. ปลุกพลังผู้จัดการ (ใช้ความสามารถของลูก)
            m1.CalculateBonus(); 

            // 4. ปริ้นสลิปหลังได้โบนัส
            m1.PrintSlip(); // ได้ 80,000!
        }
    }
}
```

**สิ่งที่ได้เรียนรู้ (Architecture Lesson):**
การตัดสินใจว่าตัวแปรไหนควรเป็น `private` หรือ `protected` คือศิลปะขั้นสูงของสถาปนิกซอฟต์แวร์!
- ถ้าคุณกลัวว่าลูกหลานจะซน แล้วเผลอเขียนทับตัวแปรจนระบบพัง (เช่น เลขบัญชีธนาคาร): จงใช้ `private` ล็อกตายไปเลย!
- ถ้าคุณอยากให้ลูกหลานเป็นอิสระ สามารถหยิบตัวแปรนี้ไปขยำรวมกับสูตรใหม่ๆ ได้ (เช่น สูตรเงินเดือน): จงใช้ `protected` เปิดทางให้สายเลือดเดียวกันครับ!
