# เฉลย: Inheritance Basics (พันธุกรรมและการสืบทอดสายเลือด)

## 🎯 เฉลย Mini Exercise

### **โจทย์ที่ 1: ตรรกะวิบัติ (The Broken IS-A)**
**เป้าหมาย:** แยกแยะระหว่างความสัมพันธ์แบบ IS-A และ HAS-A

**เฉลยคำตอบระดับสถาปนิก:**
ในฝั่งคอมพิวเตอร์ โค้ดนี้ Compile ผ่าน 100% ครับ! เพราะคอมไพเลอร์ไม่ได้รู้ภาษาคน มันไม่แคร์ว่ารถจะเป็นเครื่องยนต์หรือไม่ 
**แต่ในเชิงสถาปัตยกรรม นี่คือหายนะและผิดกฎอย่างร้ายแรงครับ!**
- **กฎ IS-A:** รถยนต์ (Car) ไม่ได้ "เป็น" เครื่องยนต์ (Engine) ชนิดหนึ่ง! ดังนั้นการใช้เครื่องหมายโคลอน `:` เพื่อสืบทอดพันธุกรรม จึงผิดตรรกะแบบไม่น่าให้อภัย
- **วิธีที่ถูกต้อง (Composition / HAS-A):** รถยนต์ "มี" เครื่องยนต์อยู่ข้างในต่างหาก! ดังนั้นเราต้องสร้างคลาส Engine แล้วเอามันไปยัดเป็นตัวแปร (Field) หนึ่งอยู่ข้างในคลาส Car แทนครับ:
  ```csharp
  class Car {
      public Engine MyEngine; // HAS-A Relationship
  }
  ```

### **โจทย์ที่ 2: ห่วงโซ่อาหาร (The Inheritance Chain)**
**เป้าหมาย:** ทำความเข้าใจการสะสมพันธุกรรมแบบลูกโซ่

**เฉลย:**
ตัวแปร `s` จะมี Property ให้เรียกใช้งานรวมทั้งหมด **3 ตัว** ครับ!
ได้แก่: `Area` (มรดกจากปู่), `Width` (มรดกจากพ่อ), และ `Edge` (ของตัวเอง)
**เหตุผล:** C# อนุญาตให้สายเลือดไหลต่อกันเป็นทอดๆ (Multi-level Inheritance) ทำให้คลาสที่อยู่ล่างสุดของห่วงโซ่อาหาร จะดูดซับเอาพลังทั้งหมดของบรรพบุรุษทุกรุ่นที่อยู่สูงกว่ามารวมไว้ที่ตัวเองอย่างสมบูรณ์แบบครับ!

## 🔥 เฉลย Challenge (วิศวกรโรงงานผลิตรถยนต์ The Vehicle Factory)

**เป้าหมาย:** สร้างโครงสร้างตระกูล (Taxonomy) จัดหมวดหมู่คุณสมบัติร่วมและคุณสมบัติเฉพาะของออบเจกต์

**กระบวนการคิดระดับสถาปนิก (Class Hierarchy Design):**
ในโลกความเป็นจริง ยานพาหนะทุกคันบนโลกต้องมี "ยี่ห้อ" และมีความเร็ว (Speed) แถมต้องบีบแตรได้เหมือนกัน เราจึงเอาของพวกนี้ไปจับยัดไว้ในคลาสแม่ (Vehicle) เพื่อที่ลูกๆ (Car และ Motorcycle) จะได้ไม่ต้องมานั่งเขียนตัวแปรและเมธอดพวกนี้ซ้ำสองครับ! โค้ดจะสั้นลงทันตาเห็น!

```csharp
using System;

namespace FactorySimulator
{
    // =======================================================
    // 1. คลาสแม่สุด (Base Class)
    // =======================================================
    class Vehicle
    {
        public string Brand { get; set; }
        public int Speed { get; set; }

        public void Honk()
        {
            Console.WriteLine($"🔊 [{Brand}] บีบแตร ปี๊นๆ!");
        }
    }

    // =======================================================
    // 2. คลาสลูกสายที่ 1: รถยนต์ (Derived Class)
    // =======================================================
    class Car : Vehicle // สืบทอดจาก Vehicle
    {
        public int NumberOfDoors { get; set; }

        public void TurnOnAC()
        {
            Console.WriteLine($"❄️ เปิดแอร์เย็นฉ่ำในรถ {NumberOfDoors} ประตู!");
        }
    }

    // =======================================================
    // 3. คลาสลูกสายที่ 2: มอเตอร์ไซค์ (Derived Class)
    // =======================================================
    class Motorcycle : Vehicle // สืบทอดจาก Vehicle เช่นกัน
    {
        public bool HasSidecar { get; set; }

        public void Wheelie()
        {
            Console.ForegroundColor = ConsoleColor.Red;
            Console.WriteLine($"🏍️ ยกล้อโชว์สาว! ซิ่งด้วยความเร็ว {Speed} กม./ชม.");
            Console.ResetColor();
        }
    }

    // =======================================================
    // EXECUTION
    // =======================================================
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== โรงงานผลิตยานพาหนะอัจฉริยะ ===\n");

            // 1. ปล่อยรถยนต์ออกจากโรงงาน
            Car honda = new Car();
            honda.Brand = "Honda"; // สืบทอดมา
            honda.Speed = 120;     // สืบทอดมา
            honda.NumberOfDoors = 4; // ของตัวเอง

            Console.WriteLine(">> ทดสอบรถยนต์:");
            honda.Honk();     // เรียกท่าของแม่
            honda.TurnOnAC(); // เรียกท่าของลูก

            Console.WriteLine("\n----------------------------------\n");

            // 2. ปล่อยมอเตอร์ไซค์ออกจากโรงงาน
            Motorcycle yamaha = new Motorcycle();
            yamaha.Brand = "Yamaha"; // สืบทอดมา
            yamaha.Speed = 150;      // สืบทอดมา
            yamaha.HasSidecar = false; // ของตัวเอง

            Console.WriteLine(">> ทดสอบมอเตอร์ไซค์:");
            yamaha.Honk();     // เรียกท่าของแม่
            yamaha.Wheelie();  // เรียกท่าของลูก

            // 💥 garfield.TurnOnAC(); 
            // แน่นอนว่ามอเตอร์ไซค์จะเรียก TurnOnAC ไม่ได้! เพราะมันอยู่คนละสายเลือดกัน!
        }
    }
}
```

**สิ่งที่ได้เรียนรู้ (Architecture Lesson):**
ถ้าวันหน้าบริษัทคุณต้องการสร้าง "รถพ่วง (Truck)" เพิ่ม คุณก็แค่สร้าง `class Truck : Vehicle`... คุณจะได้ยี่ห้อและความเร็วแถมมาใช้ฟรีๆ ทันที! โดยไม่ต้องไปแตะต้องโค้ดของแม่ รถยนต์ หรือมอเตอร์ไซค์เดิมเลย! (ระบบจะเสถียรมาก โอกาสเกิดบั๊กในโค้ดเก่าเท่ากับศูนย์) นี่แหละครับความสง่างามของ OOP!
