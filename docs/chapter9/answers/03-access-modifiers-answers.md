# เฉลย: Access Modifiers (ยามรักษาความปลอดภัย)

## 🎯 เฉลย Mini Exercise

### **โจทย์ที่ 1: เติมคำในช่องว่าง (Access Modifiers)**
**เป้าหมาย:** ตอบกฎเหล็กของการรักษาความปลอดภัยข้อมูล

**เฉลย:**
1. "ตัวแปร (Fields)" ที่เก็บสถานะของระบบ ควรตั้งเป็น **`private`** เสมอ! (นี่คือกฎศักดิ์สิทธิ์ของการเขียนโค้ด) เพื่อป้องกันไม่ให้ใครมาล้วงตู้เซฟของเราโดยตรง
2. "ฟังก์ชัน (Methods)" ที่เตรียมไว้ให้คลาสอื่นเรียกใช้งาน ควรตั้งเป็น **`public`**! (เป็นหน้าต่างบริการหรือช่องทางสื่อสารสำหรับคนนอก)
3. ถ้าลืมพิมพ์คำนำหน้า ตัวแปรนั้นจะถูกเซ็ตเป็น **`private`** ทันทีโดยอัตโนมัติ (C# ถือคติ "ปิดประตูให้หมดก่อน" เผื่อโปรแกรมเมอร์เผลอเรอครับ)

### **โจทย์ที่ 2: ช่องโหว่ของโรงพยาบาล (The Naked Variable)**
**เป้าหมาย:** อัปเกรดตัวแปรที่โป๊เปลือย (public) ให้เป็นระบบที่มีด่านตรวจคนเข้าเมือง (Private + Validation Method)

**เฉลยและวิธีทำ:**
```csharp
using System;

class Patient 
{
    // 1. จับยัดเข้าเซฟ (เปลี่ยนเป็น private และตั้งชื่อมี _ นำหน้าตามมาตรฐานไมโครซอฟต์)
    private int _age;

    // 2. สร้างพนักงานหน้าเคาน์เตอร์มาคอยรับเรื่อง (public Method)
    public void SetAge(int newAge) 
    {
        // 3. ติดตั้งด่านสกัดจับข้อมูลขยะ (Validation)
        if (newAge < 0) 
        {
            Console.WriteLine("🚨 ระบบขัดข้อง: อายุของคนไข้ห้ามติดลบ!");
            return; // ดีดตัวออก เตะคนนอกทิ้ง ไม่ยอมบันทึก
        }
        
        // 4. ตรวจสอบผ่าน ยอมให้อัปเดตได้
        _age = newAge;
        Console.WriteLine($"✅ อัปเดตอายุคนไข้เป็น: {_age} ปีเรียบร้อย");
    }
}
```

## 🔥 เฉลย Challenge (หมอเถื่อนและยาผีบอก The HP Validator)

**เป้าหมาย:** สร้างแคปซูลต่อต้านการโกง (Anti-Cheat System) สำหรับตัวละครฮีโร่ โดยห้ามเลือดติดลบ และห้ามฮีลทะลุหลอด Max HP เด็ดขาด

**กระบวนการคิดระดับสถาปนิก (Data Protection):**
การสร้าง Object ที่ปลอดภัย หมายความว่า ไม่ว่าเพื่อนร่วมงาน (คนเขียน Main) จะพยายามส่งข้อมูลมั่วๆ บ้าบอแค่ไหนเข้ามา... แคปซูลของเราก็ต้องไม่พังทลายลงมา (Robustness) การเขียน `if` สกัดจับขอบเขต (Boundary Checks) จึงเป็นหน้าที่ของคลาสตัวมันเองครับ

```csharp
using System;

namespace EncapsulationAntiCheat
{
    class Hero
    {
        // =======================================================
        // 1. DATA (ความลับสุดยอด ห้ามแฮกเกอร์ใช้โปรแกรมแกะค่า)
        // =======================================================
        private int _hp;
        private int _maxHp = 100; // ค่าคงที่ หลอดเลือดตันที่ 100

        // =======================================================
        // 2. BEHAVIORS (ช่องทางตอบโต้กับโลกภายนอกที่อนุญาต)
        // =======================================================
        
        public void ShowStatus()
        {
            // โชว์หน้าปัดให้ผู้เล่นดู
            Console.WriteLine($"[ฮีโร่] พลังชีวิตปัจจุบัน: {_hp} / {_maxHp}");
        }

        // --- ระบบโดนโจมตี ---
        public void TakeDamage(int damage)
        {
            if (damage <= 0) return; // กรองคนส่งค่าติดลบมาเล่นๆ
            
            _hp -= damage;
            Console.WriteLine($"💥 โดนอัดเข้าให้! เสียเลือด {damage} หน่วย");

            // ด่านตรวจจับ: ถ้าเลือดติดลบ ให้ชุบกลับมาล็อกไว้ที่ 0 (ตายสนิท)
            if (_hp < 0)
            {
                _hp = 0;
            }
        }

        // --- ระบบฮีลเลือด (ด่านปราบเซียน) ---
        public void Heal(int amount)
        {
            if (amount <= 0) return; // กรองของปลอม

            _hp += amount;
            Console.WriteLine($"✨ ซดยาเพิ่มพลัง! ฮีลขึ้น {amount} หน่วย");

            // 🌟 ด่านตรวจจับไฮไลต์: ถ้าเลือดกระฉูดทะลุหลอด Max!
            if (_hp > _maxHp)
            {
                // ปัดเศษทิ้ง แล้วล็อกไว้ที่ Max ทันที!
                _hp = _maxHp;
                Console.WriteLine("   -> (ข้อความระบบ: พลังชีวิตเต็มหลอดแล้ว!)");
            }
        }
    }

    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบป้องกันแฮกเกอร์ (Anti-Cheat) ===\n");

            Hero arthur = new Hero();
            
            // ฮีลเริ่มต้นให้เต็มร้อย (เพราะค่า Default ของ int คือ 0)
            arthur.Heal(100); 
            arthur.ShowStatus(); // 100/100

            // -----------------------------------------------------------
            // ทดสอบสมรภูมิโหด
            // -----------------------------------------------------------
            Console.WriteLine("\n>> มอนสเตอร์โผล่มาตบ!");
            arthur.TakeDamage(40);
            arthur.ShowStatus(); // 60/100

            Console.WriteLine("\n>> กินน้ำยาอมฤตฮีลเลือด 500 หยด!");
            arthur.Heal(500); 
            arthur.ShowStatus(); // ต้องหยุดที่ 100/100! (ระบบป้องกันทำงานสำเร็จ)

            Console.WriteLine("\n>> ท้ายทอยกระแทกพื้น โดนดาเมจ 999!");
            arthur.TakeDamage(999);
            arthur.ShowStatus(); // ต้องหยุดที่ 0/100! (ห้ามติดลบ)
        }
    }
}
```

**สิ่งที่ได้เรียนรู้ (Architecture Lesson):**
ยินดีด้วยครับ! คุณเพิ่งเขียนโมเดลตรวจสอบข้อมูล (Model Validation) เสร็จสมบูรณ์แล้ว! 
คุณสังเกตไหมว่า ถ้าผมเปลี่ยนชื่อ `Hero` เป็น `BankAccount`... เปลี่ยนฟิลด์ `_hp` เป็น `_balance`... และเปลี่ยนคำว่า `Heal` เป็น `Deposit`
**มันคือลอจิกเดียวกันกับระบบการเงินของธนาคารเป๊ะเลยครับ!**
การเรียนเรื่องการสร้างด่านสกัดขยะด้วย Access Modifiers คือพื้นฐานของการพัฒนาซอฟต์แวร์ระดับสากล ไม่ว่าจะเขียนเกมหรือเว็บแอปพลิเคชันก็ตาม!
