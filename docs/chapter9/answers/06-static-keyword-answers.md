# เฉลย: Static Keyword (สิ่งดำรงอยู่ร่วมกัน)

## 🎯 เฉลย Mini Exercise

### **โจทย์ที่ 1: เครื่องมือแปลงหน่วย (The Converter)**
**เป้าหมาย:** สร้าง Utility Class ที่เป็นส่วนกลางของจักรวาล (ไม่ต้อง `new`)

**เฉลยและวิธีทำ:**
เคล็ดลับคือการแปะยันต์ `static` ไปทั้ง 2 จุด ทั้งที่ป้ายชื่อ Class และหน้า Method ครับ!
```csharp
// 1. ล็อกคลาส ห้ามใครมา new เด็ดขาด
static class TemperatureConverter 
{
    // 2. เสกเมธอดส่วนกลาง
    public static double CelsiusToFahrenheit(double c) 
    {
        // ⚠️ ทริคคณิตศาสตร์: ถ้าพิมพ์ 9/5 เฉยๆ มันจะได้เศษส่วนจำนวนเต็ม (คือเลข 1)
        // ต้องใส่ 9.0 / 5.0 เพื่อบอกให้มันคำนวณเป็นทศนิยมด้วย!
        return (c * 9.0 / 5.0) + 32; 
    }
}
```

### **โจทย์ที่ 2: งูกินหางแห่งความตาย (The Static Trap)**
**เป้าหมาย:** ทำความเข้าใจวิกฤติเมื่อป้ายหน้าโรงเรียน พยายามดึงสมองส่วนตัวของนักเรียนมาปริ้น!

**เฉลยคำตอบระดับวิศวกร:**
คอมไพเลอร์จะด่าว่า **"An object reference is required for the non-static field, method, or property 'Robot.Name'"** ครับ!
**สาเหตุความพินาศ:** 
ฟังก์ชัน `Speak()` ถูกตั้งให้เป็น `static` (มันคือโทรโข่งที่แขวนอยู่หน้าโรงเรียน) แต่มันดันพยายามสั่งปริ้นตัวแปร `Name` ซึ่งเป็นตัวแปรแบบ Instance (ป้ายชื่อบนอกเสื้อนักเรียน)
คอมพิวเตอร์มันจะงงเป็นไก่ตาแตกว่า... *"คุณสั่งให้โทรโข่งโรงเรียนประกาศชื่อ... แต่โรงเรียนนี้มีหุ่นยนต์ 1,000 ตัว! จะให้กูประกาศชื่อของตัวไหนวะ!?"*
(ตัวแปร Static มันห้อยอยู่บนฟ้า มันไม่สามารถเอื้อมมือลงมาแตะตัวแปร Instance ที่อยู่บนดินได้เลย หากไม่ระบุตัวตนให้ชัดเจนครับ!)

## 🔥 เฉลย Challenge (ธนาคารโลก The Global Bank Rate)

**เป้าหมาย:** ประจักษ์ถึงอำนาจมืดของตัวแปร `static` ที่แค่แก้จุดเดียว แต่กระเทือนส่งผลกระทบสะเทือนไปถึงบ้านทุกหลังบนคฤหาสน์ Heap!

**กระบวนการคิดระดับสถาปนิก (Global Shared State):**
ปกติเงินใครก็เงินมัน (`Balance` ไม่เป็น static) แต่ถ้าเป็นเรื่องนโยบายส่วนกลาง เช่น ดอกเบี้ย (`InterestRate`) หรือภาษีมูลค่าเพิ่ม เราจะต้องแชร์ใช้ร่วมกัน!
นี่คือโจทย์คลาสสิกเพื่อดูว่า ผู้สมัครงานเข้าใจความต่างระหว่าง "ข้อมูลระดับออบเจกต์" ปะทะ "ข้อมูลระดับคลาส" หรือไม่!

```csharp
using System;

namespace GlobalBankingSystem
{
    class SavingsAccount
    {
        // =======================================================
        // 1. INSTANCE DATA (เงินใครเงินมัน ป้ายชื่อใครป้ายชื่อมัน)
        // =======================================================
        public string Owner { get; set; }
        public double Balance { get; set; }

        // =======================================================
        // 2. STATIC DATA (นโยบายรัฐบาล เป็นก้อนกลางก้อนเดียวในจักรวาล!)
        // =======================================================
        // ค่าเริ่มต้น 5%
        public static double InterestRate { get; set; } = 0.05; 

        // -------------------------------------------------------
        // CONSTRUCTOR
        // -------------------------------------------------------
        public SavingsAccount(string owner, double balance)
        {
            this.Owner = owner;
            this.Balance = balance;
        }

        // -------------------------------------------------------
        // BEHAVIOR (พฤติกรรมระดับบุคคล ที่เอื้อมมือไปแตะนโยบายกลางได้)
        // -------------------------------------------------------
        public void AddInterest()
        {
            // คำนวณดอกเบี้ย: เอาเงินส่วนตัว ไปคูณนโยบายกลาง
            // สังเกตว่าตัวแปรแบบปกติ สามารถดึง static มาใช้งานได้อย่างสบายใจ!
            double interestAmount = this.Balance * InterestRate;
            
            this.Balance += interestAmount; // ทบต้น

            Console.WriteLine($"[ธนาคาร] อัปเดตยอดของ {Owner} (บวกดอกเบี้ย {interestAmount} บาท)");
            Console.WriteLine($"  -> ยอดสุทธิกลายเป็น: {this.Balance} บาท");
        }
    }

    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบคำนวณดอกเบี้ยธนาคารกลาง (Global Bank) ===\n");

            // 1. สร้างบัญชีให้คน 2 คน
            SavingsAccount john = new SavingsAccount("John", 1000);
            SavingsAccount alice = new SavingsAccount("Alice", 2000);

            // 2. สั่งรันคำนวณดอกเบี้ย (ตอนนี้นโยบายกลางอยู่ที่ 5%)
            Console.WriteLine(">> ประกาศจ่ายดอกเบี้ยอัตรา 5%");
            john.AddInterest();  // 1000 + 50 = 1050
            alice.AddInterest(); // 2000 + 100 = 2100

            // -----------------------------------------------------------
            // 🌟 3. วินาทีประวัติศาสตร์: รัฐบาลประกาศหั่นดอกเบี้ย! 🌟
            // -----------------------------------------------------------
            Console.WriteLine("\n🚨 [ข่าวด่วน] ธนาคารกลางประกาศลดอัตราดอกเบี้ยเหลือ 1%!");
            
            // เราไม่เรียกตัวแปรผ่าน john หรือ alice แต่เรียกตรงๆ กลางเวหาผ่านชื่อคลาส!
            SavingsAccount.InterestRate = 0.01; 

            // 4. สั่งรันดอกเบี้ยรอบใหม่
            Console.WriteLine("\n>> ประกาศจ่ายดอกเบี้ยรอบใหม่ (อัตรา 1%)");
            
            // John ยอดยกมาคือ 1050 ... คูณดอกเบี้ยใหม่ 1% จะได้ 10.5
            // พลังของ static จะทำให้บรรทัดนี้ไปหยิบอัตราใหม่มาใช้ทันที!
            john.AddInterest();  // 1050 + 10.5 = 1060.5
        }
    }
}
```

**สิ่งที่ได้เรียนรู้ (Architecture Lesson):**
ความน่ากลัวของการใช้ `static` เยอะเกินไปคือ **"มันแก้ข้ามหัวกันได้ง่ายมาก"** ครับ! 
ถ้ามีโปรแกรมเมอร์แอบเปลี่ยนค่า `InterestRate` ระหว่างทาง... มันจะกระทบคนทั้งระบบเลย (เรียกว่าปัญหา Global State Mutation) 
สถาปนิกที่เก่งกาจ จะพยายามใช้ `static` เฉพาะกับตัวแปรที่ "อ่านได้อย่างเดียวและห้ามแก้ (Readonly / Const)" เป็นหลัก เพื่อให้โปรแกรมมีความน่าเชื่อถือ 100% ครับ!
