# เฉลย: Fields & Properties (ด่านตรวจคนเข้าเมือง)

## 🎯 เฉลย Mini Exercise

### **โจทย์ที่ 1: วิเคราะห์โค้ดพัง (The Property Trap)**
**เป้าหมาย:** สังเกตและอธิบายหายนะของ Infinite Recursion จากการชี้เป้าผิดที่

**เฉลยคำตอบระดับวิศวกร:**
เมื่อกดรัน โปรแกรมจะล่มพร้อมด่าว่า **"StackOverflowException" (แรมทะลุหลอด)** ครับ!
**สาเหตุ:** 
ในบล็อก `set` โปรแกรมเมอร์เขียนว่า `HP = value;` 
การพยายามยัดของใส่ `HP` มันจะเป็นการ "ตีระฆังเรียกประตู set ของ Property HP ให้ทำงาน"... 
พอมันเข้าไปทำใน `set` มันก็เจอคำสั่ง `HP = value;` อีก! มันก็ปลุก `set` อีกรอบ! ปลุกกันไปมาเป็นล้านๆ รอบจนคอมพิวเตอร์น็อกครับ! (เรียกว่างูกินหางตัวเอง)
**วิธีแก้ที่ถูกต้อง:** 
ต้องสร้างตัวแปรลับหลังบ้านมาช่วยรับของครับ เช่น `private int _hp;` แล้วตอนเซ็ตให้สั่ง `_hp = value;` แทน!

**เป้าหมาย:** ยุบรวม Boilerplate โค้ดยุค 90s ให้กลายเป็น C# Property สมัยใหม่

**เฉลย:**

## 🔥 เฉลย Challenge (เครื่องปรับอากาศอัจฉริยะ The Smart Thermostat)

**เป้าหมาย:** สร้างระบบ IoT ที่ปกป้องตัวเองจากการป้อนอุณหภูมิที่ไร้สาระ โดยใช้ `get/set` เป็นตัวคัดกรองข้อมูลดิบ (Data Sanitization)

**กระบวนการคิดระดับสถาปนิก (Data Clamping):**
โจทย์ข้อนี้จำลองการทำ "Clamping (การหนีบค่าให้อยู่ในกรอบ)" 
เวลาลูกค้าพิมพ์ค่ามั่วๆ จากหน้าเว็บ (เช่น 100 องศา) เราไม่ยอมให้แอปพลิเคชันพัง แต่เราจะใช้ประตู `set` เป็น "ตะแกรงร่อนขยะ" ใครส่ง 100 มา เราบังคับปรับเป็น 30 ทันที! ทำให้ Object ของเรามีสเตตัสที่ถูกต้องอยู่เสมอ 100% ครับ

```csharp
using System;

namespace SmartHomeIoT
{
    class Thermostat
    {
        // 1. Auto-Property: ให้อ่าน/เขียนชื่อเครื่องแอร์ได้อิสระ
        public string DeviceName { get; set; }

        // 2. ตัวแปรเบื้องหลัง (Backing Field) ที่ใช้กักเก็บอุณหภูมิของจริง
        private float _temperature;

        // 3. Full Property: ด่านตรวจคนเข้าเมืองสุดโหด
        public float TemperatureC
        {
            get 
            { 
                return _temperature; // ขาออก: ยอมให้ดูค่าอุณหภูมิปัจจุบันได้
            }
            set 
            {
                // ขาเข้า: value คือของที่คนข้างนอกส่งมา เราต้องตรวจสอบก่อน!
                if (value < 16)
                {
                    Console.ForegroundColor = ConsoleColor.Cyan;
                    Console.WriteLine($"[ระบบ] ตรวจพบการตั้งค่า {value}°C -> หนาวเกินไป! ล็อกค่าไว้ที่ 16°C");
                    Console.ResetColor();
                    _temperature = 16; // บังคับหนีบค่าล่างสุด
                }
                else if (value > 30)
                {
                    Console.ForegroundColor = ConsoleColor.Red;
                    Console.WriteLine($"[ระบบ] ตรวจพบการตั้งค่า {value}°C -> ร้อนเกินไป! ล็อกค่าไว้ที่ 30°C");
                    Console.ResetColor();
                    _temperature = 30; // บังคับหนีบค่าสูงสุด
                }
                else
                {
                    // ถ้าอุณหภูมิปกติดี (16-30) ก็ปล่อยผ่านให้เก็บของเข้าโกดังได้เลย
                    _temperature = value;
                    Console.WriteLine($"[ระบบ] อัปเดตอุณหภูมิเป็น {_temperature}°C สำเร็จ");
                }
            }
        }
    }

    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบควบคุมเครื่องปรับอากาศ (Smart Thermostat) ===\n");

            // เสกแอร์ขึ้นมา 1 ตัว
            Thermostat bedroomAc = new Thermostat();
            bedroomAc.DeviceName = "แอร์ห้องนอนใหญ่";

            // -----------------------------------------------------------
            // ทดสอบยิงข้อมูลเข้าประตู Set
            // -----------------------------------------------------------
            Console.WriteLine(">> แฟนสั่งตั้งอุณหภูมิ 25 องศา (ปกติ)");
            bedroomAc.TemperatureC = 25; // วิ่งเข้าเงื่อนไข else
            
            Console.WriteLine("\n>> เจ้านายบ้าพลัง สั่งตั้งแอร์ -5 องศา! (หนาวสุดขั้ว)");
            bedroomAc.TemperatureC = -5; // วิ่งเข้าเงื่อนไข < 16
            
            Console.WriteLine("\n>> เด็กเล่นรีโมท สั่งแอร์ 100 องศา! (ไฟไหม้)");
            bedroomAc.TemperatureC = 100; // วิ่งเข้าเงื่อนไข > 30

            // -----------------------------------------------------------
            // ทดสอบเปิดดูค่าผ่านประตู Get
            // -----------------------------------------------------------
            Console.WriteLine($"\n[รายงานผล] อุณหภูมิแอร์ตัวนี้ปัจจุบันคือ: {bedroomAc.TemperatureC}°C");
            // ผลลัพธ์สุดท้าย แอร์จะรอดปลอดภัยอยู่ที่ 30 องศาครับ!
        }
    }
}
```

**สิ่งที่ได้เรียนรู้ (Architecture Lesson):**
การดักข้อมูล `if/else` แบบนี้ สมัยก่อนโปรแกรมเมอร์ต้องเอาไปเขียนดักไว้ในฟังก์ชันหน้าจอ UI (เช่น หน้าเว็บ) ซึ่งถ้ามีหน้าเว็บหลายหน้า โค้ดนี้ก็ต้องก๊อปปี้ไปวางหลายรอบ 
แต่ในวิถีของ OOP... เราจับมันยัดใส่ไว้ที่ "ต้นขั้วของกล่อง (Property)" เลย! หมายความว่าไม่ว่าแอปมือถือ, หน้าเว็บ, หรือบอทแชท จะส่งคำสั่งมา... มันจะต้องผ่านตะแกรงร่อนเดียวกันนี้ 100% เสมอ! นี่คือการลดโค้ดซ้ำซ้อน (DRY) ขั้นสูงสุดครับ!
