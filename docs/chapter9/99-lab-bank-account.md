## 📖 ภาพรวมของโปรแกรม (Program Overview)

คุณได้รับภารกิจให้สร้างระบบแกนกลาง (Core Banking) สำหรับธนาคารแห่งใหม่ ระบบนี้จะอนุญาตให้สาขา (ในที่นี้คือ Main) สร้างสมุดบัญชีเล่มใหม่ให้ลูกค้าได้ และลูกค้าสามารถสั่งฝากหรือถอนเงินได้ แต่ต้องอยู่ภายใต้ **"กฎเหล็ก"** ของธนาคาร (เช่น ห้ามถอนจนเงินติดลบ, ห้ามฝากเงินติดลบ, และแฮกเกอร์ห้ามแอบเข้าถึงตัวเลขยอดเงินโดยตรง)

### ตัวอย่างหน้าจอเมื่อรันโปรแกรมสำเร็จ

```text
================================================
          [ CORE BANKING SYSTEM ]
================================================
[ 🏦 ระบบ ] เปิดบัญชีสำเร็จ! (A-1001) ชื่อบัญชี: John Doe
[ 🏦 ระบบ ] เปิดบัญชีสำเร็จ! (A-1002) ชื่อบัญชี: Alice Smith

================================================
[ 💸 จำลองธุรกรรม (Transactions) ]
================================================
>> John Doe พยายามฝากเงิน -500 บาท
❌ [ธนาคาร] ปฏิเสธ: จำนวนเงินฝากต้องมากกว่า 0 บาท

>> John Doe ฝากเงิน 1500 บาท
✅ [A-1001] ฝากเงิน: 1500 บาท | ยอดคงเหลือ: 1500.00 บาท

>> Alice Smith พยายามถอนเงิน 2000 บาท
❌ [A-1002] ปฏิเสธ: ยอดเงินในบัญชีไม่พอ (มียอดเพียง 0 บาท)

>> John Doe ถอนเงิน 300 บาท
✅ [A-1001] ถอนเงิน: 300 บาท | ยอดคงเหลือ: 1200.00 บาท

================================================
[ 📊 สรุปสถิติประจำวัน (Daily Report) ]
================================================
จำนวนบัญชีทั้งหมดที่ถูกสร้างในระบบ: 2 บัญชี
ยอดเงินคงเหลือ John Doe (A-1001) : 1200.00 บาท
ยอดเงินคงเหลือ Alice Smith (A-1002) : 0.00 บาท
```

## 📋 กฎกติกาและเกณฑ์การประเมิน (Business Requirements)

คุณต้องวางโครงสร้างสถาปัตยกรรม (Architecture) ของคลาส `BankAccount` ให้เป๊ะตามนี้เท่านั้น:

1. **หมวดการห่อหุ้มสถานะ (State & Encapsulation):**
   - มี `public string AccountName { get; private set; }` (อ่านได้อย่างเดียว ห้ามเปลี่ยนชื่อเจ้าของบัญชีทีหลัง)
   - มี `public string AccountNumber { get; private set; }` (เลขบัญชี อ่านได้อย่างเดียว)
   - มี `private decimal _balance;` (ตัวแปรลับสุดยอด เก็บเงิน)
   - มี `public decimal Balance { get { return _balance; } }` (Property แบบอ่านได้อย่างเดียว สำหรับโชว์ยอดให้หน้าจอ)
   - มี `public static int TotalAccountsCreated { get; private set; } = 0;` (ตัวนับว่าธนาคารออกสมุดไปกี่เล่มแล้ว)

2. **หมวดพิธีกรรมแรกเกิด (Constructor):**
   - ต้องสร้าง Constructor ที่รับพารามิเตอร์ 2 ตัวคือ `name` และ `accNumber`
   - บังคับเอาพารามิเตอร์ไปใส่ใน Property ที่เกี่ยวข้อง
   - กำหนดค่าเริ่มต้นยอดเงิน `_balance = 0` เสมอ
   - สั่งให้ตัวแปรกลาง `TotalAccountsCreated` บวกเพิ่ม 1 ทุกครั้งที่มีการสร้างบัญชี!

3. **หมวดช่องทางทำธุรกรรม (Methods):**
   - เมธอด `public void Deposit(decimal amount)`: 
     - **ด่านตรวจ:** ถ้ายอด `amount` ต่ำกว่าหรือเท่ากับ 0 ให้พ่นแจ้งเตือน `❌ ปฏิเสธ` แล้วหนีออกจากฟังก์ชัน (`return`)
     - ถ้าผ่าน ให้เอาไปบวกเข้ากับ `_balance` และโชว์ข้อความ `✅ ฝากเงิน...`
   - เมธอด `public void Withdraw(decimal amount)`:
     - **ด่านตรวจที่ 1:** ถ้าถอนติดลบ ให้พ่นแจ้งเตือนปฏิเสธ
     - **ด่านตรวจที่ 2:** ถ้า `amount > _balance` ให้พ่นข้อความ `❌ ปฏิเสธ ยอดไม่พอ` แล้วหนีออก
     - ถ้าผ่านด่าน ให้หักออกจาก `_balance` และโชว์ข้อความ `✅ ถอนเงิน...`

4. **หมวดการรันระบบ (The Orchestration in Main):**
   - สร้างบัญชี "John Doe" รหัส "A-1001"
   - สร้างบัญชี "Alice Smith" รหัส "A-1002"
   - ลองทำธุรกรรมฝาก/ถอน/ทำล้มเหลว ให้เกิดข้อความคล้ายกับหน้าจอตัวอย่าง
   - ปริ้นสรุปสถิติตอนท้าย โดยใช้ `BankAccount.TotalAccountsCreated` ดึงจำนวนรวมออกมา

## 🔥 Challenge (ภารกิจท้าทายขีดจำกัด)

สำหรับวิศวกรซอฟต์แวร์ที่ต้องการเข้าสู่บริษัทระดับโลก (The Big Tech):

**ภารกิจระบบตรวจสอบเส้นทางเงิน (The Audit Logger):**
ในโลกแห่งความจริง ธนาคารต้องมี "สมุดบัญชีเดินสะพัด (Transaction History)" 
**อัปเกรดระบบของคุณดังนี้:**
1. เพิ่มตัวแปร `private string[] _history;` เอาไว้เก็บประวัติความยาวสัก 10 ช่อง (จองพื้นที่ด้วยคำสั่ง `new string[10]` ใน Constructor)
2. เพิ่มตัวแปร `private int _historyCount = 0;` เอาไว้นับว่าตอนนี้เขียนไปกี่ช่องแล้ว
3. ในเมธอด ฝาก และ ถอน (เฉพาะกรณีที่ทำงานสำเร็จเท่านั้น!) ให้สร้างข้อความประวัติ เช่น `"+ ฝาก 1500 (ยอด 1500)"` หรือ `"- ถอน 300 (ยอด 1200)"`
4. เอาข้อความประวัตินั้น ไปยัดใส่ Array ถัดไป: `_history[_historyCount] = ข้อความ;` แล้วตามด้วย `_historyCount++;`
5. **ด่านสุดท้าย:** สร้างเมธอด `public void PrintStatement()` 
   - ให้มันวนลูป `for` กวาดข้อมูลใน `_history` ออกมาปริ้นออกจอให้สวยงาม!

ถ้าคุณสามารถทำระบบ Audit Logger ด้วย Array ยัดรวมเข้าไปในสถาปัตยกรรม OOP ได้ล่ะก็... คุณหลุดพ้นจากความเป็นผู้เริ่มต้น (Beginner) ไปสู่ความเป็นระดับกลาง (Intermediate) แล้วอย่างเป็นทางการครับ! เตรียมไปรับเงินเดือนครึ่งแสนได้เลย!
