# 💻 Lab: Health Data Collector

> 💡 **เป้าหมายของ Lab นี้:** เป็นเวลาที่คุณจะได้นำความรู้เชิงทฤษฎีในเรื่อง Variables (การสร้างกล่องหน่วยความจำ), Data Types (การเลือกขนาดกล่องให้ถูกชนิด) และ Type Conversion (ศิลปะการแปลงข้อมูลอย่างปลอดภัย) มาบูรณาการเข้าด้วยกันเพื่อสร้างโปรแกรม **เก็บรวบรวมและวิเคราะห์ข้อมูลสุขภาพผู้ป่วยแบบอัตโนมัติ (Automated Health Data Analyzer)** ระดับพื้นฐาน

## 📖 ภาพรวมของโปรแกรม (Program Overview)

ในฐานะที่คุณได้รับบทบาทเป็น Software Developer ของโรงพยาบาล คุณต้องสร้าง Console Application ที่โต้ตอบกับเจ้าหน้าที่พยาบาล โปรแกรมจะทำการสัมภาษณ์ข้อมูลพื้นฐานของคนไข้ทีละขั้นตอน (ชื่อ, อายุ, น้ำหนัก, ส่วนสูง) จากนั้นโปรแกรมจะใช้ความสามารถของคณิตศาสตร์ในการประมวลผลดัชนีมวลกาย (BMI - Body Mass Index) ของคนไข้ออกมาโดยอัตโนมัติ และแสดงรายงานแบบสรุปยอดที่อ่านง่ายและสวยงามด้วยเทคนิค String Interpolation

## 📋 กฎกติกาและเกณฑ์การประเมิน (Business Requirements)

1. **การตั้งชื่อตัวแปร (Naming Convention):**
   - ตัวแปรทุกตัวที่เก็บข้อมูลคนไข้ จะต้องตั้งชื่อด้วยรูปแบบ **camelCase** อย่างเคร่งครัด
   - ชื่อตัวแปรจะต้องสื่อความหมายชัดเจน (เช่น ห้ามใช้ `x`, `y`, `z` เด็ดขาด)
2. **การจัดการชนิดข้อมูล (Type Strictness):**
   - **ชื่อ-นามสกุล:** ต้องถูกเก็บไว้ในประเภทโครงสร้างที่รองรับข้อความยาวๆ
   - **อายุ:** เนื่องจากมนุษย์ไม่มีอายุเป็นจุดทศนิยม และไม่เกิน 255 ปี (ถ้าอยากประหยัด Memory ลองใช้ type ให้เหมาะสม) แต่เพื่อความง่ายอนุญาตให้ใช้ตัวเลขจำนวนเต็มมาตรฐานได้
   - **น้ำหนัก และ ส่วนสูง:** ต้องมีจุดทศนิยมได้ ดังนั้นต้องใช้กล่องข้อมูลชนิดทศนิยมทั่วไปที่กินทรัพยากรประมวลผลน้อย (Hint: Scientific calculation type)
3. **การแปลงข้อมูล (Safe Parsing Requirement):**
   - การรับค่าตัวเลขทั้งหมด (อายุ, น้ำหนัก, ส่วนสูง) จะต้องทำการแปลงข้อมูลผ่าน **`.Parse()`** เพื่อเปลี่ยนข้อความดิบให้เป็นตัวเลขที่นำไปคำนวณได้
4. **สูตรคณิตศาสตร์ (Algorithmic Logic):**
   - การคำนวณ BMI ใช้สูตรสากลคือ: `BMI = น้ำหนัก(kg) / [ส่วนสูง(m) * ส่วนสูง(m)]`
   - *จุดดักควาย (Gotcha!):* สังเกตว่าตอนรับ Input ส่วนสูง โปรแกรมขอรับเป็น "เซนติเมตร" แต่สูตร BMI ของแพทย์ต้องการส่วนสูงเป็น "เมตร" คุณต้องหาทางแปลงเซนติเมตรเป็นเมตรในโค้ดด้วยตัวเอง!
5. **การจัดการการแสดงผล (Output Formatting):**
   - ผลลัพธ์ BMI ที่คำนวณได้มักจะมีทศนิยมยาวเป็นหางว่าว (เช่น 23.82882882...) คุณต้องหาวิธีจำกัดการแสดงผลให้เหลือเพียงทศนิยม 2 ตำแหน่งเท่านั้น

## 📝 ขั้นตอนการพัฒนา (Step-by-Step Implementation)

### ขั้นตอนที่ 1: วางโครงร่างโปรเจกต์และสร้าง UI ส่วนหัว
- [ ] สร้าง Directory โฟลเดอร์ใหม่ และรันคำสั่ง `dotnet new console` เพื่อเริ่มต้นโปรเจกต์
- [ ] ลบโค้ด Hello World ทิ้งให้หมด
- [ ] เขียนคำสั่ง `Console.WriteLine()` เพื่อสร้างกล่องข้อความต้อนรับของโรงพยาบาลตามตัวอย่างให้สวยงาม

### ขั้นตอนที่ 2: สร้างระบบสัมภาษณ์ประวัติส่วนตัว (Personal Info)
- [ ] แสดงข้อความหัวข้อ `[ระบุข้อมูลประจำตัวคนไข้]`
- [ ] ใช้ `Console.Write` ถามชื่อ และรับค่าด้วย `Console.ReadLine()` ไปเก็บไว้ในตัวแปรชนิดสายอักขระ (ตั้งชื่อตัวแปรให้เหมาะสม)
- [ ] ถามอายุ และรับค่าผ่าน `ReadLine` จากนั้น **ต้องทำการแปลง (Parse)** ให้กลายเป็นตัวเลขจำนวนเต็ม เพื่อเก็บใส่ตัวแปรให้ถูกต้อง

### ขั้นตอนที่ 3: สร้างระบบสัมภาษณ์สรีระ (Physical Metrics)
- [ ] แสดงข้อความหัวข้อ `[ระบุข้อมูลสรีระร่างกาย]`
- [ ] ถามน้ำหนัก (รองรับทศนิยม) แปลงข้อความที่ผู้ใช้กรอกผ่าน `double.Parse()` และเก็บใส่ตัวแปร
- [ ] ถามส่วนสูงเป็น "เซนติเมตร" (รองรับทศนิยม) และทำการแปลงผ่าน `double.Parse()` เช่นเดียวกัน

### ขั้นตอนที่ 4: กระบวนการคิดคำนวณในสมองคอมพิวเตอร์ (The BMI Algorithm)
ขั้นตอนนี้คือแก่นแท้ของซอฟต์แวร์ เพราะเป็นจุดที่เกิด "การสร้างมูลค่าเพิ่ม (Value Creation)" ให้กับข้อมูลดิบ
- [ ] **Data Transformation:** สร้างตัวแปรใหม่เพื่อเก็บค่า "ส่วนสูงในหน่วยเมตร" (ทบทวนความรู้คณิตศาสตร์ประถม: 1 เมตร = 100 เซนติเมตร ดังนั้น ถ้ามีหน่วยเซนติเมตร จะทำยังไงให้เป็นเมตร?)
- [ ] **Business Logic:** สร้างตัวแปรมารอรับค่า BMI (ชนิด double) แล้วลงมือเขียนสูตรคณิตศาสตร์: เอาน้ำหนักตั้ง หารด้วย (ส่วนสูงหน่วยเมตร คูณ ส่วนสูงหน่วยเมตร)
  *(คำเตือน: วงเล็บมีความสำคัญในคณิตศาสตร์คอมพิวเตอร์ ทำในวงเล็บให้เสร็จก่อนเสมอ!)*

### ขั้นตอนที่ 5: นำเสนอรายงานฉบับสมบูรณ์ (Report Generation)
- [ ] สร้างเส้นกั้นกรอบรายงานให้ดูเป็นทางการ
- [ ] สร้างรหัสอ้างอิงปลอมๆ โดยใช้การนำสายอักขระและตัวแปรอายุมาต่อกัน (เช่น `"PATIENT-" + age`)
- [ ] ใช้ **String Interpolation (`$""`)** ในการแสดงข้อความทั้งหมด
- [ ] **จุดสำคัญ:** ตอนแสดงผลตัวแปร BMI ให้ใส่ฟอร์แมตติ้งโค้ด `{bmiVariable:F2}` เพื่อสั่งให้ C# ช่วยทำการปัดเศษและล็อคให้แสดงทศนิยมเพียงแค่ 2 ตำแหน่งตามเกณฑ์ทางการแพทย์

### ขั้นตอนที่ 6: กระบวนการตรวจสอบคุณภาพ (Quality Assurance Testing)
ในฐานะนักพัฒนา คุณต้องทดสอบโปรแกรมของคุณว่ามันให้ผลลัพธ์ที่ถูกต้องหรือไม่
- [ ] ทดลองรันโปรแกรม
- [ ] กรอกน้ำหนัก: `65`, ส่วนสูง: `170`
- [ ] ถ้าโปรแกรมทำงานถูกต้องร้อยเปอร์เซ็นต์ มันควรจะแสดงผลค่า BMI เท่ากับ **`22.49`**
- [ ] ถ้าผลลัพธ์เป็น 0.0022 หรือตัวเลขแปลกประหลาด แปลว่าในขั้นตอนที่ 4 ของคุณมีปัญหาเรื่องการใส่วงเล็บคณิตศาสตร์ หรือการแปลงหน่วยผิดพลาด ให้กลับไปแก้ไขโค้ด!

## 🔥 Challenge (ภารกิจท้าทายขีดจำกัด)

สำหรับวิศวกรซอฟต์แวร์ผู้หิวกระหายความสมบูรณ์แบบ ลองอัปเกรดโปรแกรมของคุณด้วยภารกิจเหล่านี้:

**ภารกิจที่ 1: ระบบกันกระแทกขั้นสุดยอด (Bulletproof Inputs)**
หากนางพยาบาลเกิดอาการง่วงนอน แล้วเผลอพิมพ์น้ำหนักคนไข้ว่า "หกสิบห้า" แทนที่จะพิมพ์ "65" โปรแกรมของคุณที่ใช้ `.Parse()` จะเกิดอาการเส้นเลือดในสมองแตกและ Crash ค้างตายไปทันที!
**คำท้า:** จงเปลี่ยนกระบวนการรับค่าอายุ น้ำหนัก และส่วนสูงทั้งหมด ให้ไปใช้กลไกการแปลงตัวเลขแบบไร้รอยต่อ **`TryParse`** (ที่คุณเพิ่งเรียนไปในหัวข้อสุดท้าย) หากนางพยาบาลกรอกผิด ให้เก็บค่าเริ่มต้นเป็น 0 แทนที่จะปล่อยให้โปรแกรมพัง

**ภารกิจที่ 2: ระบบวิเคราะห์อัตราการเผาผลาญ (BMR Calculator Expansion)**
ค่า BMI อย่างเดียวอาจจะไม่พอสำหรับนักโภชนาการ
**คำท้า:** ให้เพิ่มการคำนวณ **BMR (Basal Metabolic Rate - อัตราการเผาผลาญพลังงานพื้นฐาน)** เข้าไปในโปรแกรม
- สูตรของ BMR (แบบง่ายสำหรับผู้ชาย) คือ: `66 + (13.7 x น้ำหนักกิโล) + (5 x ส่วนสูงเซนติเมตร) - (6.8 x อายุ)`
- โปรดสังเกตว่าสูตรนี้ใช้ส่วนสูงเป็น "เซนติเมตร" (ไม่ต้องแปลงเป็นเมตรแบบ BMI)
- สร้างตัวแปรใหม่, คำนวณตามสูตร, และนำไปแสดงผลต่อจากค่า BMI ในหน้ารายงาน!

*(ถ้าคุณฝ่าฟัน Challenge เหล่านี้ไปได้ คุณก็มีความพร้อมเต็มเปี่ยมในการก้าวเข้าสู่บทถัดไปของการเขียนโปรแกรม ที่เต็มไปด้วยเงื่อนไขและการตัดสินใจที่ซับซ้อนยิ่งกว่าเดิม!)*
