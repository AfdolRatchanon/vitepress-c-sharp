# Variables & Declaration (ตัวแปรและการประกาศ)

> 💡 **เป้าหมาย:** เข้าใจกระบวนการจัดการหน่วยความจำเบื้องต้นของคอมพิวเตอร์ เรียนรู้วิธีจองพื้นที่เพื่อเก็บข้อมูลด้วย "ตัวแปร (Variables)" กฎการตั้งชื่อระดับมืออาชีพ และพื้นฐานการรับส่งข้อมูลกับผู้ใช้งานผ่าน Console

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. ความลับหลังม่าน: ตัวแปร (Variable) ทำงานอย่างไรในหน่วยความจำ?

หลายคนมักจะเปรียบเทียบตัวแปรว่าเป็น **"กล่องเก็บของ"** ซึ่งเป็นคำอธิบายที่ดีสำหรับผู้เริ่มต้น แต่ในฐานะที่เราจะเป็นวิศวกรซอฟต์แวร์ เราต้องมองให้ลึกไปถึงระดับฮาร์ดแวร์

เมื่อคุณเขียนโค้ดประกาศตัวแปร สิ่งที่เกิดขึ้นจริงๆ คือคอมพิวเตอร์จะวิ่งไปหา **RAM (Random Access Memory)** หาพื้นที่ว่างๆ จองพื้นที่นั้นไว้ ตั้งชื่อแปะป้ายให้มัน และเอาข้อมูลของคุณไปใส่ไว้ในพื้นที่นั้น

```text
แผนภาพจำลองการทำงานของ RAM (Memory Allocation)

 RAM Memory Blocks
 ┌────────┬────────┬────────┬────────┬────────┐
 │ 0x00A1 │ 0x00A2 │ 0x00A3 │ 0x00A4 │ 0x00A5 │  <-- ตำแหน่งหน่วยความจำ (Address)
 ├────────┼────────┼────────┼────────┼────────┤
 │        │  "John"│   18   │        │        │  <-- ข้อมูลที่ถูกเก็บ (Value)
 └────────┴───▲────┴───▲────┴────────┴────────┘
              │        │
         [ชื่อ: name] [ชื่อ: age]                 <-- ชื่อตัวแปร (Variable Name)
         [ชนิด: string] [ชนิด: int]               <-- ชนิดข้อมูล (Data Type)
```

**ทำไมต้องระบุ "ชนิดข้อมูล (Data Type)"?**
ในภาษา C# ซึ่งเป็น **Strongly Typed Language** คุณจำเป็นต้องบอกคอมพิวเตอร์อย่างชัดเจนว่าคุณจะเก็บอะไร (เช่น ตัวเลข, ข้อความ) เพราะ:
1. คอมพิวเตอร์จะได้รู้ว่าต้องจองพื้นที่ RAM ขนาดเท่าไหร่ (ตัวเลขเล็กๆ ใช้พื้นที่น้อย ข้อความยาวๆ ใช้พื้นที่มาก)
2. ป้องกันโปรแกรมเมอร์เผลอนำตัวเลขไปผสมกับข้อความแบบมั่วซั่ว ซึ่งจะทำให้โปรแกรมทำงานผิดพลาด (Type Safety)

### 2. ไวยากรณ์การประกาศตัวแปร (Declaration Syntax)

การสร้างตัวแปรใน C# มีองค์ประกอบที่ขาดไม่ได้ 3 ส่วน:
1. `Data Type` (ชนิดข้อมูล)
2. `Identifier` (ชื่อตัวแปร)
3. `Assignment` (การกำหนดค่า bằngเครื่องหมาย `=`)

**รูปแบบมาตรฐาน:**
```csharp
DataType variableName = value;
```

คุณสามารถแยกขั้นตอน "การจองพื้นที่" และ "การใส่ข้อมูล" ออกจากกันได้ หรือจะทำพร้อมกันเลยก็ได้:

```csharp
// วิธีที่ 1: จองกล่องก่อน แล้วค่อยเอาของใส่ทีหลัง (Declaration then Assignment)
string playerName;       // จองพื้นที่ชื่อ playerName ชนิดข้อความ
playerName = "ฮีโร่007"; // นำคำว่า "ฮีโร่007" ไปใส่ในกล่องนั้น

// วิธีที่ 2: จองกล่องพร้อมใส่ของทันที (Initialization) - นิยมใช้มากที่สุด
int playerLevel = 1;

// การเปลี่ยนแปลงค่า (Re-assignment) - ค่าเก่าจะถูกทำลายทิ้งทันที!
playerLevel = 2; // ตอนนี้ค่า 1 หายไปจาก RAM แล้ว กลายเป็น 2 แทน
```

### 3. Naming Conventions (กฎและมารยาทในการตั้งชื่อ)

การตั้งชื่อตัวแปรเป็นศิลปะอย่างหนึ่ง โค้ดที่ดีต้องอ่านแล้วเข้าใจทันทีเหมือนอ่านหนังสือภาษาอังกฤษ

**กฎข้อบังคับของภาษา C# (ถ้าฝ่าฝืนคือ Error รันไม่ผ่าน):**
- ห้ามขึ้นต้นด้วยตัวเลข (เช่น `1stPlayer` ❌)
- ห้ามเว้นวรรค (เช่น `player name` ❌)
- ห้ามใช้สัญลักษณ์พิเศษ ยกเว้นขีดล่าง `_` (เช่น `player@name` ❌)
- ห้ามใช้คำสงวนของภาษา (เช่น `int`, `string`, `class`, `if` ❌)

**มาตรฐานอุตสาหกรรม (Best Practices - ถ้าฝ่าฝืนรันผ่าน แต่เพื่อนร่วมงานจะโกรธ):**
ในโลกของ C# เราใช้มาตรฐานการตั้งชื่อหลักๆ 2 แบบ:

| รูปแบบ | วิธีการเขียน | ใช้ตอนไหนใน C#? | ตัวอย่าง |
| :--- | :--- | :--- | :--- |
| **camelCase** | คำแรกพิมพ์เล็ก คำถัดไปขึ้นต้นด้วยพิมพ์ใหญ่ | ใช้กับ **ตัวแปรทั่วไป (Local Variables)** และ **พารามิเตอร์** | `studentName`, `totalScore`, `isAlive` |
| **PascalCase** | ขึ้นต้นด้วยพิมพ์ใหญ่ทุกคำ | ใช้กับ **ชื่อ Class**, **ชื่อ Method**, **ชื่อไฟล์** | `Program`, `CalculateGrade`, `StudentInfo` |

*ในบทนี้เราโฟกัสที่การสร้างตัวแปรทั่วไป ดังนั้นเราจะใช้ **camelCase** เป็นหลัก*

### 4. การทำงานกับ Input/Output อย่างมืออาชีพ

คุณรู้จัก `Console.WriteLine()` ไปแล้วสำหรับการแสดงผล (Output)
ทีนี้เราจะมาเรียนรู้คำสั่งรับข้อมูล (Input) ซึ่งคือ **`Console.ReadLine()`**

**กลไกการทำงานของ `Console.ReadLine()` (Blocking Call):**
เมื่อโปรแกรมทำงานมาถึงคำสั่ง `ReadLine()` โปรแกรมจะ **หยุดชะงัก (Block)** และรอคอยอย่างไม่มีกำหนด จนกว่าผู้ใช้จะพิมพ์อะไรบางอย่างลงไปใน Terminal แล้วกดปุ่ม `Enter` 

```text
Flow การรับ Input:

  โปรแกรมรัน
      │
      ▼
Console.Write("ชื่อของคุณ: ");
      │
      ▼
string name = Console.ReadLine();  <--- [ โปรแกรมหยุดรอที่บรรทัดนี้ ! ]
      │                                    ^
      │                                    │ (ผู้ใช้พิมพ์ "John" แล้วกด Enter)
      ▼
Console.WriteLine("สวัสดี " + name);
```

**ข้อควรระวังขั้นวิกฤติ:** ข้อมูลทุกสิ่งที่ได้กลับมาจาก `Console.ReadLine()` จะมี Data Type เป็น **`string` (ข้อความ) เสมอ** ต่อให้ผู้ใช้จะพิมพ์เลข `123` คอมพิวเตอร์ก็จะมองว่าเป็นข้อความ `"123"` (ซึ่งคุณจะเอาไปบวกลบคูณหารไม่ได้ จนกว่าจะแปลงร่างมันในหัวข้อถัดๆ ไป)

### 5. String Interpolation (การแทรกตัวแปรในข้อความยุคใหม่)

ในบทแรก คุณใช้วิธีนำข้อความมาต่อกันด้วยเครื่องหมายบวก `+` (เรียกว่า String Concatenation)
```csharp
string name = "John";
int age = 18;
Console.WriteLine("ฉันชื่อ " + name + " อายุ " + age + " ปี");
```

แต่วิธีนี้ทำให้โค้ดอ่านยาก และเกิดบั๊กเรื่องเว้นวรรคผิดพลาดได้ง่าย 
ใน C# สมัยใหม่ เรามีเทคนิคที่เรียกว่า **String Interpolation** เพียงแค่เติมเครื่องหมาย `$` ไว้หน้าสุดของเครื่องหมายคำพูด คุณจะสามารถยัดตัวแปรเข้าไปในวงเล็บปีกกา `{}` ได้เลย!

```csharp
Console.WriteLine($"ฉันชื่อ {name} อายุ {age} ปี"); // สวยงามและอ่านง่ายกว่ามหาศาล!
```

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่างที่ 1: การประกาศตัวแปรและการจัดการ RAM อย่างเห็นภาพ

โค้ดนี้จะสาธิตการเปลี่ยนแปลงค่าในหน่วยความจำแบบเป็นลำดับขั้นตอน (Sequential)

::: code-group
```csharp [Program.cs]
using System;

namespace VariableDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== การติดตามค่าของตัวแปร ===");

            // 1. สร้างตัวแปรเริ่มต้น (Initial State)
            string currentWeapon = "ดาบไม้";
            int attackPower = 5;

            // แสดงสถานะเริ่มต้น (ใช้ String Interpolation)
            Console.WriteLine($"[Lv.1] คุณถือ: {currentWeapon} (พลังโจมตี: {attackPower})");

            // 2. จำลองสถานการณ์อัปเกรดอาวุธ
            // เราจะทำการ Re-assign ค่าใหม่ทับค่าเก่า
            currentWeapon = "ดาบเหล็กกล้า";
            attackPower = 25;

            // แสดงสถานะที่เปลี่ยนไป
            Console.WriteLine($"[Lv.5] คุณถือ: {currentWeapon} (พลังโจมตี: {attackPower})");

            // 3. การโอนถ่ายข้อมูลระหว่างตัวแปร
            // สร้างกล่องใหม่ และเอาของจากกล่องเก่ามาคัดลอก (Copy) ใส่กล่องใหม่
            int baseDamage = attackPower;
            
            // อัปเกรดพลังให้ attackPower แต่ไม่กระทบ baseDamage
            attackPower = attackPower + 10; 

            Console.WriteLine($"[Lv.10] พลังโจมตีปัจจุบัน: {attackPower} (พลังพื้นฐานตอน Lv.5 คือ: {baseDamage})");
        }
    }
}
```
:::

**Expected Output:**
```
=== การติดตามค่าของตัวแปร ===
[Lv.1] คุณถือ: ดาบไม้ (พลังโจมตี: 5)
[Lv.5] คุณถือ: ดาบเหล็กกล้า (พลังโจมตี: 25)
[Lv.10] พลังโจมตีปัจจุบัน: 35 (พลังพื้นฐานตอน Lv.5 คือ: 25)
```

### ตัวอย่างที่ 2: ระบบลงทะเบียนแบบ Interactive

โค้ดนี้สาธิตการโต้ตอบกับผู้ใช้ด้วย `ReadLine` และการจัดรูปแบบ `Output` อย่างมืออาชีพ

::: code-group
```csharp [Program.cs]
using System;

namespace RegistrationSystem
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=======================================");
            Console.WriteLine("  ยินดีต้อนรับสู่ระบบสมัคสมาชิก C# Club  ");
            Console.WriteLine("=======================================");

            // ขั้นตอนที่ 1: ขอข้อมูลชื่อ
            // สังเกตว่าเราใช้ Write (ไม่มี Line) เพื่อให้เคอร์เซอร์พิมพ์อยู่บรรทัดเดียวกัน
            Console.Write("กรุณากรอกชื่อจริงของคุณ: ");
            string firstName = Console.ReadLine();

            // ขั้นตอนที่ 2: ขอข้อมูลนามสกุล
            Console.Write("กรุณากรอกนามสกุล: ");
            string lastName = Console.ReadLine();

            // ขั้นตอนที่ 3: ขออีเมล
            Console.Write("กรุณาระบุ Email: ");
            string emailAddress = Console.ReadLine();

            Console.WriteLine("\n[กำลังบันทึกข้อมูลลงฐานข้อมูล...]\n");

            // ขั้นตอนที่ 4: สรุปข้อมูลโดยใช้ String Interpolation
            Console.WriteLine("--- สรุปข้อมูลสมาชิก ---");
            Console.WriteLine($"ชื่อ-นามสกุล : {firstName} {lastName}");
            Console.WriteLine($"อีเมลติดต่อ   : {emailAddress}");
            Console.WriteLine("----------------------");
            Console.WriteLine($"ขอบคุณคุณ {firstName} ที่ร่วมเป็นส่วนหนึ่งกับเรา!");
        }
    }
}
```
:::

**Expected Output (กรณีผู้ใช้กรอกข้อมูล):**
```
=======================================
  ยินดีต้อนรับสู่ระบบสมัคสมาชิก C# Club  
=======================================
กรุณากรอกชื่อจริงของคุณ: แฮรี่
กรุณากรอกนามสกุล: พอตเตอร์
กรุณาระบุ Email: harry@hogwarts.edu

[กำลังบันทึกข้อมูลลงฐานข้อมูล...]

--- สรุปข้อมูลสมาชิก ---
ชื่อ-นามสกุล : แฮรี่ พอตเตอร์
อีเมลติดต่อ   : harry@hogwarts.edu
----------------------
ขอบคุณคุณ แฮรี่ ที่ร่วมเป็นส่วนหนึ่งกับเรา!
```

ให้ระบุว่าบรรทัดไหนผิด ผิดเพราะอะไร และเขียนแก้ไขให้ถูกต้อง

```csharp
string 1stHeroName = "IronMan";
int hero Level = 50;
string superPower;
superPower = "บินได้"
string currentMission = 'ปกป้องโลก';
```

### **โจทย์ที่ 2: ระบบตอบกลับแชทบอท (Chatbot Intro)**
เขียนโปรแกรมแชทบอทง่ายๆ ที่:
1. ให้บอททักทายและถามชื่อผู้ใช้
2. รับชื่อผู้ใช้มาเก็บในตัวแปร `userName`
3. ถามว่าผู้ใช้มาจากจังหวัดอะไร
4. รับจังหวัดมาเก็บในตัวแปร `provinceName`
5. ตอบกลับด้วย String Interpolation (`$""`) ว่า `"โอ้โห! คุณ [ชื่อ] เดินทางมาจากจังหวัด [จังหวัด] ไกลมากเลย ยินดีที่ได้รู้จักครับ!"`

::: details 💡 คำใบ้และแนวทาง (Hints)
- **สำหรับโจทย์ 1:** 
  - ชื่อตัวแปรห้ามขึ้นต้นด้วยตัวเลข
  - ห้ามมีการเว้นวรรคในชื่อตัวแปร
  - ทุกบรรทัดคำสั่งต้องจบด้วยเซมิโคลอน `;`
  - ข้อความยาวๆ (string) ต้องใช้เครื่องหมาย `" "` (Double quote) ไม่ใช่ `' '` (Single quote ซึ่งเป็นของ char)
- **สำหรับโจทย์ 2:** ใช้ `Console.Write` ถามคำถาม และใช้ `Console.ReadLine` รับข้อมูลมาใส่ตัวแปร จากนั้นใช้ `Console.WriteLine($"ข้อความ {ตัวแปร}");` ในการแสดงผล
:::

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: การสลับค่าตัวแปร (Variable Swapping)**

สมมติว่าคุณมีกล่องใส่ของอยู่ 2 กล่อง:
- `string leftHand = "ดาบ";`
- `string rightHand = "โล่";`

**ความท้าทาย:** ให้คุณเขียนโค้ดเพื่อ "สลับของ" ระหว่างสองมือนี้ (ให้มือซ้ายถือโล่ และมือขวาถือดาบ) 
*ข้อห้าม:* ห้ามพิมพ์คำว่า `"ดาบ"` หรือ `"โล่"` ซ้ำลงไปในโค้ดอีก (ห้ามฮาร์ดโค้ด) คุณต้องใช้การย้ายข้อมูลผ่านตัวแปรเท่านั้น 

*คำใบ้:* ถ้าคุณมีแก้วน้ำโค้ก และแก้วน้ำสไปรท์ คุณจะสลับน้ำในแก้วทั้งสองโดยไม่ให้น้ำปนกันได้อย่างไร? (คุณจำเป็นต้องใช้แก้วเปล่าใบที่ 3 มาช่วยไหม?) ลองแปลงหลักการนั้นมาเป็นโค้ดดู!

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** ทำไม C# ถึงบังคับให้เราตั้งชื่อตัวแปรให้ตรงตาม Data Type เสมอ? (เช่น ห้ามเอา "ข้อความ" ใส่ในตัวแปร `int`) สิ่งนี้มีประโยชน์อย่างไรเมื่อเทียบกับภาษาอย่าง Python หรือ JavaScript ที่จับของใส่ได้มั่วๆ?
**แนวคำตอบ:** C# เป็นภาษาแบบ Strongly Typed ข้อดีคือมันช่วย "ป้องกันบั๊กตั้งแต่ตอนเขียนโค้ด (Compile-time)" สมมติว่าคุณเขียนระบบคำนวณเงินเดือน ถ้าภาษาไม่เข้มงวด คุณอาจเผลอเอาข้อความไปรวมกับตัวเลข แล้วระบบอาจจะทำงานผิดพลาดไปจนถึงมือลูกค้า (Runtime Error) แต่ใน C# มันจะฟ้องตัวแดงใน VS Code และไม่ยอมให้คุณ Build โปรแกรมตั้งแต่แรก ทำให้โปรแกรมมีเสถียรภาพสูงมาก เหมาะกับระบบขนาดใหญ่ (Enterprise Systems)

**คำถาม 2:** โค้ดต่อไปนี้ทำงานได้หรือไม่? ถ้าได้ ผลลัพธ์คืออะไร? ถ้าไม่ได้ เพราะอะไร?
```csharp
string myName;
Console.WriteLine(myName);
```
**แนวคำตอบ:** **ทำงานไม่ได้ (Compile Error)!** C# มีกฎความปลอดภัยอีกข้อที่เรียกว่า "Definite Assignment" หมายความว่า "ตัวแปรทุกตัว ต้องถูกกำหนดค่าก่อนนำไปใช้งานเสมอ" ในตัวอย่างนี้ มีการจองกล่อง `myName` ไว้ แต่ยังไม่ได้เอาอะไรใส่เลย (เป็นกล่องเปล่า) คอมพิวเตอร์จึงปฏิเสธที่จะนำกล่องเปล่านั้นไปสั่งพิมพ์ เพื่อป้องกันการเข้าถึงข้อมูลขยะ (Garbage Data) ในหน่วยความจำนั่นเอง
