# เฉลย: Type Conversion & Casting

## 🎯 เฉลย Mini Exercise

### **โจทย์ที่ 1: เส้นทางแห่งการสูญเสียข้อมูล**
**โจทย์ที่ให้มา:**
```csharp
double startingPoint = 5.8;
int step1 = (int)startingPoint;
double step2 = step1;
int finalResult = (int)step2 + 2;
Console.WriteLine(finalResult);
```

**การทดเลขในใจทีละบรรทัด (Execution Trace):**
1. `startingPoint` ถูกสร้างเป็น `double` มีค่า **`5.8`**
2. บรรทัดที่ 2: บังคับ Explicit Cast ยัด `5.8` ลงไปใน `int` กฎการแปลงคือ "หั่นจุดทศนิยมทิ้งทั้งหมดโดยไม่มีการปัดเศษ" ดังนั้น `step1` จะมีค่า **`5`** เท่านั้น! (เศษ .8 หายสาบสูญไปในหลุมดำเรียบร้อย)
3. บรรทัดที่ 3: เกิดกระบวนการ Implicit Cast เอาค่า `5` จาก `int` ไปใส่กล่องใหญ่อย่าง `double` มันก็แค่วางไปเฉยๆ ค่าของ `step2` จึงกลายเป็น **`5.0`** (เศษ .8 ที่ตายไปแล้ว ไม่มีทางฟื้นคืนชีพกลับมาได้!)
4. บรรทัดที่ 4: บังคับคาสต์ `5.0` เป็น `int` ก็จะได้ `5` แล้วนำไปบวกกับ `2` ผลลัพธ์สุดท้ายที่เก็บใน `finalResult` คือ **`7`**
5. โปรแกรมพิมพ์ **`7`** ออกมาบนหน้าจอ

### **โจทย์ที่ 2: แปลงโค้ดมรณะ (The Crash Fixer)**
**โจทย์ที่ให้มา:** โค้ดที่ใช้ `int.Parse()` ทื่อๆ แล้วทำให้โปรแกรมพังถ้ายูสเซอร์กรอกข้อมูลขยะ

**กระบวนการแก้ปัญหา (Refactoring Process):**
เราต้องเปลี่ยนจากการใช้ปืนใหญ่ (Parse ที่ทำลายล้างโปรแกรมถ้าล้มเหลว) ไปเป็นตาข่ายนิรภัย (TryParse) เพื่อรับแรงกระแทกจากความโง่เขลาของผู้ใช้แทน

**โค้ดที่ได้รับการปกป้องแล้ว (Defensive Code):**
```csharp
Console.WriteLine("=== ระบบสุ่มแจกรางวัล (Crash-Proof Version) ===");
Console.Write("กรุณาป้อนรหัสลุ้นรางวัล (ตัวเลขเท่านั้น): ");
string userInput = Console.ReadLine();

// 1. สร้างตัวแปร int ว่างๆ มารอรับของจากการแปลง (ถ้าแปลงไม่สำเร็จ ตัวแปรนี้จะถูกเซ็ตเป็น 0 อัตโนมัติ)
int luckyNumber;

// 2. เรียกใช้ TryParse โยน userInput เข้าไป และสั่งให้ออกผลลัพธ์มาที่ luckyNumber
// TryParse จะรีเทิร์นค่า true/false มาให้ เราจึงจับใส่ตัวแปร bool ไว้ตรวจสอบ
bool isValid = int.TryParse(userInput, out luckyNumber);

// 3. ใช้การตัดสินใจ (ซึ่งจะเรียนเจาะลึกในบทถัดไป) หรือใช้ Ternary Operator
if (isValid)
{
    // ถ้าแปลงได้สวยงาม ก็ทำงานต่อ
    Console.WriteLine($"ยินดีด้วย! เลขที่คุณส่งชิงโชคคือ [ {luckyNumber} ]");
    Console.WriteLine("ระบบกำลังนำเลขของคุณเข้าเครื่องจับฉลาก...");
}
else
{
    // ถ้าแปลงไม่ได้ ก็แค่เตือนผู้ใช้ ไม่ปล่อยให้แอปพังคามือ
    Console.WriteLine("❌ ERROR: สิ่งที่คุณกรอกมาไม่ใช่ตัวเลขครับ!");
    Console.WriteLine("กรุณารันโปรแกรมใหม่ และกรอกเฉพาะตัวเลขเท่านั้น");
}
```

## 🔥 เฉลย Challenge (มนตร์ตราแห่งการจัดรูปแบบ String Formatting Magic)

**โจทย์:** ค้นหาและใช้ Standard Numeric Format Strings แปลงตัวเลข `1250000.55` ให้เป็นข้อความ 3 รูปแบบโดยไม่ต้องบวกอักษรเพิ่มเอง
**เฉลยและวิธีทำ:**

นี่คือความลับของ C# ที่น้อยคนจะใช้เป็น แต่ถ้าคุณใช้มัน คุณจะประหยัดเวลาการแต่งหน้าต่าง UI ไปได้มหาศาล โค้ดของ C# มีระบบ Formatting อัจฉริยะฝังมาใน `.ToString()` เรียบร้อยแล้ว!

```csharp
Console.WriteLine("=================================");
Console.WriteLine("  📈 REPORT FORMATTING MACHINE 📈  ");
Console.WriteLine("=================================");

double money = 1250000.55;

Console.WriteLine($"ค่าดิบในระบบ (Raw Value): {money}");
Console.WriteLine("---------------------------------");

// ภารกิจแบบที่ 1: สกุลเงิน (Currency)
// ตัว C ย่อมาจาก Currency จะเติมสัญลักษณ์เงินของภูมิภาค (เช่น ฿ หรือ $) และใส่คอมม่าให้สวยงาม
string format1 = money.ToString("C");
Console.WriteLine($"รูปแบบสกุลเงิน (C) : {format1}");

// ภารกิจแบบที่ 2: ตัวเลขกลมๆ แบ่งหลักพัน (Number - No Decimal)
// ตัว N ย่อมาจาก Number, เลข 0 ด้านหลังคือระบุว่า "ขอทศนิยม 0 ตำแหน่ง" 
// (ถ้าเป็น N2 คือเอาทศนิยม 2 ตำแหน่ง)
// สังเกตว่าระบบจะแอบ 'ปัดเศษขึ้น' ให้ด้วยจาก .55 กลายเป็นปัดหลักหน่วย!
string format2 = money.ToString("N0");
Console.WriteLine($"รูปแบบจำนวนเต็ม (N0): {format2}");

// ภารกิจแบบที่ 3: เปอร์เซ็นต์ (Percentage)
// ตัว P ย่อมาจาก Percentage ระบบจะเอาตัวเลขของคุณไปคูณ 100 ให้อัตโนมัติ แล้วเติมเครื่องหมาย % ต่อท้ายให้ทันที
// ตัวอย่างเช่น ถ้าคุณมี 0.15 แล้วสั่ง ToString("P") มันจะกลายเป็น 15.00%
string format3 = money.ToString("P");
Console.WriteLine($"รูปแบบเปอร์เซ็นต์ (P) : {format3}");

Console.WriteLine("=================================");
```

**ผลลัพธ์ที่จะแสดงบนหน้าจอ:**
*(หมายเหตุ: สัญลักษณ์เงิน ฿ หรือ $ ขึ้นอยู่กับว่าคอมพิวเตอร์ของคุณตั้งภาษาของระบบ Windows ไว้ที่ประเทศอะไร)*
```text
=================================
  📈 REPORT FORMATTING MACHINE 📈
=================================
ค่าดิบในระบบ (Raw Value): 1250000.55
---------------------------------
รูปแบบสกุลเงิน (C) : ฿1,250,000.55
รูปแบบจำนวนเต็ม (N0): 1,250,001
รูปแบบเปอร์เซ็นต์ (P) : 125,000,055.00 %
=================================
```

**เคล็ดลับ:** คุณสามารถใช้รูปแบบเหล่านี้แบบย่อใน String Interpolation (เครื่องหมาย `$`) ได้เลย โดยไม่ต้องพิมพ์คำว่า `.ToString` ให้ยาวเหยียด เช่น `$"จำนวนเงิน: {money:C}"` หรือ `$"อัตราดอกเบี้ย: {interest:P}"` ซึ่งเป็นที่นิยมทำกันมากที่สุดในหมู่โปรแกรมเมอร์ C# หน้าใหม่และหน้าเก่า!
