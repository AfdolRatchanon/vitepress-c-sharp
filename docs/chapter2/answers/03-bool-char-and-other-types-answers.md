# เฉลย: bool, char และ Other Types

## 🎯 เฉลย Mini Exercise

### **โจทย์ที่ 1: นักวิเคราะห์ตรรกะ (Logic Analyst)**
เป้าหมายคือการแปลงภาษามนุษย์ให้เป็นตัวแปร `bool` ที่ชื่อสื่อความหมายได้อย่างถูกต้องตามมาตรฐาน (Naming Convention)

**เฉลย:**
1. **"สมาชิกคนนี้ยังไม่ได้ทำการยืนยันอีเมล"**
   - ตัวแปรต้องถามถึงสถานะ "ยืนยันแล้วหรือยัง"
   - `bool isEmailVerified = false;` (หรือ `hasVerifiedEmail = false;`)
2. **"เอกสารสลิปโอนเงินแนบมาเรียบร้อยแล้ว"**
   - ตัวแปรถามถึง "มีสลิปไหม"
   - `bool hasPaymentSlip = true;` (หรือ `isPaymentAttached = true;`)
3. **"ผู้ใช้อนุญาตให้ระบบส่งรับข่าวสารทาง SMS ได้"**
   - ตัวแปรถามถึง "สามารถรับข่าวสารได้ไหม / ยอมรับหรือยัง"
   - `bool canSendSms = true;` (หรือ `isSmsSubscribed = true;`, `hasConsentedSms = true;`)
4. **"รหัสผ่านที่ตั้งมาสั้นเกินไป ไม่ปลอดภัย"**
   - ตัวแปรมุ่งเน้นไปที่ความปลอดภัย
   - `bool isPasswordSecure = false;` (หรือ `isPasswordWeak = true;`)

*(แนวคิดหลัก: พยายามอย่าตั้งชื่อตัวแปรที่ติดลบ (Negative form) เช่น `bool isNotVerified` เพราะเมื่อคุณนำไปซ้อนกับเครื่องหมาย `!` มันจะกลายเป็น `!!isNotVerified` (ปฏิเสธซ้อนปฏิเสธ) ซึ่งทำให้อ่านโค้ดแล้วปวดหัวมากๆ ควรตั้งในเชิงบวกแล้วใช้ค่า false แทนจะดีกว่า)*

### **โจทย์ที่ 2: ตามหาบั๊กตัวอักษรแฝด (Twin String Bug)**
**โจทย์ที่ให้มา:**
```csharp
char studentGrade;
studentGrade = "A";   // Error 1
studentGrade = 'A+';  // Error 2
```

**วิเคราะห์สาเหตุ (Why it fails):**
- **บรรทัดที่ 2 (`"A"`):** คุณพยายามเอา `string` (ข้อความ) ซึ่งเกิดจากการห่อหุ้มด้วย Double Quote มายัดใส่ในกล่อง `char` ซึ่งคอมไพเลอร์มองว่ามันข้ามสายพันธุ์กันอย่างสิ้นเชิง แม้ในความเป็นจริงมันจะมีแค่ตัวเดียวก็ตาม
- **บรรทัดที่ 3 (`'A+'`):** คุณใช้ Single Quote ถูกต้องตามกฎของ `char` แล้ว แต่อย่าลืมกฎเหล็กข้อที่สองคือ **"char เก็บได้เพียงความยาว 1 อักขระเป๊ะๆ"** การพิมพ์ทั้ง A และเครื่องหมาย + รวมเป็น 2 อักขระ มันจึงระเบิดทันที

**การแก้ไขที่ถูกต้อง (The Fix):**
ถ้าต้องการเก็บแค่ตัวอักษรเดียว ก็ใช้ Single Quote แต่ถ้าเกรดของมหาลัยคุณดันมีประจุบวกลบด้วย คุณ "จำเป็น" ต้องอัปเกรดไปใช้ชนิดข้อมูลแบบ ข้อความ (`string`) แทน!

```csharp
// ทางเลือกที่ 1: ถ้ายืนยันจะใช้ char ก็ต้องหั่นให้เหลือตัวเดียว
char studentGrade = 'A';

// ทางเลือกที่ 2: อัปเกรดเป็น string เพื่อให้เก็บเครื่องหมายได้
string advancedGrade = "A+";
```

## 🔥 เฉลย Challenge (การถอดรหัสผ่านความลับ The Caesar Cipher)

**โจทย์:** เขียนระบบแปลงตัวอักษรโรมัน โดยเลื่อนตำแหน่งไปทางขวา 3 ขั้น (เช่น 'H' -> 'K')
**เฉลยและวิธีทำเชิงลึก:**

กลไกนี้พึ่งพาระบบสถาปัตยกรรมของคอมพิวเตอร์ที่เรียกว่า "การหลอมรวมคณิตศาสตร์เข้ากับตัวอักษร (Character Math)" เพราะเบื้องหลังตัวอักษรทุกตัวในโลก ล้วนแต่เป็นตัวเลข (รหัส Unicode) 

```csharp
Console.WriteLine("======================================");
Console.WriteLine("      🛡️ CAESAR CIPHER MACHINE 🛡️     ");
Console.WriteLine("======================================");

Console.Write("กรุณาป้อนตัวอักษรลับ 1 ตัว (A-Z): ");

// 1. รับข้อความ (string) จากผู้ใช้ และทำการกรีดเอาเฉพาะตัวอักษรตัวแรกสุดมา (Index 0) 
// พร้อมทั้งแปลงให้เป็นพิมพ์ใหญ่ทั้งหมด (ToUpper) ป้องกันความผิดพลาดของรหัส ASCII
char secretLetter = Console.ReadLine().ToUpper()[0];

Console.WriteLine($"\n> วิเคราะห์รหัส: คุณป้อนตัว '{secretLetter}'");

// 2. กระบวนการเข้ารหัส (Encryption Process)
// เราจะดึงตัวเลข (ASCII/Unicode Code) ของตัวอักษรนั้นออกมา
// เช่น 'A' มีรหัส 65, 'H' มีรหัส 72
int originalCode = (int)secretLetter;

// 3. ปรับเลื่อนค่าไปทางขวา 3 ตำแหน่ง (บวกเพิ่ม)
int encryptedCode = originalCode + 3;

// 4. แปลงรหัสตัวเลขนั้น กลับมาสวมหน้ากากตัวอักษรอีกครั้ง
char encryptedLetter = (char)encryptedCode;

// สรุปผล
Console.WriteLine($"[ระบบกำลังประมวลผล] เลื่อนรหัสจาก {originalCode} ไปเป็น {encryptedCode}");
Console.WriteLine("======================================");
Console.WriteLine($"🗝️ ตัวอักษรที่ถูกเข้ารหัสแล้วคือ: [ {encryptedLetter} ]");
Console.WriteLine("======================================");
```

**การทดสอบ (Unit Testing):**
- ถ้าผู้ใช้พิมพ์ `H` (รหัส 72) + 3 = 75 ซึ่งแปลกลับมาได้ตัวอักษร **`K`**
- ถ้าผู้ใช้พิมพ์ `A` (รหัส 65) + 3 = 68 ซึ่งแปลกลับมาได้ตัวอักษร **`D`**

*(ข้อสังเกตของบั๊กระดับสูง: โค้ดด้านบนนี้ใช้งานได้ดี แต่ถ้าผู้ใช้พิมพ์ตัว `Z` (รหัส 90) พอบวก 3 จะกลายเป็น 93 ซึ่งรหัส 93 ในตาราง ASCII คือเครื่องหมาย `]` กลายเป็นบั๊กทันที! ในการเข้ารหัส Caesar Cipher ของจริง คุณจะต้องเขียนลูป `if` ดักไว้ด้วยว่า ถ้าบวกแล้วทะลุตัว Z ให้มันวนกลับ (Wrap Around) ไปเริ่มที่ตัว A ใหม่นั่นเอง!)*
