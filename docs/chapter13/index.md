# บทที่ 13: Exception Handling & File I/O (การรับมือข้อผิดพลาดและการจัดการไฟล์)

> 💡 **ภาพรวม:** ยินดีต้อนรับสู่โลกแห่งความจริง! ในโลกที่สมบูรณ์แบบ โค้ดของคุณจะทำงานได้ไร้ที่ติ... แต่ในโลกความจริง ผู้ใช้จะกรอกตัวหนังสือแทนตัวเลข, อินเทอร์เน็ตจะหลุด, และฮาร์ดดิสก์จะเต็ม! บทนี้คุณจะได้เรียนรู้วิชา "สร้างตาข่ายนิรภัย (Exception Handling)" เพื่อไม่ให้โปรแกรมแครชตายสนิท พร้อมกับวิชา "เซฟข้อมูลลงฮาร์ดดิสก์ (File I/O)" เพื่อให้ข้อมูลของคุณคงอยู่ตลอดไปแม้จะปิดเครื่อง!

## 🎯 สิ่งที่จะได้เรียนรู้ในบทนี้

- ✅ **Try/Catch/Finally:** คาถาสร้างตาข่ายนิรภัย ดักจับ Error ไม่ให้โปรแกรมเด้งออก (Crash)
- ✅ **Exception Types:** รู้จักกับปีศาจ Error สายพันธุ์ต่างๆ และวิธีวางกับดักดักจับให้ถูกตัว
- ✅ **Custom Exceptions:** การสร้าง Error เป็นชื่อของบริษัทคุณเอง (เช่น `InsufficientFundsException`)
- ✅ **using & IDisposable:** การเปิดใช้งานทรัพยากร (เช่น ไฟล์ หรือ ดาต้าเบส) แล้วสั่ง "ปิดทิ้งอัตโนมัติ" เพื่อป้องกัน Memory Leak
- ✅ **File I/O:** วิชาการอ่าน/เขียนไฟล์ข้อความ (Text File) ลงบนฮาร์ดดิสก์ 

## 🔗 Prerequisite (ความรู้ที่ต้องมีก่อน)

> ⚠️ บทนี้คือด่านสำคัญของการทำแอปพลิเคชันจริง คุณต้องมีความรู้เหล่านี้:
- **ตัวแปรและ If-Else (บท 2-3):** เพราะเราต้องเช็คเงื่อนไขและจัดการกับข้อมูล
- **OOP Inheritance (บท 10):** การสร้าง Custom Exception ต้องใช้ความรู้เรื่องการ "สืบทอดสายเลือด" จากคลาส `Exception` ของไมโครซอฟต์

## 🚀 ทำไมสองเรื่องนี้ถึงอยู่บทเดียวกัน?

หลายคนอาจจะสงสัยว่า "ทำไมเรื่อง Error ถึงต้องมาเรียนพร้อมกับเรื่อง File I/O?"
คำตอบจากสถาปนิกคือ: **"เพราะการยุ่งกับ File คือจุดที่ทำให้โปรแกรมเกิด Error บ่อยที่สุดในโลกครับ!"**

- จะอ่านไฟล์... อ้าว ไฟล์โดนลบไปแล้ว! (`FileNotFoundException`)
- จะเซฟไฟล์... อ้าว ฮาร์ดดิสก์เต็ม! (`IOException`)
- จะแก้ไฟล์... อ้าว มีโปรแกรมอื่นเปิดไฟล์นี้ค้างไว้อยู่! (`UnauthorizedAccessException`)

ถ้าคุณไม่รู้วิชาสร้างตาข่ายนิรภัย (Exception Handling)... โปรแกรมของคุณจะจอแดงและเด้งปิดตัวเองทุกครั้งที่ผู้ใช้ทำอะไรแผลงๆ กับไฟล์ครับ! เตรียมตัวติดอาวุธชิ้นสำคัญที่สุดของการเป็นโปรแกรมเมอร์มืออาชีพกันได้เลย!
