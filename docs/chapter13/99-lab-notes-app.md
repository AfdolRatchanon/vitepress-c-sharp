# 💻 Lab: สร้างสมุดโน้ตส่วนตัว (The Notes App)

> 💡 **เป้าหมายของ Lab นี้:** ถึงเวลาลงสนามจริง! คุณจะต้องรวบรวมทักษะทั้งหมดในบทที่ 13 ทั้งการสร้างตาข่ายนิรภัย (Try/Catch) คาถากำจัดทรัพยากร (using) และการจัดการฮาร์ดดิสก์ (File I/O) มาหลอมรวมกัน เพื่อสร้างแอปพลิเคชัน "สมุดจดบันทึกประจำวัน" ที่จดจำข้อมูลของคุณได้ตลอดไป แม้จะปิดคอมพิวเตอร์ก็ตาม!

## 📖 ภาพรวมของโปรแกรม (Program Overview)

ผู้ใช้สามารถ "เพิ่มโน้ตใหม่", "เปิดอ่านโน้ตทั้งหมด", และ "ล้างประวัติโน้ตทิ้ง" ได้ 
**จุดสำคัญที่สุด:** ไฟล์ฮาร์ดดิสก์นั้นเปราะบางมาก (บางทีอาจจะถูกล็อกอยู่ หรือโฟลเดอร์โดนลบ) คุณต้องกางตาข่ายนิรภัยป้องกันแอปแครช (Crash) ในทุกจังหวะที่มีการยุ่งกับไฟล์!

## 📋 กฎกติกาและเกณฑ์การประเมิน (Business Requirements)

**ข้อกำหนดทางสถาปัตยกรรมระดับ Enterprise:**
คุณต้องแยกโค้ดออกเป็นคลาส `NoteManager` แบบจัดการเฉพาะไฟล์ และมี `try/catch` ครอบลอจิกจุดเสี่ยงไว้ทั้งหมด!

### โครงสร้างโค้ดที่ต้องสร้าง:

1. **คลาสผู้จัดการสมุด (The Engine): `NoteManager`**
   - มี Field ซ่อนไว้ตัวนึง: `private readonly string _filePath = "my_notes.txt";`
   
   - **ภารกิจ 1:** สร้างเมธอด `public void AddNote(string message)`
     - เอา `message` มาประกอบร่างกับ `DateTime.Now.ToString("yyyy-MM-dd HH:mm:ss")` ให้ดูสวยงาม (ตัวอย่าง: `[2026-10-01 14:30] ข้อความ...`)
     - **เงื่อนไขบังคับ:** ต้องใช้ `StreamWriter` ควบคู่กับคำสั่ง `using` และสั่ง `append: true` เพื่อต่อท้ายไฟล์!
     - **ตาข่ายนิรภัย:** ครอบโค้ดด้วย `try/catch` เพื่อดักจับ `Exception` ถ้าพัง ให้ `throw` ระเบิดลูกใหม่ที่เขียนว่า `"บันทึกโน้ตล้มเหลว: ..."` ออกไป!
   
   - **ภารกิจ 2:** สร้างเมธอด `public void ReadAllNotes()`
     - **เงื่อนไขบังคับ:** เช็คก่อนด้วย `File.Exists` ว่ามีไฟล์ไหม? ถ้าไม่มี ให้ปา `throw new FileNotFoundException("ยังไม่มีสมุดโน้ตเลย ไปเขียนก่อนนะ!");`
     - ใช้ `StreamReader` คู่กับ `using` (หรือ `File.ReadAllText` ก็ได้ อนุโลม) เพื่อปริ้นข้อความทั้งหมดโชว์หน้าจอ
     - **ตาข่ายนิรภัย:** ต้องมี `try/catch` ครอบไว้เช่นกัน
     
   - **ภารกิจ 3:** สร้างเมธอด `public void ClearNotes()`
     - ใช้ `File.Delete(_filePath)` ลบไฟล์ทิ้งให้สิ้นซาก
     - ครอบ `try/catch` ไว้ด้วย เผื่อไฟล์มันถูกโปรแกรมอื่นเปิดล็อกไว้อยู่ (มันจะดักได้)

2. **การทำงานของหน้าจอ (The UI Level): `Program.cs (Main)`**
   - วนลูป `while(true)` คอยรับตัวเลขจากผู้ใช้
   - ใช้ `switch-case` ในการเรียกใช้ฟังก์ชันจาก `NoteManager`
   - **จุดสำคัญสูงสุด:** คุณต้องกางตาข่าย `try/catch` รอบวงเล็บของ `switch-case` อีก 1 ชั้น! เพื่อคอยตักตวง "ระเบิด (Exceptions)" ทุกชนิดที่ `NoteManager` อาจจะปาขึ้นมา แล้วปริ้นคำด่าโชว์สีแดงสวยๆ แทนที่จะปล่อยให้แอปแครชจอฟ้าพังพินาศ!

## ⏱️ เวลาที่ใช้ในการทำ Lab: 45 นาที

**เคล็ดลับระดับสถาปนิก:**
ระบบนี้คือการจำลอง **N-Tier Architecture** ครับ
- `NoteManager` คือชั้น Backend (Data Access Layer) หน้าที่ของมันคือยุ่งกับ Database (ไฟล์) ถ้ายุ่งไม่ได้ มันจะปาระเบิด (`throw`) ออกมา (ห้ามให้มันใช้ `Console.WriteLine` ด่าผู้ใช้ตรงนี้เด็ดขาด โค้ดจะเน่า)
- `Main` คือชั้น Frontend (UI Layer) หน้าที่ของมันคือการเอาตาข่ายมารอรับระเบิด (catch) แล้วคลี่ดูข้อความ นำมาปริ้นโชว์ให้ผู้ใช้เห็นด้วยสีสันสวยงาม (นี่คือหลักการที่สะอาดที่สุดในโลก OOP ครับ!)

## 📝 ขั้นตอนการพัฒนา (Step-by-Step Implementation)

### ขั้นตอนที่ 1: ปั้นฐานข้อมูลหลังบ้าน (The NoteManager)
- [ ] สร้างคลาส `NoteManager` แปะ Path ไฟล์ไว้เป็นตัวแปร
- [ ] เขียนเมธอด `AddNote` โดยใช้ท่าไม้ตาย `using (StreamWriter w = new StreamWriter(path, true))`
- [ ] เขียนเมธอด `ReadAllNotes` และอย่าลืมเอา `if (!File.Exists(path))` ไปดักทางระเบิดหน้าแหกไว้ด้วย!
- [ ] เขียนเมธอด `ClearNotes` ลบไฟล์ทิ้ง

### ขั้นตอนที่ 2: วางตาข่ายนิรภัยประจำฟังก์ชัน
- [ ] ในแต่ละฟังก์ชันของ `NoteManager` จงเอา `try / catch (Exception ex)` ไปห่อหุ้มของพวกนั้นซะ 
- [ ] ถ้าระเบิดลงใน catch ให้คุณสั่ง `throw new Exception("เกิดปัญหา: " + ex.Message);` (ส่งผ่านระเบิดขึ้นไปหาหน้าบ้าน)

### ขั้นตอนที่ 3: ประกอบร่างหน้าบ้าน (UI Console)
- [ ] สร้างเมนู Console กดหมายเลข
- [ ] รับค่าตัวเลขผ่าน `int.Parse` (ระวังจุดพังตรงนี้ด้วย! ถ้ากรอกตัวหนังสือมันจะแครช จงใช้ try/catch ดัก `FormatException` แยกต่างหากเลย!)
- [ ] กางตาข่ายรับระเบิด `Exception` ทั่วไป ไว้ตรงกรอบนอกสุด เพื่อปริ้นข้อความสีแดงเตือนผู้ใช้งานเวลาไฟล์ระเบิด!

## 🔥 Challenge (บททดสอบไฟนอลสถาปนิก)

สำหรับผู้ที่อยากเห็นว่า "โลก API สมัยใหม่" เขาทำงานกันอย่างไร!

**โจทย์ระดับบอส: การจับคู่จัดกลุ่มและบันทึกเป็นฐานข้อมูลเสมือน (The Structured DB Format)**
การเขียนใส่ไฟล์เป็น String เปล่าๆ นั้นมัน "โง่" ครับ เอาไปค้นหาต่อก็ยาก! 

**ภารกิจลับ:**
1. จงใช้คลาสลับของ C# ในการ "สับข้อความ (Split)" ตอนที่คุณใช้เมธอด `ReadAllNotes()`
2. สมมติฐานข้อมูลคุณเก็บด้วยฟอร์แมตขั้นคั่น: `2026-10-01|สวัสดีชาวโลก` (ใช้ตัว `|` ขั้นระหว่างเวลากับข้อความ)
3. ตอนที่ฟังก์ชันหน้าบ้านสั่งโหลด... ให้ลอจิกหลังบ้านใช้ `StreamReader.ReadLine()` ดูดมาทีละบรรทัด...
4. จากนั้นเอามีดสับข้อความ: `string[] parts = line.Split('|');`
5. แยก `parts[0]` เป็นเวลา, แยก `parts[1]` เป็นข้อความ
6. จับมันปริ้นโชว์หน้าจอแบบจัดโครงสร้าง (เช่น เปลี่ยนสีเวลาเป็นสีเหลือง ข้อความเป็นสีขาว)

นี่คือจุดกำเนิดของการทำฐานข้อมูล **CSV (Comma Separated Values)** ก่อนที่มนุษยชาติจะก้าวไปรู้จักกับ JSON Database ครับ! ท่านี้ฝึกกระบวนทัศน์การฉีกข้อมูล (Parsing) ได้ดีเยี่ยมสุดๆ ลองลุยดูครับ!
