# บทที่ 6: Arrays & Collections (โกดังเก็บข้อมูล)

> 💡 **ภาพรวม:** ขอต้อนรับเข้าสู่โลกของ "โครงสร้างข้อมูล (Data Structures)"! ในบทที่ผ่านมา คุณเก่งเรื่องการประมวลผลลอจิก (Loops & Methods) แต่ถ้าคุณต้องรับมือกับข้อมูลจำนวนมหาศาล (เช่น ชื่อนักเรียน 1,000 คน) การสร้างตัวแปร `string student1, student2, ...` จนถึง 1,000 ตัว ถือเป็นการกระทำที่ไร้สติ! บทนี้จะสอนให้คุณสร้าง "ตู้คอนเทนเนอร์" เพื่อบรรจุข้อมูลนับล้านชิ้น และวิธีใช้เครื่องมือในการบริหารจัดการมัน

## 🎯 สิ่งที่จะได้เรียนรู้ในบทนี้

- ✅ อาร์เรย์ 1 มิติ (1D Arrays) - โกดังขนาดคงที่ และกลไก Memory แบบ Reference
- ✅ อาร์เรย์ 2 มิติ (2D Arrays) - การสร้างแผนที่ตาราง (Rows & Columns)
- ✅ `List<T>` - ดาบเลเซอร์แห่งยุคใหม่ (Dynamic Arrays ที่ยืดหดได้ตามใจชอบ)
- ✅ `Dictionary<K,V>` - โครงสร้างตู้เซฟ (Key-Value Pairs) สำหรับการค้นหาด้วยความเร็วแสง
- ✅ `Queue` & `Stack` - กฎการต่อแถวแบบมาก่อนได้ก่อน (FIFO) และมาหลังได้ก่อน (LIFO)
- ✅ การประยุกต์ใช้ลูปพิเศษ `foreach` ในการกวาดล้างข้อมูล

## 🔗 Prerequisite (ความรู้ที่ต้องมีก่อน)

> ⚠️ บทนี้คือจุดตัดที่สำคัญมาก หากขาดพื้นฐานต่อไปนี้ คุณจะเดินหลงทางในเขาวงกตข้อมูล:
- **`for` และ Nested Loops (บทที่ 4):** เครื่องมือหลักในการวิ่งไล่หาข้อมูลในโกดัง
- **Methods (บทที่ 5):** เราจะส่งผ่านโครงสร้างข้อมูลขนาดใหญ่ (Reference Type) ข้ามไปมาระหว่างฟังก์ชัน
- **Data Types (บทที่ 2):** โกดังแต่ละตู้ จะต้องระบุว่ามันรับข้อมูลชนิดไหน (`int`, `string`) และห้ามปะปนกัน!

## 🚀 จุดเปลี่ยนจาก "สถาปนิก" สู่ "นักจัดการฐานข้อมูล"

ทำไมเราถึงต้องเรียน Data Structures?
1. **ประสิทธิภาพ (Performance):** การเลือกโกดังที่ผิดประเภท อาจทำให้โปรแกรมช้าลงล้านเท่า! ถ้าคุณแค่จะเก็บของเข้าแถว การใช้ Array จะเร็วกว่าจรวด แต่ถ้าคุณต้องการลบคนกลางแถวทิ้งบ่อยๆ การใช้ Array คือนรกแตก (เพราะคุณต้องขยับคนด้านหลังทุกคนเดินหน้ามา 1 ก้าว) 
2. **รากฐานของ Database:** ก่อนที่คุณจะไปเชื่อมต่อกับ SQL Database เพื่อดึงข้อมูลลูกค้ามาโชว์บนหน้าเว็บ ข้อมูลเหล่านั้นทั้งหมดจะต้องถูกดึงมาพักไว้ใน `List` หรือ `Dictionary` ใน RAM เครื่องเราเสมอ!
3. **การเข้าสู่วงการอัลกอริทึม (Algorithms):** ไม่มีการค้นหาเส้นทางแบบ AI (A* Search) ใดที่สร้างได้โดยไม่ใช้ `Queue`, และไม่มีระบบ Undo/Redo ใน Photoshop ใดที่ไม่ได้ใช้ `Stack`

เตรียมตัวสวมบทบาทหัวหน้าคลังสินค้า แล้วมาระเบิดพลังจัดระเบียบข้อมูลกันครับ!
