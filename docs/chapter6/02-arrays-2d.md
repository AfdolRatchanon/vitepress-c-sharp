# Arrays (2D) - ตารางแบนราบ

> 💡 **เป้าหมาย:** สยายปีกออกจากโลกมิติเดียว เข้าสู่โลกของ 2 มิติ (แกน X และ Y) เรียนรู้วิธีสร้างโครงสร้างกระดานหมากรุก แผนที่ พิกัดตาราง และการใช้ Nested Loops ในการกวาดข้อมูลแบบปูพรมตาราง (Row-Major Order)

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. ทำไมต้องใช้ Array 2 มิติ?

หากอาร์เรย์ 1 มิติ (1D Array) คือ **"ตู้ล็อคเกอร์แถวเรียงหนึ่ง"**
อาร์เรย์ 2 มิติ (2D Array) ก็คือ **"ตู้ไปรษณีย์ในคอนโดที่มีหลายชั้น และหลายห้องในแต่ละชั้น"** (ตารางตู้)

ในโลกของการพัฒนาซอฟต์แวร์จริง เราเจองานที่เป็น 2 มิติตลอดเวลา:
- **กระดานเกม:** กระดานหมากรุก (8x8), ตารางเกมเศรษฐี, พิกัดในเกม 2D
- **เอกสารและข้อมูล:** โปรแกรมประเภทตารางคำนวณ (Excel) ที่มี "แถว (Rows)" และ "คอลัมน์ (Columns)"
- **ภาพดิจิทัล (Image Processing):** รูปภาพทุกรูปที่คุณเห็นในจอคอมพิวเตอร์ คือตาราง 2 มิติที่เก็บค่าสี RGB ในแต่ละพิกเซล (X, Y)

---

### 2. ไวยากรณ์แห่งโลก 2 มิติ (Declaration & Initialization)

ใน C# การเพิ่มมิติที่ 2 ทำได้ง่ายๆ เพียงแค่สอดแทรก **"เครื่องหมายจุลภาค (Comma `,`)"** เข้าไปไว้ตรงกลางวงเล็บก้ามปู `[,]` เพื่อบ่งบอกว่าเรามี 2 แกนนะ!

```csharp
// รูปแบบที่ 1: สั่งเทปูนสร้างโกดังเปล่าๆ ขนาด 3 แถว และ 4 คอลัมน์ (รวม 12 ช่องว่างๆ)
// แกนหน้า (3) คือ แถวแนวตั้ง (Rows / Y) | แกนหลัง (4) คือ คอลัมน์แนวนอน (Cols / X)
int[,] matrix = new int[3, 4]; 

// การระบุตำแหน่ง (Indexing) ต้องใส่เลข 2 ตัวเสมอ (แถวที่เท่าไหร่, คอลัมน์ที่เท่าไหร่)
matrix[0, 0] = 50;  // ช่องแรกสุด มุมซ้ายบน
matrix[2, 3] = 99;  // ช่องสุดท้าย มุมขวาล่างสุด
// 💥 matrix[3, 4] = 100; // แครช! (IndexOutOfRange) เพราะมีแค่แถว 0,1,2 และ คอลัมน์ 0,1,2,3


// รูปแบบที่ 2: สร้างและใส่ข้อมูลเข้าไปพร้อมกันเลย (Hardcoded Matrix)
// สังเกตการใช้ปีกกาซ้อนปีกกาให้ดูเหมือนตาราง
string[,] ticTacToe = 
{
    { "X", "O", "X" },  // แถวที่ 0
    { "-", "X", "O" },  // แถวที่ 1
    { "O", "-", "X" }   // แถวที่ 2
};
```

*(ข้อสังเกต: ในภาษา C# มีอาร์เรย์อีกประเภทที่เรียกว่า Jagged Array `[][]` หรือ "อาร์เรย์ซ้อนอาร์เรย์" ซึ่งอนุญาตให้แต่ละแถวยาวไม่เท่ากันได้ (เช่น ฟันปลา) แต่สำหรับ 2D Array `[,]` ทุกแถวจะถูกบังคับให้มีความกว้างเท่ากันเป๊ะเสมอ เป็นตารางสี่เหลี่ยมสมบูรณ์)*

### 3. กระบวนการปูพรมตาราง (Row-Major Traversal)

ในการลากข้อมูลออกจาก Array 1 มิติ เราใช้ `for` loop แค่ชั้นเดียว 
ดังนั้น การดึงข้อมูลจาก Array 2 มิติ เราก็หนีไม่พ้นต้องใช้ **"Nested Loops (ลูปซ้อนลูปแบบที่เรียนในบทที่ 4)"** เข้ามาช่วย 

สิ่งที่โปรแกรมเมอร์มือใหม่มักพลาดคือ การดึงค่าความยาว (Length) ของ 2D Array!
- ถ้าคุณใช้ `matrix.Length` มันจะให้ผลรวมของห้องทั้งหมด (เช่น 3x4 = 12) ซึ่งเอามาเข้า `for` loop ตรงๆ ไม่ได้!
- C# จึงมีอาวุธลับชื่อว่า **`.GetLength(มิติ)`**
  - `matrix.GetLength(0)` -> ขอความยาวของแกนที่ 0 (จำนวนแถวแนวตั้ง / Rows)
  - `matrix.GetLength(1)` -> ขอความยาวของแกนที่ 1 (จำนวนคอลัมน์แนวนอน / Cols)

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่างที่ 1: ระบบจัดการโรงภาพยนตร์ (Cinema Seating System)

โค้ดนี้จำลองระบบการจองที่นั่งในโรงภาพยนตร์ (1 คือมีคนจองแล้ว, 0 คือว่าง) 
และสาธิตวิธีการใช้ Nested Loop ปูพรมวาดแผนผังที่นั่งออกมาบนหน้าจอ

::: code-group
```csharp [Program.cs]
using System;

namespace CinemaSystem
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบแผนผังโรงภาพยนตร์ ===\n");

            // สร้าง Array 2D เก็บสถานะที่นั่ง (0 = ว่าง, 1 = จองแล้ว)
            // โรงหนังขนาดย่อม: 4 แถว (A,B,C,D), แถวละ 5 ที่นั่ง
            int[,] seatingPlan = 
            {
                { 1, 1, 0, 1, 1 }, // แถวที่ 0 (ติดจอสุด)
                { 0, 0, 0, 0, 0 }, // แถวที่ 1
                { 1, 0, 1, 0, 1 }, // แถวที่ 2
                { 1, 1, 1, 1, 1 }  // แถวที่ 3 (หลังสุด จองเต็ม)
            };

            // -----------------------------------------------------------
            // กระบวนการวาดแผนผัง (Traversal)
            // -----------------------------------------------------------
            Console.WriteLine("    [ หน้าจอภาพยนตร์ ]");
            Console.WriteLine("--------------------------");

            // ลูปชั้นนอก: รับผิดชอบการขยับไปทีละ "แถว (Row)" -> แกน 0
            for (int row = 0; row < seatingPlan.GetLength(0); row++)
            {
                // ตกแต่งพิมพ์ตัวอักษรนำหน้าแถว (ทริค: เอาตัว 'A' ไปบวกกับเลข row แล้วแปลงกลับเป็น char)
                Console.Write($"แถว {(char)('A' + row)}: ");

                // ลูปชั้นใน: รับผิดชอบการพิมพ์ที่นั่งไปตามยาวของ "คอลัมน์ (Col)" -> แกน 1
                for (int col = 0; col < seatingPlan.GetLength(1); col++)
                {
                    // อ่านค่าจาก Array 2D
                    int status = seatingPlan[row, col];

                    // แปลงตัวเลขเป็นกราฟิกสวยๆ
                    if (status == 1)
                    {
                        Console.ForegroundColor = ConsoleColor.Red;
                        Console.Write("[X] "); // แดง = เต็ม
                    }
                    else
                    {
                        Console.ForegroundColor = ConsoleColor.Green;
                        Console.Write("[O] "); // เขียว = ว่าง
                    }
                    Console.ResetColor();
                }

                // จบ 1 แถวปุ๊บ ต้องสั่งขึ้นบรรทัดใหม่เสมอ เพื่อวาดแถวถัดไป!
                Console.WriteLine(); 
            }
            Console.WriteLine("--------------------------");
        }
    }
}
```
:::

### ตัวอย่างที่ 2: เกมจิ้มระเบิด (Minesweeper Coordinates)

ตัวอย่างนี้ชี้ให้เห็นจุดบอดที่หลายคนสับสนระหว่าง ระบบพิกัด (X,Y) ในวิชาคณิตศาสตร์ กับ ระบบตาราง (Row,Col) ในวิชาคอมพิวเตอร์!

::: code-group
```csharp [Program.cs]
using System;

namespace MineSweeperCoordinates
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบยิงขีปนาวุธ (Targeting System) ===");

            // สร้างทะเลขนาด 5x5
            string[,] ocean = new string[5, 5];

            // เติมน้ำทะเล (~) ให้เต็มทุกช่อง ด้วย Nested Loop
            for (int r = 0; r < ocean.GetLength(0); r++)
            {
                for (int c = 0; c < ocean.GetLength(1); c++)
                {
                    ocean[r, c] = "~";
                }
            }

            // สมมติผู้ใช้สั่งยิงจรวดไปที่พิกัด X = 3, Y = 1 (ตามหลักกราฟคณิตศาสตร์)
            int targetX = 3; 
            int targetY = 1;

            // ⚠️ อันตราย! ใน Array 2D เราอ้างอิงเป็น [Row, Col]
            // แถว (Row) คือการขยับลงล่าง (มันคือแกน Y)
            // คอลัมน์ (Col) คือการขยับไปขวา (มันคือแกน X)
            // ดังนั้นเวลาหยอดข้อมูล ต้อง **สลับตำแหน่ง** เอา Y ขึ้นก่อน X นะครับ!!
            ocean[targetY, targetX] = "💥"; 

            // วาดทะเลออกมาดู
            for (int r = 0; r < ocean.GetLength(0); r++)
            {
                for (int c = 0; c < ocean.GetLength(1); c++)
                {
                    Console.Write(ocean[r, c] + " ");
                }
                Console.WriteLine();
            }
        }
    }
}
```
:::
**ผลลัพธ์การวาด:** พิกัด 💥 จะตกอยู่ที่ แถว 1 (นับบรรทัดลงมา) และ คอลัมน์ 3 (นับจากซ้ายไปขวา) อย่างแม่นยำ

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

### **โจทย์ที่ 1: ชนขอบตารางทะลุมิติ**
พิจารณาการจองพื้นที่นี้: `int[,] grid = new int[10, 5];`
ถ้าคุณต้องการเข้าถึง **"ช่องขวาล่างสุด (ช่องสุดท้ายของตาราง)"** โค้ดคำสั่ง `grid[ ?, ? ] = 99;` ควรเติมตัวเลขอะไรลงไปในเครื่องหมายคำถาม? (ห้ามตอบ 10, 5 เด็ดขาดนะ!)

### **โจทย์ที่ 2: ระบบตรวจตั๋วผี (Matrix Summation)**
**ภารกิจ:** จงประกาศ Array 2 มิติ ให้อยู่ในรูปแบบนี้:
```csharp
int[,] tickets = {
    { 5, 2, 8 },
    { 1, 0, 3 }
};
```
ให้คุณเขียนโปรแกรมบวกเลขทั้งหมดทุกกล่องในตาราง 2D นี้รวมเข้าด้วยกัน (5+2+8+1+0+3) โดยใช้ Nested Loop และตัวแปร Accumulator (`int total = 0`) เพื่อหาคำตอบว่าขายตั๋วไปได้ทั้งหมดกี่ใบ? 

::: details 💡 คำใบ้และแนวทาง (Hints)
- **สำหรับโจทย์ 1:** อย่าลืมทฤษฎี Zero-based! แถวมี 10 แถว (แปลว่า 0-9) คอลัมน์มี 5 แถว (แปลว่า 0-4) ดังนั้นมุมขวาล่างสุดคือ `grid[9, 4]` ครับ!
- **สำหรับโจทย์ 2:** ลูปนอกคือ `row < tickets.GetLength(0)` และลูปในคือ `col < tickets.GetLength(1)` ข้างในปีกกาในสุด ให้สั่ง `total += tickets[row, col];` เรียบง่ายแต่ทรงพลัง!
:::

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: การบวกเมทริกซ์ทะลุแกน (Matrix Diagonal Summation)**

นี่คือข้อสอบมาตรฐานสำหรับโปรแกรมเมอร์สาย Data Science / เกมกราฟิก 
เส้นทแยงมุม (Diagonal Line) ของตารางสี่เหลี่ยมจัตุรัส คือเส้นที่ลากจากมุมซ้ายบน เฉียงลงมาทะลุมุมขวาล่าง

**ภารกิจของคุณ:** 
1. สร้างตาราง 2D (Matrix) ขนาด 3x3 พร้อมข้อมูลตัวเลขอะไรก็ได้ เช่น:
```csharp
int[,] matrix = {
    { 1, 2, 3 },
    { 4, 5, 6 },
    { 7, 8, 9 }
};
```
2. ให้เขียนโปรแกรมคำนวณ "ผลรวมเฉพาะตัวเลขที่อยู่ในแนวเส้นทแยงมุมหลัก (Main Diagonal)" 
   (ในตัวอย่างข้างบน เส้นทแยงมุมคือเลข **1, 5, 9** รวมกันต้องได้คำตอบคือ 15)

**คำแนะนำระดับสถาปนิก:**
- วิธีถึกทน (Brute Force): ใช้ Nested Loop กวาดมันทุกช่อง 3x3 = 9 รอบ แล้วเช็คเงื่อนไข `if (row == col)` (เพราะเส้นทแยงมุมคือจุดที่พิกัด X กับ Y มันเท่ากันเป๊ะ! เช่น (0,0), (1,1), (2,2)) 
- **วิธีระดับเทพ (O(N) Optimization):** ถ้าสังเกตดีๆ... ทำไมต้องวิ่งตั้ง 9 รอบด้วยล่ะ? ในเมื่อเรารู้ว่ามันคือพิกัดที่ตัวเลขเดียวกัน (0,0) (1,1) (2,2) 
  คุณสามารถใช้ **`for` loop ชั้นเดียวพอ!** วิ่งแค่ 3 รอบ (ตัวแปร `i`) แล้วดึงพิกัดมาบวกกันแบบนี้เลย `total += matrix[i, i];` 

ลองประยุกต์ใช้วิธีระดับเทพดูสิครับ แล้วคุณจะเข้าใจว่าทำไมวิชา Algorithm ถึงมีความสำคัญ!

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** ในเอกสารอ้างอิงของ C# บอกไว้ว่า `GetLength(0)` ใช้ดึงค่ามิติที่ 0 ... แล้วทำไมเราไม่ใช้ `matrix.Length` เหมือนใน Array 1 มิติล่ะ? ถ้าเราฝืนใช้มันจะคืนค่าอะไรกลับมาให้เรา?
**แนวคำตอบ:** ฝืนใช้ได้ครับ ไม่ Error! แต่ `matrix.Length` ของ 2D Array จะคืนค่า **"จำนวนช่องเก็บข้อมูลทั้งหมด (Total Elements)"** 
เช่น ถ้าตารางขนาด 3x4 ค่า `Length` จะพ่นเลข `12` ออกมา! ซึ่งถ้าคุณเอาเลข 12 ไปยัดในเงื่อนไขลูป `for (int i=0; i<12)` มันไม่สามารถเอาตัวแปร `i` ไปแทนค่าพิกัด `matrix[i, ?]` ได้เลย เพราะมันมี 2 แกน นี่คือเหตุผลที่เราถูกบังคับให้ใช้ `GetLength(มิติ)` เพื่อรีดความยาวของแกนใครแกนมันออกมาเข้าสมการ Nested Loop นั่นเองครับ

**คำถาม 2:** โค้ดด้านล่างนี้ เกิดข้อผิดพลาดร้ายแรง (Compile Error) หรือไม่?
```csharp
int[,] map = {
    { 1, 2, 3 },
    { 4, 5 }
};
```
**แนวคำตอบ:** **พินาศ 100% (Compile Error)!**
อย่างที่ย้ำไว้ในทฤษฎีข้างต้น: Array 2 มิติแท้ๆ แบบ `[,]` มีกฎเหล็กแห่งสถาปัตยกรรมว่า **"ต้องเป็นตารางสี่เหลี่ยมผืนผ้า/จัตุรัสที่สมบูรณ์เท่านั้น"** แถวที่ 1 มี 3 ตัว แถวที่ 2 ดันมีแค่ 2 ตัว ทำให้ตารางแหว่ง ซึ่งคอมไพเลอร์ของ C# ไม่ยอมรับสถาปัตยกรรมพังๆ แบบนี้เด็ดขาด (แต่ถ้าอยากเขียนแบบแหว่งๆ ฟันปลาได้จริงๆ ต้องหนีไปใช้เทคนิคขั้นสูงที่เรียกว่า Jagged Array `[][]` ครับ)
:::
