# Arrays (1D) - แถวตอนเรียงหนึ่ง

> 💡 **เป้าหมาย:** ทำความรู้จักกับโกดังเก็บข้อมูลแบบที่เก่าแก่ ทรงพลัง และเร็วที่สุดในโลกคอมพิวเตอร์! เรียนรู้วิธีการจองพื้นที่ในหน่วยความจำแบบต่อเนื่อง (Contiguous Memory) กฎเหล็กแห่งการนับเลขที่เริ่มจาก 0 และวิธีวิ่งไล่ตามข้อมูลด้วยลูป `for` และ `foreach`

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. ปัญหาของการไม่มีกล่องรวมข้อมูล (The Variable Hell)

สมมติคุณได้รับโจทย์ให้เขียนโปรแกรมหาค่าเฉลี่ยคะแนนสอบของนักเรียน 5 คน
คุณอาจจะประกาศตัวแปร: `int score1, score2, score3, score4, score5;` 
แล้วจับมาบวกกันหาร 5... ฟังดูเหมือนจะโอเค แต่ถ้าพรุ่งนี้โรงเรียนมีนักเรียนเพิ่มเป็น 1,000 คนล่ะ? คุณจะต้องนั่งพิมพ์ตัวแปร 1,000 ตัวเลยหรือ?

**Array (อาร์เรย์)** ถูกสร้างขึ้นมาเพื่อแก้ปัญหานี้ มันคือ **"กล่องพลาสติกที่มีหลายๆ ช่องติดกัน"** (เหมือนกล่องใส่ยาประจำวัน) โดยที่คุณตั้งชื่อให้กล่องนั้นแค่ชื่อเดียว!

### 2. ไวยากรณ์แห่งการจองพื้นที่ (Array Declaration & Initialization)

ในการสร้างอาร์เรย์ คุณต้องใช้สัญลักษณ์ **ก้ามปู `[ ]`** วางต่อท้าย Data Type เพื่อบอกคอมไพเลอร์ว่า "ฉันไม่ได้ขอสร้างกล่องใบเดียวนะ แต่ฉันขอสร้างเป็นแผง!"

และที่สำคัญที่สุด: **"อาร์เรย์ในภาษา C# คือ Reference Type เสมอ!"** 
มันไม่ได้ถูกสร้างใน Stack Memory เหมือนตัวเลขทั่วไป แต่มันถูกสร้างใน Heap Memory (พื้นที่กว้างใหญ่) ดังนั้นคุณจึงต้องใช้คีย์เวิร์ด **`new`** เสมอ เพื่อสั่งให้เครื่องจักรไป "เทปูนสร้างโกดัง" ใน Heap

```csharp
// รูปแบบที่ 1: สั่งเทปูนสร้างโกดัง 5 ช่อง (แต่ยังไม่ได้เอาของใส่ ช่องว่างๆ จะมีค่าเป็น 0 อัตโนมัติ)
int[] scores = new int[5];

// รูปแบบที่ 2: สร้างโกดังพร้อมกับระบุของที่จะใส่ไปเลยทันที (ไม่ต้องบอกตัวเลข 5 คอมมันนับให้เอง)
string[] days = new string[] { "Mon", "Tue", "Wed", "Thu", "Fri" };

// รูปแบบที่ 3: แบบย่อขั้นสุด (ใช้บ่อยที่สุดเมื่อรู้ข้อมูลล่วงหน้า)
double[] temperatures = { 35.5, 36.2, 37.0, 36.8 };
```

### 3. ปรัชญา Zero-Based Indexing (ทำไมคอมพิวเตอร์ถึงนับเลขเริ่มที่ 0)

ในสายตามนุษย์ เราเริ่มนับสิ่งของจากชิ้นที่ 1 (One-based)
แต่ในโลกของคอมพิวเตอร์ (รวมถึง C#, Java, C++, JavaScript) **ช่องแรกสุดของอาร์เรย์ จะมีหมายเลขตำแหน่ง (Index) เท่ากับ 0 เสมอ!**

**ทำไมต้องเป็นเลข 0? (The Pointer Offset Theory):**
คำว่า "Index" ในภาษาคอมพิวเตอร์ แปลตรงตัวว่า **"ระยะกระจัด (Offset)"** หรือ "ต้องเดินถัดไปอีกกี่ก้าว จากจุดเริ่มต้น?"
- ช่องแรกสุด (อยู่ที่ประตูโกดังพอดี) -> ต้องเดินอีกกี่ก้าว? ตอบ: เดินอีก 0 ก้าว (Index 0)
- ช่องที่ 2 -> ต้องเดินไปอีกกี่ก้าวจากประตู? ตอบ: เดิน 1 ก้าว (Index 1)

ดังนั้น ถ้าอาร์เรย์มีขนาด 5 ช่อง (Size = 5) หมายเลขห้อง (Index) จะมีแค่ **`0, 1, 2, 3, 4`** เท่านั้น!
ถ้าคุณพยายามเข้าห้องเบอร์ 5 (`scores[5]`) โปรแกรมคุณจะแครชตายทันทีด้วยข้อหาลักลอบบุกรุกพื้นที่คนอื่น (Error ยอดฮิตชื่อว่า `IndexOutOfRangeException`)

### 4. อาวุธใหม่: การกวาดข้อมูลด้วย `foreach` Loop

ในบทที่ 4 เราใช้ `for` loop ในการนับหมายเลขห้อง (Index) เพื่อเข้าถึงข้อมูล
แต่นักออกแบบภาษา C# รู้ว่าโปรแกรมเมอร์ขี้เกียจมานั่งนับ 0 ถึง length-1 จึงสร้างลูปอำนวยความสะดวกที่ชื่อว่า **`foreach` (ทำทีละอันในกลุ่ม)**

```csharp
string[] names = { "John", "Jane", "Bob" };

// ใช้ for loop แบบคลาสสิก (ข้อดี: รู้ว่าของชิ้นนี้อยู่ตำแหน่ง (Index) ที่เท่าไหร่)
for (int i = 0; i < names.Length; i++) {
    Console.WriteLine(names[i]);
}

// ใช้ foreach แบบล้ำสมัย (ข้อดี: สั้น อ่านง่าย / ข้อเสีย: เราจะไม่รู้เลขห้อง Index ของมัน)
foreach (string person in names) {
    Console.WriteLine(person);
}
```

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่างที่ 1: ระบบจัดการคลังแสง (Inventory System)

โค้ดนี้สาธิตวัฏจักรชีวิตของ Array ทั้งหมด: การสร้างช่องว่าง, การสวมข้อมูล, การแก้ไขข้อมูลเฉพาะจุด, และการกวาดล้างโชว์ข้อมูลด้วยลูป

::: code-group
```csharp [Program.cs]
using System;

namespace InventoryDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบกระเป๋าตัวละคร (RPG Inventory) ===\n");

            // 1. สร้างโกดังกระเป๋าขนาด 4 ช่องว่างๆ (เก็บได้แต่ string)
            string[] inventory = new string[4];
            
            // 2. จับไอเทมยัดลงช่องต่างๆ (จำไว้! ช่องแรกคือเลข 0)
            inventory[0] = "ดาบไม้เก่าๆ";
            inventory[1] = "ยาฟื้นพลัง (Red Potion)";
            inventory[2] = "ศิลาเวทมนตร์";
            inventory[3] = "แผนที่ขาดๆ";
            
            // 💥 ระวัง! ถ้าคุณเผลอเขียน inventory[4] = "แหวน"; โปรแกรมจะแครชทันที!

            // 3. จำลองเหตุการณ์: ผู้เล่นกินยาฟื้นพลังไปแล้ว! (ช่องที่ 1 ต้องเปลี่ยนสถานะ)
            Console.WriteLine("[ระบบ] ผู้เล่นกินยาฟื้นพลังไปแล้ว!");
            inventory[1] = "[ช่องว่าง]"; // เสียบข้อมูลใหม่ทับลงไปที่เดิม

            // 4. การแสดงผล (Traversal)
            Console.WriteLine($"\nกระเป๋ามีขนาดทั้งหมด {inventory.Length} ช่อง:");
            Console.WriteLine("-----------------------------------");

            // ใช้ for loop เพราะเราอยากให้มันแสดงหมายเลขช่อง (Index) คู่ไปด้วย
            for (int i = 0; i < inventory.Length; i++)
            {
                Console.WriteLine($"ช่องที่ {i} : {inventory[i]}");
            }
            Console.WriteLine("-----------------------------------");
        }
    }
}
```
:::

### ตัวอย่างที่ 2: ระบบคัดกรองคะแนนและ Sorting

ในความเป็นจริง Array ไม่ได้มีดีแค่เก็บของ แต่ Class `System.Array` ของไมโครซอฟต์ มีวิชาเวทมนตร์ (Static Methods) ที่ช่วยให้เราจัดการโกดังได้อย่างง่ายดาย

::: code-group
```csharp [Program.cs]
using System;
using System.Linq; // นำเข้าเครื่องมือคำนวณขั้นสูง (จะเรียนลึกซึ้งในบท LINQ)

namespace ScoreAnalyzer
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบวิเคราะห์สถิติคะแนนสอบ ===");

            // ชุดข้อมูลดิบ (ไม่ได้เรียงลำดับ)
            double[] scores = { 45.5, 92.0, 78.5, 100.0, 55.0, 88.5 };

            Console.WriteLine("\n[1] คะแนนดิบทั้งหมด:");
            // เราใช้ foreach ได้เลย เพราะเราแค่อยากจะพิมพ์มันออกมา ไม่สนใจพิกัด Index
            foreach (double s in scores)
            {
                Console.Write($"{s} | ");
            }
            Console.WriteLine();

            // ----------------------------------------------------
            // เวทมนตร์ของ System.Array (เครื่องมือจัดระเบียบ)
            // ----------------------------------------------------
            
            // Array.Sort() จะเข้าไปสลับของในกล่องให้เรียงจาก "น้อย ไป มาก" อย่างถาวร!
            Array.Sort(scores); 

            // สมมติเราอยากได้คะแนน "มาก ไป น้อย" (Top Scores) เราก็เอาที่เรียงแล้วมากลับหลังหันซะ!
            Array.Reverse(scores);

            Console.WriteLine("\n[2] จัดอันดับ Top Scores (มากไปน้อย):");
            for (int i = 0; i < scores.Length; i++)
            {
                // ตกแต่งนิดหน่อย อันดับ 1, 2, 3 ก็เอา Index (i) มาบวก 1 ซะ
                Console.WriteLine($"อันดับ {i + 1}: {scores[i]} คะแนน");
            }

            // ----------------------------------------------------
            // เวทมนตร์ของ System.Linq (เครื่องมือวิเคราะห์ยอดฮิต)
            // ----------------------------------------------------
            Console.WriteLine("\n[3] บทสรุปสถิติ (Statistics):");
            Console.WriteLine($"> คะแนนสูงสุด: {scores.Max()}");
            Console.WriteLine($"> คะแนนต่ำสุด: {scores.Min()}");
            Console.WriteLine($"> คะแนนเฉลี่ย: {scores.Average():N2}");
        }
    }
}
```
:::

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

### **โจทย์ที่ 1: เพชฌฆาต IndexOutOfRange**
พิจารณาโค้ดด้านล่างนี้ มีบั๊กเพชฌฆาตซ่อนอยู่ 1 จุดที่คอมไพเลอร์มองไม่เห็น (รันผ่าน) แต่พอกดรันแล้วแอปจะแครชเด้งออกทันที (Runtime Error) จงหาให้เจอและแก้ไขให้ถูกต้อง!
```csharp
int[] numbers = { 10, 20, 30, 40, 50 };

for (int i = 0; i <= numbers.Length; i++)
{
    Console.WriteLine(numbers[i]);
}
```

### **โจทย์ที่ 2: ระบบตรวจหาสระภาษาอังกฤษ (The Vowel Hunter)**
**ภารกิจ:** จงประกาศ Array แบบ `char[]` เพื่อเก็บสระภาษาอังกฤษทั้ง 5 ตัว (a, e, i, o, u)
จากนั้นให้รับข้อความ Input ภาษาอังกฤษจากผู้ใช้ (สมมติว่าเป็นคำว่า "Hello")
ให้ใช้ `foreach` loop วิ่งวนตรวจดูอักษรทุกตัวในข้อความนั้น (ความลับ: `string` ใน C# สามารถมองเป็นอาร์เรย์ของ `char` ได้เลย!) 
และให้โปรแกรมสรุปออกมาว่า "ในคำนั้น มีสระรวมกันทั้งหมดกี่ตัว"

::: details 💡 คำใบ้และแนวทาง (Hints)
- **สำหรับโจทย์ 1:** บั๊ก Off-by-one! ดูที่สัญลักษณ์ `<= numbers.Length` ให้ดี Array มี 5 ตัว ความยาว (Length) คือ 5 ถ้าใช้ `<=` มันจะวิ่งรอบสุดท้ายที่ Index=5 ซึ่งใน Array มีแค่ช่อง 0, 1, 2, 3, 4 (รวม 5 ตัว) ไม่มีช่องเบอร์ 5! ทางแก้คือลบเครื่องหมาย `=` ทิ้งไปซะ
- **สำหรับโจทย์ 2:** คุณจะต้องใช้ Loop 2 ชั้น! ลูปนอกคือ `foreach (char c in userInput.ToLower())` และลูปในคือการเอาอักษรตัวนั้นไปเช็คว่ามันตรงกับใครใน Array 5 ตัว (หรือใช้เวทมนตร์ `Array.IndexOf()` หรือ `vowels.Contains(c)` ก็ได้ถ้าอยากโกง)
:::

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: การสลับร่างข้อมูลโดยไม่ง้อไลบรารี (The Pure Array Reversal)**

ในตัวอย่างที่แล้ว เราใช้คำสั่งสบายๆ อย่าง `Array.Reverse(scores)` เพื่อกลับหลังหันข้อมูล 
แต่นักเรียนวิทยาการคอมพิวเตอร์ (Computer Science) ต้องเข้าใจเบื้องหลังการสลับข้อมูลด้วยตัวเอง!

**ภารกิจของคุณ:** 
ให้สร้าง Array ของตัวเลข `int[] data = { 1, 2, 3, 4, 5, 6, 7 };`
**ห้ามใช้** `Array.Reverse()` เด็ดขาด! ให้คุณใช้ `for` loop และวิชา "แก้วน้ำใบที่ 3 (Temporary Variable)" เพื่อสลับข้อมูลทั้งหมดกลับหลังหัน (เป้าหมายคือให้มันกลายเป็น { 7, 6, 5, 4, 3, 2, 1 })

**คำแนะนำจากสถาปนิก (Algorithm Logic):**
- ถ้าคุณวิ่ง `for` loop กวาดหมดตั้งแต่ช่อง 0 ยันช่องสุดท้าย เพื่อสลับหน้าไปหลัง... ท้ายที่สุด อาร์เรย์ของคุณจะ "กลับมาหน้าตาเหมือนเดิมเป๊ะ!" (เพราะมันสลับซ้อนสลับ)
- คุณต้องวิ่ง `for` loop แค่ **"ครึ่งเดียว (Length / 2)"** เท่านั้น!
  - รอบแรก: หยิบกล่องซ้ายสุด (index `0`) สลับกับกล่องขวาสุด (index `Length - 1`)
  - รอบสอง: หยิบกล่องถัดมา (index `1`) สลับกับกล่องรองสุดท้าย (index `Length - 1 - 1`)
  - ทยอยบีบเข้าหากันตรงกลางเรื่อยๆ จนจบ!

ใครทำโจทย์ข้อนี้ได้ แสดงว่าคุณมีสายเลือด Algorithm ของโปรแกรมเมอร์สายฮาร์ดคอร์ไหลเวียนอยู่เต็มเปี่ยม!

ถ้าคุณมีอาร์เรย์ 10 ช่อง และต้องการเพิ่มเป็น 15 ช่อง สิ่งที่คอมพิวเตอร์ทำเบื้องหลัง (เช่นในภาษา C) คือการ "เทปูนสร้างโกดังใหม่เบอร์ 15 ช่องว่างๆ", แล้วเอารถบรรทุกมา "ขนของจากโกดัง 10 ช่องเก่า ไปยัดใส่โกดังใหม่", แล้วค่อย "ทิ้งระเบิดทำลายโกดังเก่าทิ้งไป" (ซึ่งเปลืองพลังงาน CPU สูงมาก) 
ในบทต่อไปของ C# เราจึงมีนวัตกรรมที่ชื่อว่า `List<T>` (อาร์เรย์ยางยืด) ซึ่งมันจะจัดการวงจรทุบโกดังย้ายของแบบนี้ให้เราอยู่เบื้องหลังแบบอัตโนมัติ โดยที่เราไม่ต้องลงมือเขียนโค้ดย้ายของเอง!

**คำถาม 2:** ถ้าเราโยน Array ผ่าน Parameter ของ Method (เช่น `ProcessData(int[] myData)`) ข้อมูลที่ถูกส่งเข้าไปจะเป็นการ ถ่ายเอกสาร (Pass by Value) หรือ เป็นการส่งกุญแจ (Pass by Reference)?
**แนวคำตอบ:** **เป็นการส่งกุญแจ (Pass by Reference)!**
ย้อนกลับไปตอนต้นบท เราคุยกันว่า Array เป็น Reference Type ดังนั้นเมื่อเราโยนมันเข้าไปใน Method คอมไพเลอร์ไม่ได้ก๊อปปี้ตัวเลข 1,000 ตัวไปให้ Method นะ (เพราะมันจะแลกมาด้วยความช้าสุดยอด) แต่มันจะก๊อปปี้ **"พิกัดแผนที่ (Pointer)"** ส่งไปให้แทน!
ดังนั้น หาก Method ดันทะลึ่งไปสั่ง `myData[0] = 999;` ข้อมูลช่องแรกใน Main ของคุณจะเปลี่ยนเป็น 999 ถาวรทันที! (จงระวังจุดนี้ให้ดีเวลาส่ง Array ให้คนอื่นใช้งาน)
