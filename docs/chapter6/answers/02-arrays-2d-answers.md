# เฉลย: Arrays (2D) - ตารางแบนราบ

## 🎯 เฉลย Mini Exercise

### **โจทย์ที่ 1: ชนขอบตารางทะลุมิติ**
**คำถาม:** ถ้าจองที่ `int[,] grid = new int[10, 5];` จะเข้าไปหยิบช่องขวาล่างสุด (ช่องสุดท้าย) ต้องพิมเลขอะไร? `grid[?, ?]`

**เฉลย:**
ตามกฎ Zero-based Indexing 
- แกน Y (แถว / Row) มี 10 ช่อง -> นับตั้งแต่เลข 0 ถึง **9**
- แกน X (คอลัมน์ / Col) มี 5 ช่อง -> นับตั้งแต่เลข 0 ถึง **4**

ดังนั้น มุมขวาล่างสุด (สุดท้ายของตาราง) จะอยู่ที่พิกัด: **`grid[9, 4]`** เสมอ!
(สูตรลับวิศวกร: ช่องสุดท้ายคือ `[Row.Length - 1, Col.Length - 1]`)

### **โจทย์ที่ 2: ระบบตรวจตั๋วผี (Matrix Summation)**
**เป้าหมาย:** บวกรวมตัวเลขทุกกล่องใน 2D Array ให้กลายเป็นยอดสุทธิ

**เฉลยและวิธีทำ:**
เราต้องใช้กระบวนการ "ปูพรมค้นหา (Row-Major Traversal)" โดยวิ่งลูปทะลวงเข้าไปทีละแถว และในแต่ละแถวก็วิ่งทะลวงคอลัมน์จากซ้ายไปขวาให้จบ

```csharp
using System;

class Program
{
    static void Main()
    {
        // ฐานข้อมูลยอดขายตั๋ว (แถว=พนักงาน, คอลัมน์=ยอดแต่ละวัน)
        int[,] tickets = {
            { 5, 2, 8 },
            { 1, 0, 3 }
        };

        // ตัวแปรสะสมยอด (Accumulator)
        int totalTickets = 0;

        // ลูปชั้นนอก (Rows)
        for (int row = 0; row < tickets.GetLength(0); row++)
        {
            // ลูปชั้นใน (Cols)
            for (int col = 0; col < tickets.GetLength(1); col++)
            {
                // นำค่าในกล่องปัจจุบัน สาดทับเข้าไปในยอดรวม
                totalTickets += tickets[row, col];
            }
        }

        Console.WriteLine($"[ระบบบัญชี] ยอดรวมการขายตั๋วทั้งหมด: {totalTickets} ใบ");
    }
}
```

## 🔥 เฉลย Challenge (การบวกเมทริกซ์ทะลุแกน Matrix Diagonal Summation)

**เป้าหมาย:** เขียนโปรแกรมเพื่อหาผลบวก "เฉพาะตัวเลขในแนวเส้นทแยงมุมหลัก" (มุมซ้ายบน ทะลุ มุมขวาล่าง) โดยใช้ประสิทธิภาพสูงสุด!

**กระบวนการคิดระดับสถาปนิก (The Algorithm Optimization):**
ตารางตัวอย่างคือ
1 2 3
4 5 6
7 8 9
เป้าหมายคือบวกเลข 1, 5, 9 เข้าด้วยกัน

- **ถ้าคุณเป็นมือใหม่ (Brute Force):** คุณจะใช้ Nested Loop `for(row) > for(col)` เพื่อวิ่งครบทั้ง 9 ช่อง แล้วใส่ `if (row == col)` เพื่อดักจับพิกัด (0,0), (1,1), (2,2) เอามาบวกกัน 
วิธีนี้ทำงานได้จริง... แต่มัน **"สูญเปล่า"** มาก! (เพราะคอมพิวเตอร์ต้องเดินสำรวจช่องอื่นๆ ที่เหลืออีก 6 ช่องไปฟรีๆ โดยไม่เกิดประโยชน์) ถ้าตารางใหญ่ขนาด 10,000 x 10,000 คอมพิวเตอร์จะต้องเดินตั้ง 100 ล้านรอบ!

- **ถ้าคุณเป็นโปรแกรมเมอร์ Data Science (O(N) Optimization):** คุณจะมองเห็น "ความสมมาตรทางคณิตศาสตร์" ว่าพิกัดที่เราต้องการ มันมีเลขแกน X และแกน Y "เท่ากันเป๊ะ" เสมอ! 
ดังนั้น เราใช้ลูปแค่ชั้นเดียว แล้วเอาตัวแปร `i` โยนให้ทั้งสองแกนไปเลย!

```csharp
using System;

namespace DiagonalSum
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบหาผลรวมเส้นทแยงมุม (Diagonal Matrix) ===\n");

            int[,] matrix = {
                { 1, 2, 3 },
                { 4, 5, 6 },
                { 7, 8, 9 }
            };

            int diagonalSum = 0;

            // -----------------------------------------------------------
            // THE OPTIMIZED ALGORITHM (ความเร็วระดับแสง!)
            // -----------------------------------------------------------
            // วิ่งแค่ 3 รอบ (ตามความยาวของตาราง)
            for (int i = 0; i < matrix.GetLength(0); i++)
            {
                // ดึงพิกัด (0,0) -> (1,1) -> (2,2) มาใช้งานโดยตรง!
                int currentValue = matrix[i, i];
                
                diagonalSum += currentValue;

                Console.WriteLine($"[Log] ขุดเจอตัวเลข {currentValue} ที่พิกัด ({i},{i})");
            }

            Console.WriteLine($"\n✅ สรุปผลรวมเส้นทแยงมุมหลัก = {diagonalSum}");
            // คำตอบต้องได้ 15 เป๊ะ!
        }
    }
}
```

โปรแกรมเมอร์ที่บริษัทชั้นนำต้องการ คือคนที่เขียนโค้ดแล้วตั้งคำถามกับตัวเองเสมอว่า "เราสามารถทำให้มันหาคำตอบได้เร็วกว่านี้ ประหยัด CPU กว่านี้ ได้อีกหรือไม่?"
