## 📖 ภาพรวมของโปรแกรม (Program Overview)

ในฐานะคุณครูประจำวิชา C# คุณต้องเก็บคะแนนสอบของนักเรียนแต่ละคน ซึ่งตลอดเทอมจะมีการสอบหลายรอบ (คะแนนเก็บ, มิดเทอม, ไฟนอล)
- ถ้าใช้ Array คุณจะต้องเจอปัญหา "สอบกี่รอบไม่รู้" ทำให้จองกล่องไม่ได้
- ถ้าใช้ List เฉยๆ คุณก็ไม่รู้ว่าคะแนนก้อนไหนเป็นของใคร
- ถ้าใช้ Dictionary `(Key=ชื่อ, Value=คะแนน)` แบบบทที่แล้ว เด็ก 1 คนก็จะมีได้แค่ 1 คะแนนสอบเท่านั้น!

**ความลับขั้นสุดยอด:** เราสามารถยัด "โกดัง (List)" ลงไปใน "ตู้เซฟ (Dictionary)" ได้! 
โครงสร้างข้อมูลของเราจะมีหน้าตาแบบนี้: **`Dictionary<string, List<double>>`** 
(แปลว่า: ถ้าคุณเอา กุญแจชื่อนักเรียน ไปไขตู้ คุณจะไม่ได้ตัวเลข... แต่คุณจะได้ **กล่องยางยืดทั้งใบ** ที่มีคะแนนสอบทั้งหมดของนักเรียนคนนั้นหล่นออกมา!)

### ตัวอย่างหน้าจอเมื่อรันโปรแกรมสำเร็จ

```text
======================================
     [ STUDENT GRADE MANAGER ]

[1] 📝 เพิ่มรายชื่อนักเรียนใหม่
[2] ➕ กรอกคะแนนสอบให้นักเรียน
[3] 📊 สรุปผลการเรียนทั้งหมด
[4] ❌ ออกจากระบบ
กรุณาเลือกเมนู: 2

>> กรุณากรอกชื่อนักเรียน: สมหญิง

✅ บันทึกคะแนน 85.5 ให้ 'สมหญิง' สำเร็จ!

(กด Enter เพื่อกลับไปเมนูหลัก)

// -- เมื่อกดเข้าเมนู 3 (สรุปผลการเรียน) --
======================================
       [ สรุปผลการเรียนทั้งหมด ]
======================================
🧑 สมชาย -> คะแนน: [ 70.0 | 80.0 | ] | ค่าเฉลี่ย: 75.00
👩 สมหญิง -> คะแนน: [ 85.5 | 92.0 | ] | ค่าเฉลี่ย: 88.75
--------------------------------------
```

## 📋 กฎกติกาและเกณฑ์การประเมิน (Business Requirements)

คุณต้องพัฒนาเมธอด (Methods) ต่างๆ ขึ้นมาเพื่อรับผิดชอบแต่ละเมนูอย่างเป็นสัดส่วน โดยมีกฎดังนี้:

1. **โครงสร้างข้อมูลหลัก (The Core Database):**
   - ต้องประกาศตัวแปร `Dictionary<string, List<double>> studentDB` เป็นศูนย์กลาง
   - *(เนื่องจากเรายังไม่เรียนเรื่อง Class ระดับสูง อนุญาตให้ประกาศ Dictionary ตัวนี้ไว้ใน `Main` แล้วค่อยโยนแบบ Pass-by-Reference หรือส่งตัวแปรลงไปให้ Method อื่นๆ ใช้งาน)*

2. **เมธอด `RegisterStudent` (เมนู 1):**
   - รับ `Dictionary` และชื่อนักเรียนที่รับจากแป้นพิมพ์
   - ต้องเช็คก่อนว่า `ContainsKey` หรือไม่? ถ้ามีแล้วให้ด่าว่า "มีนักเรียนชื่อนี้ในระบบแล้ว"
   - ถ้ายังไม่มี ให้ทำการ `Add(ชื่อนักเรียน, new List<double>())` (สำคัญมาก! ต้องเอา List เปล่าๆ ผูกติดไปกับชื่อเด็กด้วย ไม่งั้นจะเกิด NullReferenceException ตอนกรอกคะแนน)

3. **เมธอด `AddScore` (เมนู 2):**
   - รับ `Dictionary`, ชื่อนักเรียน, และคะแนนสอบ (`double`)
   - ต้องเช็คว่าเด็กคนนี้มีตัวตนไหม ถ้าไม่มีให้ฟ้อง Error
   - ถ้ามี ให้ดึง List ของเด็กคนนั้นออกมา แล้วใช้คำสั่ง `.Add(คะแนน)` เพื่อยัดเลขลงไปท้ายแถว!

4. **เมธอด `ShowReport` (เมนู 3):**
   - ต้องใช้ `foreach` วิ่งไล่กวาดนักเรียนทุกคนในระบบ
   - ภายในลูปของเด็กแต่ละคน ต้องมีลูป `foreach` ชั้นที่ 2 เพื่อวิ่งกวาด "คะแนนทั้งหมด" ใน List ของเด็กคนนั้น
   - ให้คำนวณ "ค่าเฉลี่ย (Average)" ของเด็กคนนั้นออกมาโชว์ให้สวยงาม (ถ้าเด็กคนนั้นยังไม่เคยสอบเลยแม้แต่รอบเดียว ให้โชว์คำว่า "N/A" หรือ 0 แทน เพื่อป้องกันการหารด้วยศูนย์)

**ภารกิจ The Auto-Sorting Leaderboard:**
