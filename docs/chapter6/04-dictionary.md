# `Dictionary<K,V>` (บัญชีหนังหมา)

> 💡 **เป้าหมาย:** ทำความรู้จักกับสุดยอดโครงสร้างข้อมูลที่ "ค้นหาข้อมูลได้ไวที่สุดในโลก" (ความเร็วคงที่ O(1) Time Complexity) ผ่านสถาปัตยกรรม "แม่กุญแจ (Key) และ ลูกกุญแจ (Value)" ซึ่งเปรียบเสมือนสมุดหน้าเหลือง หรือระบบพจนานุกรมในชีวิตจริง

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. วิกฤติของ List ในการค้นหาข้อมูล (The Linear Search Bottleneck)

สมมติคุณมี `List` เก็บประวัติพนักงาน 100,000 คน วันหนึ่งเจ้านายสั่งให้ค้นหา "เงินเดือนของพนักงานรหัส EMP-999"
ถ้าใช้ List คุณต้องเขียนลูป `foreach` กวาดสายตาไล่อ่านรหัสพนักงานทีละคนตั้งแต่คนที่ 1 ไปจนถึงคนที่ 100,000 (เรียกการค้นหาแบบนี้ว่า Linear Search หรือ $O(N)$) ซึ่งทำให้โปรแกรมช้ามากเมื่อข้อมูลระดับ Big Data

โครงสร้าง **`Dictionary<TKey, TValue>`** เกิดมาเพื่อฆ่าปัญหานี้! มันคือกล่องมหัศจรรย์ที่ทำงานเหมือนล็อกเกอร์ฝากของ:
คุณเอารหัสบัตร (Key) ไปแสกนที่ตู้ปุ๊บ ตู้จะดีดเปิดและให้ของข้างใน (Value) แก่คุณ "ทันทีทันใด" โดยไม่ต้องไปนั่งไล่เปิดตู้คนอื่นเลย! (ความเร็วคงที่ $O(1)$ เร็วทะลุแสง!)

---

### 2. ไวยากรณ์แห่งคู่หู Key-Value (The Architecture)

Dictionary บังคับให้คุณต้องประกาศชนิดข้อมูล (Type) 2 ก้อนเสมอ:
- **`TKey` (กุญแจ):** ป้ายชื่อที่เอาไว้ใช้ค้นหา **(กฎเหล็ก: ห้ามมีกุญแจซ้ำกันในโกดังเด็ดขาด!)** ส่วนใหญ่มักเป็น `string` หรือ `int`
- **`TValue` (ของรางวัล):** ข้อมูลจริงที่เราอยากเก็บไว้ (กุญแจคนละดอก สามารถไขได้ของรางวัลซ้ำกันได้ ไม่ผิดกฎ)

```csharp
// ต้อง using System.Collections.Generic;

// สร้างสมุดโทรศัพท์: Key เป็นชื่อคน (string) -> Value เป็นเบอร์โทร (string)
Dictionary<string, string> phonebook = new Dictionary<string, string>();

// เพิ่มข้อมูล (Add) 
// สังเกตว่าวงเล็บต้องโยนของเข้าไป 2 ชิ้นคู่กันเสมอ
phonebook.Add("John", "081-123-4567");
phonebook.Add("Alice", "099-987-6543");

// 💥 ระวังแครช! ถ้าคุณฝืน Add("John", "011..."); 
// โปรแกรมจะระเบิด เพราะมีกุญแจชื่อ John อยู่ในระบบแล้ว!
```

---

### 3. การดึงของออกและกลไกดักจับ (Retrieval & Safe Checks)

การจะล้วงของออกมาจาก Dictionary ง่ายเหมือนการหยิบของจาก Array เลย! 
เพียงแค่เปลี่ยนจากการใส่พิกัดเลขช่อง `[0]` มาเป็นการใส่ชื่อกุญแจเข้าไปตรงๆ `["John"]`

```csharp
// การดึงข้อมูลออกมาใช้ (เหมือน Array แต่ใช้ตัวอักษรค้นหาได้!)
string johnsNumber = phonebook["John"]; 

// 💥 แต่ระวังอีกแล้ว! ถ้าคุณขอไขกุญแจที่ไม่มีอยู่จริง เช่น phonebook["David"];
// โปรแกรมจะระเบิดหน้าหงาย (KeyNotFoundException) ทันที!
```

วิศวกรระดับโปร จะไม่ไขกุญแจสุ่มสี่สุ่มห้า แต่จะต้อง "เคาะประตูก่อน" ด้วยคำสั่ง **`ContainsKey`** (เช็คว่ามีกุญแจนี้ไหม) หรือใช้วิชาขั้นสูง **`TryGetValue`** (ไขไปด้วยเช็คไปด้วย)

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่างที่ 1: ระบบจัดการคะแนนผู้เล่นออนไลน์ (Server Scoreboard)

โค้ดนี้สาธิตวิธีการตรวจสอบข้อมูล, อัปเดตข้อมูลเก่า, และเพิ่มข้อมูลใหม่ ลงใน Dictionary อย่างรัดกุม

::: code-group
```csharp [Program.cs]
using System;
using System.Collections.Generic;

namespace DictionaryDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== บัญชีคะแนนสะสม (Score Registry) ===\n");

            // Key = ชื่อผู้เล่น (string) | Value = คะแนน (int)
            Dictionary<string, int> serverScores = new Dictionary<string, int>();

            // 1. เพิ่มผู้เล่นใหม่เข้าสู่ระบบ
            serverScores.Add("Arthur", 1500);
            serverScores.Add("Merlin", 3200);

            // 2. จำลองเหตุการณ์: ผู้เล่นสู้กันได้คะแนนเพิ่ม
            string targetPlayer = "Arthur";

            // ก่อนจะบวกคะแนน ต้องตรวจเช็คให้ชัวร์ก่อนว่า Arthur มีตัวตนในระบบจริง (Guard check)
            if (serverScores.ContainsKey(targetPlayer))
            {
                // เข้าถึงเพื่อเอาของเก่ามาบวกเพิ่ม แล้วยัดเก็บลงกล่องเดิม
                serverScores[targetPlayer] += 50; 
                Console.WriteLine($"[อัปเดต] {targetPlayer} ได้คะแนนเพิ่มเป็น {serverScores[targetPlayer]}");
            }
            
            // 3. จำลองเหตุการณ์: เพิ่มคนใหม่ หรือ แก้ไขคนเก่า ด้วย Indexer []
            // การใช้เครื่องหมายปีกกา [ ] ในการ Add ถือเป็นท่าไม้ตายที่ปลอดภัยที่สุด:
            // - ถ้า "ไม่มี" Lancelot อยู่ในระบบ -> มันจะ Add ให้ใหม่เลย
            // - ถ้า "มี" Lancelot อยู่แล้ว -> มันจะไม่แครช! แต่มันจะเขียนค่าใหม่ "ทับ (Overwrite)" ของเก่าไปเลย
            serverScores["Lancelot"] = 999; 
            
            Console.WriteLine("\n[สรุปกระดานผู้นำทั้งหมด]");
            
            // 4. การกวาดล้างโชว์ข้อมูลด้วย foreach
            // ข้อมูลที่หลุดออกมาจาก Dictionary แต่ละรอบ จะอยู่ในรูปแบบของวัตถุแพคคู่ (KeyValuePair)
            foreach (KeyValuePair<string, int> player in serverScores)
            {
                // เข้าถึง Key และ Value แยกส่วนกันได้
                Console.WriteLine($"> {player.Key} : {player.Value} แต้ม");
            }

            // แถม: ถ้าอยากวนลูปเฉพาะ "รายชื่อผู้เล่น (Key ล้วนๆ)" ไม่สนคะแนน
            /*
            foreach (string name in serverScores.Keys) {
                Console.WriteLine(name);
            }
            */
        }
    }
}
```
:::

### ตัวอย่างที่ 2: ระบบพจนานุกรมอัจฉริยะ (The `TryGetValue` Hack)

การใช้ `ContainsKey` ตามด้วยการล้วงของด้วย `[ ]` เป็นการเสียเวลาแสกนตู้ 2 รอบ 
สถาปนิกขั้นเทพจะใช้เมธอด **`TryGetValue` (ลองพยายามไขกุญแจดู)** ซึ่งรวมร่างฟีเจอร์ "ดักจับบวกล้วงของผ่านสายยาง out" ไว้ในบรรทัดเดียว!

::: code-group
```csharp [Program.cs]
using System;
using System.Collections.Generic;

namespace SmartDictionary
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== พจนานุกรมคำศัพท์ IT ===\n");

            Dictionary<string, string> dict = new Dictionary<string, string>()
            {
                // การกำหนดค่าเริ่มต้นแบบกลุ่ม (Collection Initializer)
                { "API", "ช่องทางเชื่อมต่อระหว่างซอฟต์แวร์" },
                { "Bug", "ข้อผิดพลาดในโปรแกรม" },
                { "Cache", "พื้นที่เก็บข้อมูลชั่วคราวเพื่อความเร็ว" }
            };

            Console.Write("กรุณาป้อนคำศัพท์ที่ต้องการค้นหา (เช่น API, Bug): ");
            string searchKey = Console.ReadLine();

            // ท่าไม้ตาย! TryGetValue จะพยายามหากุญแจ searchKey ในกล่อง
            // ถ้า "หาเจอ": คืนค่า true และโยนความหมายผ่านท่อ out มาใส่กล่อง definition
            // ถ้า "หาไม่เจอ": คืนค่า false รอดตายจากการแครชแบบฉิวเฉียด
            if (dict.TryGetValue(searchKey, out string definition))
            {
                Console.ForegroundColor = ConsoleColor.Green;
                Console.WriteLine($"[ค้นพบ] ความหมาย: {definition}");
            }
            else
            {
                Console.ForegroundColor = ConsoleColor.Red;
                Console.WriteLine($"[ไม่พบคำศัพท์] ขออภัย ไม่มีคำว่า '{searchKey}' ในฐานข้อมูล");
            }
            Console.ResetColor();
        }
    }
}
```
:::

### **โจทย์ที่ 1: ตู้ล็อกเกอร์กุญแจผี**
ถ้าเราสร้าง Dictionary เอาไว้เก็บราคาสินค้า: `Dictionary<string, double> prices = new Dictionary<string, double>();`
แล้วเกิดโค้ดแบบนี้ขึ้นมา:
```csharp
prices.Add("Water", 10.0);
prices["Cola"] = 15.0;
prices.Add("Water", 12.0);
prices["Cola"] = 20.0;
```
จงวิเคราะห์ว่า เมื่อรันโปรแกรมนี้ถึงบรรทัดสุดท้าย จะเกิดอาการแครชพังทลายขึ้นหรือไม่? ที่บรรทัดไหน? และถ้าสมมติว่าเอาบรรทัดที่แครชออกไป ราคาสุดท้ายของ "Cola" จะเป็นเท่าไหร่?

### **โจทย์ที่ 2: เครื่องนับคะแนนเสียงโหวต (The Election Counter)**
**ภารกิจ:** จงประกาศ `Dictionary<string, int>` ชื่อ `votes` (ชื่อคนเป็นกุญแจ, คะแนนโหวตเป็นแต้ม)
สมมติผู้คนแห่มาโหวตคะแนนโดยโยนชื่อเข้ากล่อง: `"Bob", "Alice", "Bob", "Charlie", "Alice", "Alice"`
ให้คุณเขียน `foreach` ลูปกวาดสายตาข้อมูลสมมติชุดนี้ แล้วเอาไปสับใส่ Dictionary โดยมีเงื่อนไขว่า:
- ถ้าชื่อนี้ "เพิ่งโหวตครั้งแรก (ไม่มีในระบบ)" ให้เพิ่มชื่อเข้าไป และตั้งแต้มเป็น 1
- ถ้าชื่อนี้ "เคยโหวตแล้ว (มีอยู่ในตู้เซฟแล้ว)" ให้บวกแต้มเก่าเพิ่มอีก 1
*(คำใบ้: ลองใช้ ContainsKey ดักหน้าประตูก่อนไขตู้เซฟดูสิ! โค้ดโจทย์ข้อนี้คือหัวใจหลักของการประมวลผล Big Data และ AI เชียวนะ)*

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: เครื่องนับสถิติความถี่คำศัพท์ (The Word Frequency Analyzer)**

นี่คือโปรแกรมตระกูลสืบสวนสอบสวนที่เอาไว้วิเคราะห์ว่า นักการเมืองหรืออินฟลูเอนเซอร์ ชอบพูดคำว่าอะไรบ่อยที่สุด!

**ภารกิจของคุณ:** 
1. รับ Input เป็นประโยคภาษาอังกฤษยาวๆ 1 ประโยค เช่น `"apple banana apple orange banana apple"` (สมมติว่าเป็นคำคั่นด้วยช่องว่าง)
2. สับประโยคนั้นให้ขาดออกจากกันเป็นชิ้นๆ (ใบ้ให้: ใช้คำสั่ง `string[] words = inputMessage.Split(' ');` จะช่วยสับช่องว่างแล้วป้อนลงอาร์เรย์ให้เอง)
3. กางตาราง `Dictionary<string, int>` ขึ้นมา 1 ใบ
4. วิ่งลูปกวาดชิ้นส่วนคำที่ถูกสับ แล้วเอามาใช้วิธี **"เช็คชื่อและบวกแต้ม (เหมือนโจทย์เครื่องนับคะแนนเสียงเป๊ะๆ)"** เพื่อหาว่าคำไหนโผล่มากี่รอบ
5. พอปั้น Dictionary จนครบถ้วน ให้ใช้ลูป `foreach` วงสุดท้าย วาดรายงานสรุปสถิติออกมาบนหน้าจอ

ถ้าทำได้ โปรแกรมคุณจะตอบได้เลยว่า "คำว่า apple พูดซ้ำ 3 ครั้ง, banana 2 ครั้ง, orange 1 ครั้ง"
การเปลี่ยนข้อมูลดิบ (Raw text) ให้กลายเป็นสถิติสรุป (Aggregated dictionary) คือก้าวแรกของการเข้าสู่สายงาน Data Scientist!

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** ในเอกสารคู่มือระบุว่า "Dictionary อาศัยกลไกที่เรียกว่า Hash Table ในการค้นหาด้วยความเร็ว O(1)" การทำ Hashing คืออะไร และทำไมมันถึงหาของเจอทันทีโดยไม่ต้องเรียงคิว?
**แนวคำตอบ:** **มันคือเวทมนตร์ของการแปลงชื่อเป็นรหัสพิกัด!** 
เมื่อเราโยนกุญแจ `"John"` เข้าไป มันไม่ได้เอาคำว่า John ไปเรียงไว้ในโกดัง แต่มันจะเอาคำว่า John ไปเข้า "เครื่องบดเนื้อ (Hash Function)" เพื่อสับเป็นตัวเลขทางคณิตศาสตร์ที่ตายตัว เช่น `0x4A5B` และคอมพิวเตอร์ก็จัดสรรกล่องเบอร์ `0x4A5B` เตรียมรอไว้เลย 
เมื่อไหร่ที่คุณอยากหาของ ก็แค่ป้อนคำว่า "John" เข้าเครื่องบดเนื้ออีกรอบ มันก็จะได้รหัส `0x4A5B` เป๊ะๆ ชี้ทะลุพิกัดไปถึงกล่องใบนั้นในแรมทันที! (ไม่ต้องสนใจเลยว่าในโกดังจะมีคนอีกหมื่นคนขวางอยู่ เพราะฉันรู้รหัสประตูวาปของฉัน!)

**คำถาม 2:** โค้ดด้านล่างนี้ เกิดปัญหาประสิทธิภาพที่เลวร้ายระดับหายนะ (Performance Issue) อย่างไร? และสถาปนิกซอฟต์แวร์อย่างคุณจะแก้มันยังไงให้เหลือแค่บรรทัดเดียว?
```csharp
if (myDict.ContainsKey("Username"))
{
    string user = myDict["Username"];
    Console.WriteLine(user);
}
```
**แนวคำตอบ:** **มันคือการ "ชกตู้เซฟฟรี 1 ที" แบบสูญเปล่า!**
คอมพิวเตอร์เสียแรงเอากุญแจ "Username" ไปปั่นหาพิกัด (Hashing) ในกระบวนการ `ContainsKey` 1 รอบ (เจอแล้วว่ามีของ)... แต่คุณกลับทิ้งผลลัพธ์นั้นไป แล้วบังคับให้โปรแกรมไปเข้ากระบวนการปั่นหากุญแจและชี้พิกัด `myDict["Username"]` ใหม่อีก 1 รอบเต็มๆ เพื่อหยิบของออกมา (ช้าลง 2 เท่าโดยไม่จำเป็น!)
**วิธีแก้ระดับเซียน:** เราถูกสอนวิชาดักของจากท่อ out มาแล้ว ดังนั้น เปลี่ยนโค้ดทั้งหมดนี้ด้วยบรรทัดเดียวคือ `if (myDict.TryGetValue("Username", out string user))` เท่านั้นจบ! เครื่องจักรจะหากุญแจรอบเดียว คว้าของติดมือมาด้วยเลย แถมประหยัดบรรทัดด้วย!
:::
