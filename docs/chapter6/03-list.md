# `List<T>` (โกดังยางยืด)

> 💡 **เป้าหมาย:** ทลายขีดจำกัดที่น่าอึดอัดที่สุดของ Array (การล็อคขนาดตายตัว) เข้าสู่นวัตกรรมแห่งยุคใหม่ที่เรียกว่า "Dynamic Array" ที่สามารถยืดขยายขนาดของตัวเองได้อัตโนมัติ พร้อมด้วยสารพัดเครื่องมือช่วยเหลือในการ เพิ่ม แทรก ลบ และค้นหา!

## 📖 ทฤษฎีและแนวคิดระดับลึก (In-Depth Theory & Concepts)

### 1. วิกฤติของ Array ทื่อๆ (The Fixed-Size Nightmare)

ในโลกของการพัฒนาเกม หรือระบบฐานข้อมูลจริง เราแทบจะ **ไม่รู้เลย** ว่าวันพรุ่งนี้จะมีผู้เล่นสมัครสมาชิกเพิ่มกี่คน หรือผู้เล่นจะเก็บไอเทมเข้ากระเป๋าไปกี่ชิ้น 
ถ้าเราใช้ `int[] players = new int[100];` วันใดที่คนสมัครคนที่ 101 เข้ามา ระบบก็จะแครชทันที (IndexOutOfRange)

ทางแก้คือต้องใช้ **`List<T>` (อาร์เรย์ยางยืด)** ซึ่งไมโครซอฟต์ได้สร้างขึ้นมาเพื่อเป็น "Wrapper (ตัวห่อหุ้ม)" ให้กับ Array อีกทีหนึ่ง 

---

### 2. เวทมนตร์เบื้องหลังการ "ยืดหด" (The Doubling Algorithm)

หลายคนคิดว่า `List` เป็นเวทมนตร์วิเศษ แต่ความจริงแล้ว **ไส้ในของ List ก็คือ Array ธรรมดาๆ นี่แหละ!**
เพียงแต่มันมีระบบ AI (Algorithm) แฝงอยู่เบื้องหลังคอยดูแลให้เรา:
1. ตอนแรกที่เราสร้าง List เปล่าๆ มันจะแอบสร้าง Array ขนาดจิ๋ว (เช่น 4 ช่อง) ไว้รอ
2. พอเราเติมของจนเต็ม 4 ช่องปุ๊บ แล้วพยายามยัดของชิ้นที่ 5 เข้าไป...
3. List จะสั่ง **"ระเบิดโกดังเก่าทิ้ง แอบเทปูนสร้างโกดังใหม่ที่ขนาดใหญ่เป็น 2 เท่า (8 ช่อง)"** จากนั้นก็เอารถบรรทุกขนข้อมูลเก่ามาเรียงไว้ให้ แล้วค่อยยัดของชิ้นที่ 5 ลงไป! (กระบวนการนี้เกิดขึ้นระดับไมโครวินาที โดยที่เรามองไม่เห็นเลย)

### 3. ไวยากรณ์แห่ง Generics `<T>` (ตัวกำหนด Data Type)

คุณอาจสงสัยว่าตัว `<T>` หลังคำว่า List คืออะไร? มันย่อมาจาก **Type (ชนิดข้อมูล)** 
มันเป็นฟีเจอร์ระดับสูง (Generics) เพื่อบอกคอมไพเลอร์ว่า "โกดังยางยืดใบนี้ อนุญาตให้เก็บของประเภทไหน?" (เพื่อป้องกันการเอาขยะมาปนกับอาหาร)

```csharp
// ต้องใส่ using System.Collections.Generic; ไว้บนสุดของไฟล์เสมอ!

// โกดังเก็บตัวเลข (เติมได้ไม่จำกัด)
List<int> scores = new List<int>();

// โกดังเก็บข้อความ
List<string> names = new List<string>();

// การใส่ข้อมูลเริ่มต้นให้ List (Collection Initializer)
List<double> prices = new List<double> { 19.9, 50.5, 99.0 };
```

**⚠️ ความแตกต่างที่ต้องจำให้ขึ้นใจ:**
Array ใช้คำว่า `Length` (ความยาวของโกดัง) 
แต่ List จะใช้คำว่า **`Count`** (จำนวนของที่มีอยู่จริงในโกดัง ณ ตอนนี้)

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่างที่ 1: ระบบตะกร้าสินค้า (Dynamic Shopping Cart)

โค้ดนี้สาธิตฟังก์ชันพื้นฐาน 4 จตุรเทพของ List ได้แก่ `Add` (ต่อท้าย), `Insert` (แทรกกลาง), `Remove` (ทำลายทิ้ง), และ `Clear` (ล้างบาง)

::: code-group
```csharp [Program.cs]
using System;
using System.Collections.Generic; // สิ่งนี้ขาดไม่ได้เด็ดขาดเมื่อใช้ List!

namespace DynamicCartDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบตะกร้าสินค้าอัจฉริยะ ===\n");

            // สร้างตะกร้ายางยืดว่างๆ 1 ใบ
            List<string> cart = new List<string>();

            // 1. กระบวนการ Add (โยนของใส่ท้ายตะกร้า)
            cart.Add("แอปเปิ้ล");
            cart.Add("กล้วย");
            cart.Add("ส้ม");

            Console.WriteLine($"[1] มีของในตะกร้า {cart.Count} ชิ้น");

            // 2. กระบวนการ Insert (แทรกคิวแซงหน้าคนอื่น)
            // สมมติแม่สั่งให้รีบซื้อ "นมสด" ด่วนที่สุด เอาไปเสียบไว้ที่ตำแหน่ง 0 เลย!
            // (ผลคือ แอปเปิ้ลและเพื่อนๆ จะถูกดันถอยหลังไป 1 ก้าวอัตโนมัติ)
            cart.Insert(0, "นมสด"); 
            Console.WriteLine($"[2] หลังแทรกคิวด่วน มีของ {cart.Count} ชิ้น");

            // 3. กระบวนการ Remove (หยิบของทิ้ง)
            // ไม่เอาแอปเปิ้ลแล้ว 
            // (ถ้ามีแอปเปิ้ลหลายลูก มันจะลบตัวแรกสุดที่เจอทิ้งตัวเดียว)
            cart.Remove("แอปเปิ้ล");

            // ถ้าอยากลบแบบระบุพิกัดที่แน่นอน (ลบของช่องที่ 2 ทิ้ง)
            cart.RemoveAt(2); // ณ ตอนนี้ช่องที่ 2 คือ "ส้ม" (เพราะนมสดเสียบมาดันแอปเปิ้ลหายไปแล้ว)

            Console.WriteLine("\n--- เช็คบิล (สินค้าที่เหลืออยู่) ---");
            
            // 4. การ Traverse ข้อมูล (อ่านออกมาโชว์)
            // List สามารถถูกเรียกใช้ [index] ได้เหมือน Array เป๊ะๆ!
            for (int i = 0; i < cart.Count; i++)
            {
                Console.WriteLine($"ชิ้นที่ {i + 1}: {cart[i]}");
            }

            // ใช้ foreach ก็ได้เช่นกัน (สะดวกกว่ามาก)
            /*
            foreach (string item in cart) {
                Console.WriteLine(item);
            }
            */

            // 5. กระบวนการล้างแค้น (Clear) เผาตะกร้าทิ้ง
            cart.Clear();
            Console.WriteLine($"\n[3] หลังเทของทิ้งหมด ตะกร้ามีของ = {cart.Count} ชิ้น");
        }
    }
}
```
:::

### ตัวอย่างที่ 2: ระบบ AI กรองข้อความ (The Word Filter)

นอกจากการเพิ่มและลดข้อมูลแล้ว `List` ยังมีคลังแสงสำหรับ "วิเคราะห์" ข้อมูลที่ทรงพลังกว่า Array หลายขุม (เช่น `.Contains()`)

::: code-group
```csharp [Program.cs]
using System;
using System.Collections.Generic;

namespace AutoFilterSystem
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== ระบบบอทกรองคำหยาบ (Chat Filter) ===\n");

            // สร้างฐานข้อมูลคำต้องห้าม
            List<string> bannedWords = new List<string> { "โง่", "เลว", "บ้า" };

            Console.Write("ผู้ใช้งานพิมพ์ข้อความ: ");
            string inputMessage = Console.ReadLine(); // สมมติพิมพ์ "แกมันบ้าจริงๆ"

            // ฟีเจอร์ Contains ช่วยให้เราหาของในกล่อง 100 ใบได้ในบรรทัดเดียว!
            bool isBanned = false;

            // ตรวจสอบทีละคำหยาบในฐานข้อมูล ว่ามีปรากฏอยู่ในข้อความคนพิมพ์ไหม
            foreach (string badWord in bannedWords)
            {
                if (inputMessage.Contains(badWord))
                {
                    isBanned = true;
                    break; // เจอคำด่าคำแรกก็แบนเลย ไม่ต้องหาต่อ
                }
            }

            if (isBanned)
            {
                Console.ForegroundColor = ConsoleColor.Red;
                Console.WriteLine("🚨 ระบบ AI ตรวจพบการใช้ถ้อยคำไม่สุภาพ ข้อความถูกระงับ!");
            }
            else
            {
                Console.ForegroundColor = ConsoleColor.Green;
                Console.WriteLine($"💬 ส่งข้อความสำเร็จ: {inputMessage}");
            }
            Console.ResetColor();
        }
    }
}
```
:::

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

### **โจทย์ที่ 1: แกะรอยความแตกต่าง**
ถ้าคุณเขียนโค้ดเพื่อสร้างโครงสร้างเก็บข้อมูลไว้ดังนี้:
1. `string[] names = new string[0];`
2. `List<string> nameList = new List<string>();`
เมื่อคุณเขียนคำสั่ง `names[0] = "John";` และ `nameList[0] = "John";` โปรแกรมจะทำงานสำเร็จหรือไม่? เพราะอะไรโครงสร้าง 2 แบบที่ดูว่างเปล่าเหมือนกัน ถึงมีพฤติกรรมต่างกันตอนเริ่มต้น?

### **โจทย์ที่ 2: เครื่องตัดเลขคู่ (Odd Filter)**
**ภารกิจ:** จงสร้าง `List<int>` และเติมเลข `1` ถึง `10` ลงไป
จากนั้น ให้คุณเขียนโปรแกรมวิ่งลูปไปใน List นี้ แล้วจัดการ **"เตะ (Remove)"** เลขที่เป็น "เลขคู่ (2, 4, 6, 8, 10)" ทิ้งไปให้หมด 
สุดท้ายให้ใช้ `foreach` วงที่สอง พิมพ์ตัวเลขที่หลงเหลืออยู่ใน List ออกมา (ควรจะเหลือแค่ 1, 3, 5, 7, 9)

::: details 💡 คำใบ้และแนวทาง (Hints)
- **สำหรับโจทย์ 1:** พังพินาศ (Crash) ทั้งคู่ครับ! Array ขนาด 0 แปลว่าคุณเทปูนสร้างโกดังทิพย์ (ไม่มีช่องให้เดินเข้าด้วยซ้ำ) ส่วน List ที่สร้างมาใหม่ๆ `Count` ของมันคือ 0 แปลว่ามันเป็นกล่องเปล่า การจะอ้างอิงตำแหน่ง `[0]` ใน List คุณต้องใช้ `nameList.Add("John");` เพื่อยัดของลงไปก่อนเท่านั้น! ห้ามใช้ `[0]` อ้างอิงกล่องทิพย์เด็ดขาด
- **สำหรับโจทย์ 2 (กับดักมรณะ!):** ถ้าคุณใช้ `foreach` หรือ `for` เดินลูป *ไปข้างหน้า (i++)* แล้วคุณสั่ง `RemoveAt(i)` โปรแกรมคุณจะแครชข้ามบรรทัด (หรือทำงานข้ามกล่อง)! เพราะเวลาคุณลบของ 1 ชิ้น ของข้างหลังจะถูก "ดูดให้ขยับมาข้างหน้า" สวนทางกับคุณที่เดินไปข้างหลัง ทำให้คุณข้ามของบางชิ้นไป (Skipping Bug) 
  *วิธีแก้ระดับสถาปนิก:* เวลาลบของออกจาก List รวดเดียวเยอะๆ **คุณต้องใช้ for loop นับถอยหลัง `(for int i = Count - 1; i >= 0; i--)` เท่านั้น!!** หรือใช้พลังพระเจ้า `.RemoveAll(x => x % 2 == 0);` ในบรรทัดเดียวไปเลย!
:::

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: ระบบจัดการคิวงานสุดหิน (The To-Do List Engine)**

นี่คือหัวใจของแอปพลิเคชันอย่าง Trello หรือ Asana 
**ภารกิจของคุณ:** จงสร้างโปรแกรม Console ที่มีระบบเมนูวนลูป `while(true)` ที่สามารถจัดการ List งาน 3 อย่างได้:

1. **เมนูเพิ่มงาน (Add Task):** ให้พิมพ์ชื่องาน แล้วยัดใส่ `List<string>`
2. **เมนูแสดงงาน (View Tasks):** ถ้า List ว่างเปล่า ให้บอกว่า "ไม่มีงานค้าง" แต่ถ้ามี ให้พิมพ์ออกมาทีละบรรทัด พร้อมตัวเลขนำหน้า (เช่น `[1] ซักผ้า`, `[2] เขียนโค้ด`) *อย่าลืมว่า Index เริ่มที่ 0 แต่โชว์คนดูต้องบวก 1*
3. **เมนูลบงาน (Complete Task):** ให้ผู้ใช้กรอก "หมายเลขนำหน้างาน (1, 2...)" ที่ทำเสร็จแล้ว ระบบต้องไปควานหาแล้วเอาออกจาก List (ระวัง! ผู้ใช้อาจจะกรอกเลข 99 ทั้งๆ ที่มีงานแค่ 2 ชิ้น คุณต้องดัก `if` ไว้กันแครชด้วย!)
4. **เมนูออก (Exit)** 

โปรแกรมนี้จะใช้ความรู้ตั้งแต่บทที่ 3 (Switch) บทที่ 4 (While Loop) ผสมกับ List ในบทนี้ เป็นสุดยอดข้อสอบประเมินผลกลางภาคที่สมบูรณ์แบบครับ!

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** ในเมื่อ `List<T>` เป็นโกดังยางยืดที่เก่งกว่า Array ทุกด้าน ทำไม Microsoft ถึงไม่ลบ Array ทิ้งไปจากภาษา C# เลยล่ะ? แล้วเราควรใช้ Array ในสถานการณ์ไหน?
**แนวคำตอบ:** **เพราะ "ความเร็ว (Performance) และ ความประหยัดแรม (Memory)" ครับ!**
List เป็นสถาปัตยกรรมระดับสูงที่ต้องจ่ายภาษี (Overhead) เบื้องหลังการทำงานมันต้องแบกฟังก์ชันจุกจิก และมีการจองแรมเผื่อล่วงหน้า (เผื่อจะขยาย) ทำให้มันอ้วนกว่าและทำงานช้ากว่า Array ในระดับเสี้ยวไมโครวินาที
- **จงใช้ Array** กับข้อมูลที่คุณ "รู้จำนวนล่วงหน้าแน่นอน และไม่มีวันเปลี่ยนแปลง" เช่น ชื่อวันทั้ง 7 ในสัปดาห์, รหัสไปรษณีย์ 77 จังหวัด, หรือข้อมูลภาพ Pixel 4K บนหน้าจอ (ซึ่งต้องการความเร็วระดับเทพ)
- **จงใช้ List** กับข้อมูลที่คุณ "ไม่รู้อนาคต" เช่น รายการลูกค้าที่กำลังเข้ามาสมัคร, สินค้าในตะกร้า, หรือกระสุนปืนในเกมที่ถูกยิงออกไปเรื่อยๆ

**คำถาม 2:** ถ้าสมมติว่า List ของเรามีขนาดปัจจุบัน (Capacity) อยู่ที่ 1,024 ช่อง แล้วเราดัน `Add` ของชิ้นที่ 1,025 เข้าไป เบื้องหลังคอมพิวเตอร์จะต้องทำอะไรบ้าง? (ตอบอ้างอิงจาก The Doubling Algorithm)
**แนวคำตอบ:** นี่คือเรื่องที่วิศวกรทุกคนต้องพึงระวัง (Performance Spike) 
สิ่งที่คอมพิวเตอร์ต้องทำคือ: (1) หาที่ว่างในแรมใหม่ขนาดใหญ่เป็นสองเท่าคือ 2,048 ช่อง (2) ทำการก๊อปปี้ข้อมูลเก่า 1,024 ตัว ลากไปวางในกล่องใหม่ทีละชิ้นๆ (3) นำของชิ้นที่ 1,025 ใส่ลงไป (4) สั่งเผาทำลายโกดังเก่า 1,024 ช่องทิ้งให้เกลี้ยง 
ทั้งหมดนี้จะทำให้กราฟการทำงานของ CPU ของคุณ "กระตุกพุ่งปรี๊ด" 1 จังหวะ! (วิธีแก้ระดับโปร: ถ้าเราพอจะรู้ล่วงหน้าว่าจะมีคนประมาณล้านคน เราสามารถสั่ง `List<string> users = new List<string>(1000000);` เพื่อจองล่วงหน้าตอนแรกสุดทีเดียวเลย จะช่วยเซฟ CPU ไปได้มหาศาลครับ!)
:::
