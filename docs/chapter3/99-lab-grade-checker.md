# 💻 Lab: Grade Checker Program

> 💡 **เป้าหมายของ Lab นี้:** โปรเจกต์นี้จะเป็นบททดสอบความเข้าใจทั้งหมดจากบทที่ 1 ถึง 3 เราจะสร้าง **ระบบตัดเกรดอัตโนมัติ (Automated Grade Checker)** ที่รับข้อมูลคะแนนของนักศึกษา ทำการตรวจสอบขอบเขตความถูกต้องของข้อมูล (Validation) และใช้ตรรกะแบบหลายชั้นเพื่อแยกแยะเกรด พร้อมทั้งแสดงผลลัพธ์เป็นหน้าตารายงานที่สวยงามด้วย ASCII Art!

---

## 📖 ภาพรวมของโปรแกรม (Program Overview)

โปรแกรมจะต้องโต้ตอบกับผู้ใช้งาน โดยสอบถามชื่อและคะแนนสอบ จากนั้นใช้ `if-else` ตรวจสอบความถูกต้องของคะแนน (ต้องไม่ต่ำกว่า 0 และไม่เกิน 100) หากคะแนนถูกต้อง จะนำไปประมวลผลเพื่อออกเกรดตามเกณฑ์มาตรฐาน และสรุปผลรายงานให้ผู้ใช้ทราบ

### ตัวอย่างหน้าจอเมื่อรันสำเร็จ (Happy Path)

```text
=================================================
       🎓 UNIVERSITY GRADE SYSTEM V.1.0 🎓
=================================================
กรุณากรอกชื่อนักศึกษา: อัษฎาวุธ ใจดี
กรุณากรอกคะแนนที่ได้ (0-100): 85

[ กำลังประมวลผลข้อมูล... ]

-------------------------------------------------
              📄 OFFICIAL TRANSCRIPT 📄
-------------------------------------------------
  ชื่อนักศึกษา : อัษฎาวุธ ใจดี
  คะแนนรวม   : 85 / 100
  ระดับผลการเรียน : A
  สถานะ      : 🌟 ผ่านเกณฑ์ยอดเยี่ยม!
-------------------------------------------------
```

### ตัวอย่างหน้าจอเมื่อข้อมูลผิดพลาด (Error Handling)

```text
=================================================
       🎓 UNIVERSITY GRADE SYSTEM V.1.0 🎓
=================================================
กรุณากรอกชื่อนักศึกษา: สมชาย โชคร้าย
กรุณากรอกคะแนนที่ได้ (0-100): 105

[ กำลังประมวลผลข้อมูล... ]

❌ ERROR: ข้อมูลคะแนนไม่ถูกต้อง!
กรุณาระบุคะแนนในช่วง 0 ถึง 100 เท่านั้น ระบบได้ทำการระงับการออกเกรด
```

## 📋 กฎกติกาและเกณฑ์การประเมิน (Business Rules)

คุณต้องใช้คำสั่งตรรกะ `if, else if, else` ในการแยกสายการประเมินผลตามเกณฑ์ต่อไปนี้อย่างเคร่งครัด:

1. **การตรวจสอบความถูกต้อง (Validation):**
   - คะแนนต้องไม่อยู่ในจุดที่ **น้อยกว่า 0** `(< 0)`
   - คะแนนต้องไม่อยู่ในจุดที่ **มากกว่า 100** `(> 100)`
   - หากเข้าข่ายผิดปกติ ให้แสดงข้อความ Error และหยุดการให้เกรดทันที

2. **เกณฑ์การให้เกรด (Grading Criteria):**
   - คะแนนตั้งแต่ **80 ถึง 100** -> ได้เกรด **A** (สถานะ: ผ่านเกณฑ์ยอดเยี่ยม!)
   - คะแนนตั้งแต่ **70 ถึง 79** -> ได้เกรด **B** (สถานะ: ผ่านเกณฑ์ดี)
   - คะแนนตั้งแต่ **60 ถึง 69** -> ได้เกรด **C** (สถานะ: ผ่านเกณฑ์ปานกลาง)
   - คะแนนตั้งแต่ **50 ถึง 59** -> ได้เกรด **D** (สถานะ: ผ่านเกณฑ์ขั้นต่ำ)
   - คะแนนตั้งแต่ **0 ถึง 49** -> ได้เกรด **F** (สถานะ: ไม่ผ่านเกณฑ์ ต้องลงเรียนใหม่)

## 📝 ขั้นตอนการพัฒนา (Step-by-Step Implementation)

### ขั้นตอนที่ 1: สร้างโปรเจกต์และเตรียมสภาพแวดล้อม
- [ ] เปิด Terminal แล้วรันคำสั่งเพื่อสร้างแอปพลิเคชันคอนโซลใหม่:
  ```bash
  mkdir lab-grade-checker
  cd lab-grade-checker
  dotnet new console
  code .
  ```
- [ ] เมื่อ VS Code เปิดขึ้นมา ให้เปิดไฟล์ `Program.cs` แล้วลบโค้ดเดิมทิ้งทั้งหมด

### ขั้นตอนที่ 2: วางโครงร่างส่วนติดต่อผู้ใช้ (UI & Input)
- [ ] สร้าง ASCII Art เป็นแบนเนอร์ส่วนหัวของโปรแกรม (ใช้ `Console.WriteLine` หลายๆ บรรทัด หรือใช้ verbatim string `@""`)
- [ ] ประกาศตัวแปร `string studentName;` แล้วรับชื่อจากผู้ใช้ด้วย `Console.ReadLine()`
- [ ] ประกาศตัวแปร `int score;` แล้วรับคะแนนจากผู้ใช้ อย่าลืมใช้ `int.Parse()` เพื่อแปลงข้อความเป็นตัวเลข

### ขั้นตอนที่ 3: ออกแบบตัวแปรสำหรับเก็บผลลัพธ์
- [ ] เพื่อความเป็นระเบียบและทำให้โค้ดแก้ไขง่าย ให้สร้างตัวแปรเอาไว้รอรับค่าผลลัพธ์ตั้งแต่เนิ่นๆ ก่อนเข้าสู่เงื่อนไข `if-else`
  ```csharp
  // สร้างตัวแปรว่างๆ ไว้รอรับเกรด
  string grade = "";
  string statusMsg = "";
  ```

### ขั้นตอนที่ 4: สร้างด่านตรวจสอบความถูกต้อง (Data Validation)
- [ ] ใช้ `if` โครงสร้างชั้นนอกสุด เพื่อเช็คว่าคะแนนเป็นไปได้หรือไม่
  ```csharp
  // ถ้าคะแนน < 0 "หรือ" คะแนน > 100
  if (score < 0 || score > 100)
  {
      // แสดงข้อความ Error สีแดง (ถ้าสามารถทำสีได้) หรือข้อความปกติ
      Console.WriteLine("❌ ERROR: ข้อมูลคะแนนไม่ถูกต้อง!");
  }
  else
  {
      // ถ้าข้อมูลถูกต้อง ถึงจะเข้าสู่การประมวลผลตัดเกรดในขั้นตอนต่อไป
      // นำโค้ดในขั้นตอนที่ 5 มาใส่ในบล็อก else นี้ทั้งหมด (เรียกว่า Nested If)
  }
  ```

### ขั้นตอนที่ 5: จัดการตรรกะการตัดเกรดด้วย else if chain
- [ ] ภายในบล็อก `else` ของขั้นตอนที่ 4 ให้เริ่มต้นโซ่การตรวจสอบคะแนน โดยเรียงลำดับจากสูงไปต่ำ (หรือต่ำไปสูงก็ได้ แต่ต้องสม่ำเสมอ)
  ```csharp
  if (score >= 80)
  {
      grade = "A";
      statusMsg = "🌟 ผ่านเกณฑ์ยอดเยี่ยม!";
  }
  else if (score >= 70)
  {
      grade = "B";
      statusMsg = "ผ่านเกณฑ์ดี";
  }
  // ทยอยเพิ่ม else if สำหรับ C, D 
  // ...
  else
  {
      // ตัวเลือกสุดท้ายคือ F ไม่ต้องเช็คเงื่อนไขแล้วเพราะเหลือแค่ < 50 อย่างเดียว
      grade = "F";
      statusMsg = "💥 ไม่ผ่านเกณฑ์ ต้องลงเรียนใหม่";
  }
  ```

### ขั้นตอนที่ 6: แสดงรายงานผลการเรียน (Output & Report)
- [ ] หลังจากจบปีกกาของกระบวนการทั้งหมด (ที่อยู่ภายในบล็อกความถูกต้อง) ให้สั่งปรินท์รายงาน `OFFICIAL TRANSCRIPT` อย่างสวยงาม
- [ ] ประกอบตัวแปร `studentName`, `score`, `grade`, และ `statusMsg` เข้าด้วยกัน

### ขั้นตอนที่ 7: ทดสอบทุกกรณีที่เป็นไปได้ (Unit Testing แบบ Manual)
ในฐานะนักพัฒนาซอฟต์แวร์ที่ดี คุณต้องทดสอบโปรแกรมให้ครบทุกสถานการณ์
- [ ] รันโปรแกรม กรอกคะแนน **85** -> คาดหวัง: **A**
- [ ] รันโปรแกรม กรอกคะแนน **50** -> คาดหวัง: **D** (เช็คขอบเขตรอยต่อ ขอบเขตบั๊คบ่อยสุด)
- [ ] รันโปรแกรม กรอกคะแนน **49** -> คาดหวัง: **F**
- [ ] รันโปรแกรม กรอกคะแนน **-5** -> คาดหวัง: **ERROR**
- [ ] รันโปรแกรม กรอกคะแนน **101** -> คาดหวัง: **ERROR**

ถ้าทุกอย่างแสดงผลตรงตามเงื่อนไข แปลว่าโปรแกรมของคุณสมบูรณ์แบบร้อยเปอร์เซ็นต์! 🎉

## 🔥 Challenge (โจทย์ท้าทายสุดขีด!)

สำหรับผู้เรียนที่ผ่านแล็บพื้นฐานอย่างรวดเร็วและต้องการบททดสอบความลึกล้ำทางโค้ดดิ้ง ลองทำโจทย์เสริมเหล่านี้ดู:

**ภารกิจที่ 1: ระบบกันโปรแกรมพัง (Crash Protection)**
เมื่อทดสอบโปรแกรม ถ้าคุณเผลอกรอกคะแนนเป็นตัวอักษร เช่น พิมพ์คำว่า "ร้อย" ลงไป คำสั่ง `int.Parse()` จะทำงานผิดพลาดและแอปพลิเคชันจะพัง (Unhandled Exception) คายบรรทัดสีแดงออกมายาวเหยียด
**คำท้า:** ให้คุณนำความรู้จากหัวข้อ `TryParse` (ท้ายบทที่ 2) เข้ามาผสมผสาน ดักจับให้ได้ว่าสิ่งที่ผู้ใช้พิมพ์มา "เป็นตัวเลขแน่ๆ ใช่หรือไม่" ถ้าพิมพ์ตัวอักษรมา ให้แสดง Error ว่า "รูปแบบไม่ถูกต้อง กรุณากรอกเป็นตัวเลข" แทนการปล่อยให้โปรแกรมพัง

**ภารกิจที่ 2: ระบบคำนวณแต้ม GPA (Grade Point)**
ระบบมหาวิทยาลัยไม่ได้เก็บเกรดเป็นตัวอักษรอย่างเดียว แต่นำเกรดไปผูกกับคะแนนหน่วยกิต (A = 4.0, B = 3.0, C = 2.0, D = 1.0, F = 0.0) 
**คำท้า:** จงใช้ **Switch Expression (จากบทเรียนที่เพิ่งผ่านมา)** รับตัวแปร `grade` (เช่น "A") เข้าไป แล้วแปลงให้กลายเป็นตัวเลขทศนิยม (`double gpa`) ออกมา จากนั้นเพิ่มบรรทัดแสดง GPA ในหน้า Transcript ตัวอย่าง: `เกรดเฉลี่ยที่ได้: 4.0`

**ภารกิจที่ 3: ระบบให้กำลังใจตามเกรด**
ใช้ตรรกศาสตร์ **Ternary Operator (`?:`)** บรรทัดเดียวจบ ในการตัดสินใจว่า: 
"ถ้า gpa >= 2.0 ให้ออกข้อความ `(สถานะปลอดภัย)` แต่ถ้าน้อยกว่านั้น ให้ออกข้อความ `(สถานะโปรเบชั่น/เสี่ยงรีไทร์)`"
และนำข้อความนี้ไปแปะท้ายสุดของ Transcript
