# เฉลย: if / else (การตัดสินใจ)

## 🎯 เฉลย Mini Exercise

### **โจทย์ที่ 1: เครื่องตรวจจับความเร็ว (Speed Camera)**
**เฉลยและวิธีทำ:**
ในการวางโครงสร้าง `if-else if` สำหรับการเช็คตัวเลขเป็นช่วงๆ กฎที่สำคัญที่สุดคือ **ต้องเรียงเงื่อนไขอย่างเป็นระเบียบ จากสูงสุดลงต่ำสุด (หรือต่ำสุดขึ้นสูงสุด)** ไม่เช่นนั้นโปรแกรมจะถูกหลอกและดักจับเงื่อนไขผิดตัวได้

```csharp
Console.WriteLine("=== ระบบกล้องตรวจจับความเร็วอัจฉริยะ ===");
Console.Write("ความเร็วรถที่วิ่งผ่าน (km/h): ");
int speed = int.Parse(Console.ReadLine());

// เริ่มเช็คจากค่าที่อันตรายที่สุด หรือค่าสูงสุดก่อน
if (speed > 120)
{
    Console.WriteLine("โดนใบสั่ง: ขับรถเร็วเกินกำหนด (ปรับ 1,000 บาท)");
}
else if (speed >= 80) // ถ้าหลุดรอด if แรกมาได้ แปลว่า speed <= 120 แน่นอน 100%
{
    // ตรงนี้จริงๆ คือ (speed >= 80 && speed <= 120) แต่เราละเว้นข้างหลังได้เพราะ else if กรองมาให้แล้ว
    Console.WriteLine("ความเร็วปกติ ขับขี่ปลอดภัย");
}
else
{
    // เหลือแค่กรณีสุดท้ายคือวิ่งช้ากว่า 80 แน่นอน ไม่ต้องเขียนเงื่อนไขอีก
    Console.WriteLine("ขับช้าเกินไปในเลนขวา (ปรับ 500 บาท)");
}
```

### **โจทย์ที่ 2: หักล้างโค้ดที่ผิดพลาด (Bug Hunting)**
**เฉลยและวิธีทำ:**
โค้ดเจ้าปัญหา:
```csharp
int score = 100;
if (score >= 50) { Console.WriteLine("สอบผ่าน"); }
if (score < 50) { Console.WriteLine("สอบตก"); }
else { Console.WriteLine("ข้อมูลผิดพลาด"); }
```
**วิเคราะห์บั๊ก:**
สาเหตุที่เป็นบั๊กเพราะว่ามี `if` สองตัวแยกกันอย่างอิสระ (ไม่ใช่สาย `else if` เดียวกัน)
1. เมื่อ `score = 100` มันเช็ค `if (100 >= 50)` พบว่าเป็นจริง จึงพิมพ์ **"สอบผ่าน"** ออกมา 1 รอบ
2. จบคำสั่งแรก มันเดินหน้าต่อไปเจอ `if (100 < 50)` พบว่าเป็นเท็จ มันจึงกระโดดลงไปทำบล็อก `else` ของคำสั่งชุดที่สองทันที
3. บล็อก `else` สั่งให้พิมพ์ **"ข้อมูลผิดพลาด"** ออกมาเป็นรอบที่สอง! ทำให้ผู้ใช้สับสน

**การแก้ไขที่ถูกต้อง (Refactoring):**
ต้องจับทั้งหมดมัดรวมกันเป็นสายเดียวด้วยกาวที่ชื่อว่า `else if`

```csharp
int score = 100;

if (score >= 50 && score <= 100) 
{
    Console.WriteLine("สอบผ่าน");
}
else if (score >= 0 && score < 50)
{
    Console.WriteLine("สอบตก");
}
else
{
    // สำหรับคะแนนแปลกๆ เช่น กรอก -5 หรือ 105 เข้ามา
    Console.WriteLine("ข้อมูลผิดพลาด");
}
```

## 🔥 เฉลย Challenge (เกมทายตัวเลขสุดโหด)

**โจทย์:** สร้างตรรกะเกมทายตัวเลข 42 โดยมีระบบคำใบ้ระดับขั้น
**เฉลยระดับโค้ดคุณภาพสูง (Production Quality):**

ความยากของโจทย์นี้คือการพิจารณากรณีที่ตัวเลข "ใกล้เคียง (ต่างไม่เกิน 5)" ซึ่งสามารถอยู่ได้ทั้งด้านบวกและลบ (เช่น 38 หรือ 47) การใช้ค่าสัมบูรณ์ (Absolute) `Math.Abs()` จะทำให้โค้ดเราคลีนมาก

```csharp
Console.WriteLine("======================================");
Console.WriteLine("      🕵️ NUMBER GUESSING GAME 🕵️     ");
Console.WriteLine("======================================");

// กำหนดเลขเป้าหมายแบบค่าคงที่
const int SecretNumber = 42;

Console.Write("ทายตัวเลขปริศนา (1 - 100): ");
int guess = int.Parse(Console.ReadLine());

Console.WriteLine("\n[กำลังตรวจสอบคำทายของคุณ...]\n");

// [ด่านที่ 1]: ทายถูกแบบเป๊ะๆ 100% 
if (guess == SecretNumber)
{
    Console.WriteLine("🎉 ยินดีด้วย! คุณทายถูกต้องแม่นยำมาก! 🎉");
}
// [ด่านที่ 2]: ทายไม่ถูก แต่เฉียดไปนิดเดียว (ไม่เกิน 5 แต้ท)
// Math.Abs() จะแปลงค่าติดลบให้กลายเป็นค่าบวกเสมอ
else if (Math.Abs(SecretNumber - guess) <= 5)
{
    Console.WriteLine("😅 เกือบถูกแล้ว! เฉียดไปนิดเดียวจริงๆ ลองอีกนิดนะ");
    
    // โบนัสพิเศษ: ใบ้เพิ่มว่าเฉียดไปทางไหน (Nested if)
    if (guess > SecretNumber) {
        Console.WriteLine("  (คำใบ้เพิ่มเติม: ลดลงอีกนิดนึง)");
    } else {
        Console.WriteLine("  (คำใบ้เพิ่มเติม: เพิ่มขึ้นอีกนิดนึง)");
    }
}
// [ด่านที่ 3]: ทายผิดแบบห่างไกล (เกิน 5 แต้มขึ้นไป) จึงต้องแยกกรณี
else if (guess > SecretNumber)
{
    Console.WriteLine("❌ คุณทายตัวเลขสูงเกินไปมาก โดดลงมาหน่อย!");
}
else
{
    // ไม่ต้องใช้ else if (guess < SecretNumber) แล้ว เพราะเป็นทางเลือกสุดท้าย
    Console.WriteLine("❌ คุณทายตัวเลขต่ำเกินไป เร่งเครื่องขึ้นมาหน่อย!");
}

Console.WriteLine("======================================");
```

**สิ่งที่เรียนรู้จากเฉลยนี้:**
1. การสร้างเกมที่โต้ตอบและฉลาด ไม่ได้ยากอย่างที่คิด เพียงแค่จัดโครงสร้างทางเดิน `if-else if-else` ให้คลุมทุกความเป็นไปได้
2. การใช้เครื่องมือเสริมทางคณิตศาสตร์อย่าง `Math.Abs()` เข้ามาประยุกต์ ช่วยประหยัดบรรทัดโค้ดที่ต้องเขียนเช็คแบบแมนนวล (`guess >= 37 && guess <= 47`) ไปได้อย่างมหาศาล
3. การวาง Nested if ข้างใน `else if` เป็นเทคนิคยอดฮิตในการสร้าง "ความฉลาดระดับรอง" ให้กับเงื่อนไขหลักที่ผ่านเข้ารอบมาแล้ว
