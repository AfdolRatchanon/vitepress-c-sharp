# เฉลย: Ternary & Null Operators

## 🎯 เฉลย Mini Exercise

**โจทย์ที่ 1:** แปลงประโยค "ถ้าตัวแปร money >= price คืนค่า true มิฉะนั้น false" เป็น Ternary Operator บรรทัดเดียว
**เฉลย:**
```csharp
int money = 500;
int price = 650;

// [วิธีที่ 1: เขียนแบบตรงตัว]
bool canBuy = (money >= price) ? true : false;
```
*(เคล็ดลับระดับโปร: ในความเป็นจริง โค้ดด้านบนทำงานถูกต้อง แต่โปรแกรมเมอร์ที่มีประสบการณ์จะไม่เขียนแบบนั้น เพราะตัวสมการ `(money >= price)` มันให้ผลลัพธ์เป็น `true/false` อยู่แล้ว จึงสามารถเขียนรวบรัดได้เลยว่า `bool canBuy = money >= price;` แต่อย่างไรก็ตาม การตอบแบบวิธีที่ 1 คือถูกต้องตามจุดประสงค์ของโจทย์ที่ต้องการให้ฝึกใช้เครื่องหมาย `?:`)*

**โจทย์ที่ 2:** อธิบายผลลัพธ์ของ `??`
```csharp
int? userScore = null;
int defaultScore = 0;
int result = userScore ?? defaultScore;
```
**เฉลย:**
ตัวแปร `result` จะมีค่าเป็น **`0`**
**เหตุผล:** ตัวดำเนินการ Null-Coalescing (`??`) จะทำการพิจารณาค่าทางฝั่งซ้ายมือก่อน (`userScore`) 
- ถ้าฝั่งซ้าย **มีค่าไม่ใช่ null** มันจะดึงค่านั้นมาใช้ทันที
- แต่ถ้าฝั่งซ้าย **เป็น null (ความว่างเปล่า)** มันจะหันไปหยิบค่าทางฝั่งขวามือมาใช้แทน (Fallback)
ในกรณีนี้ `userScore` เป็น `null` อย่างชัดเจน มันจึงไปดึงค่าตัวแปร `defaultScore` ซึ่งมีค่า `0` มายัดใส่ใน `result`

## 🔥 เฉลย Challenge

**โจทย์:** Nested Ternary (การซ้อนทับเครื่องหมาย Ternary ยาวๆ เพื่อหาเกรด)
"ถ้า >= 80 ได้ A, ถ้าไม่ใช่ เช็คต่อว่า >= 50 ได้ P, ถ้าไม่ใช่เลยได้ F"
**เฉลย:**
โครงสร้างของ Ternary คือ `(เงื่อนไข) ? (ถ้าจริง) : (ถ้าเท็จ)`
การทำ Nested หมายถึงเราเอาเครื่องหมาย Ternary อีกอัน ไปยัดใส่ตรงตำแหน่ง "ถ้าเท็จ" ของอันแรก

```csharp
int score = 65;

// เขียนแบบซ้อนกัน (เรียงบรรทัดเพื่อให้อ่านง่าย)
string grade = (score >= 80) ? "A" : 
               (score >= 50) ? "P" : 
               "F";

Console.WriteLine("เกรดที่ได้คือ: " + grade);
```

**การวิเคราะห์กลไก:**
1. คอมพิวเตอร์เช็ค `(65 >= 80)` -> เท็จ
2. เนื่องจากเป็นเท็จ มันจึงข้ามค่า `"A"` แล้วตกมาทำคำสั่งที่อยู่หลังโคลอน (`:`)
3. คำสั่งหลังโคลอน ดันเป็นเครื่องหมาย Ternary ซ้อนอีกชุด! มันจึงทำงานประเมิน `(65 >= 50)` -> จริง
4. เนื่องจากเป็นจริง มันจึงคืนค่า `"P"` และจบการทำงานทันที

*(ข้อควรระวัง: โค้ดด้านบนนี้ถูกต้องและทำงานได้เร็วกว่า `if-else` ด้วยซ้ำ แต่ในทีมทำงานจริง หลายคนจะปฏิเสธการรีวิวโค้ด (Code Review Reject) หากเขียนซ้อนกันลึกเกิน 2 ชั้น เพราะมันทำให้เพื่อนร่วมงานอ่านยากจนปวดหัว!)*
