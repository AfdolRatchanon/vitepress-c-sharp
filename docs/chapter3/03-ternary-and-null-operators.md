# Ternary & Null Operators

> 💡 **เป้าหมาย:** ทำความรู้จักกับ "ทางลัด" ในการตัดสินใจของ C# และวิธีรับมือกับ "ความว่างเปล่า" (Null) อย่างมืออาชีพ เครื่องหมายเหล่านี้จะทำให้โค้ดของคุณสั้นลงอย่างน่าอัศจรรย์

## 📖 ทฤษฎีและแนวคิด (Theory & Concepts)

### 1. Ternary Operator (เงื่อนไขบรรทัดเดียว `?:`)

Ternary Operator คือทางลัดของคำสั่ง `if-else` ใช้สำหรับ **กำหนดค่าให้ตัวแปรตามเงื่อนไขที่กำหนด** โดยเขียนจบในบรรทัดเดียว

```text
โครงสร้างของ Ternary Operator:

  ตัวแปร = (เงื่อนไข) ? ค่าถ้าเป็นจริง : ค่าถ้าเป็นเท็จ ;

ตัวอย่าง:
  int score = 80;
  string result = (score >= 50) ? "สอบผ่าน" : "สอบตก";
  
  // แปลว่า: ถ้า score >= 50 (จริง) ให้ result = "สอบผ่าน"
  //        แต่ถ้าไม่ใช่ (เท็จ) ให้ result = "สอบตก"
```

### 2. ข้อมูลที่มีค่าว่าง (Nullable Types: `?`)

ใน C# ตัวแปรทั่วไป (เช่น `int`, `double`, `bool`) **ไม่สามารถ** มีค่าว่าง (Null) ได้ มันต้องมีค่าเสมอ (เช่น เริ่มต้นที่ 0 หรือ false)
แต่ในโลกจริง บางครั้ง "ข้อมูลอาจจะไม่มีอยู่" เช่น ผู้ใช้ยังไม่ได้กรอกเบอร์โทรศัพท์

หากเราต้องการให้ตัวแปรเก็บค่า `null` ได้ เราต้องใส่ **`?`** ต่อท้ายชนิดข้อมูล:
```csharp
int age = null;        // ❌ Error: int ห้ามเป็น null
int? ageOrNull = null; // ✅ ถูกต้อง: int? คือ Nullable int (เก็บ int ก็ได้ เก็บ null ก็ได้)

string name = null;    // string เป็น Reference Type สามารถเก็บ null ได้โดยธรรมชาติตั้งแต่เวอร์ชันเก่า
                       // (แต่ใน C# 8.0+ แนะนำให้ใส่ ? เป็น string? หากต้องการให้เก็บ null แบบปลอดภัย)
```

### 3. Null-Coalescing Operator (`??`)

เมื่อมีตัวแปรที่อาจเป็น `null` ได้ เรามักจะต้องการ **"ค่าเริ่มต้น (Default Value)"** ไว้ใช้ในกรณีที่มันว่างเปล่า เราสามารถใช้ `??` เพื่อจัดการเรื่องนี้อย่างง่ายดาย

```text
โครงสร้าง:
  ค่าที่ต้องการ = ตัวแปรที่อาจจะว่าง ?? ค่าสำรองถ้ามันว่าง ;

ตัวอย่าง:
  string? userInput = null;
  string finalName = userInput ?? "Guest"; 

  // แปลว่า: ตรวจสอบ userInput ถ้ามีค่า ให้เอาค่านั้นมาใช้
  //        แต่ถ้าเป็น null ให้เอา "Guest" ไปใช้แทน
```

### 4. Null-Conditional Operator (`?.`)

เมื่อคุณต้องการเข้าถึงข้อมูลหรือ Method ของ Object ที่อาจจะว่างเปล่า (null) ถ้าคุณเรียกใช้ตรงๆ โปรแกรมจะพัง (NullReferenceException)
การใส่ `?.` หมายความว่า "ถ้ามันไม่ว่าง ค่อยเข้าไปเรียกใช้ แต่ถ้าว่าง ก็ปล่อยผ่านคืนค่าเป็น null ไป ไม่ต้องพัง"

```csharp
string? message = null;

// int length = message.Length;   // ❌ プログラムพัง! เพราะ message เป็น null ไม่มีคุณสมบัติ Length ให้ดึง
int? length = message?.Length;    // ✅ ถ้า message ว่าง length จะเป็น null ไปด้วยอย่างปลอดภัย
```

## 💻 ตัวอย่างโค้ด (Code Implementation)

### ตัวอย่างที่ 1: การใช้ Ternary Operator ลดรูปโค้ด

::: code-group
```csharp [Program.cs]
Console.Write("กรุณากรอกอายุของคุณ: ");
int age = int.Parse(Console.ReadLine());

// [1] วิธีเขียนแบบเก่า (if-else ปกติ - จะเรียนในหัวข้อถัดไป)
// string status;
// if (age >= 18) {
//     status = "ผู้ใหญ่";
// } else {
//     status = "เด็ก";
// }

// [2] วิธีเขียนแบบ Ternary (สั้นและกระชับ)
string status = (age >= 18) ? "ผู้ใหญ่" : "เด็ก";

Console.WriteLine("สถานะของคุณคือ: " + status);
```
:::

**Expected Output:**
```
กรุณากรอกอายุของคุณ: 15
สถานะของคุณคือ: เด็ก
```

### ตัวอย่างที่ 2: การจัดการค่าว่างด้วย `??`

::: code-group
```csharp [Program.cs]
Console.WriteLine("=== ระบบสมัครสมาชิก ===");

// จำลองสถานการณ์: ระบบฐานข้อมูลคืนค่าเบอร์โทรกลับมาเป็น null (ไม่ได้กรอก)
string? phoneFromDatabase = null;

// ถ้า phoneFromDatabase เป็น null ให้ใช้ "ไม่มีข้อมูลติดต่อ" แทน
string displayPhone = phoneFromDatabase ?? "ไม่มีข้อมูลติดต่อ";

Console.WriteLine("เบอร์โทรศัพท์: " + displayPhone);

// ---------------------------
// ทีนี้ลองจำลองว่ามีข้อมูลจริงๆ
string? emailFromDatabase = "user@example.com";

// เนื่องจากไม่เป็น null มันจะเอา "user@example.com" ไปใช้
string displayEmail = emailFromDatabase ?? "ไม่ได้ระบุอีเมล";

Console.WriteLine("อีเมล: " + displayEmail);
```
:::

**Expected Output:**
```
=== ระบบสมัครสมาชิก ===
เบอร์โทรศัพท์: ไม่มีข้อมูลติดต่อ
อีเมล: user@example.com
```

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

- **โจทย์ที่ 1:** เขียนโค้ดแปลงคำสั่งจำลองต่อไปนี้ ให้ใช้ Ternary Operator (`?:`) ให้จบในบรรทัดเดียว
  *เงื่อนไข: "ถ้าตัวแปร `money` มากกว่าหรือเท่ากับ `price` ให้คืนค่า `true` (ซื้อได้) แต่ถ้าน้อยกว่าให้คืนค่า `false` (ซื้อไม่ได้)"*
  กำหนดตัวแปรให้:
  ```csharp
  int money = 500;
  int price = 650;
  bool canBuy = // เติมโค้ด Ternary ตรงนี้
  ```

- **โจทย์ที่ 2:** อธิบายผลลัพธ์ของโค้ดบรรทัดที่ 3 ว่าตัวแปร `result` จะได้ค่าอะไรไปเก็บไว้?
  ```csharp
  int? userScore = null;
  int defaultScore = 0;
  int result = userScore ?? defaultScore;
  ```

::: details 💡 คำใบ้ (Hint)
- โจทย์ 1: `(เงื่อนไข) ? ค่าถ้าเป็นจริง : ค่าถ้าเป็นเท็จ;` โดยค่าที่ออกต้องเป็นประเภท `bool`
- โจทย์ 2: สังเกตว่า `userScore` ฝั่งซ้ายเป็นอะไร? ถ้าเป็น `null` เครื่องหมาย `??` จะทำงานฝั่งขวา
:::

## 🔥 Challenge (โจทย์ท้าทาย!)

- **โจทย์:** เราสามารถต่อกัน (Chain) เครื่องหมาย Ternary ได้เหมือน `else if` (เรียกว่า Nested Ternary) ลองแปลประโยคนี้เป็น Ternary บรรทัดเดียว:
  "ถ้าคะแนน (score) >= 80 ได้เกรด 'A', ถ้าไม่ใช่ ให้ดูว่า >= 50 ไหม ถ้าใช่ได้เกรด 'P' (Pass), ถ้าไม่ใช่เลยได้เกรด 'F'"
  ลองเขียนดูว่ามันจะออกมาหน้าตาแบบไหน! (ข้อควรระวัง: แม้จะเขียนได้ แต่มันอาจจะอ่านยากมากในชีวิตจริง)

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจ

**คำถาม 1:** Ternary Operator `?:` สามารถเอาไปแทนที่ `if-else` ได้ทุกกรณีเลยหรือไม่?
**แนวคำตอบ:** **ไม่ได้** Ternary Operator เหมาะสำหรับแทนที่ `if-else` เฉพาะในกรณีที่เป็น **"การกำหนดค่า (Assignment) คืนกลับ (Return) เท่านั้น"** เช่น การเลือกค่าใดค่าหนึ่งมาใส่ตัวแปร แต่ถ้าคุณต้องการให้เงื่อนไขนั้นไปสั่ง "ทำงานกระบวนการซับซ้อน" (เช่น Print ข้อความ 3 บรรทัด หรือ บันทึกไฟล์) คุณจะต้องใช้ `if-else` แบบปกติ (Statements)

**คำถาม 2:** โค้ด `int? count = null;` ตัวอักษร `?` หลัง `int` มีความหมายว่าอย่างไร?
**แนวคำตอบ:** เป็นการบอก Compiler ให้สร้างเป็นชนิดข้อมูล **Nullable Value Type** ซึ่งหมายความว่า ตัวแปร `count` สามารถเก็บตัวเลขจำนวนเต็มตามปกติก็ได้ และสามารถรับค่า `null` (ความว่างเปล่า) เพิ่มเติมได้ด้วย (โดยปกติแล้วตัวแปรแบบ Value Type อย่าง int, double, bool จะรับค่า null ไม่ได้)
:::
