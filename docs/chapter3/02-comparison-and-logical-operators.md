# Comparison & Logical Operators

> 💡 **เป้าหมาย:** คอมพิวเตอร์ตัดสินใจได้โดยการเปรียบเทียบข้อมูล (เช่น เงินพอไหม? อายุถึงเกณฑ์หรือเปล่า?) บทนี้เราจะเรียนรู้วิธีตั้งคำถามกับคอมพิวเตอร์ และนำคำถามหลายๆ ข้อมาเชื่อมตรรกะกัน

## 📖 ทฤษฎีและแนวคิด (Theory & Concepts)

### 1. Comparison Operators (ตัวดำเนินการเปรียบเทียบ)

ตัวดำเนินการเหล่านี้ใช้เปรียบเทียบข้อมูลฝั่งซ้ายและฝั่งขวา โดยจะให้ผลลัพธ์เป็นค่าความจริง (`bool` คือ `true` หรือ `false`) เสมอ

| เครื่องหมาย | ความหมาย | ตัวอย่าง | ผลลัพธ์ (`int x = 5`) |
| :---: | :--- | :--- | :--- |
| `==` | เท่ากับ (Equal) | `x == 5` | `true` |
| `!=` | ไม่เท่ากับ (Not Equal) | `x != 5` | `false` |
| `>` | มากกว่า (Greater than) | `x > 3` | `true` |
| `<` | น้อยกว่า (Less than) | `x < 5` | `false` |
| `>=` | มากกว่าหรือเท่ากับ | `x >= 5` | `true` |
| `<=` | น้อยกว่าหรือเท่ากับ | `x <= 4` | `false` |

> **⚠️ ข้อควรระวังขั้นวิกฤติ:**
> อย่าสับสนระหว่าง `=` (ตัวเดียว) กับ `==` (สองตัว)
> - `=` ใช้สำหรับ **"กำหนดค่า"** (เอาของใส่กล่อง) เช่น `int a = 10;`
> - `==` ใช้สำหรับ **"เปรียบเทียบ"** (ถามว่าเท่ากันไหม?) เช่น `bool isEqual = (a == 10);`

### 2. Logical Operators (ตัวดำเนินการทางตรรกศาสตร์)

เมื่อเรามีคำถามหลายคำถาม และต้องการให้คอมพิวเตอร์พิจารณาร่วมกัน เราจะใช้เครื่องหมายตรรกศาสตร์:

```text
1. AND (และ) ใช้เครื่องหมาย &&
   - จะเป็น true ได้ ต้อง "จริงทั้งคู่"
   - เช่น "ต้องมีอายุ 18 ปีขึ้นไป" && "ต้องมีบัตรประชาชน"

2. OR (หรือ) ใช้เครื่องหมาย ||
   - จะเป็น true ได้ ขอแค่ "จริงอย่างใดอย่างหนึ่ง"
   - เช่น "จ่ายด้วยเงินสด" || "จ่ายด้วยบัตรเครดิต" (อย่างไหนก็ได้)

3. NOT (ไม่/นิเสธ) ใช้เครื่องหมาย !
   - กลับค่าความจริง (จริงเป็นเท็จ, เท็จเป็นจริง)
   - เช่น !isBanned (แปลว่า ไม่ถูกแบน)
```

#### Truth Table (ตารางค่าความจริง)
เพื่อให้เห็นภาพชัดเจน นี่คือตารางสรุปผลของ `&&` และ `||`

| A (เงื่อนไข 1) | B (เงื่อนไข 2) | A && B (AND) | A \|\| B (OR) |
| :---: | :---: | :---: | :---: |
| `true` | `true` | **`true`** | `true` |
| `true` | `false` | `false` | **`true`** |
| `false` | `true` | `false` | **`true`** |
| `false` | `false` | `false` | `false` |

### 3. Short-Circuit Evaluation (การประเมินแบบลัด)

C# มีความฉลาดในการทำงานที่เรียกว่า **Short-Circuit**:
- สำหรับ `&&`: ถ้ารู้ว่าฝั่งซ้ายเป็น `false` มันจะ **ไม่อ่านฝั่งขวาเลย** (เพราะฝั่งซ้ายเท็จปุ๊บ ผลรวมเป็นเท็จทันที)
- สำหรับ `||`: ถ้ารู้ว่าฝั่งซ้ายเป็น `true` มันจะ **ไม่อ่านฝั่งขวาเลย** (เพราะฝั่งซ้ายจริงปุ๊บ ผลรวมเป็นจริงทันที)
สิ่งนี้ช่วยให้โปรแกรมทำงานเร็วขึ้นและป้องกัน Error ในฝั่งขวาได้

## 💻 ตัวอย่างโค้ด (Code Implementation)

### ตัวอย่างที่ 1: การเปรียบเทียบพื้นฐาน

::: code-group
```csharp [Program.cs]
int playerHealth = 100;
int enemyDamage = 120;

// คำนวณเลือดที่เหลือ
int remainingHealth = playerHealth - enemyDamage;

// เปรียบเทียบ
bool isDead = remainingHealth <= 0;
bool isPerfect = remainingHealth == 100;

Console.WriteLine("เลือดที่เหลือ: " + remainingHealth);
Console.WriteLine("ตัวละครตายหรือไม่? " + isDead);
Console.WriteLine("เลือดเต็มหรือไม่? " + isPerfect);
```
:::

**Expected Output:**
```
เลือดที่เหลือ: -20
ตัวละครตายหรือไม่? True
เลือดเต็มหรือไม่? False
```

---

### ตัวอย่างที่ 2: การใช้ตรรกะ AND / OR (การตรวจสอบสิทธิ์)

::: code-group
```csharp [Program.cs]
Console.WriteLine("=== ระบบตรวจบัตรเข้าผับ ===");
int age = 19;
bool hasIDCard = true;
bool hasVipPass = false;

// เงื่อนไข: อายุต้อง >= 20 "และ" ต้องพกบัตร
bool canEnterNormally = (age >= 20) && (hasIDCard == true);

// เงื่อนไขพิเศษ: เข้าได้แบบปกติ "หรือ" มีบัตร VIP
bool finalDecision = canEnterNormally || hasVipPass;

Console.WriteLine("อายุ: " + age);
Console.WriteLine("พกบัตรประชาชน: " + hasIDCard);
Console.WriteLine("บัตร VIP: " + hasVipPass);
Console.WriteLine("------------------");
Console.WriteLine("ได้รับอนุญาตให้เข้าร้าน: " + finalDecision);
```
:::

**Expected Output:**
```
=== ระบบตรวจบัตรเข้าผับ ===
อายุ: 19
พกบัตรประชาชน: True
บัตร VIP: False
------------------
ได้รับอนุญาตให้เข้าร้าน: False
```

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

- **โจทย์ที่ 1:** สมมติคุณกำลังเขียนเกม พิจารณากฎการโจมตีคริติคอลดังนี้:
  "การโจมตีจะติดคริติคอล (Critical) ก็ต่อเมื่อ ความแม่นยำ (`accuracy`) มากกว่า 80 **และ** ไม่ถูกศัตรูบล็อค (`isBlocked` เป็น false)"
  ให้เขียนเฉพาะบรรทัดที่คำนวณตัวแปร `bool isCritical` โดยใช้ `&&` และ `!` ให้ถูกต้อง (สมมติว่ามีการประกาศ `accuracy` และ `isBlocked` ไว้แล้ว)

- **โจทย์ที่ 2:** เดาผลลัพธ์ (True หรือ False) ของตัวแปรเหล่านี้ด้วยตาเปล่า:
  ```csharp
  int x = 10;
  int y = 5;
  bool result1 = (x > y) && (x == 10);
  bool result2 = (x < 5) || (y != 5);
  bool result3 = !(x == 10);
  ```

::: details 💡 คำใบ้ (Hint)
- โจทย์ 1: `ไม่ถูกศัตรูบล็อค` แปลงเป็นโค้ดคือ `!isBlocked` หรือ `isBlocked == false`
- โจทย์ 2: 
  - `(10 > 5)` คือ true, `(10 == 10)` คือ true -> true && true = ?
  - `(10 < 5)` คือ false, `(5 != 5)` คือ false -> false || false = ?
  - `!(10 == 10)` คือ !(true) = ?
:::

## 🔥 Challenge (โจทย์ท้าทาย!)

- **โจทย์:** ทฤษฎี De Morgan's Laws ทางตรรกศาสตร์กล่าวไว้ว่า `!(A && B)` มีค่าความจริงเท่ากับ `!A || !B` เสมอ
  เขียนโปรแกรมเพื่อจำลองและพิสูจน์ทฤษฎีนี้ใน C# โดย:
  1. สร้างตัวแปร `bool A = true;` และ `bool B = false;`
  2. คำนวณฝั่งซ้าย `!(A && B)` เก็บใส่ตัวแปร `leftSide`
  3. คำนวณฝั่งขวา `!A || !B` เก็บใส่ตัวแปร `rightSide`
  4. พิมพ์ผลลัพธ์ของทั้งคู่ และใช้ `==` เปรียบเทียบ `leftSide == rightSide` ว่ามันมีค่าตรงกันจริงหรือไม่

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจ

**คำถาม 1:** ระหว่าง `&&` (Logical AND) กับ `&` (Bitwise AND) มีความแตกต่างกันอย่างไรในการใช้งานเปรียบเทียบเงื่อนไข?
**แนวคำตอบ:** `&&` มีกลไก **Short-Circuit** (ถ้าฝั่งซ้ายเป็นเท็จ ฝั่งขวาจะไม่ถูกอ่านและประมวลผลเลย) ทำให้โปรแกรมทำงานเร็วกว่าและปลอดภัยกว่า (เช่น การเช็คว่าตัวแปรไม่เป็น Null ก่อนจะเรียกใช้เมธอด) ส่วน `&` (ตัวเดียว) คอมพิวเตอร์จะบังคับประมวลผลทั้งฝั่งซ้ายและขวาเสมอแม้ว่าจะรู้ผลลัพธ์แล้วก็ตาม

**คำถาม 2:** โค้ด `bool isValid = age = 18;` ถูกหรือผิด?
**แนวคำตอบ:** **ผิด (Compile Error)!** เครื่องหมาย `=` ตัวเดียวคือการกำหนดค่า ดังนั้นคอมพิวเตอร์พยายามจะเอาเลข 18 ใส่กล่อง age (ได้ผลเป็น int) จากนั้นพยายามเอา int ยัดใส่ bool `isValid` ซึ่งเป็นไปไม่ได้ ที่ถูกต้องต้องใช้เครื่องหมายเปรียบเทียบ `==` คือ `bool isValid = age == 18;` หรือ `bool isValid = (age == 18);`
:::
