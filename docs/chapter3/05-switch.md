# switch & switch expression (การแยกกรณี)

> 💡 **เป้าหมาย:** ทำความรู้จักกับ `switch` statement ซึ่งเป็นทางเลือกที่สะอาดตาและมีประสิทธิภาพมากกว่าการเขียน `else if` ยาวๆ เรียงต่อกันเป็นหางว่าว รวมถึงศึกษาไวยากรณ์สมัยใหม่สุดล้ำอย่าง `Switch Expression` ที่มาใน C# 8.0+

## 📖 ทฤษฎีและแนวคิด (Theory & Concepts)

### 1. ปัญหาของ else if ที่ยาวเกินไป

เมื่อโปรแกรมของเราต้องเผชิญกับการ "ตรวจสอบค่าของตัวแปรตัวเดิมตัวเดียวเปรียบเทียบกับค่าที่เป็นไปได้หลายๆ ค่า" การใช้ `if-else if` จะทำให้โค้ดดูซ้ำซ้อนและอ่านยากมาก

ลองดูตัวอย่างนี้:
```csharp
int dayOfWeek = 3;

if (dayOfWeek == 1) { Console.WriteLine("Monday"); }
else if (dayOfWeek == 2) { Console.WriteLine("Tuesday"); }
else if (dayOfWeek == 3) { Console.WriteLine("Wednesday"); }
else if (dayOfWeek == 4) { Console.WriteLine("Thursday"); }
// ... เขียนแบบนี้ไปเรื่อยๆ จนถึง 7
```
การต้องพิมพ์คำว่า `dayOfWeek ==` ซ้ำๆ ทุกบรรทัดทำให้เสียเวลา และโค้ดดูไม่สวยงามเอาเสียเลย นี่คือจุดที่ `switch` ก้าวเข้ามาแก้ปัญหา

---

### 2. โครงสร้างของ switch statement แบบดั้งเดิม (Classic Switch)

`switch` จะรับตัวแปรมา 1 ตัว แล้วนำไปจับคู่ (Match) กับคำว่า `case` แต่ละอัน หากตรงกับ `case` ไหน ก็จะทำงานในบล็อกนั้น

```text
โครงสร้างไวยากรณ์ (Syntax):

  switch (ตัวแปรที่ต้องการตรวจสอบ)
  {
      case ค่าที่1:
          // ทำงานเมื่อตัวแปรมีค่าเท่ากับ ค่าที่1
          break; // ต้องมีคำสั่ง break เพื่อกระโดดออกจาก switch เสมอ!

      case ค่าที่2:
          // ทำงานเมื่อตัวแปรมีค่าเท่ากับ ค่าที่2
          break;

      default:
          // ทำงานเมื่อค่าไม่ตรงกับ case ไหนเลย (ทำหน้าที่เหมือน else ตัวสุดท้าย)
          break;
  }
```

**⚠️ กฎเหล็กของ Switch ใน C#:**
แตกต่างจากภาษา C/C++ หรือ JavaScript ใน C# คุณ **ห้ามลืมคำสั่ง `break;` เด็ดขาด** หากคุณเขียนเคสแล้วมีโค้ดอยู่ข้างใน C# จะบังคับให้คุณต้องสั่ง `break;` (หรือ `return;`) เสมอเพื่อป้องกันข้อผิดพลาดที่เรียกว่า "Implicit Fall-through" (การไหลลงไปทำเคสถัดไปโดยไม่ได้ตั้งใจ ซึ่งเป็นบั๊กยอดฮิตในอดีต)

---

### 3. การรวมกรณี (Explicit Fall-through / Stacking Cases)

แม้ C# จะห้ามไม่ให้ไหลผ่านเคสที่มีโค้ด แต่ C# **อนุญาตให้คุณวาง `case` เปล่าๆ ซ้อนกันได้** เพื่อให้หลายๆ เงื่อนไขทำงานด้วยโค้ดชุดเดียวกัน (เปรียบเสมือนการใช้ตรรกะ `OR` หรือ `||`)

```csharp
switch (menuOption)
{
    case "A":
    case "a":
    case "1":
        // โค้ดตรงนี้จะทำงานไม่ว่าผู้ใช้จะพิมพ์ A, a หรือ 1
        Console.WriteLine("คุณเลือกเมนูที่หนึ่ง");
        break;
}
```

### 4. Switch Expression (ไวยากรณ์ยุคใหม่ของ C# 8.0+)

ในยุคปัจจุบัน (C# 8.0 เป็นต้นมา) ไมโครซอฟต์ได้เพิ่มความสามารถที่เรียกว่า **Switch Expression** ซึ่งเป็นวิวัฒนาการขั้นสุดของการเขียน `switch` ให้เหลือเพียงโค้ดสั้นๆ และคืนค่า (Return value) ออกมาใส่ตัวแปรได้โดยตรง

รูปแบบเดิมมักจะเขียนแนวนี้:
```csharp
string result;
switch (number) {
    case 1: result = "One"; break;
    case 2: result = "Two"; break;
    default: result = "Unknown"; break;
}
```

**รูปแบบใหม่ (Switch Expression):**
ตัดคำว่า `case`, `break` ทิ้งทั้งหมด และใช้เครื่องหมาย `=>` (Arrow) แทน
คำว่า `default` จะถูกแทนที่ด้วยเครื่องหมาย `_` (Discard)

```csharp
string result = number switch
{
    1 => "One",
    2 => "Two",
    _ => "Unknown"
};
```
*เห็นไหมว่ามันสั้น คลีน และสวยงามกว่าเดิมมาก!*

### 5. Pattern Matching ใน Switch (ความสามารถระดับสูง C# 9.0+)

นอกจากการเช็คค่าว่า "เท่ากับ" ไหมแล้ว `switch` ยุคใหม่ยังรองรับการเขียนแบบเช็คช่วงตัวเลข (Relational Patterns) และเช็คตรรกะผสมได้ด้วย (เหมือน `if-else` ทุกประการ แต่สวยกว่า)

```csharp
// ตัวอย่างการใช้ Relational Pattern ใน switch
string status = temperature switch
{
    < 0 => "น้ำแข็ง",
    >= 0 and <= 100 => "น้ำ",
    > 100 => "ไอน้ำ"
};
```

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่างที่ 1: การใช้ Switch ดั้งเดิมเพื่อสร้างเมนูร้านอาหาร

::: code-group
```csharp [Program.cs]
Console.WriteLine("=== ยินดีต้อนรับสู่ร้านกาแฟ C# ===");
Console.WriteLine("1. Espresso (เอสเปรสโซ่) - 50 บาท");
Console.WriteLine("2. Latte (ลาเต้) - 60 บาท");
Console.WriteLine("3. Mocha (มอคค่า) - 65 บาท");
Console.Write("กรุณาเลือกหมายเลขเมนู: ");

int choice = int.Parse(Console.ReadLine());

// ใช้ Switch ตรวจสอบทางเลือก
switch (choice)
{
    case 1:
        Console.WriteLine("คุณเลือก Espresso");
        Console.WriteLine("กำลังชง... เข้มข้นตาสว่างแน่นอน");
        break; // ต้องเบรกเสมอ

    case 2:
        Console.WriteLine("คุณเลือก Latte");
        Console.WriteLine("กำลังชง... นุ่มละมุนกลมกล่อม");
        break;

    case 3:
        Console.WriteLine("คุณเลือก Mocha");
        Console.WriteLine("กำลังชง... หอมช็อกโกแลตฟินๆ");
        break;

    default:
        // default จะทำงานเมื่อตัวเลขไม่ตรงกับ 1, 2, 3 ด้านบนเลย
        Console.WriteLine("❌ ขออภัย หมายเลขเมนูไม่ถูกต้อง");
        Console.WriteLine("กรุณาทำรายการใหม่อีกครั้ง");
        break;
}

Console.WriteLine("--- จบการทำงาน ---");
```
:::

### ตัวอย่างที่ 2: การซ้อน Case (Stacking) ทำงานร่วมกัน

สมมติเราต้องการตรวจสอบว่าตัวอักษรที่ผู้ใช้กรอก เป็น สระ (Vowel) หรือ พยัญชนะ (Consonant)

::: code-group
```csharp [Program.cs]
Console.Write("กรุณาป้อนตัวอักษรภาษาอังกฤษ 1 ตัว: ");
string input = Console.ReadLine().ToLower(); // แปลงเป็นพิมพ์เล็กหมดเพื่อความง่าย

// ดึงตัวอักษรตัวแรกมาเช็ค (เผื่อผู้ใช้พิมพ์มาหลายตัว)
char letter = input[0];

switch (letter)
{
    // ซ้อนเคส 5 ตัวเข้าด้วยกัน (ถ้าเป็น a, e, i, o, u ตัวใดตัวหนึ่ง จะวิ่งเข้าโค้ดเดียวกัน)
    case 'a':
    case 'e':
    case 'i':
    case 'o':
    case 'u':
        Console.WriteLine($"ตัวอักษร '{letter}' เป็น สระ (Vowel)");
        break;

    // ถ้าไม่ตรงกับข้างบนเลย จะวิ่งเข้า default
    default:
        Console.WriteLine($"ตัวอักษร '{letter}' เป็น พยัญชนะ (Consonant)");
        break;
}
```
:::

### ตัวอย่างที่ 3: โชว์พลังของ Switch Expression และ Pattern Matching

โค้ดนี้จำลองระบบวิเคราะห์เกรดแบบง่ายๆ โดยใช้ C# ยุคใหม่ ซึ่งจะเห็นว่าโค้ดสั้นและสวยงามมาก

::: code-group
```csharp [Program.cs]
Console.WriteLine("=== ระบบวิเคราะห์และให้คำปรึกษาคะแนนสอบ ===");
Console.Write("กรุณากรอกคะแนนที่ได้ (0-100): ");
int score = int.Parse(Console.ReadLine());

// สังเกตการใช้ switch expression ร่วมกับ relational patterns (<, >=) และ logical patterns (and)
string feedback = score switch
{
    // กรณีคะแนนเต็ม
    100 => "สมบูรณ์แบบ! คุณคือยอดมนุษย์",
    
    // กรณีอยู่ในช่วง 80-99
    >= 80 and < 100 => "ยอดเยี่ยมมาก รักษามาตรฐานไว้นะ",
    
    // กรณีอยู่ในช่วง 50-79
    >= 50 and < 80 => "สอบผ่าน! แต่ยังสามารถพัฒนาให้ดีกว่านี้ได้อีก",
    
    // กรณีตก (0-49)
    >= 0 and < 50 => "ไม่ต้องเสียใจ ลองพยายามใหม่อีกครั้งนะ เราเป็นกำลังใจให้",
    
    // กรณีข้อมูลผิดพลาด (กรอกติดลบ หรือเกิน 100) ตัว _ (discard) จะรองรับทั้งหมด
    _ => "ข้อผิดพลาด: คะแนนไม่อยู่ในช่วง 0-100"
};

// นำผลลัพธ์ที่ได้ไปแสดงผล
Console.WriteLine("คำแนะนำจากระบบ: " + feedback);
```
:::

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)

### **โจทย์ที่ 1: ระบบแปลชื่อวัน (Day Translator)**
ให้รับค่า Input ตัวเลข `1` ถึง `7` จากผู้ใช้ โดยมีความหมายคือ วันจันทร์ (1) ถึง วันอาทิตย์ (7)
ให้ใช้โครงสร้าง `switch` แบบดั้งเดิม (Classic switch ที่มี `case` และ `break`) เพื่อแปลงตัวเลขเหล่านั้นให้เป็นชื่อวันภาษาไทย เช่น ถ้ากด `1` ให้พิมพ์ `"วันจันทร์"` ถ้ากดค่าอื่นนอกเหนือจากนี้ให้พิมพ์ `"หมายเลขวันไม่ถูกต้อง"`

### **โจทย์ที่ 2: ระบบจำแนกขนาดเสื้อ (T-Shirt Sizer)**
กำหนดให้ผู้ใช้ป้อนขนาดรอบอก (เป็นเซนติเมตร) ลงในตัวแปร `int chestSize`
ให้คุณใช้ **Switch Expression (วิถี C# ยุคใหม่)** ในการจัดหมวดหมู่ไซส์เสื้อตามนี้:
- ขนาดน้อยกว่า 90 ซม. ได้ไซส์ `"S"`
- ขนาดตั้งแต่ 90 ถึง 100 ซม. ได้ไซส์ `"M"`
- ขนาดตั้งแต่ 101 ถึง 110 ซม. ได้ไซส์ `"L"`
- ขนาดตั้งแต่ 111 ซม. ขึ้นไป ได้ไซส์ `"XL"`
ให้นำค่าไซส์ที่ได้ไปเก็บในตัวแปร `string tshirtSize` และแสดงผลออกทางหน้าจอ

::: details 💡 คำใบ้และแนวทาง (Hints)
- **สำหรับโจทย์ 1:** อย่าลืมใส่ `break;` หลังจบแต่ละ `case` และใช้ `default:` ในการจัดการกับตัวเลขแปลกๆ ที่ไม่ใช่ 1-7
- **สำหรับโจทย์ 2:** โครงสร้างคือ `string tshirtSize = chestSize switch { < 90 => "S", >= 90 and <= 100 => "M", ... และจบด้วย _ => "Unknown" };`
:::

## 🔥 Challenge (โจทย์ท้าทายสุดโหด!)

**โจทย์: ระบบจัดการค่าจัดส่งสินค้า (Shipping Calculator)**

บริษัทขนส่งแห่งหนึ่งมีนโยบายคิดค่าจัดส่งอิงตาม **"โซนพื้นที่"** และ **"ประเภทลูกค้า"** ผสมกัน
ให้เขียนโปรแกรมรับข้อมูล 2 ตัว:
1. `string zone` (รับค่าเป็น "BKK" สำหรับกรุงเทพฯ, หรือ "UPC" สำหรับต่างจังหวัด)
2. `bool isPremiumMember` (ถ้าเป็นสมาชิกพรีเมียมให้ค่าส่งถูกลง)

**ตารางค่าจัดส่ง:**
| โซน (zone) | สมาชิกปกติ | สมาชิกพรีเมียม (isPremiumMember == true) |
| :--- | :--- | :--- |
| `"BKK"` | 50 บาท | 0 บาท (ส่งฟรี) |
| `"UPC"` | 100 บาท | 30 บาท |

**ความท้าทาย:** ให้คุณใช้ **Switch Expression แบบซ้อนทับ (Tuple Pattern Matching)** ในการไขโจทย์นี้! ซึ่งเป็นเทคนิคระดับสูงที่ช่วยให้เช็ค 2 ตัวแปรพร้อมกันได้ใน switch เดียว 
*คำใบ้:* รูปแบบของ Tuple Pattern จะหน้าตาประมาณนี้:
```csharp
int shippingCost = (zone, isPremiumMember) switch
{
    ("BKK", true) => 0,
    ("BKK", false) => 50,
    // ลองเติมเคสที่เหลือให้สมบูรณ์ดู!
};
```
ลองเติมโค้ดและรันทดสอบดูว่าคุณเข้าใจเทคนิคยุคใหม่นี้หรือไม่!

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** อะไรคือเหตุผลหลักที่ไมโครซอฟต์ออกแบบภาษา C# ให้บังคับต้องใส่คำว่า `break;` ในทุกๆ case ของโครงสร้าง switch ดั้งเดิม (ไม่ให้ไหลลื่นแบบภาษา C)?
**แนวคำตอบ:** ในภาษา C ยุคเก่า หากโปรแกรมเมอร์ลืมใส่ `break;` โปรแกรมจะไหลทะลุไปทำงานใน case ถัดไปแบบต่อเนื่อง (Fall-through) ซึ่ง 95% ของกรณีที่เกิดขึ้น ล้วนเกิดจาก "ความไม่ตั้งใจ" และทำให้เกิดบั๊กร้ายแรงในระบบที่ตามหาได้ยากมาก ไมโครซอฟต์จึงออกแบบ C# ให้เข้มงวดเรื่องนี้ โดยถ้ามีโค้ดในเคสไหน จะต้องสั่งให้หยุดและกระโดดออกอย่างชัดเจนเสมอด้วย `break;` เพื่อลดโอกาสเกิดข้อผิดพลาดจากมนุษย์

**คำถาม 2:** โครงสร้าง `switch` และ `if-else if` สามารถใช้งานแทนกันได้ 100% หรือไม่? แล้วควรเลือกใช้อันไหนตอนไหน?
**แนวคำตอบ:**
- โดยทฤษฎีแล้ว ทุกสิ่งที่ `switch` ทำได้ คุณสามารถใช้ `if-else if` ทำแทนได้ทั้งหมด 100% 
- **แต่** `switch` จะเปล่งประกายและดีกว่ามาก เมื่อคุณต้องการ "ตรวจสอบค่าแบบเฉพาะเจาะจง (Exact Match)" ของตัวแปร **ตัวเดียว** กับชุดข้อมูลที่มีหลายๆ ค่า (เช่น เมนู 1 ถึง 10, ชื่อวันทั้ง 7 วัน, รหัสไปรษณีย์) เพราะมันทำให้อ่านง่ายกว่าและรันได้เร็วกว่าในระดับหน่วยความจำ (Compiler สามารถสร้าง Jump Table ได้)
- ส่วน `if-else` จะเหมาะกว่าเมื่อเงื่อนไขมีความซับซ้อนเกี่ยวพันกับตัวแปร **หลายตัว** พร้อมๆ กัน (เช่น `if (a > 10 && b == 5 || c != null)`) ซึ่งแบบนี้เอาไปเขียนใน switch คลาสสิกไม่ได้
:::
