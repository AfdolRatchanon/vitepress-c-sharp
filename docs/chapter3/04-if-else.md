# if / else (การตัดสินใจและแยกสายการทำงาน)

> 💡 **เป้าหมาย:** ทำความเข้าใจและเชี่ยวชาญการใช้โครงสร้างควบคุมทิศทางแบบเงื่อนไข `if`, `else if` และ `else` ซึ่งเป็นหัวใจสำคัญของการสร้างความฉลาด (Logic) ให้กับแอปพลิเคชันทุกประเภท

## 📖 ทฤษฎีและแนวคิด (Theory & Concepts)

### 1. ความสำคัญของการแยกสาย (Branching)

ก่อนหน้านี้ โปรแกรมของเราทำงานแบบ **เส้นตรง (Sequential Flow)** คือทำตั้งแต่บรรทัดแรกจนถึงบรรทัดสุดท้ายโดยไม่ข้ามเลยสักบรรทัดเดียว แต่ในโลกความเป็นจริง ซอฟต์แวร์ต้องมีการ "ตัดสินใจ" ว่าจะทำงานชุดไหน และจะข้ามการทำงานชุดไหนไป ตามเงื่อนไขที่ผู้ใช้ป้อนเข้ามา

โครงสร้าง **`if-else`** เปรียบเสมือน **"ทางแยก"** บนถนน ที่ป้ายบอกทางคือ "เงื่อนไขทางตรรกศาสตร์ (Boolean Expression)" 

```text
แผนภาพจำลองการทำงานแบบเส้นตรง (Sequential)
    │
    ▼
[ ทำงาน A ]
    │
    ▼
[ ทำงาน B ]
    │
    ▼
[ ทำงาน C ]


แผนภาพจำลองการทำงานแบบแยกสาย (Branching ด้วย if-else)
    │
    ▼
  < เงื่อนไข: อายุ >= 18 หรือไม่? >
    │                      │
   [จริง]                 [เท็จ]
    │                      │
    ▼                      ▼
[ อนุญาตให้เข้า ]   [ แสดงข้อความปฏิเสธ ]
    │                      │
    └──────────┬───────────┘
               │
               ▼
       [ ทำงานต่อไป... ]
```

### 2. โครงสร้าง if แบบพื้นฐานที่สุด (Simple if)

โครงสร้าง `if` เดี่ยวๆ จะประเมินเงื่อนไข ถ้าเงื่อนไขเป็น `true` จะทำงานในปีกกา `{ }` แต่ถ้าเงื่อนไขเป็น `false` จะข้ามปีกกานั้นไปเลยโดยไม่ทำอะไร

**ไวยากรณ์ (Syntax):**
```csharp
if (เงื่อนไขที่เป็น bool)
{
    // โค้ดที่จะทำเมื่อเงื่อนไขเป็นจริง (true)
}
// โค้ดด้านล่างนี้จะทำงานเสมอ ไม่ว่าเงื่อนไขด้านบนจะจริงหรือเท็จ
```

**ตัวอย่างการคิดแบบคอมพิวเตอร์:**
"ถ้า (if) ฝนตก (isRaining == true) ให้ { กางร่ม }"
(ถ้าฝนไม่ตก ก็แค่เดินผ่านไป ไม่ต้องทำอะไร)

### 3. โครงสร้าง if-else (มีทางเลือกสำรอง)

เมื่อเราต้องการให้โปรแกรมทำ "อย่างใดอย่างหนึ่ง" เสมอ คือถ้าเงื่อนไขแรกไม่เป็นจริง ก็ให้ไปทำอีกอย่างหนึ่งแทน เราจะใช้ `else` เข้ามาช่วย

**ไวยากรณ์ (Syntax):**
```csharp
if (เงื่อนไข)
{
    // ทำส่วนนี้ถ้าเงื่อนไขเป็น จริง (true)
}
else
{
    // ทำส่วนนี้ถ้าเงื่อนไขเป็น เท็จ (false)
}
```

```text
ตัวอย่างการคิดด้วย if-else:
  ถ้า (if) มีเงิน > 500 บาท
  {
      กินชาบู
  }
  มิฉะนั้น (else)
  {
      กินมาม่า
  }
```

---

### 4. โครงสร้าง if-else if-else (เงื่อนไขหลายชั้น)

เมื่อชีวิตไม่ได้มีแค่ 2 ทางเลือก (ขาวกับดำ) แต่มีหลายช่วง เช่น การตัดเกรด A, B, C, D, F เราสามารถนำ `else if` มาเชื่อมต่อกันได้กี่ชั้นก็ได้ โดยที่เงื่อนไขจะถูกประเมินจากบนลงล่าง **ถ้าเจออันไหนจริงก่อน มันจะทำอันนั้น แล้วกระโดดออกทันที (ไม่เช็คอันล่างต่อแล้ว)**

**ไวยากรณ์ (Syntax):**
```csharp
if (เงื่อนไขที่ 1)
{
    // ทำเมื่อเงื่อนไข 1 จริง
}
else if (เงื่อนไขที่ 2)
{
    // ทำเมื่อเงื่อนไข 1 เท็จ แต่เงื่อนไข 2 จริง
}
else if (เงื่อนไขที่ 3)
{
    // ทำเมื่อเงื่อนไข 1 และ 2 เท็จ แต่เงื่อนไข 3 จริง
}
else
{
    // ทำเมื่อไม่มีเงื่อนไขใดเลยข้างบนที่เป็นจริง (ตัวเลือกสุดท้าย)
}
```

---

### 5. Nested if (เงื่อนไขซ้อนเงื่อนไข)

เราสามารถเขียน `if` ซ้อนเข้าไปข้างใน `if` อีกชั้นหนึ่งได้ (เหมือนการเข้าไปในห้อง แล้วเจอประตูอีกบานที่ต้องไขกุญแจ) 

**ไวยากรณ์ (Syntax):**
```csharp
if (เงื่อนไขหลัก)
{
    if (เงื่อนไขย่อย)
    {
        // จะทำงานตรงนี้ได้ ต้องผ่านทั้งเงื่อนไขหลักและย่อย
    }
}
```
*ข้อควรระวัง: การซ้อน if ลึกเกินไป (เช่น 4-5 ชั้น) จะทำให้โค้ดเกิดสิ่งที่เรียกว่า Arrow Code (โค้ดรูปลูกศรที่เยื้องเข้าไปลึกๆ) ซึ่งจะอ่านยากมาก ควรหลีกเลี่ยงโดยการใช้ Logical Operators (`&&`, `||`) ช่วยรวบเงื่อนไข*

### 6. Scope (ขอบเขตของตัวแปรใน Block)

กฎที่สำคัญมากในภาษา C# คือ **"ตัวแปรที่ถูกสร้างในปีกกา `{}` ใด จะมีชีวิตอยู่แค่ในปีกกานั้นเท่านั้น"**

```csharp
int outsideValue = 10;

if (true)
{
    int insideValue = 5;
    // มองเห็น outsideValue (เพราะอยู่ข้างนอก)
    Console.WriteLine(outsideValue + insideValue); 
}

// ❌ Error! มองไม่เห็น insideValue เพราะมันถูกทำลายไปแล้วเมื่อจบปีกกาของ if
// Console.WriteLine(insideValue); 
```

## 💻 ตัวอย่างโค้ดเชิงลึก (In-Depth Implementation)

### ตัวอย่างที่ 1: การใช้ Simple if (เช็คข้อยกเว้น)

::: code-group
```csharp [Program.cs]
Console.WriteLine("=== ระบบคำนวณราคาสินค้า ===");

Console.Write("กรอกราคาสินค้า: ");
double price = double.Parse(Console.ReadLine());

// เริ่มต้นสมมติว่าไม่มีค่าจัดส่ง
double shippingFee = 0.0;

// ถ้าซื้อไม่ถึง 500 บาท จะโดนคิดค่าส่ง 50 บาท
if (price < 500)
{
    shippingFee = 50.0;
    Console.WriteLine("⚠️ แจ้งเตือน: ยอดซื้อไม่ถึง 500 บาท มีค่าจัดส่ง 50 บาท");
}

// ไม่ว่าจะเข้า if หรือไม่ บรรทัดด้านล่างนี้จะทำงานเสมอ
double total = price + shippingFee;

Console.WriteLine("-----------------------");
Console.WriteLine("ราคาสินค้า: " + price + " บาท");
Console.WriteLine("ค่าจัดส่ง: " + shippingFee + " บาท");
Console.WriteLine("ยอดรวมทั้งสิ้น: " + total + " บาท");
```
:::

**การวิเคราะห์ผลลัพธ์:**
- ถ้ากรอก `300`: เงื่อนไข `300 < 500` เป็นจริง โปรแกรมจะเข้าปีกกา กำหนดค่าส่งเป็น 50 และแสดงคำเตือน
- ถ้ากรอก `1000`: เงื่อนไข `1000 < 500` เป็นเท็จ โปรแกรมจะข้ามปีกกาไป ค่าจัดส่งจึงคงค่าเริ่มต้นไว้ที่ 0.0

### ตัวอย่างที่ 2: การใช้ if-else (ทางแยก 2 ทาง)

::: code-group
```csharp [Program.cs]
Console.WriteLine("=== ระบบล็อกอิน ===");

// จำลองรหัสผ่านที่ถูกต้องในระบบ
const string CorrectPassword = "secret_password_123";

Console.Write("กรุณากรอกรหัสผ่าน: ");
string userInput = Console.ReadLine();

{

    // สามารถจำลองการดึงข้อมูลส่วนตัวมาแสดง
    Console.WriteLine("- ข้อมูลบัญชี: ปกติ");

else
{

    Console.WriteLine("ระบบจะบันทึกความพยายามในการเข้าระบบที่ผิดปกติครั้งนี้ไว้");
}

Console.WriteLine("--- จบการทำงานของระบบล็อกอิน ---");
```

### ตัวอย่างที่ 3: การใช้ if-else if-else (จัดหมวดหมู่)

การใช้ else if เหมาะอย่างยิ่งสำหรับการจัดกลุ่มข้อมูลที่มีช่วง (Range) แน่นอน
*เคล็ดลับ: ต้องจัดเรียงเงื่อนไขจาก มากไปน้อย หรือ น้อยไปมาก อย่างเป็นระเบียบ ไม่อย่างนั้นตรรกะจะรวน*

::: code-group
```csharp [Program.cs]
Console.WriteLine("=== เครื่องประเมินอายุและช่วงวัย ===");

Console.Write("โปรดระบุอายุของคุณ: ");
int age = int.Parse(Console.ReadLine());

// สังเกตการเรียงลำดับ: เช็คจากค่าน้อย (เด็ก) ไล่ขึ้นไปค่ามาก (ผู้ใหญ่)
if (age < 0)
{
    // ป้องกันคนกรอกเลขติดลบ
    Console.WriteLine("ข้อผิดพลาด: อายุไม่สามารถติดลบได้");
}
else if (age <= 12)
{
    // ถ้าผ่าน if แรกมาได้ แสดงว่า age >= 0 แน่นอน
    Console.WriteLine("สถานะ: วัยเด็ก (Child)");
    Console.WriteLine("คำแนะนำ: ควรดื่มนมเยอะๆ และนอนพักผ่อนให้เพียงพอ");
}
else if (age <= 19)
{
    // ถ้ามาถึงตรงนี้ แสดงว่า age ต้องมากกว่า 12 
    Console.WriteLine("สถานะ: วัยรุ่น (Teenager)");
    Console.WriteLine("คำแนะนำ: อยู่ในช่วงค้นหาตัวเอง สนุกกับชีวิตให้เต็มที่");
}
else if (age <= 59)
{
    Console.WriteLine("สถานะ: วัยทำงาน (Adult)");
    Console.WriteLine("คำแนะนำ: อย่าลืมแบ่งเวลาให้ครอบครัวและการออกกำลังกาย");
}
else
{
    // ทุกกรณีที่อายุ >= 60 จะตกมาที่ else ทั้งหมด
    Console.WriteLine("สถานะ: วัยเกษียณ (Senior)");
    Console.WriteLine("คำแนะนำ: รักษาสุขภาพ ทานอาหารที่มีประโยชน์");
}
```
:::

### ตัวอย่างที่ 4: Nested if (การกรองข้อมูลแบบละเอียด)

กรณีที่เราต้องการเช็คสิทธิ์หลายขั้นตอน เช่น จะกู้เงินได้ ต้องมีเงินเดือนผ่านเกณฑ์ก่อน และอายุงานต้องผ่านเกณฑ์ด้วย

::: code-group
```csharp [Program.cs]
Console.WriteLine("=== ระบบพิจารณาสินเชื่อเบื้องต้น ===");

Console.Write("เงินเดือนของคุณ: ");
double salary = double.Parse(Console.ReadLine());

Console.Write("อายุงาน (เดือน): ");
int workMonths = int.Parse(Console.ReadLine());

// ด่านที่ 1: ตรวจสอบรายได้
if (salary >= 15000)
{
    // ผ่านด่านที่ 1 เข้าสู่ด่านที่ 2: ตรวจสอบอายุงาน
    if (workMonths >= 6)
    {
        Console.WriteLine("🎉 ยินดีด้วย! คุณผ่านเกณฑ์การขอสินเชื่อ");
    }
    else
    {
        // เงินเดือนผ่าน แต่อายุงานไม่ผ่าน
        Console.WriteLine("❌ คุณสมบัติไม่ผ่าน: อายุงานของคุณน้อยกว่า 6 เดือน");
        Console.WriteLine("กรุณากลับมาสมัครใหม่เมื่อทำงานครบ 6 เดือน");
    }
}
else
{
    // ไม่ผ่านตั้งแต่ด่านแรก (เงินเดือนไม่ถึง)
    Console.WriteLine("❌ คุณสมบัติไม่ผ่าน: เงินเดือนขั้นต่ำต้อง 15,000 บาทขึ้นไป");
    // สังเกตว่าถ้าเงินเดือนไม่ถึง ระบบจะไม่สนใจเช็คอายุงานเลย (ประหยัดเวลา)
}
```

### **โจทย์ที่ 1: เครื่องตรวจจับความเร็ว (Speed Camera)**
ให้คุณเขียนโปรแกรมจำลองกล้องตรวจจับความเร็วบนทางด่วน โดยรับข้อมูลความเร็วของรถ (int) จากผู้ใช้ แล้วตรวจสอบตามกฎดังนี้:
1. ถ้าความเร็ว **เกิน 120** ให้แสดงข้อความ `"โดนใบสั่ง: ขับรถเร็วเกินกำหนด (ปรับ 1,000 บาท)"`
2. ถ้าความเร็วอยู่ระหว่าง **80 ถึง 120** ให้แสดงข้อความ `"ความเร็วปกติ ขับขี่ปลอดภัย"`
3. ถ้าความเร็วน้อยกว่า **80** ให้แสดงข้อความ `"ขับช้าเกินไปในเลนขวา (ปรับ 500 บาท)"`

### **โจทย์ที่ 2: หักล้างโค้ดที่ผิดพลาด (Bug Hunting)**
โปรแกรมเมอร์มือใหม่เขียนโค้ดเพื่อเช็คผลสอบผ่าน/ตก แต่มีบั๊กร้ายแรงซ่อนอยู่ ให้อ่านโค้ดด้านล่างแล้วบอกว่าทำไมเวลาป้อนคะแนน `100` โปรแกรมถึงบอกว่า "ตก" และคุณจะแก้ไขโค้ดนี้อย่างไร?

```csharp
int score = 100;

if (score >= 50)
{
    Console.WriteLine("สอบผ่าน");
}
if (score < 50)
{
    Console.WriteLine("สอบตก");
}
else
{
    Console.WriteLine("ข้อมูลผิดพลาด");
}
```

::: details 💡 คำใบ้และแนวทาง (Hints)
- **สำหรับโจทย์ 1:** คุณต้องใช้โครงสร้าง `if`, `else if`, และ `else` ให้พิจารณาว่าควรเอาเงื่อนไขไหนขึ้นก่อน (เช็ค `> 120` ก่อน แล้วค่อย `else if (speed >= 80)` แล้วที่เหลือปล่อยลง `else`)
- **สำหรับโจทย์ 2:** สังเกตว่ามี `if` สองตัวที่แยกขาดจากกัน (ไม่ใช่ `else if`) เมื่อ `score = 100` มันจะผ่าน `if` ตัวแรกพิมพ์ "สอบผ่าน" จากนั้นมันจะมาเจอ `if` ตัวที่สอง ซึ่ง `100 < 50` เป็นเท็จ มันจึงกระโดดเข้า `else` ของตัวที่สอง แล้วพิมพ์ "ข้อมูลผิดพลาด" ออกมาด้วย! วิธีแก้คือต้องรวบให้เป็นสายเดียวกันด้วย `else if`
:::

**โจทย์: เกมทายตัวเลข (Number Guessing Logic)**

ให้คุณตั้งตัวแปรคงที่ (const) ชื่อ `SecretNumber = 42;` เป็นตัวเลขปริศนา
จากนั้นให้รับ Input ตัวเลขจากผู้ใช้ (คำทาย) แล้วให้โปรแกรมโต้ตอบตามเงื่อนไขนี้:
1. ถ้าผู้ใช้ทายถูกเป๊ะ (`==`) ให้แสดง `"ยินดีด้วย! คุณทายถูกต้อง!"`
2. ถ้าผู้ใช้ทายผิด แต่ **ใกล้เคียงมาก (บวกลบไม่เกิน 5)** (เช่น ทาย 38 ถึง 47) ให้แสดง `"เกือบถูกแล้ว! เฉียดไปนิดเดียว"`
3. ถ้าทายผิดแบบห่างไกล ให้เช็คต่อว่า ทายมากไป หรือ ทายน้อยไป 
   - ถ้ามากไป แสดง `"คุณทายตัวเลขสูงเกินไป"`
   - ถ้าน้อยไป แสดง `"คุณทายตัวเลขต่ำเกินไป"`

*คำแนะนำ: ในข้อ 2 การตรวจสอบระยะห่างบวกลบไม่เกิน 5 สามารถใช้ฟังก์ชันคณิตศาสตร์ `Math.Abs(SecretNumber - guess) <= 5` (ฟังก์ชันค่าสัมบูรณ์) หรือจะเขียนเงื่อนไขแบบดิบๆ ด้วยตรรกศาสตร์ `&&` ก็ได้ ลองคิดหาวิธีดู!*

## 🗣️ ทบทวน (Review)

::: details ❓ คำถามทบทวนความเข้าใจเชิงลึก

**คำถาม 1:** ในกรณีที่เรามีเงื่อนไข `if` โดยไม่ได้ใส่ปีกกา `{}` ครอบไว้ จะเกิดอะไรขึ้น? (ตัวอย่าง: `if (x > 0) Console.WriteLine("Positive"); Console.WriteLine("Always");`)
**แนวคำตอบ:** ในภาษา C# (และตระกูล C) หากไม่ใส่ปีกกา `{}` คำสั่ง `if` จะมีผลควบคุมเฉพาะ **คำสั่งบรรทัดถัดไปเพียง 1 บรรทัดเท่านั้น** 
จากตัวอย่าง คำว่า "Positive" จะถูกพิมพ์เฉพาะเมื่อ x > 0 แต่คำว่า "Always" จะถูกพิมพ์ **เสมอ** ไม่ว่า x จะเป็นอะไรก็ตาม เพราะมันอยู่นอกเหนือการควบคุมของ if แล้ว
*คำเตือน: นี่เป็นสาเหตุของบั๊กที่พบบ่อยมาก (เรียกว่า Apple's goto fail bug) นักพัฒนาที่ดีควรใส่ปีกกา `{}` เสมอแม้ว่าจะมีโค้ดแค่บรรทัดเดียวก็ตาม เพื่อป้องกันความผิดพลาดในอนาคต*

**คำถาม 2:** โครงสร้าง `if - if - if` กับ `if - else if - else if` ให้ผลลัพธ์ต่างกันอย่างไรในการประมวลผล?
**แนวคำตอบ:** 
- แบบ `if - if - if` (เรียงต่อกัน): คอมพิวเตอร์จะประเมินและเช็คเงื่อนไข **ทุกตัว** เสมอ แม้ว่าตัวแรกจะเป็นจริงไปแล้วก็ตาม (ทำให้เสียเวลาประมวลผล และอาจทำให้เกิดการรันคำสั่งซ้ำซ้อนถ้าเงื่อนไขเหลื่อมล้ำกัน)
- แบบ `if - else if` (เชื่อมต่อกันเป็นสาย): คอมพิวเตอร์จะประเมินไล่จากบนลงล่าง **เมื่อเจอเงื่อนไขใดที่เป็นจริง มันจะทำงานแค่บล็อกนั้น และข้าม (Skip) การเช็คเงื่อนไขที่เหลือทั้งหมดด้านล่างทันที** (ทำให้ทำงานได้เร็วและถูกต้องตามตรรกะที่เป็นทางเลือกที่ต้องเลือกทางใดทางหนึ่งเท่านั้น)
:::
