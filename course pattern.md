# 📘 C# Zero to Hero: Content Pattern & Standard

คู่มือและมาตรฐานการเขียนเนื้อหาสำหรับเอกสารประกอบการเรียนการสอนวิชาภาษา C# แบบ Zero to Hero เน้น Console Application เป็นหลัก

---

## ⚙️ กฎการสร้างเนื้อหาบทเรียน (Content Generation Rules)

เมื่อต้องสร้างเนื้อหาใหม่ ผู้เขียน/ผู้สร้าง ต้องปฏิบัติตามกฎเหล็กดังนี้:

1. **Naming Convention ตรงตามโครงสร้าง:** ใช้รูปแบบโฟลเดอร์รายบท เช่น `chapter1/` และไฟล์ย่อย `01-topic.md`
2. **เน้นบริบท Console App:** ตัวอย่างโค้ดทุกชิ้นต้องสามารถรันได้ใน Console Application เป็นหลัก ไม่อิงกับ Framework ภายนอก
3. **C# เป็น Tech Stack หลัก:** โฟกัสที่ภาษา C# และ .NET Runtime เป็นแกนกลาง ห้ามนำ Library ภายนอกมาสอนในเนื้อหาพื้นฐาน
4. **Comprehensive Theory & Visuals:** ทฤษฎีต้อง "อัดแน่นและละเอียดที่สุด" และต้องอธิบายโครงสร้างผ่าน **ASCII Art** ให้เห็นภาพชัดเจน
5. **Usable Code Examples:** โค้ดตัวอย่างต้องสมบูรณ์ คัดลอกแล้วรันได้ทันทีโดยไม่ต้องแก้ไขเพิ่มเติม
6. **Creative yet Accessible Exercises:** โจทย์ Mini Exercise ต้องมีความคิดสร้างสรรค์ (แตกต่างจากตัวอย่าง) แต่ **ห้ามยากเกินขอบเขตของทฤษฎีที่เพิ่งเรียน** เพื่อให้ทำหน้าที่เป็น "การทบทวน" อย่างแท้จริง
7. **เนื้อหาต้องครบองค์ประกอบ:** ทุกไฟล์ต้องมี ทฤษฎี + ASCII Art อย่างน้อย 1 ชิ้น + ตัวอย่างโค้ดที่รันได้ + Mini Exercise อย่างน้อย 2 ข้อ
8. **Progressive Difficulty ระหว่างบท:** แต่ละบทต้อง **build on** ความรู้จากบทก่อนหน้า ห้ามใช้ concept ที่ยังไม่ได้สอน และ `index.md` ของทุกบทต้องระบุ prerequisite ไว้ชัดเจน
9. **ความยาวและความลึกซึ้งของเนื้อหา (Depth & Length):** เนื้อหาของแต่ละไฟล์ (รวมถึงไฟล์ทฤษฎี Lab และเฉลย) ต้องมีความยาวและอัดแน่นไปด้วยความรู้ โดยกำหนดให้ **ต้องมีอย่างน้อย 300 บรรทัดขึ้นไปต่อ 1 ไฟล์** เพื่อให้ครอบคลุมทฤษฎีเชิงลึก, โฟลว์ชาร์ต ASCII, Edge Cases, การอธิบายโค้ดที่ละเอียดยิบ และ Best Practices

## 📁 Naming Convention (โครงสร้างโฟลเดอร์และไฟล์)

โครงสร้างหลักจะแบ่งตาม **บท (Chapter)** -> **หัวข้อย่อย (Topic)**

| ประเภท | รูปแบบโฟลเดอร์/ไฟล์ | ตัวอย่าง |
| :--- | :--- | :--- |
| โฟลเดอร์บท | `chapterX` | `chapter1/` |
| ไฟล์ภาพรวมบท | `index.md` | `chapter1/index.md` |
| ไฟล์เนื้อหาทฤษฎี | `XX-topic-name.md` (มีเลขลำดับ) | `chapter1/01-hello-world.md` |
| ไฟล์แล็บปฏิบัติ | `99-lab-topic.md` (ใช้เลข **99 เสมอ** เพื่อให้ไฟล์ Lab ไปอยู่ท้ายสุดของ Sidebar เสมอ ไม่ว่าบทนั้นจะมีกี่หัวข้อ) | `chapter1/99-lab-calculator.md` |
| ไฟล์เฉลย | `answers/XX-topic-answers.md` (อยู่ใน subfolder `answers/`) | `chapter1/answers/09-lab-calculator-answers.md` |

> ⚠️ **หมายเหตุเรื่องไฟล์เฉลย:** โฟลเดอร์ `answers/` จะ **ไม่ถูกเชื่อมต่อกับ Sidebar** โดยตรง การเข้าถึงเฉลยและวิธีเปิดไฟล์เหล่านี้จะระบุไว้ใน `README.md` เท่านั้น ห้ามใส่ลิงก์เฉลยในเนื้อหาบทเรียนโดยเด็ดขาด

**ตัวอย่างโครงสร้างบทแรก:**
```
chapter1/
├── index.md                          ← ภาพรวมบท: Introduction to C#
├── 01-what-is-csharp.md              ← C# คืออะไร, .NET คืออะไร
├── 02-setup-environment.md           ← ติดตั้ง .NET SDK, VS Code
├── 03-hello-world.md                 ← โปรแกรมแรก, โครงสร้าง Program.cs
├── 04-variables-datatypes.md         ← ตัวแปรและชนิดข้อมูล
├── 99-lab-calculator.md              ← Lab: สร้าง Console Calculator (เสมออยู่ท้ายสุด)
└── answers/
    ├── 03-hello-world-answers.md     ← เฉลย (ไม่แสดงใน Sidebar)
    ├── 04-variables-answers.md
    └── 99-lab-calculator-answers.md
```

## 🏅 Content Standard — Layout Blueprint

โครงสร้างหน้าเว็บ (Markdown) ต้องใช้ Layout ด้านล่างนี้เป็นแกนกลางในการสร้างไฟล์เนื้อหาทุกครั้ง เพื่อความสม่ำเสมอ:

````markdown
# ชื่อหัวข้อ

> 💡 **เป้าหมาย:** สรุปสั้นๆ 1-2 บรรทัดว่าหัวข้อนี้สำคัญอย่างไรและนักเรียนจะได้เรียนรู้อะไร

## 📖 ทฤษฎีและแนวคิด (Theory & Concepts)
[อธิบายทฤษฎีพื้นฐานที่นักเรียนต้องรู้ก่อนลงมือเขียนโค้ด โดยต้องลงรายละเอียดจากง่ายไปยาก พร้อมวาดภาพประกอบแบบ ASCII Art]

```text
[ตัวอย่าง ASCII Art อธิบายการทำงาน]
+---------------------+
|   Source Code (.cs) |
+---------------------+
          |
          v  (คอมไพล์ด้วย dotnet build)
+---------------------+
|   IL (Bytecode)     |
+---------------------+
          |
          v  (รันด้วย .NET Runtime / CLR)
+---------------------+
|   Console Output    |
+---------------------+
```

## 💻 ตัวอย่างโค้ด (Code Implementation)
[หลังจากอธิบายทฤษฎีแล้ว ให้แสดงตัวอย่างโค้ดที่รันได้จริงใน Console Application]

::: code-group
```csharp [Program.cs]
// [1] คอมเมนต์อธิบายบรรทัดสำคัญ
Console.WriteLine("Hello, C#!");
```
:::

**Expected Output:**
```
Hello, C#!
```

## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ (Mini Exercise)
[โจทย์ที่สั่ง **ต้องแตกต่าง** จากตัวอย่างโค้ดด้านบนอย่างสิ้นเชิง เพื่อบังคับให้นักเรียนประยุกต์ใช้ความรู้ (Apply) ไม่ใช่แค่ทำซ้ำ (Duplicate)]
- **โจทย์ที่ 1:** ให้นักเรียนสร้างโปรแกรมที่...
- **โจทย์ที่ 2:** ให้นักเรียนแก้ไขโปรแกรมเพื่อ...

::: details 💡 คำใบ้ (Hint)
[ใส่คำใบ้เล็กน้อยเพื่อช่วยนักเรียนทบทวน โดยให้ซ่อนเอาไว้ในกล่อง details]
:::

## 🔥 Challenge (โจทย์ท้าทาย!)
[เพิ่มความท้าทายให้นักเรียนคิดด้วยตัวเอง]
- **โจทย์:** ให้นักเรียนลองสร้าง... โดย **ห้าม** เลื่อนกลับไปดูโค้ดตัวอย่างด้านบน ให้ใช้ความจำและความเข้าใจเขียนขึ้นมาใหม่ทั้งหมด

## 🗣️ ทบทวน (Review)
::: details ❓ คำถามทบทวนความเข้าใจ
**คำถาม:** ทำไมเราถึงต้องใช้... ?
**แนวคำตอบ:** เพราะ...
:::
````

## 📝 Lab Pattern (โครงสร้างแล็บปฏิบัติประจำบท)

สำหรับไฟล์ `XX-lab-topic.md` ซึ่งจะเป็นการนำเนื้อหาย่อยทั้งหมดในบทนั้นๆ มารวมกันเพื่อสร้างโปรแกรม Console จริง โครงสร้างจะเน้นไปที่เป้าหมายและขั้นตอนการทำ:

````markdown
# 💻 Lab: ชื่อโปรแกรมประจำบท

> 💡 **เป้าหมาย:** สังเคราะห์ความรู้ทั้งหมดที่เรียนในบทนี้ออกมาเป็น Console Application ที่ใช้งานได้จริง

## 📖 ภาพรวมของโปรแกรม (Program Overview)
[อธิบายว่าจะสร้างโปรแกรมอะไร รับ Input อะไร แสดง Output อะไร พร้อมวาด ASCII Art แสดง Flow การทำงาน]

```text
[Flow ของโปรแกรม]
START
  |
  v
รับ Input จากผู้ใช้
  |
  v
ประมวลผล
  |
  v
แสดงผลลัพธ์
  |
END
```

## ⏱️ เวลาที่ใช้: [ระบุเวลา เช่น 60 นาที]

## 📝 ขั้นตอนการทำงาน (Step-by-Step)
### ขั้นตอนที่ 1: [หัวข้อ]
- [ ] Task 1...
- [ ] Task 2...

### ขั้นตอนที่ 2: [หัวข้อ]
- [ ] Task 1...

## 🔥 Challenge (โจทย์ท้าทาย สำหรับคนที่ทำเสร็จก่อนเวลา!)
- **โจทย์:** [โจทย์ที่สั่งให้เพิ่มฟีเจอร์ โดยใช้ความรู้เดิม แต่มีความซับซ้อนขึ้น]
````

## 📋 Chapter Index Blueprint (โครงสร้าง `index.md` ภาพรวมบท)

ไฟล์ `index.md` ทุกไฟล์ต้องประกอบด้วย 4 ส่วนนี้เสมอ เพื่อให้นักเรียนเข้าใจภาพรวมและเตรียมตัวได้ก่อนเรียน:

````markdown
# บทที่ X: ชื่อบท

> 💡 **ภาพรวม:** อธิบายสั้นๆ ว่าบทนี้สอนอะไร และนักเรียนจะสามารถทำอะไรได้หลังจากเรียนจบ

## 🎯 สิ่งที่จะได้เรียนรู้ในบทนี้
- ✅ [หัวข้อที่ 1]
- ✅ [หัวข้อที่ 2]
- ✅ [หัวข้อที่ 3]

## 🔗 Prerequisite (ความรู้ที่ต้องมีก่อน)
> ⚠️ นักเรียนควรเข้าใจสิ่งต่อไปนี้ก่อนเริ่มบทนี้:
- [ความรู้จากบทก่อนหน้าที่จำเป็น เช่น "ตัวแปรและชนิดข้อมูล (บทที่ 1)"]
- [ถ้าเป็นบทแรกให้ระบุว่า "ไม่มี — เริ่มจาก Zero ได้เลย"]

## 📚 หัวข้อในบทนี้
| ลำดับ | หัวข้อ | ไฟล์ |
| :---: | :--- | :--- |
| 01 | [ชื่อหัวข้อ] | `01-topic.md` |
| 02 | [ชื่อหัวข้อ] | `02-topic.md` |
| 🧪 Lab | [ชื่อ Lab] | `99-lab-topic.md` |
````

## 🚫 Anti-Patterns (สิ่งที่ต้องหลีกเลี่ยง)

| ❌ ห้ามทำ | ✅ ควรทำแทน |
| :--- | :--- |
| โค้ดล้วนไม่มีทฤษฎี | ต้องอธิบายทฤษฎีและมี **ASCII Art** เพื่อให้เห็นภาพรวมก่อนเสมอ |
| ไม่แสดง Expected Output ของโค้ด | ต้องมีบล็อก `Expected Output:` ท้ายทุกตัวอย่างโค้ดเสมอ |
| โจทย์แบบฝึกหัดเหมือนกับตัวอย่างโค้ดเป๊ะ | โจทย์ (Mini Exercise) ต้อง **แตกต่าง** จากตัวอย่าง เพื่อบังคับให้ประยุกต์ใช้ความรู้ |
| ใช้ตัวอย่างที่ซับซ้อนเกินบทที่สอน | ตัวอย่างโค้ดต้องอยู่ใน **ขอบเขตของทฤษฎีในบทนั้นเท่านั้น** |
| ชื่อไฟล์ยาวและไม่มีลำดับ | ใช้ `01-topic.md`, `02-topic.md` เพื่อจัดเรียงใน Sidebar ให้สวยงาม |
| ไม่มีโจทย์ฝึกปฏิบัติในแต่ละหัวข้อ | ต้องมีแท็บ `## 🎯 โจทย์ฝึกปฏิบัติเสริมความเข้าใจ` ทุกครั้ง |
| ใส่ลิงก์เฉลยในเนื้อหาบทเรียนหรือ Sidebar | เฉลยต้องอยู่ใน `answers/` subfolder เท่านั้น **ห้ามระบุที่อยู่ในเนื้อหา** — วิธีเข้าถึงเฉลยให้เขียนไว้ที่ `README.md` แห่งเดียว |
| ใช้ Library หรือ NuGet Package ในบทพื้นฐาน | ยึด C# Standard Library เท่านั้น จนกว่าจะถึงบทที่กำหนด |
